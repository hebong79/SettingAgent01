## 추가한 테스트 케이스 (test/roiEdit.test.ts, `describe('resizeRect 변(edge) 드래그 n/s/w/e (설계 (f))')`)
기준 rect `{x:0.2, y:0.2, w:0.4, h:0.4}` (left=0.2/top=0.2/right=0.6/bottom=0.6).

| # | 케이스 | 입력 | 기대 | 직교축 불변 |
|---|--------|------|------|-------------|
| 1 | n 상변 하향 | `('n',0,0.1)` | y=0.3, h=0.3 | x=0.2, w=0.4 |
| 2 | n 상변 상향 | `('n',0,-0.1)` | y=0.1, h=0.5 | x=0.2, w=0.4 |
| 3 | s 하변 하향 | `('s',0,0.1)` | y=0.2, h=0.5 | x=0.2, w=0.4 |
| 4 | w 좌변 좌향 | `('w',-0.1,0)` | x=0.1, w=0.5 | y=0.2, h=0.4 |
| 5 | e 우변 우향 | `('e',0.1,0)` | x=0.2, w=0.5 | y=0.2, h=0.4 |
| 6 | 직교 델타 무시 | `('n',0.3,0.1)` / `('e',0.1,0.3)` | n: x·w 불변 / e: y·h 불변 | — |
| 7 | e clamp 경계 | `({x:0.9,y:0.9,w:0.05,h:0.05},'e',0.5,0)` | x+w ≤ 1, w>0 | — |
| 8 | w 음수 clamp | `({x:0.1,...},'w',-0.5,0)` | x ≥ 0, w>0(최소폭) | — |
| 9 | n 뒤집힘 정규화 | `('n',0,0.5)` | y≈0.6, w·h>0 | x=0.2, w=0.4 |
| 10 | e 좌우 뒤집힘 | `('e',-0.5,0)` | x≈0.1, w>0 | y=0.2, h=0.4 |

- 6번은 케이스 하나에 n·e 두 검증을 포함 → describe 블록 총 11 it (32-21=11).
- 각 방향 케이스마다 **관련 없는 직교축(n/s→x·w, e/w→y·h) 고정**을 assert 로 명시 검증.
- clamp(7·8)·뒤집힘 정규화(9·10)로 `clamp01Rect` + `Math.min/abs` 흡수 경로 검증.

## 경계면 교차 비교
- 소비처: `app.js` mousemove 에서 `dragState.handle`(코너 nw/ne/sw/se 또는 신규 n/s/w/e)을 `resizeRect` 로 그대로 전달. handle 문자열 도메인이 넓어진 것뿐, 시그니처·반환 shape `{x,y,w,h}` 불변 → 소비처 파싱 불일치 없음.
- `resizeRect` 반환은 항상 `clamp01Rect` 통과 → `updateSlotRoi` 가 기대하는 `[0,1]` 정규화 rect 계약 유지.

## 수동 동작확인 필요 (유닛테스트 대상 아님 — 통과 위장 금지)
`app.js` `hitTestEdge` / mousedown 결선은 DOM(overlay 캔버스·마우스 좌표) 의존이라 vitest 유닛으로 검증하지 않음. 다음은 브라우저 뷰어에서 **수동 확인 필수**:

1. **변 히트**: 슬롯 선택 후 4개 변(n/s/w/e) 라인 근처 클릭·드래그 시 해당 변만 이동(크기 변경)하는지.
2. **코너 우선순위**: 코너(nw/ne/sw/se) 근처에서는 변이 아닌 코너 히트가 잡혀 대각 2변이 동시 이동하는지(`hitTestHandle` 선호출 + `inX/inY` 코너 구간 배제 이중 안전).
3. **선택 슬롯 한정**: 선택되지 않은 슬롯/빈 영역에서는 변 드래그가 발동하지 않는지.
4. **얇은 박스**: 폭이 2·tol 미만인 박스에서 상/하변 무히트(코너로만 조작 가능)로 동작하는지 — 무해 확인.
5. **커서 피드백**: 설계 (e)항대로 이번 범위 미적용. 커서 모양(ns/ew-resize) 변화 없음 — 정상.

## 발견 이슈
- 없음. 구현(`resizeRect` 케이스 추가)이 설계 (f)항 수식·기대값과 일치. 정규화·clamp 경로도 기대대로 동작.
