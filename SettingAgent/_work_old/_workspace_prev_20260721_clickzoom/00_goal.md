# Goal — 개별 center+zoom 실카 성공률 개선 (거짓 성공 제거 + 줌 사다리)

실행모드: **B(goal/loop, 관찰형 성공기준)** · 착수: 2026-07-21

## Goal (관찰 가능한 성공)
실카(휴컴스 `RealPtzSource`)에서 `개별 center+zoom` 으로 임의의 차량(특히 **먼 차량**)을 클릭하면
**클릭한 바로 그 차량**의 번호판이 화면 중앙에 오고 목표 폭(0.20)까지 확대된다.
- 클릭 대상 ≠ 조준 대상인 **거짓 성공("완료" 보고)** 이 0 이어야 한다.
- 실패는 실패로 보고한다(명시적 reason).

## 배경 진단 (코드 확정 — 재조사 불필요)
1. **거짓 성공의 코드적 원인**: `src/calibrate/platePtz.ts:209` `centerOnPlate` 의 **최초 대상 선정만
   `radius=null`** → 반경 기각 없음. `controlMath.ts:16 pickNearestPlate` 는 무조건 최근접 1개를 반환.
   → 클릭한 (먼·작은) 판이 광각에서 미검출이면 **다른 판을 조용히 채택**, 폐루프가 그 차에 정상 수렴,
   `ok:true` "완료" 보고. 마스터 실측 목격: 좌측 끝 차량 클릭 → 중앙 왼쪽 차량이 센터+줌됨.
   2회차 이후 반복에는 `matchRadiusNorm=0.08` 기각이 있으나 **첫 선정에만 없다**.
2. **10% 성공률의 근본 원인**: center+zoom 경로(`PtzCalibrator.centerOnPoint:166`)가 **조준보다 검출을
   먼저** 요구한다. 광각에서 먼 번호판은 화소 부족으로 LPD 미검출 → 시작조차 못 함.
   ("가까운 것만 성공" 증상의 정체)
3. **구조적 비대칭**: `개별 center`(`aimPointToCenter:206`)는 실카 네이티브
   `ptz_centering setcenter`(`RealPtzSource.centerOnPoint:237`)를 써서 정확한데,
   `center+zoom` 은 이 네이티브를 **전혀 쓰지 않고** 시뮬 실측 게인(`-62`/`-35.5`,
   주석에 "cam1 시뮬 실측 — 다른 카메라에서 타당하다는 근거 없음" 명시)에 의존한다.

## Requirements (루프 내내 유지되는 불변 제약)
1. **[정직성 — 선행 필수]** `centerOnPlate` 최초 대상 선정에 반경 게이트 적용. 클릭 지점 근처에 판이
   없으면 다른 판을 대신 잡지 말고 `no_plate` 로 실패. 반경은 파라미터화하고 기본값 근거를 남긴다
   (광각에서 과도하게 좁으면 정상 케이스를 죽인다). 위장 성공 금지 원칙.
2. **[본편 — 줌 사다리]** center+zoom 경로 재구성(가산 — 기존 경로 파괴 금지):
   ① 클릭점 조준: `aimPointToCenter` 재사용(실카=네이티브 setcenter / 시뮬=기하 게인)
   ② 줌 사다리: zoom 단계 확대(`maxZoomStepRatio=1.5` 와 정합) + 각 칸에서 캡처+LPD
   ③ 검출 시 판중심 재중심 — **실카는 네이티브 setcenter 한 방**(게인/probe/damp 불요),
      네이티브 없는 시뮬은 기존 게인 1스텝 폴백
   ④ `plateWidth` 가 `targetPlateWidth(0.20)` 수렴까지 ②~③ 반복
   근거: zoom-in 은 광학중심 보존 → ①로 중앙에 온 대상은 확대해도 중앙에 남는다.
3. **[정직한 한계]** 최대 줌에서도 미검출(가림·각도·오염)이면 위장 성공 없이 명시 사유로 실패
   (예: `plate_not_found_at_max_zoom`). LPD 서비스 한계는 한계로 보고.
4. **[회귀 금지]** 시뮬레이터 center+zoom **98% 유지가 하드 제약**. 네이티브 분기 유무로 갈라
   기존 게인 경로를 보존한다.
5. **[경로 정합]** 실카 소스 라우팅(`POST /calibrate/point` 의 `source` → `CameraSourceClient`)이
   사다리 **전 구간**에서 유지. 뷰어가 보고 있는 소스 = 명령 대상.
6. **[테스트]** vitest 최소 4건: ①반경 밖 판만 있을 때 `no_plate`(다른 판 채택 안 함)
   ②네이티브 유무에 따른 setcenter/게인 분기 ③최대 줌 미검출 시 명시 사유 실패
   ④시뮬 경로 기존 동작 불변.

## 성공 확인 방법 (경험적 — 마스터 실카 클릭)
1. 실카 라이브뷰에서 `개별 center+zoom` 선택.
2. **먼 차량(화면 좌/우 끝)** 클릭 → 조준된 차량이 클릭한 그 차인지 육안 대조.
3. 실패 시 UI `종료(reason)` 문자열 채록.
4. 체감 소요 시간 기록(사다리 칸수 × 정착시간).
