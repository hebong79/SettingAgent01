### 3-2. 구현자 주장 검증 — 두 수치의 판정이 갈린다

**(a) "p3 idx=14, 여유 0.0761" → 검증됨(입력 차이).**
내 실측은 **0.0706**, 구현자는 **0.0761**. 불일치처럼 보이나 **입력이 다르다**: 구현자는 동결 픽스처 `test/fixtures/occupancyAnchor/`, 나는 지시대로 `_qa_data_final2/`(라이브)를 썼다. 두 p3 JSON 을 diff 한 결과 **검출 좌표 자체가 다르다**(예: `rect.y` 0.2778 vs 0.2926, `confidence` 0.9674 vs 0.9735 — 별개 검출 실행). 공정을 기하기 위해 **구현자의 픽스처로 동일 절차를 재실행**했다:

```
== 구현자 픽스처(test/fixtures/occupancyAnchor) 기준 ==
  p1: 최소여유 idx=1  maxY=0.8028 여유=0.1972
  p2: 최소여유 idx=13 maxY=0.8008 여유=0.1992
  p3: 최소여유 idx=14 maxY=0.9239 여유=0.0761   ← 구현자 수치 정확히 재현
```
→ **구현자의 0.0761 은 자기 입력에서 정확하다.** "최소 여유 인스턴스는 p3 idx=14" 라는 결론은 **두 입력 모두에서 동일**하게 성립한다. 이 항목은 이견 없음.

**(b) "downRatio≈0.66 부근에서 첫 절단" → 반증됨. 사실이 아니다.**

임계값을 추정이 아니라 **스윕으로 실측**했다(0.60→0.01 간격 증가, 전 프리셋·전 인스턴스에서 `maxY > 1` 최초 발생 지점):

| 입력 | 첫 하단 초과 downRatio | 발생 인스턴스 | maxY |
|------|------------------------|---------------|------|
| `_qa_data_final2` (라이브) | **0.94** | p3 idx=14 | 1.0012 |
| `test/fixtures/occupancyAnchor` (구현자 입력) | **0.96** | p3 idx=14 | 1.0002 |

**구현자 입력에서조차 0.96 이다 — 0.66 이 아니다.** 첫 절단 대상이 p3 idx=14 라는 예측만 맞았고, **임계 수치는 0.30 만큼(약 45%) 과소평가**됐다.

**원인 — 선형 외삽 식이 틀렸다.** 구현자는 maxY 가 downRatio 에 **비례**한다고 보고 `0.9239 × (0.66/0.60) ≈ 1.016 > 1` 로 계산한 것으로 보인다. 그러나 실제 아래 정점의 y 는

```
y = c.y + downRatio·bw·v.y ± (bw/2)·u.y
```

로, **`c.y`(번호판 중심) 항은 downRatio 와 무관**하다. downRatio 에 비례하는 것은 `|Cb−C|` 뿐이다. 따라서 올바른 기울기는 `d(maxY)/d(downRatio) = bw·v.y` 이고, 임계는

```
임계 downRatio ≈ 0.60 + 여유 / (bw·v.y)
```

idx=14 실측: `|Cb−C| = 0.129427` @ down 0.60 → `bw·|v| = 0.2157`, 여유 0.0706 → `0.60 + 0.0706/0.2076 ≈ 0.94` — **스윕 실측 0.94 와 일치**. 공식을 바로잡으면 실측과 맞는다.

**영향**: 코드는 무결하다. 안전여유는 구현자 주장보다 **훨씬 크다**(0.60 → 0.94, 약 57% 추가 증대 여력). 구현자 보고서 §4 의 "약 0.66 부근" 문구는 **문서상 오류**이므로 정정 권고. 향후 downRatio 증대 요청 시 0.66 을 상한으로 오인해 불필요하게 보수적으로 판단할 위험이 있다.

## 7. 육안 소견 (3장 전부 직접 열람)

**(a) 아래가 2배로 늘어 앞범퍼~바닥을 자연스럽게 덮는가 — 그렇다.**
세 프리셋 모두 region 아래 변이 **번호판에서 앞범퍼를 지나 차량 앞 노면까지** 내려온다. p3 의 idx=14(파랑, 근거리 대형 차량)에서 가장 뚜렷하다 — 범퍼 하단과 노면 접지선을 지나 그림자 영역까지 덮으며, 0.30 때 번호판 바로 아래에서 끊기던 어색함이 해소됐다. p1·p2 의 원거리 차량들도 범퍼 아래 노면을 일관되게 덮는다. **점유영역이 "차량이 실제 점유한 바닥"에 더 근접해졌다** — 지시 의도에 부합.

**(b) 하단이 잘리거나 어색한 곳 — 없다.**
전 17개가 4점 평행사변형 원형을 유지하며, 이미지 하단에서 잘린 곳이 하나도 없다. 최소 여유 인스턴스인 p3 idx=14 도 아래 변이 화면 하단에서 **눈에 띄는 간격(여유 0.0706 ≈ 76px @1080)** 을 두고 멈춘다 — 육안으로 "아슬아슬"해 보이지 않는다.

**(c) 이웃 침범이 늘었는가 — 아니다.**
`maxPairArea` 가 전 프리셋 **정확히 0** 이고, 육안으로도 인접 region 경계가 서로 닿지 않는다. p1 의 idx=5↔6, p2 의 idx=12↔13 이 가로로 가장 근접하나 교차하지 않는다. **증대 방향(아래)이 차량 열 배치 방향(가로)과 직교**하므로 이웃 간섭이 원리적으로 늘지 않는다는 점이 시각적으로 확인된다.

**(d) 관찰 사항(결함 아님, 참고).**
`upRatio=0.90` 이 차량 전장 전체를 덮지는 않아 region 위쪽이 **차량 앞유리~보닛 부근에서 끝나고 지붕·후미는 벗어난다**(p2 에서 뚜렷). 이는 이번 변경과 무관한 기존 성질이며 마스터 지시 범위 밖이다 — 보고만 하고 손대지 않았다.
