# RESUME — 개별 center+zoom 실카 개선 (마스터 재검증 대기)

작성: 2026-07-21 17:5x · 실행모드 B(goal/loop) · **현재 상태: 코드 완료, 마스터 실카 재검증 대기**

## 한 줄 요약
실카 `개별 center+zoom` 실패의 진짜 원인은 사다리가 아니라 **`zoomRange` 오설정**이었다.
`[0,65535]` → 실제 장비 상한 **16384**(뷰어 36 = raw 16384 = x36, 마스터 OSD 확증).

## 지금 무엇을 기다리는가
마스터의 실카 재검증 5종. 결과를 받으면 그때부터 이어서 작업한다.

| # | 클릭 대상 | 이전 결과 | 판정 기준 |
|---|---|---|---|
| 1 | 좌측 끝 먼 차량 | 옆차 (2회 모두) | 클릭한 그 차인가 |
| 2 | 우측 끝 먼 차량 **연속 2회** | 1회차만 성공 | **2회차도 되는가(핵심)** |
| 3 | 가까운 차량 | 성공 | 회귀 없는가 |
| 4 | 빈 바닥 | 정직한 실패 | 유지되는가 |
| 5 | UI 방향 버튼(센터라이징 직후) | 이전 위치로 튐 | 안 튀는가 |

받을 것: 클릭↔조준 대상 일치 여부 / 실패 시 `종료(...)` 문자열 / 체감 시간.

## 재검증 전 필수

- **브라우저 Ctrl+F5**(`web/app.js` 변경, 캐시 남음)

## 결과가 나쁘면 볼 곳 (추측 금지 — 계측이 이미 들어가 있다)
`logs/setting_*.log` 의 `cat:'centering' phase:'ladder'`:
- **`zoomCmd` vs `zoomAct`** — 명령과 장비 실측이 벌어지면 아직 줌이 안 먹는 것
- **`bytes`/`sha`** — 인접 rung 지문이 같으면 동일 프레임(이번에 이걸로 원인을 잡았다)
- **`distAtAim`** — rung 마다 일정해야 정상. 커지면 setcenter 잔차 또는 등방확대 가정 위반(펌웨어 왜곡)

## 변경 요약 (수정 1~16 + 리더 핫픽스)
- 반경 게이트(거짓 성공 제거) / 줌 사다리(조준→확대→검출→재중심)
- 게이트 반경 **zoom 스케일링**(고정 0.10 은 k배 확대에서 표적을 스스로 떨군다)
- `zoomRange [0,16384]`(config 실카 2곳) — **파장이 사다리 밖까지**: 배치 폭 수렴·UI 수동 줌·acquire 사다리 전부 25% 천장에 걸려 있었다
- 정착: 고정 sleep → 실제 정지 감지 / 타임아웃 5→15초 / 정체 조기종료
- `state.ptz` 재동기화 — **40개 라우트 전수 분류 + 미분류 0 검사로 봉인**(열거는 2번 실패했다)
- 성공 경계 변경: latch+정렬+장비상한 사실확인 시 `ok:true` + `widthShortfall` 보존

## 확정된 사실 (재조사 금지)
- 저장 zoom 17건은 **전부 시뮬 출처** → 조치 불요(로그 타임스탬프·소수점 일치·실카 배치 지표 0건으로 교차 확인)
