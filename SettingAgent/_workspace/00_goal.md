# 00_goal — 개별 center+zoom: LPD 1회 미검(plate_lost) 조기사망 제거

작성: 2026-07-22 · 리더(현자 라) · **B 모드(goal/loop)** — 마스터 구조화 프롬프트로 확정.

## Goal (관찰 가능한 성공기준)

시뮬 `cam1 / preset1` 의 주차면 **1~7 전부**가 개별(클릭) 센터라이징 `center+zoom`(`POST /calibrate/point mode=plate-zoom`)에서
**`ok:true` + `plateWidth = 0.20 ± 0.02`** 로 수렴한다.

- 현재(수정 전) 실측: slot1·2·3 = `ok:false reason=plate_lost`(zoom 1.69 그대로), slot4~7 = `ok:true`(zoom 13.1/13.9/16.0/16.4).
- **거짓 성공 금지선(1순위)**: 반경 밖 이웃 판을 대신 채택하지 않는다. 대상이 실제로 없으면 정직하게 `plate_lost`/`no_plate_near_click`.

## Loop (리더가 따를 이터레이션 절차)

1. 코드 수정(architect 설계 → developer 구현)
2. **리더 직접 실측**: 슬롯 1~7 스윕. 매 회 base PTZ(`pan 22 / tilt 6.8 / zoom 1.6934098`)로 복귀 후 `POST /calibrate/point {cam:1,preset:1,point:<판 중심>,mode:'plate-zoom'}`.
3. 실패가 남으면 **로그·프레임 재분석**(추측 금지) → 재설계·재수정 → 2로.
4. 성공 시 마감: `npx vitest run` 전건 + `npx tsc --noEmit` 클린 → qa 보고서 → 한글 문서화·영향도 분석.

### 새로 드러난 결함 2건

**P1 — 위장 성공(금지선 위반, 최우선).** `PtzCalibrator.centerOnPoint` 는 줌 단계가 실패하면 `if (z.ok) return z;` 를 그냥 지나쳐 **센터링 결과를 `ok:true` + reason 없이** 반환한다. 줌이 전혀 안 됐는데 UI 는 "개별 센터라이징 완료". 이 결함은 이번 변경 이전부터 있었고, 센터링이 고쳐지자 표면화됐다.

**P2 — 줌 단계의 zoom 데드존(tilt 디더로 회복 불가).** R1 slot1 줌 단계에서 `zoom 8.1738` 프레임은 **plates:0**(화면 전체 미검), tilt 디더 7시도(±0.024°=6px 등가) 전부 실패. 같은 pan/tilt 에서 zoom 만 바꿔 실측:

| zoom | 7.8 | 8.0 | **8.1738** | 8.25 | 8.4 | 8.6 | 9.0 | 6.0 | 10.0 |
|---|---|---|---|---|---|---|---|---|---|
| 검출 | ✗ | ✗ | **✗** | ✓ 0.137 | ✓ 0.142 | ✗ | ✓ 0.156 | ✓ 0.117 | ✓ 0.188 |

→ 줌 축에 **좁고 산발적인 데드존**이 있다. 줌 스텝 직후 캡처의 회복 축은 **tilt 가 아니라 zoom**이어야 한다(+1%(8.17→8.25)에서 회복).

### 이터레이션 3 설계(리더 직접 — 속도 위해 architect 생략, 근거는 위 실측)
1. **P1**: 줌 실패를 삼키지 말고 정직하게 반환(`ok:false` + `reason`). 카메라는 이미 센터링 위치로 옮겨진 상태이므로 그 `ptz` 를 그대로 싣는다. UI 는 기존 `종료(${reason})` 경로로 사유를 표시한다.
2. **P2**: `zoomToPlateWidth` 의 **줌 스텝 직후 캡처(C 지점)** 재포착 사다리를 **zoom 승법 디더** `×[1.01, 0.99, 1.02, 0.98, 1.04, 0.96]` 로 교체. prior 는 `predictCenterAfterZoom` 로 보정. clampZoom 으로 값이 안 변하는 시도는 건너뛴다. pan/tilt 이동 지점(A·B)은 **tilt 디더 그대로**(실측으로 이미 효과 확인).
