## 1. 배경 / 목적

앞선 백엔드 작업("LLM 차량 점유율 판정")에서 정밀수집 체크포인트마다 프리셋별 점유율을
LLM으로 판정해 SQLite `occupancy` 테이블에 저장하고 `GET /capture/runs/:id/occupancy`로
조회하는 경로가 완성되었다(문서: `20260705_103647_점유율판정_트리거타입.md`).

이번 작업은 그 **후속 프론트 소비/표시**다. 백엔드는 조회만 하고 무수정이다.
사용자가 확정한 표시 방식은 **"요약 표 + 오버레이 둘 다"**이며, 면별 `box` 좌표는 optional
제약(LLM이 box를 안 줄 수 있음 → 없으면 오버레이만 skip, 표/비율은 정상)을 전제로 한다.

확정 표시 범위:
- (A) 프리셋별 점유율 **요약 표** — 분석 탭(`#an-occupancy`) + 정밀수집 종료 결과 모달.
- (B) 프레임 위 면별 **점유/공차 박스 오버레이** — Live View 캔버스(`box` 있을 때만).
- (C) **표시 토글** `#roi-occupancy`(기존 ROI 토글과 대칭).
- (D) LLM 미가용 경고 모달 `#occ-llm-warn-modal`(`status.llmOccupancyUnavailable` 소비).

## 2. 구현 상세

### 2-1. `web/core.js` — 순수 함수 4개(가산, DOM/fetch 미참조)

`analyzeArtifact` 뒤(주차면 편집 섹션 앞)에 append. 캔버스 그리기는 순수화 불가하므로
**파싱·집계·포맷만** 순수 함수로 분리했다(box→pixel 환산은 기존 `toPixel` 재사용, 신규 없음).

| 함수 | 시그니처 | 역할 |
|---|---|---|
| `formatRatePct(rate)` | `0~1 → 'NN%'` | `Math.round(rate*100)+'%'`. `Number.isFinite` 실패(null/NaN/비수치/Infinity) → `'0%'`. (`core.js:254`) |
| `occupancyByKey(rows)` | `rows[] → { [cam:preset]: {camIdx,presetIdx,occupiedCount,total,rate,spaces[]} }` | `spacesJson`이 **문자열일 때만** `JSON.parse`, 실패/비배열/null → `spaces:[]`로 graceful 강등(throw 안 함). `presetKey(camIdx,presetIdx)`로 키잉. box 미보유 요소도 그대로 통과. (`core.js:266`) |
| `occupancyRows(occByKey)` | `→ Array<[key,cam,preset,occ,total,'NN%']>` | cam→preset ASC 정렬(숫자 뺄셈). `formatRatePct`로 6열 포맷. (`core.js:293`) |
| `occupancyAverage(occByKey)` | `→ {occupied,total,rate}` | ΣoccupiedCount/Σtotal 전체 평균. `total>0` 아니면 rate 0(0분모 방어, NaN 아님). (`core.js:302`) |

설계 의도: `occupancyByKey`가 파싱 실패/`null`을 삼켜 `spaces:[]`로 강등하는 것은
백엔드의 UNKNOWN 강등 철학과 정합하며, 부분 손상 데이터가 뷰어 전체를 깨뜨리지 않게 한다.

### 2-2. `web/app.js` — 데이터 소비·오버레이·표·경고 결선

**import·state 가산**
- import: 위 순수함수 4개.
- `state.occByKey:{}`(cam:preset→점유), `state.capFrameKey2:null`(수집 중 현재 프레임 {cam,preset}),
  `state.lastRunId:null`(fetch 대상 run). 모듈 변수 `occLlmWarnShown`·`prevCapRound` 가산.

**현재 프리셋 판별(비자명 핵심)**
정밀수집 중 Live 캔버스에 뜨는 프레임은 여러 프리셋을 순환한다. 이때 `state.cam/preset`(드롭다운
선택값)은 갱신되지 않고, 실제 표시 중 프레임의 프리셋은 `X-Cap-Cam/Preset` 응답 헤더로만 바뀐다.
occupancy는 프리셋 단위이므로 **지금 화면 프레임의 cam:preset**에 맞는 spaces를 그려야 정합한다.
- `capFrameTick`: 헤더 처리부에서 `state.capFrameKey2 = {cam,preset}` 갱신(`app.js:701`).
- `stopCapFramePolling`: 말미 `state.capFrameKey2 = null`(수집 종료 후 라이브 선택 프리셋 복귀, `app.js:723`).
- `currentFrameKey()`(`app.js:233`): `capFrameTimer && state.capFrameKey2`면 캡처 프레임 프리셋,
  아니면 `presetKey(state.cam, state.preset)`. 이 분기가 "수집 중 vs 라이브"를 캡슐화한다.

**오버레이 — mapping 가드 이전 분리(비자명 핵심)**
- `drawOccupancyOverlay(ctx)`(`app.js:243`)를 별도 함수로 뽑아 `drawRoiOverlay`의
  `clearRect` 직후·`if(state.roiHidden||!state.mapping)return` **이전**에 호출(`app.js:182`).
  근거: 점유 박스는 `state.mapping` 미최종화(수집 중)에도 유효하므로 mapping 가드에 막히면 안 된다.
- 렌더: 자체 가드 `$('roi-occupancy').checked && !state.roiHidden` → `currentFrameKey()`의
  `spaces` 순회 → `sp.box` 없으면 `continue`(graceful) → `toPixel(sp.box, overlay.width, overlay.height)`
  → 점유=빨강 `#ff4d4d` / 공차=초록 `#39ff14` 반투명 채움 + `strokeRect` + id 라벨.

**fetch 게이트(과다호출 회피)**
- `capPoll`(2s)에서만 fetch. 500ms 프레임폴에는 얹지 않는다.
- `state.lastRunId = status?.runId ?? state.lastRunId`(런 유지).
- 활성 중 **round 변화 게이트**: `round != null && round !== prevCapRound`일 때만
  `fetchOccupancy(lastRunId)`(`app.js:794`). fetch는 최신 1행 조회라 멱등 → 라운드 단위 호출도 안전.
- 활성→종료 전환: `await fetchOccupancy(lastRunId)` **먼저**(최신 점유율 확보) →
  이어서 결과 모달(`showCaptureResult`) 순서(`app.js:807`).
- `fetchOccupancy(runId)`(`app.js:735`): `GET /capture/runs/:id/occupancy` → `occupancyByKey`
  → `state.occByKey` → `drawRoiOverlay`. runId null·비200·예외 모두 graceful no-op.

**요약 표 / 결과 모달**
- `renderOccupancyAnalysis(showAvgCard)`(`app.js:1063`): runId=`state.lastRunId` 우선,
  없으면 `GET /capture/runs`[0].id 폴백 → occupancy fetch →
  `#an-occupancy`에 `buildTable(['프리셋 키','카메라','프리셋','점유','전체','점유율%'], occupancyRows)`.
  데이터 없으면 "점유율 데이터 없음 (정밀 수집 후 생성)". `showAvgCard`면
  `#an-summary`에 `summaryCard('평균 점유율', formatRatePct(occupancyAverage(...).rate))` 1장(`app.js:1095`).
- `renderAnalysis`: 산출물 없음(`!a.ok`)에도 `renderOccupancyAnalysis(false)` 호출(점유율은
  산출물과 독립 — 최종화 전에도 표시), `a.ok` 말미 `renderOccupancyAnalysis(true)`.
- `showCaptureResult`(`app.js:749`): 기존 lines 뒤에 `occupancyRows`가 있을 때
  "프리셋별 차량 점유율:" 헤더 + `프리셋 {key}: {occ}/{total} ({NN%})` 라인 append.

**경고·초기화·결선**
- `status.llmOccupancyUnavailable && !occLlmWarnShown` → `#occ-llm-warn-modal` 1회 표시(`app.js:785`).
- `capStart`: `occLlmWarnShown=false`·`prevCapRound=null`·`state.occByKey={}`(이전 런 잔여 제거).
- 이벤트: `#roi-occupancy` change → `drawRoiOverlay`, `#occ-llm-warn-close` → 모달 닫기.

## 3. 데이터 흐름

```
[백엔드] occupancy 테이블(프리셋별 최신 at_round 1행)
   → GET /capture/runs/:id/occupancy  (captureRoutes → getLatestOccupancy 그대로 반환)
   → [app.js fetchOccupancy / renderOccupancyAnalysis] fetch rows[]
   → occupancyByKey(rows)  // spacesJson JSON.parse graceful, cam:preset 키맵
   → state.occByKey
        ├─ (오버레이) drawOccupancyOverlay: occByKey[currentFrameKey()].spaces → box → toPixel → 캔버스
        ├─ (요약표)   occupancyRows(occByKey) → buildTable → #an-occupancy / 결과 모달 라인
        └─ (평균카드) occupancyAverage(occByKey).rate → formatRatePct → #an-summary
```

**fetch 트리거 3경로:** ① capPoll round 변화, ② capPoll 활성→종료 전환(모달 직전),
③ 분석 탭 진입(`renderOccupancyAnalysis`, runId 없으면 최신 run 폴백).

## 4. 좌표계

`box`는 정규화 0~1 좌표(면별 점유 영역). 기존 `toPixel(rect, imgW, imgH)`(`core.js:11`)를
그대로 재사용해 표시 픽셀로 환산한다 → floor/vehicle 오버레이와 동일 좌표계로 정합. 신규 환산 로직 없음.

## 5. UX 동작 명시 (중요 — 의도된 동작)

- **수집 진행 중 라이브 오버레이 미표시.** `capStart → clearRoiDisplay`가 `state.roiHidden=true`로
  두고, `drawOccupancyOverlay` 가드가 `!state.roiHidden`을 포함한다. 따라서 **수집이 도는 동안
  Live 캔버스에는 점유 박스가 뜨지 않는다.** 최종화(`capFinalize → roiHidden=false`)·결과 열람 이후
  라이브 선택 프리셋에 대해 점유 박스가 렌더된다. 이는 **기존 floor ROI 오버레이와 동일한 관례**이며
  설계 §4의 "clearRoiDisplay와 정합" 지침을 그대로 구현한 것으로 **의도된 동작(버그 아님)**이다.
- **요약 표·결과 모달 라인은 종료 즉시 표시.** 이들은 캔버스가 아니라 DOM 삽입이라 `roiHidden`
  가드와 무관하다. capPoll 활성→종료 전환에서 `await fetchOccupancy` 먼저 → `showCaptureResult`
  순서라 종료 시점 `state.occByKey`가 채워진 뒤 모달 라인이 렌더된다.
- **"수집 중 실시간 오버레이"가 필요하면 후속 조정 필요.** 현재 관례상 수집 중 라이브 캔버스에는
  안 뜨므로, 실시간 표시를 원하면 `roiHidden` 가드 분리 등 별도 작업이 필요하다.

## 7. 입출력 / REST 계약

- 소비 라우트: `GET /capture/runs/:id/occupancy` → 배열, 각 원소
  `{ camIdx, presetIdx, atRound, occupiedCount, total, rate(0~1), spacesJson:string|null, updatedAt }`.
- 폴백 라우트: `GET /capture/runs` → run 목록(원소 `.id`, 최신 `[0]`).
- spaces 요소 shape: `{ id:int, occupied:boolean, box?:{x,y,w,h} }`(정규화 0~1). box optional.
- 이번 작업은 **소비만** — 라우트/스키마/백엔드 무변경.
