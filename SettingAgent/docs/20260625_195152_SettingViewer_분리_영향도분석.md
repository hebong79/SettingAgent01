## 2. 신규 운영 의존

### 2.1 두 서비스 구동 + 기동 순서
분리 후 프로세스가 **2개**(SettingAgent + SettingViewer)다.
- **기동 순서: SettingAgent(:13020) 먼저 → SettingViewer(:13030)**. SettingAgent 가 `/mapping` upstream 이므로.
- SettingAgent 미가동 시 `/viewer/api/mapping` → **502**(ROI 미표시). 단 **영상·PTZ 정상**(카메라:13100 직접 호출, SettingAgent 비의존). → ROI 만 부분 저하, 뷰어 전체 다운 아님.

### 2.2 포트 충돌 점검
| 서비스 | 포트 | 충돌 |
|--------|------|------|
| SettingAgent | 13020 | — |
| **SettingViewer** | **13030(신규)** | 13020/13100 과 비충돌 |
| Unity 카메라 | 13100 | — |

신규 13030 은 기존 포트와 겹치지 않음(설계 확인). 동일 호스트 동시 구동 시 충돌 없음.

### 2.3 워크스페이스
루트 `package.json` `workspaces` 에 `"SettingViewer"` 추가(`["packages/*","SettingAgent","SettingViewer"]`). **소스 link 가 아니라 설치·스크립트 일원화 목적**(SettingViewer 는 `@parkagent/types` 미의존). 루트 `npm install` 성공 확인.

## 3. 중복 코드 리스크 및 동기화 가이드

분리 독립성의 대가로 **2벌 공존**:
| 코드 | SettingAgent(원본) | SettingViewer(사본) | 차이 |
|------|--------------------|----------------------|------|
| `CameraClient.ts` | 셋업이 사용. `requestImage/move/health/clampZoom` | 뷰어가 사용. 위 4 + `listCameras` 보유, `CapturedImage` 로컬 정의 | listCameras 유무·CapturedImage 출처 |
| `util/http.ts` | `fetchWithTimeout`+`isRetryable`/`withRetry`/`RetryOptions` | `fetchWithTimeout` 만 발췌 | 재시도 유틸 유무 |

**리스크**: 한쪽의 버그 수정·계약 변경(특히 Unity REST 의 snake_case 바디/응답 파싱)이 다른 쪽에 자동 반영되지 않는다.

**동기화 가이드**:
1. **CameraClient 의 Unity REST 경계(`cam_idx`/`preset_idx`/`img_name`/`img_bytes`) 변경 시 양쪽 사본 동시 수정**을 원칙으로 한다.
2. 변경 시 양쪽 테스트(SettingAgent `cameraClient*` 셋업 계약 + SettingViewer `cameraClientList`/`viewerRoutes`) 동시 그린 확인.
3. 중복이 부담이 되는 시점에 **공유 패키지(`@parkagent/camera-client` 류) 승격**을 별도 라운드로 검토(이번 분리 범위 밖 — 과설계 회피).

## 4. git 미초기화 (사실 — 이력 보존 영향)

- `ParkAgent/.git` 디렉터리가 비어 있어 저장소 미초기화 상태. `git mv`/`git status` 등 모든 git 명령이 `fatal: not a git repository`.
- 결과: 파일 이동이 일반 `mv` 로 수행되어 **git 이력 보존 불가**(파일 내용·구조는 계획과 일치). 테스트·동작에는 영향 없음.
- **후속 고려**: 추후 저장소 초기화 시, 이동된 파일들의 이력 추적이 끊긴 상태로 재커밋된다. 이력 연속성이 필요하면 초기화 시점에 별도 처리(예: 분리 커밋 메시지에 원본 경로 명시) 검토.

## 7. 영향도 결론

- **SettingAgent 셋업·두뇌 핵심 무영향**: CameraClient 시그니처 동결 + `/mapping` 유지 + `@parkagent/types`/SetupArtifact 불변 → 셋업 파이프라인·81 테스트 회귀 0.
- **파급은 두 서비스 경계에 국한**: ActionAgent/DMAgent·공유 도메인 타입으로의 전파 없음.
- **신규 운영 부담**: 2-프로세스 구동 + 기동 순서(SettingAgent 먼저). 포트 충돌 없음.
- **관리 대상 리스크 2건**: ① CameraClient/http 중복 2벌(동기화 가이드 §3) ② git 미초기화로 이력 보존 불가(§4).
- **미검증 잔여**: 실기기·실서버·통합구동·DOM·WebRTC(§6) — 전부 수동/E2E/후속 영역.
