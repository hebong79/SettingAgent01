## 1. 목적

주차장 **초기 데이터 셋팅용** 캘리브레이션 기능. 번호판 ROI(`setup_artifact.slots[].plateRoiByPreset`)를 보유한 주차면마다, 해당 `camIdx/presetIdx` 에서 카메라 PTZ 를 움직여:

1. **① pan/tilt 로 번호판 중심을 화면 중앙 `(0.5, 0.5)` 에 정렬**
2. **② zoom 으로 번호판 가로폭을 목표값(`targetPlateWidth`, 기본 0.2)으로 맞춤**

해당 슬롯의 최적 PTZ·번호판 폭·수렴 여부를 `data/slot_ptz.json` 에 저장한다. 이 산출물은 **ActionAgent 센터링(번호판 클로즈업)의 prior 후보**로 쓸 의도이나, 현재 소비 계약은 미확정(후속 결정 — 영향도 문서 참조).

> 입력 `setup_artifact.json` 은 **읽기 전용**. 캘리브레이션은 이 파일을 수정하지 않고, **별도 파일** `slot_ptz.json` 만 쓴다.

## 2. 제어 루프 (순서 엄수)

`PtzCalibrator.calibrateSlot(target)` — 슬롯 1건당:

```
프리셋 PTZ 로 시작 → 캡처 → LPD 검출 → 대상 prior 최근접 번호판 선택
  └ 번호판 미검출이면 skip(reason:'no_plate')

A) pan/tilt 중심정렬
   1. probe: probeStepDeg(1°) pan·tilt 1회 이동 → 변위 관측 → 부호 포함 게인 추정
                                                (변위 미미 → fallback 게인)
   2. for iter < maxIterations & !centered:
        (옵셔널) LLM adviseCentering 자문 → 클램프, occluded면 skip
        panTiltCorrection(P 제어, ±maxStepDeg 클램프) → move → 캡처 → 재검출
        개선 정체(<IMPROVE_EPS) → dampGain(게인 절반, 진동 방지)
   3. centered = isCentered(err, centerTol)

B) zoom 폭정렬  (중심 수렴 이후에만 진입)
   for iter < maxIterations & !converged:
        zoomCorrection = clampZoom(curZoom * sqrt(target/cur))
        move → 캡처 → 재검출
        zoom 으로 중심 드리프트 시 panTiltCorrection 1스텝 재중심(과보정 방지 1회)
        converged = isWidthConverged(plateWidth, targetPlateWidth, widthTol)

결과 기록: { ptz, plateWidth, centered, converged, reason? }
```

**순서가 핵심**: 반드시 pan/tilt 중심정렬(A)이 수렴한 뒤에야 zoom(B)에 진입한다. 검증에서 move 명령 시퀀스로 `lastPanTiltChange < firstZoomChange` 를 단언해 이 순서를 보장한다(QA §2.3 통과).

## 3. ★ PTZ 명령값 추적 (결정적 설계점)

시뮬 `/req_img`·`/req_move` 응답의 PTZ 가 항상 `pan:0, tilt:0, zoom:1` 로 echo 되어 **신뢰할 수 없다**. 따라서:

- 제어 상태 `ptz: Ptz` 는 **내가 `/req_move` 로 명령한 값(commanded)** 으로만 갱신한다. 응답 객체의 `pan/tilt/zoom` 은 **읽지 않는다**.
- `captureAndDetect(target, ptz)` 가 `camera.requestImage(cam, preset, ptz)` 로 **명령 PTZ override** 를 넘겨 현재 명령 화면을 얻는다.
- 게인은 **명령한 도(°) 변화 ↔ 관측된 번호판 정규화 변위**로 측정한다(응답 PTZ 무관).

검증 논리(QA §2.2): 모킹 LPD 가 응답 echo(0/0/1)가 아니라 명령 PTZ 에 따라 번호판 위치·폭을 생성하도록 했다. 만약 구현이 응답 PTZ 로 재동기화했다면 화면이 초기값에 고정되어 **영원히 수렴 불가**해야 하는데, happy-path 가 `centered:true·converged:true·plateWidth≈0.2` 로 수렴 → 구현이 명령값만 추적함을 역으로 증명.

## 4. 적응형 게인 (FOV 불요·부호 무관)

- pan↔X·tilt↔Y 의 부호 규약은 **하드코딩하지 않는다**. probe 1회 이동 전후 변위로 `estimateGain` 이 **부호까지 포함**해 게인(°/정규화)을 추정한다. → 카메라 FOV·축 방향을 사전에 알 필요 없음.
- `panTiltCorrection`: `newPan = curPan − errX·gainPan` (부호는 게인에 흡수). 1스텝 `±maxStepDeg` 로 클램프.
- `zoomCorrection`: `clampZoom(curZoom · sqrt(target/cur))`. `clampZoom` 은 CameraClient 의 1~36 배율 제한 경유. `plateWidth≈0` 방어(현재 zoom 반환).
- 진동 감쇠: 중심정렬 루프에서 오차 크기(유클리드) 개선이 `IMPROVE_EPS`(1e-3) 미만이면 `dampGain`(게인 절반).

> 구현 노트(§3): 설계 의사코드의 부호(`− errX·gain`)를 1차에 `+`로 작성했다가 모킹 수렴 실패로 발견 → `−`로 정정. 설계가 명시한 부호이며 설계 위반 아님.

## 5. 하이브리드 (결정형 + LLM 자문) · 폴백

- **엔진은 결정형 비례제어**(probe 게인 + P 제어 + zoom 공식). LLM 은 **자문**(소폭 보정 제안·수렴/가림 판정)만.
- `cfg.llmAdvise && brain.adviseCentering` 존재 시에만 자문 호출. 응답은 **호출측이 결정형 클램프**(pan/tilt `±maxStepDeg`, zoomFactor `0.5~2.0`) 후 적용.
- `null`·검증 실패 → 비례제어 폴백. `occluded=true` → 해당 슬롯 스킵(`reason:'occluded'`).
- `brain=undefined` · `llmAdvise=false` · `adviseCentering` 미구현/null 모든 경우 **순수 결정형으로 동작**(QA §2.4 통과).
- `AgentRuntime.adviseCentering`: 인라인 한글 시스템/유저 프롬프트 + 멀티모달(이미지 base64) + `chatJson`(json 모드·1회 재시도·실패 null) 재사용. **좌표를 생성하지 않고** 작은 제안값·판정만 반환.

장애 격리: 개별 슬롯 예외는 `run` 루프에서 흡수(경고 + `reason:'error'` 항목, 잡 중단 아님). 번호판 사유 코드: `no_plate`(초기 미검출)·`plate_lost`(중심정렬 중 소실)·`occluded`(자문 가림)·`error`(예외).

## 6. 출력 — `data/slot_ptz.json`

```jsonc
{
  "createdAt": "ISO8601",
  "items": [
    { "camIdx": 1, "presetIdx": 1, "slotId": "c1p1s1", "globalIdx": 1,
      "ptz": { "pan": 12.3, "tilt": -4.5, "zoom": 8.2 },
      "plateWidth": 0.198,   // 최종 번호판 정규화 가로폭
      "centered": true,      // pan/tilt 중심 수렴 여부
      "converged": true,     // zoom 폭 수렴 여부
      "reason": "no_plate"   // (옵셔널) 스킵·미수렴 사유. 정상이면 생략
    }
  ]
}
```

- `globalIdx`: `setup_artifact.globalIndex` 에서 `slotId` 역참조(없으면 `null`).
- 한 슬롯이 여러 프리셋에 ROI 를 가지면 `plateRoiByPreset` 키(`${camIdx}:${presetIdx}`)마다 1 항목으로 펼친다. 실 `setup_artifact.json` 기준 **26 항목** 펼침(QA §2.1 검증).
- 타입: `src/calibrate/types.ts` 의 `SlotPtzItem`·`SlotPtzArtifact`(SettingAgent 로컬).
- 쓰기: `slotPtzWriter.writeSlotPtz`(별도 파일 I/O, Repository 비오염).

### 미커버 (라이브 의존 — 후속 스모크 대상, 본 단계 미검증)
검증자가 명시한 한계 그대로:
1. **시뮬 실측 게인**: pan↔X·tilt↔Y 실제 부호·스케일은 시뮬 라이브에서 probe 변위가 명령 PTZ 대로 관측될 때만 검증 가능. 모킹은 규약·수렴 로직만 검증.
2. **gemma 자문 신뢰도**: `adviseCentering` 실 LLM 응답 품질은 실 모델 호출 필요. 본 단계는 fake OpenAI 호환 서버로 파싱·클램프·폴백 경로만 검증.
3. **실 PTZ 단위**: 본 기능은 시뮬 도(°) 단위 한정. 실 PTZ 매핑 미검증.
4. **브라우저 UI**: `web/{app.js,index.html}` 버튼·폴링·렌더는 정적 추가만 확인. 실제 브라우저 동작 미검증(헤드리스 라우트 계약만 app.inject 로 검증).
