## 1. 개요

### 무엇을
SettingAgent의 셋업 산출물(`setup_artifact.json`)을 사람이 눈으로 확인·검수하고, 카메라를 제어하기 위한 **웹 뷰어(SPA)**. SettingViewer(:13030)가 **정적 SPA + 카메라 프록시**를 서빙하고, ROI(/mapping)는 SettingAgent(:13020)의 GET /mapping 을 서버측 프록시한다. 브라우저는 카메라별 프리셋·주차면 ROI 오버레이·실시간 영상(≈3fps 스냅샷 폴링)·PTZ 제어를 한 화면에서 수행한다.

### 왜
- 셋업 결과(프리셋↔ROI↔전역 슬롯 인덱스)는 좌표 데이터라 텍스트만으로 검수가 어렵다 → 영상 위 오버레이로 시각 검수.
- 브라우저가 `:13020`(SettingAgent)·`:13100`(Unity)를 직접 호출하면 교차출처(CORS) 문제 + 카메라 주소가 프런트에 노출된다 → **단일 출처 프록시(SettingViewer :13030)**로 해소.

### 범위
- **포함(이번 라운드)**: 설계서 §10.1 **1~10단계** — 시뮬레이터 뷰어(프리셋/ROI/스트림/제어) + 실 PTZ 어댑터(`RealPtzSource`, Hucoms CGI, 모킹 단위검증).
- **제외**: §10.1 **11단계** RTSP→WebRTC 게이트웨이(후속). VPD/LPD 실시간 검출 박스·OCR(확장 훅만). 셋업 *실행* UI(기존 `/setup/run-from-map`이 담당).

## 2. 아키텍처

### 2.1 단일 출처 (SettingViewer :13030)
```
[브라우저 SPA] ── HTTP ──► [SettingViewer :13030]
  (정적 자산만)             ├─ GET  /viewer/*            정적 SPA(@fastify/static, SettingViewer/web/)
                           ├─ GET  /viewer/api/mapping  ─ 프록시 → SettingAgent :13020 GET /mapping
                           ├─ GET  /viewer/api/cameras  ─┐
                           ├─ GET  /viewer/api/snapshot  ├─ CameraSource 추상화
                           ├─ POST /viewer/api/move      ├─ (서버-서버)
                           ├─ POST /viewer/api/camera/login ┘
                           └─ GET  /viewer/api/health
                                      │
                          ┌───────────┼───────────────┐
              [SettingAgent :13020]  [Unity 시뮬레이터 :13100]  [Hucoms 실 PTZ :80]
               GET /mapping(산출물)   (SimulatorSource)         (RealPtzSource)
```
- 브라우저는 **오직 `:13030`** 만 호출 → CORS 불필요. 카메라 주소·자격증명은 **서버만 보유**. ROI(/mapping)는 SettingViewer 가 SettingAgent(:13020) 로 서버측 프록시.
- 라우트는 `/viewer` prefix로 격리 — SettingAgent 의 기존 셋업 라우트와 충돌 없음.

### 2.2 CameraSource 추상화
프록시 라우트는 `CameraSource` 인터페이스만 호출한다. 따라서 소스 구현(시뮬레이터/실 PTZ)을 교체해도 SPA·라우트는 무변경이다.

| 구현체 | kind | 영상 | 제어 | 단위 변환 |
|--------|------|------|------|-----------|
| `SimulatorSource` | `sim` | Unity `/req_img`(base64 JPEG) | Unity `/req_move` | 항등(뷰어=Unity 단위 동일) |
| `RealPtzSource` | `hucoms` | Hucoms snapshot CGI(바이너리 JPEG) | Hucoms PTZ CGI | 선형 매핑(뷰어 단위↔원시 정수) |

### 2.3 프록시 라우트 5개
`/viewer/api/cameras`(GET) · `/snapshot`(GET) · `/move`(POST) · `/camera/login`(POST) · `/health`(GET).
정적 서빙은 `@fastify/static`(prefix `/viewer/`, 와일드카드)이 담당하며, **API 라우트 등록 뒤에** 등록되어 우선순위가 보장된다(`/viewer/api/cameras`가 정적 파일보다 먼저 매칭).

## 3. 신규/수정 파일별 설명

### 3.1 신규 — 서버 (`SettingViewer/src/viewer/`)

#### `CameraSource.ts` — 계약(타입만, 구현체 없음)
- 타입: `Ptz{pan,tilt,zoom}`, `CameraList`(A타입 그대로, 중첩 presets 보존), `SnapshotResult{jpeg:Buffer, ptz}`, `SnapshotOpts{presetIdx?, ptz?, mode:'preset'|'manual'}`.
- 인터페이스 `CameraSource`:
  - `readonly kind: 'sim' | 'hucoms'` (onvif는 후속).
  - `listCameras()`, `snapshot(cam, opt)`, `move(cam, ptz)`.
  - `streamUrl?(cam)` — 11단계용 선택 시그니처(이번 미사용).
  - `login?(user, pass)` — 실 PTZ 소스만 구현(sim은 미구현 → 라우트가 400 반환).
  - `toNativePtz(viewerPtz)` / `fromNativePtz(native)` — 단위 변환.
- 인덱스는 전 구간 1-based(cam/preset), zoom은 뷰어 단위 1.0~36.0.

#### `SimulatorSource.ts` — 기존 CameraClient 래퍼 (`kind='sim'`)
- 생성자: `constructor(private camera: CameraClient)`.
- `snapshot(cam, opt)`: `mode==='manual'`이면 PTZ override를 `requestImage`에 동봉, `preset`이면 미동봉(PTZ 인자 `undefined`). 반환 PTZ는 캡처 응답(`captured.pan/tilt/zoom`)에서 채운다.
- `move(cam, ptz)` → `camera.move(cam, pan, tilt, zoom)`.
- `toNativePtz`/`fromNativePtz` = **항등**(같은 객체 반환).

#### `RealPtzSource.ts` — Hucoms CGI 어댑터 (`kind='hucoms'`)
- 생성자: `constructor(private cfg: CameraSourceConfig, private timeoutMs = 7000)`. `host`/`port`로 base URL 구성, `cfg.ptz` 범위가 있으면 우선, 없으면 기본 상수 폴백.
- 세션 상태: `private session: string | null`. `login(user, pass)`이 `login.cgi`(POST, form-urlencoded)로 인증 → `set-cookie`(없으면 `'authenticated'` 플래그)를 세션으로 보관. **자격증명은 보관·로그·응답에 노출하지 않음**(세션만 유지).
- `snapshot(cam, opt)`: manual 모드면 먼저 `move`로 PTZ 적용 후 snapshot CGI를 인증 세션(`cookie` 헤더)으로 GET → `arrayBuffer`→`Buffer`. PTZ는 `lastPtz`(마지막 명령값) 폴백 반환(현재-뷰 조회 CGI 미상).
- `move(cam, ptz)`: `toNativePtz`로 원시 정수 변환 → PTZ CGI GET. 성공 시 `lastPtz` 갱신.
- `toNativePtz`/`fromNativePtz`: `mapRange`(선형 + 0~1 클램프)로 뷰어 범위(pan `[-180,180]`/tilt `[-90,90]`/zoom `[1,36]`)↔원시 범위 매핑. **왕복 일치**(검증 §2.4 — 4 샘플 `toBeCloseTo` 통과).
- **상수 분리(파일 상단, 실측 보정 표시)**: `HUCOMS_LOGIN_PATH` / `HUCOMS_SNAPSHOT_PATH` / `HUCOMS_PTZ_PATH` / `HUCOMS_PTZ_PARAMS`, 기본 원시 범위 `HUCOMS_DEFAULT_*_RANGE`.

#### `sourceRegistry.ts` — `buildSourceRegistry(tools): Map<string, CameraSource>`
- `cameraSources` 미설정/빈 배열 → **하위호환**: 기존 단일 `camera` 설정으로 `SimulatorSource` 1개를 `id='sim'`으로 등록.
- 설정 있으면 항목별 kind 분기: `sim`→`baseUrl`만 교체한 `CameraClient`로 `SimulatorSource`, `hucoms`→`RealPtzSource`(camera의 `imageTimeoutMs` 주입).

#### `routes.ts` — `registerViewerRoutes(app, deps)`
- `ViewerDeps{ sources: Map<string,CameraSource>, viewer: ToolsConfig['viewer'] }`.
- zod 스키마: `CamerasQuery` · `SnapshotQuery`(`coerce`로 query 문자열→number) · `MoveBody` · `LoginBody`.
- `pickSource(id?)`: `source` 쿼리 지정 시 해당 소스, 미지정 시 **첫 소스**(삽입 순서).
- 라우트 등록 순서: API 5개 → `GET /viewer`(`/viewer/`로 302 redirect) → `@fastify/static`(와일드카드). **순서가 우선순위를 결정**한다.

## 4. 프록시 API 명세

공통: 입력 검증 실패(zod) → **400** `{error,detail}`. 소스 호출 실패(`CameraApiError` 등) → **502** `{error}`. (기존 server.ts 패턴 준수.)

| 메서드/경로 | 입력 | 처리 | 응답 |
|-------------|------|------|------|
| `GET /viewer/api/cameras` | query `source?` | `pickSource().listCameras()` | `CameraList` JSON |
| `GET /viewer/api/snapshot` | query `source?, cam:int>0, preset:int>0, mode:'preset'\|'manual', pan?, tilt?, zoom?, t?` | mode 분기. manual이면 `ptz{pan,tilt,zoom}` 동봉(zoom 클램프). | `Content-Type: image/jpeg` 바이너리 + 헤더 `X-PTZ-Pan/Tilt/Zoom`, `Cache-Control: no-store` |
| `POST /viewer/api/move` | body `source?, cam:int>0, pan, tilt, zoom` | `allowMove===false` → **403**; `controlToken` 설정 시 `X-Viewer-Token` 불일치 → **403**; 통과 시 `move(cam, ptz)`(zoom 클램프) | `{ ok: boolean }` |
| `POST /viewer/api/camera/login` | body `source(필수), user, pass` | 소스에 `login` 없으면(sim) **400** `{error:'login unsupported'}`; 있으면 `login(user,pass)`. **자격증명 미저장·응답/로그 미노출** | `{ ok: boolean }` |
| `GET /viewer/api/health` | — | — | `{ status:'ok', sources:[...id] }` |

- **에러 코드 요약**: 400(zod 실패 / source not found / login unsupported), 403(allowMove=false / controlToken 불일치), 502(소스 호출 실패 / login 예외).
- zoom 클램프: snapshot의 manual ptz.zoom과 move의 zoom 모두 `clampZoom(1~36)`. 검증서에서 99→36 확인.

### 명명 규약 (cam/preset ↔ camIdx ↔ cam_idx)
- 프런트 URL/body: `cam`/`preset` 으로 통일.
- 서버 핸들러: zod 파싱 후 `cam`/`preset`(지역 변수)로 소스에 전달. `CameraSource`의 인자명은 `cam`(number).
- Unity 변환: `cam_idx`/`preset_idx`로의 변환은 **`CameraClient` 내부 1곳에서만** 처리(프록시·소스는 1-based `cam`/`preset` 유지).

## 5. 설정 가이드

### `SettingViewer/config/viewer.config.json` → `viewer` 섹션
```jsonc
"viewer": {
  "enabled": true,      // false 면 /viewer/* 라우트 미등록(셋업 전용 운영)
  "allowMove": true,    // false 면 POST /viewer/api/move → 403 (LAN 조회 전용)
  "defaultFps": 3,      // 기본 스트림 fps(333ms 간격)
  "staticDir": "web",   // SPA 자산 폴더(resolve 로 절대경로화)
  "controlToken": ""    // 설정 시 move 요청에 X-Viewer-Token 일치 요구. 빈 값=미사용
}
```

### `cameraSources` (옵셔널 배열, 하위호환 핵심)
```jsonc
"cameraSources": [
  { "id": "sim",  "kind": "sim",    "baseUrl": "http://localhost:13100" },
  { "id": "cam1", "kind": "hucoms", "host": "192.168.0.153", "port": 80,
    "loginPath": "/cgi-bin/login.cgi",
    "snapshotUrl": "http://192.168.0.153/cgi-bin/snapshot.cgi",
    "ptz": { "panRange": [0, 36000], "tiltRange": [0, 9000], "zoomRange": [1, 36] } }
]
```
- **미설정 시**: `camera` 단일 설정으로 `SimulatorSource` 1개를 `id='sim'`으로 자동 등록 → 기존 동작 그대로 보존.
- **자격증명은 config·env에 두지 않음** — UI 입력 → 프록시 통과(세션 메모리). CGI 경로/원시 PTZ 범위는 실기기 확인 후 보정.

### 로더 하위호환 처리
- `viewer`는 `DEFAULT_TOOLS_CONFIG`에 포함되어 섹션 병합으로 처리(누락 시 기본값).
- `cameraSources`는 옵셔널 배열이라 섹션 병합 루프 밖에서 `raw.cameraSources !== undefined`일 때만 명시 대입.

## 6. 실행 / 사용법

```bash
# 워크스페이스 루트에서 1회 의존성 설치(@fastify/static·@parkagent/types 링크)
cd d:/Work/Parking3D/AgentVLA/ParkAgent && npm install

# 1) SettingAgent 기동(REST :13020, /mapping 제공)
cd SettingAgent && npm run start

# 2) SettingViewer 기동(:13030, 정적 SPA + 카메라 프록시 + /mapping 프록시)
cd SettingViewer && npm run start

# 브라우저 (SettingViewer)
http://localhost:13030/viewer/
```

### 시뮬레이터 사용 흐름
1. 소스 셀렉트에서 `sim` 선택(기본). 카메라/프리셋 셀렉트가 `/viewer/api/cameras`로 채워진다.
2. `[제어·모니터링]` 탭: `[시작]`으로 스냅샷 폴링(fps 입력) → 라이브 영상. **프리셋 모드**는 고정 위치, **수동 모드**는 PTZ 버튼/절대이동으로 자유 이동(`/req_img` PTZ override로 프리셋 복귀 방지).
3. `[주차면 검수]` 탭: 선택 프리셋의 slot ROI 오버레이(차량=시안/번호판=노랑, 라벨=globalIdx). 차량/번호판 토글.

### 실 카메라(Hucoms) 흐름
1. 소스 셀렉트에서 hucoms 소스 선택 → 로그인 박스 노출.
2. user/pass 입력 → `[로그인]`(`POST /viewer/api/camera/login`). 성공 시 세션이 서버 메모리에만 보관. **pass 필드는 즉시 클리어**, URL/로그에 평문 미노출.
3. 이후 snapshot/move가 그 세션으로 동작.

## 7. 실 PTZ(Hucoms) 실기기 보정 필요 항목 (미검증)

`RealPtzSource`의 다음 값은 **실기기 미확인 가정값**이다(192.168.0.153, HNR-2036LA 미연결 → 단위테스트는 CGI 모킹 전제). 실 장비 연결 후 실측 보정 필요:
