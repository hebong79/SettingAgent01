## 1. 기능 개요 (배경: bbox → floor ROI 전환)

기존 뷰어의 ROI 편집기는 **차량 bbox(vpd rect, `roiByPreset`)** 를 크기 조정(4변/모서리 리사이즈)하도록 결선되어 있었다. 그러나 실제 편집 니즈는 **주차면 바닥(floor) ROI** — 원근으로 인해 직사각형이 아닌 **4점 사변형(quad, `floorRoiByPreset`)** — 를 정밀하게 다듬는 것이었다.

이번 변경은 뷰어의 **편집 대상(edit target)** 을 차량 bbox 에서 floor ROI quad 로 전환하고, 사변형의 **4꼭짓점을 각각 독립적으로 드래그**하여 모양을 조정할 수 있게 한다.

핵심 성격:
- **전부 프런트엔드**(뷰어) 변경. MCP 두뇌/도구·REST·PTZ 캘리브레이션 루프에 개입하지 않는다.
- 순수 로직은 `web/core.js`(유닛테스트 대상), DOM/상태 결선은 `web/app.js`.
- **데이터 모델·저장 계약 무변경**: 이미 존재하던 `floorRoiByPreset`(4점 quad) 필드를 재사용한다. 신규 필드·신규 REST 스키마 없음.
- 차량 bbox 는 **표시·클릭 선택은 유지**하되 **편집(리사이즈)은 비활성화**된다.

### 2.1 `web/core.js` — 신규 순수 함수 3개 (`core.js:409~441`)

`updateSlotRoi`(차량 rect용) 바로 아래에 추가. 차량용 순수 함수(`resizeRect`/`clamp01Rect`/`updateSlotRoi`)는 **무변경·무삭제**.

quad 는 4×`{x,y}` 정규화(0..1) 배열, 인덱스 0..3 고정(순서 불변).

#### (1) `hitTestQuadVertex(quad, nx, ny, tolX, tolY) -> 0|1|2|3|null`
- i=0..3 순회, `Math.abs(nx-quad[i].x) <= tolX && Math.abs(ny-quad[i].y) <= tolY` 인 **첫 번째** 정점 index 반환. 없으면 `null`.
- 사각(rect) 허용범위: x/y 분리 tol(비대칭 허용). tol 경계값(`|dx| == tol`)은 `<=` 판정으로 히트.
- 방어: `!Array.isArray(quad) || quad.length < 4` → `null`.
- **DOM 미참조**: `tolX`/`tolY` 는 호출측(app.js)에서 `HANDLE_PX/overlay.width`, `HANDLE_PX/overlay.height` 로 산출해 주입한다(core 는 캔버스 크기를 모른다).

#### (2) `moveQuadVertex(quad, index, ndx, ndy) -> newQuad`
- `index` 정점만 `{x: clamp01(x+ndx), y: clamp01(y+ndy)}` 로 이동, 나머지 3정점은 **동일 객체 참조를 그대로 복사**(값·참조 모두 불변).
- `clamp01(v) = Math.min(1, Math.max(0, v))` — **점 단위 0..1 clamp**. rect 처럼 역전/최소폭 보정 없음(사변형은 정점 자유 이동이 요구사항).
- 방어: quad 부적합/`index` 0..3 밖 → `quad.slice()`(배열이면 얕은복사) 또는 원본 반환(변형 없음).
- **불변**: 반환 배열은 항상 새 배열(`!== 입력`). 원본 quad·직교 정점 객체는 불변.

#### (3) `updateSlotFloorRoi(artifact, slotId, key, quad) -> artifact`
- `updateSlotRoi` 미러. 대상 slot 의 `floorRoiByPreset[key]` 만 quad 로 **불변 교체**.
- 구현: `slots.map(s => s.slotId===slotId ? {...s, floorRoiByPreset:{...s.floorRoiByPreset,[key]:quad}} : s)` → `{...artifact, slots}`.
- **globalIndex 참조 동일성 유지**(`next.globalIndex === artifact.globalIndex`). 대상 슬롯의 차량 `roiByPreset`·타 preset key·타 슬롯 floor 모두 불변.

## 3. 동작 방식

```
슬롯 클릭 선택  →  floor 레이어(roi-floor) ON  →  선택 슬롯 floor quad 4정점에 흰/초록 핸들 렌더
    →  정점 위 mousedown(hitTestFloorVertex 히트)  →  drag 중 moveQuadVertex 로 해당 정점만 이동(0..1 clamp)
    →  updateSlotFloorRoi 로 mapping 불변 교체 + markDirty  →  mouseup 확정  →  저장(PUT /viewer/api/mapping)
```

- **개별 정점 이동**: `moveQuadVertex` 는 드래그 중인 index 정점만 이동하고 나머지 3정점은 참조까지 불변 → 사변형이 자유롭게 변형(직사각형 제약 없음).
- **clamp**: 각 좌표를 0..1 로 clamp. 캔버스 밖으로 정점이 벗어나지 않는다. rect 편집과 달리 최소폭·역전 보정은 **의도적으로 없음**(원근 사변형은 임의 형상 허용).
- **불변 갱신**: `updateSlotFloorRoi` 가 대상 슬롯만 새 객체로 교체하고 `globalIndex`·타 슬롯·차량 roi 는 참조 유지 → 리렌더 비용·회귀 위험 최소화.
- **hit-test 허용범위(tol)**: 핸들 픽셀 크기 `HANDLE_PX` 를 캔버스 폭/높이로 나눠 정규화 tol 로 변환. 캔버스 비율이 비대칭이어도 x/y 각각 올바른 픽셀 반경으로 판정.
- **편집 활성 조건**: floor 레이어 OFF 이거나 현재 preset 에 해당 슬롯의 floor quad 가 없으면 `hitTestFloorVertex` 가 `null` → 정점 드래그 자연 비활성.

## 6. 사용법

1. 뷰어 실행 후 캠/프리셋을 선택하고, 편집하려는 주차면 슬롯을 **클릭 선택**한다.
2. **floor 레이어(roi-floor) 체크박스를 ON** 한다.
3. 선택 슬롯의 floor quad 4꼭짓점에 흰 사각 + 초록(`#39ff14`) 핸들이 표시된다.
4. 원하는 꼭짓점 핸들을 **드래그**하면 그 정점만 이동하여 사변형이 변형된다(나머지 3정점 고정).
5. 편집 후 미저장 표시(markDirty)가 뜨며, 저장 버튼으로 PUT `/viewer/api/mapping` 을 통해 영속화한다.

- 차량 bbox(rect)는 선택 시 색/굵기만 강조되고 **리사이즈 핸들이 없다**(표시·선택만, 편집 불가).
- floor 레이어 OFF 이거나 현재 preset 에 floor quad 가 없는 슬롯은 정점 편집이 비활성이다.

## 8. orphan(미사용) 코드 처리 — A안(보존+주석) 확정

편집 대상 전환으로 app.js 내 차량 bbox 편집 경로가 미사용(orphan)이 되었다. 설계 §5 의 **A안(삭제하지 않고 보존, 주석만)** 을 확정 적용했다.

- **app.js 차량용 UI 함수**: `drawHandles`, `hitTestHandle`, `hitTestEdge` — 삭제하지 않고 각 상단에 `// [미사용] 차량 bbox 편집 비활성 — floor 정점 편집 전환으로 호출 제거됨(차량 bbox 재편집 복구 대비 보존).` 주석 부착.
- **app.js import**: `resizeRect`, `updateSlotRoi` 도 동일 방침(보존+주석). 설계 A안은 3함수만 명시했으나, 구현자가 이 import 도 같은 orphan 이므로 보존 번들로 일관 처리(구현 §5 편차. 재편집 복구 시 import 재추가 부담 제거가 근거).
- **core.js 차량용 순수 함수(`resizeRect`/`clamp01Rect`/`updateSlotRoi`)와 그 유닛테스트**: **무변경·무삭제**(사용자 "기존 bbox 편집 로직 손대지 않음" 지침).
- **A안 근거**: 사용자가 기존 bbox 편집 로직 보존을 명시했고, 이 함수들은 그 로직의 일부라 삭제(B안)가 취지와 충돌. `HANDLE_PX` 상수는 `drawQuadHandles`/`hitTestFloorVertex` 가 계속 사용하므로 유지.
- **트레이드오프(사실 명시)**: 데드코드가 잔존한다. typecheck exit 0·미사용 경고 없음으로 런타임/타입 무영향은 확인됐으나, 잠재적 혼동/회귀 위험은 §영향도(04) 참조.
