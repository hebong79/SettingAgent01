## 1. 배경 / 목적

기존 '결과 저장'·'결과 열기' 버튼은 **서버 `save/` 폴더**를 매개로 동작했다.

- 저장: `window.prompt`로 이름 입력 → `POST /capture/save {name, artifact}` (서버 파일 생성).
- 열기: `GET /capture/saves` 목록을 **모달**로 표시 → 항목 클릭 시 `GET /capture/saves/:name` 으로 불러와 화면 반영.

사용자 요청은 이 흐름을 **클라이언트(내 PC) 로컬 파일**을 다루는 **OS 네이티브 파일 대화상자**로 전환하는 것이다. 즉 저장/열기 위치가 서버가 아니라 사용자 로컬 파일시스템이 되도록 바꾼다.

이 변경은 전적으로 **SettingViewer 프론트엔드(브라우저 DOM + File System Access API)** 범위이며, MCP 두뇌/도구 경계 판단은 해당 없다. 하네스 컨벤션에 따라 **순수 로직은 `core.js`(vitest 검증)**, **DOM·File API 배선은 `app.js`**로 분리했다.

## 2. 변경/신규 함수 설명

### 2.1 신규 순수함수 (`web/core.js`, `core.d.ts` 선언 추가)

**`parseLoadedArtifact(text)` → `{ ok:true, artifact } | { ok:false, error }`** (core.js:737-751)

로컬에서 읽은 JSON 텍스트를 파싱하고 **최소 형태만** 검증한다.
- `JSON.parse` 실패 → `ok:false`("올바른 JSON 파일이 아닙니다(파싱 실패)").
- 최상위가 객체가 아님(배열/숫자/문자열/`null`) → `ok:false`("형식 오류: 최상위가 객체가 아닙니다").
- `presets`·`slots`·`globalIndex` 중 하나라도 배열이 아니면 → `ok:false`("형식 오류: presets/slots/globalIndex 배열이 필요합니다").
- 통과 시 파싱 객체를 그대로 `artifact`로 반환.

**검증 깊이 결정(게이트 Q3=최소 검증)**: 슬롯 내부 `roiByPreset`/`slotId` 등 깊은 검증은 하지 않는다. 근거 — (a) 브라우저에서 서버 zod(`artifactSchema.ts`)를 직접 import 불가, (b) 렌더 로직(`analyzeArtifact`/`drawRoiOverlay`)이 이미 필드 부재를 `?? []`/`?? {}`로 관용 처리, (c) 최소·단순 원칙. 목적은 "깨진/형태불량 입력으로 인한 렌더 크래시 방지".

**`defaultResultFilename(date = new Date())` → `string`** (core.js:753-760)

저장 대화상자의 제안 파일명을 생성한다. 형식 `setup_YYYYMMDD_HHmmss.json`(로컬시각, 월/일/시/분/초 2자리 제로패딩). `date` 주입으로 테스트 결정성 확보. 서버 `SaveStore.defaultSaveName`(`result_...`)과는 프리픽스·확장자 정책이 달라 재사용하지 않고 뷰어 전용으로 신설.

`core.d.ts`(248-251)에 두 함수 선언 추가 — 구현 시그니처와 1:1 일치, tsc EXIT 0 확인.

### 2.2 신규 배선 함수 (`web/app.js`, 비순수 — vitest 밖)

**`saveJsonToFile(suggestedName, text)` → `Promise<string|null>`** (app.js:419-444)
- `window.showSaveFilePicker` 지원 시: 네이티브 저장 대화상자(`suggestedName` + JSON types) → `createWritable().write().close()` → 저장 파일명(`handle.name`) 반환.
- 대화상자 취소(`AbortError`) → `null` 반환(조용히 무동작). 그 외 오류는 rethrow.
- 미지원(폴백): `Blob([text],{type:'application/json'})` + `URL.createObjectURL` + 임시 `<a download>` 클릭 + `revokeObjectURL`(다운로드 폴더 저장, 취소 개념 없음 → 항상 파일명 반환).

**`pickAndReadJsonFile()` → `Promise<string|null>`** (app.js:451-466)
- 리더 게이트 Q1 확정 = **`input[type=file]` 단일**(모든 브라우저 호환·최소). `showOpenFilePicker`는 쓰지 않음.
- 동적 `input[type=file] accept=".json,application/json"` → `change` 이벤트에서 `files[0].text()` resolve. 파일 부재 시 `resolve(null)`. 취소는 브라우저가 이벤트를 주지 않아 미해결(1회성 input, 부작용 없음).

**`applyLoadedMapping(artifact, label)` → `void`** (app.js:469-478)
- 기존 `openSaved`의 후반 렌더부 추출. `state.mapping = artifact` 주입 후 `state.roiHidden=false`/`state.selectedSlotId=null` 초기화 → `drawRoiOverlay`/`renderSlotList`/`renderSelectionInfo` 재사용 → `map-msg`="열림: {label}".

## 3. 수정 로직의 의도와 동작

### 3.1 `saveResult()` 재작성 (app.js:481-496)
1. `state.mapping` 없으면 `map-msg`="표시된 결과 없음 — 최종화 또는 결과 열기 후 저장하세요"(기존 메시지 유지) → return.
2. `name = defaultResultFilename()`, `text = JSON.stringify(state.mapping, null, 2)`.
3. `saveJsonToFile(name, text)` → `null`(취소) 조용히 return / 성공 시 `map-msg`="저장됨: {파일명}".
4. try/catch → 파일 오류 시 `map-msg`="저장 실패: {err}".

의도: 서버 POST 제거, `state.mapping`(SetupArtifact) shape 불변, 저장 매체만 로컬로 전환.

### 3.2 `openResult()` 재작성 (app.js:499-513)
1. `pickAndReadJsonFile()` → `null`(취소) 조용히 return.
2. `parseLoadedArtifact(text)` → `ok:false` 시 `map-msg`="열기 실패: {error}" return.
3. `applyLoadedMapping(r.artifact, '로컬 파일')` → 성공 시 `map-msg`="열림: 로컬 파일".
4. try/catch(File API 오류) → `map-msg`="열기 실패: {err}".

의도: 서버 목록 모달·GET 제거, 로컬 파일 텍스트를 파싱·검증한 뒤 기존 렌더 파이프라인 재사용.

### 3.3 고아 제거 (자기 변경으로 미사용화된 것만 — CLAUDE.md §3)
- `web/app.js`: `renderSaveList()`, `openSaved()` 함수 삭제 / `$('open-result-close')` 배선 삭제.
- `web/index.html`: `#open-result-modal`(`#open-result-list`·`#open-result-close` 포함) 마크업 삭제.
- `#result-save`/`#result-open` 버튼 요소는 **유지**(핸들러 내용만 교체), title 문구를 "save/" 표현 제거로 갱신:
  - `#result-save` title = "현재 결과를 로컬 JSON 파일로 저장"
  - `#result-open` title = "로컬 JSON 파일에서 결과 열기"
- 배선(app.js:1311-1312): `result-save→saveResult`, `result-open→openResult` 유지.

## 4. 입출력 / REST 계약

### 4.1 순수함수 계약
| 함수 | 입력 | 출력 |
|------|------|------|
| `parseLoadedArtifact` | `text: string` | `{ok:true, artifact:ArtifactLike}` \| `{ok:false, error:string}` |
| `defaultResultFilename` | `date?: Date` | `string` (`setup_YYYYMMDD_HHmmss.json`) |
| `saveJsonToFile` | `suggestedName, text` | `Promise<string\|null>`(취소=null) |
| `pickAndReadJsonFile` | — | `Promise<string\|null>`(취소=null) |
| `applyLoadedMapping` | `artifact, label` | `void` |

경계면 정합(QA §4 교차 확인): `parseLoadedArtifact`의 `{ok/error/artifact}` ↔ `openResult` 소비, `saveJsonToFile`/`pickAndReadJsonFile`의 `string|null` 취소 규약 ↔ 소비측 `== null` 가드 일치.

### 4.2 REST 계약 변화
- **뷰어→서버 REST 호출 제거**: `POST /capture/save`, `GET /capture/saves`, `GET /capture/saves/:name` 을 뷰어가 더 이상 호출하지 않음.
- **서버 라우트/스키마 자체는 무변경**(삭제 아님, 영향도 §참조). REST 계약 문서상 파괴적 변경 없음 — 소비자만 사라진 상태.

### 6.1 미검증 항목 (통과 위장 금지 — 그대로 기록)

아래 **DOM·File System Access API 배선 7종**은 jsdom 없이 vitest(node 환경) 밖이라 자동 검증 불가. **코드 레벨(정적)로만 확인됨. 실제 동작은 Chromium(localhost) 브라우저 육안 스모크 필요 — 미수행 상태 잔존**:

1. 결과 없음 → '결과 저장' → "표시된 결과 없음" 안내, 대화상자 안 뜸.
2. 결과 표시 후 저장 → 네이티브 저장 대화상자, 제안명 `setup_YYYYMMDD_HHmmss.json`, "저장됨: {파일명}".
3. 저장 대화상자 취소 → `AbortError`→null, 조용히 무동작.
4. '결과 열기' → `input[type=file]` → JSON 선택 → 3종 ROI 오버레이+슬롯목록, "열림: 로컬 파일".
5. 깨진/형태불량 JSON 선택 → "열기 실패: ...", 화면 유지(로직 자체는 vitest 커버, 배선만 수동).
6. 열기 대화상자 취소 → 무동작.
7. 폴백(비Chromium) → 저장=`<a download>` 다운로드 폴더, 열기=input 동일 경로.

자동화하려면 Playwright 등 브라우저 e2e 필요(현 범위 밖). 시나리오 2·4의 "결과 표시" 전제(finalize 또는 열기)는 별도 필요. 이 변경은 Unity/VPD/LPD/LPR/VLA REST 를 호출하지 않는 순수 로컬 파일 IO → 외부 서비스 기동 불필요.
