## 1. 기능 개요

SettingAgent 뷰어의 주차면(슬롯) 차량 ROI 박스 편집에서, 기존에는 **코너(nw/ne/sw/se) 핸들**만 드래그해 크기를 조절할 수 있었다. 이번 변경으로 박스의 **4개 변(n=상변, s=하변, w=좌변, e=우변)** 라인 근처를 클릭·드래그하면 **해당 한 변만** 이동하여 크기를 조절할 수 있다.

- 코너 드래그: 기존과 동일하게 대각 2변이 동시 이동.
- 변 드래그: 한 변만 직교 방향으로 이동(폭 또는 높이만 변화).
- 동작 조건: **선택된 슬롯**에 한정. 선택되지 않은 슬롯·빈 영역에서는 발동하지 않는다.
- 우선순위: 코너 히트가 변 히트보다 **우선**. 코너 근처에서는 항상 코너(2변) 리사이즈가 잡힌다.
- 순수 프론트엔드 캔버스 상호작용이며 MCP 두뇌·MCP 도구·REST 계약과 무관하다.

## 2. 변경 파일 · 함수 상세

### 2.1 `web/core.js` — `resizeRect(rect, handle, ndx, ndy)`

순수 함수. `switch (handle)` 블록의 기존 코너 case(`nw/ne/sw/se`) 바로 위에 변 case 4개를 삽입했다.

```js
switch (handle) {
  case 'n': top += ndy; break;   // 상변 상하 이동
  case 's': bottom += ndy; break; // 하변 상하 이동
  case 'w': left += ndx; break;   // 좌변 좌우 이동
  case 'e': right += ndx; break;  // 우변 좌우 이동
  case 'nw': left += ndx; top += ndy; break;
  case 'ne': right += ndx; top += ndy; break;
  case 'sw': left += ndx; bottom += ndy; break;
  case 'se': right += ndx; bottom += ndy; break;
  default: break;
}
```

- 시그니처·반환 shape(`{x,y,w,h}`)·이후 정규화(min/abs)·`clamp01Rect` 호출은 **전부 불변**.
- 변 뒤집힘(예: 상변을 하변 아래로 끌어 `top>bottom`)·경계 초과·최소폭 붕괴는 기존 정규화/클램프가 그대로 흡수한다. 추가 방어 코드 없음(4줄만 추가).

### 2.2 `web/app.js` — `hitTestEdge(nx, ny)` 신규

`hitTestHandle`(코너 히트) 바로 아래, `selectSlot` 위에 신규 배치. 반환은 `'n'|'s'|'e'|'w'|null`.

```js
function hitTestEdge(nx, ny) {
  if (!state.selectedSlotId || !state.mapping) return null;
  const key = presetKey(state.cam, state.preset);
  const slot = (state.mapping.slots ?? []).find((s) => s.slotId === state.selectedSlotId);
  const vrect = slot?.roiByPreset?.[key];
  if (!vrect) return null;
  const w = overlay.width || frame.clientWidth || 1;
  const h = overlay.height || frame.clientHeight || 1;
  const tol = HANDLE_PX / w;   // x 허용오차(정규화)
  const tolY = HANDLE_PX / h;  // y 허용오차(정규화)
  const left = vrect.x, right = vrect.x + vrect.w;
  const top = vrect.y,  bottom = vrect.y + vrect.h;
  const inX = nx >= left + tol && nx <= right - tol;   // 코너 구간 배제
  const inY = ny >= top + tolY && ny <= bottom - tolY; // 코너 구간 배제
  if (Math.abs(ny - top)    <= tolY && inX) return 'n';
  if (Math.abs(ny - bottom) <= tolY && inX) return 's';
  if (Math.abs(nx - left)   <= tol  && inY) return 'w';
  if (Math.abs(nx - right)  <= tol  && inY) return 'e';
  return null;
}
```

- 좌표계·분모·허용오차 산출은 `hitTestHandle`과 동일(`HANDLE_PX=8`, `tol=HANDLE_PX/w`, `tolY=HANDLE_PX/h`).
- 판정 원리: 각 변에 대해 **직교 거리 ≤ 허용오차** AND **평행축이 선분 범위 내(`inX`/`inY`)**.
- 코너 배제: `inX`/`inY`가 선분 양끝을 `tol`/`tolY`만큼 안쪽으로 좁혀, 코너 히트박스와 겹치는 구간을 변 판정에서 제외한다. 코너 우선순위를 보장하는 이중 안전장치(호출 순서 + 구간 배제).

### 2.3 `web/app.js` — `wireOverlayEditing` mousedown 결선

코너 우선, 미스 시 변 판정으로 결선을 확장했다(2줄 변경).

```js
let handle = hitTestHandle(nx, ny);          // 코너 우선
if (!handle) handle = hitTestEdge(nx, ny);   // 없으면 변
if (handle && state.selectedSlotId) {
  const slot = (state.mapping.slots ?? []).find((s) => s.slotId === state.selectedSlotId);
  const startRect = slot?.roiByPreset?.[key];
  if (startRect) {
    dragState = { handle, slotId: state.selectedSlotId, key, startRect: { ...startRect }, last: { nx, ny } };
    e.preventDefault();
    return;
  }
}
```

- `const handle =` → `let handle =` 변경 + `if (!handle) handle = hitTestEdge(...)` 한 줄 추가.
- `dragState` 구조(`{handle, slotId, key, startRect, last}`)·이후 슬롯 선택/해제·mousemove·mouseup 로직은 **무수정**. `handle` 문자열 도메인만 `n/s/e/w`로 넓어졌다.

## 3. 동작 방식

### 3.1 히트 우선순위 (mousedown)

```
mousedown(nx, ny)
  ├─ hitTestHandle(nx, ny)  → 코너(nw/ne/sw/se) 반경 내면 코너 반환
  ├─ (null이면) hitTestEdge(nx, ny) → 변(n/s/e/w) 라인 근처면 변 반환
  ├─ handle 있고 선택 슬롯 있으면 → dragState 시작(리사이즈)
  └─ handle 없으면 → hitTestSlots 로 슬롯 선택/해제
```

코너 근처는 (a) `hitTestHandle`이 먼저 호출되고, (b) `hitTestEdge`가 `inX/inY`로 코너 구간을 배제하므로, 두 단계 모두에서 코너가 우선된다.

### 3.2 resizeRect 수식

좌표계는 `left=x, top=y, right=x+w, bottom=y+h`. 델타 `(ndx, ndy)`는 마우스 이동 정규화 델타.

| handle | 적용 | 효과 |
|--------|------|------|
| `n`(상변) | `top += ndy` | 폭 불변, 상변만 상하 이동 → 높이 변화 |
| `s`(하변) | `bottom += ndy` | 폭 불변, 하변만 상하 이동 → 높이 변화 |
| `w`(좌변) | `left += ndx` | 높이 불변, 좌변만 좌우 이동 → 폭 변화 |
| `e`(우변) | `right += ndx` | 높이 불변, 우변만 좌우 이동 → 폭 변화 |

적용 후 공통 정규화:
```
x = min(left, right); y = min(top, bottom);
w = |right - left|;   h = |bottom - top|;
return clamp01Rect({x, y, w, h});
```
→ 뒤집힘은 `min/abs`가, 경계·최소폭(MIN=0.001)은 `clamp01Rect`가 흡수.

### 3.3 드래그 라이프사이클

- mousedown: `dragState` 시작(코너/변 공통 경로).
- mousemove: `resizeRect(cur, dragState.handle, ndx, ndy)` → `updateSlotRoi`로 `state.mapping` 갱신 → `drawRoiOverlay`로 실시간 미리보기. `dragState.last` 갱신.
- mouseup: `dragState=null`, `markDirty()`(미저장 표시), 재렌더. **디스크 영속화는 발생하지 않음** — 별도 "저장"(`PUT /mapping`) 시 반영.

## 4. 입출력 / 계약

- `resizeRect(rect: NormalizedRect, handle: string, ndx: number, ndy: number): NormalizedRect` — 시그니처 불변. `handle` 도메인이 코너 4종 → 코너 4종 + 변 4종으로 확장. 반환은 항상 `clamp01Rect` 통과 → `[0,1]` 정규화 rect.
- `updateSlotRoi`는 대상 슬롯의 `roiByPreset[key]`만 교체하고 slot 집합·`globalIndex`는 불변으로 유지.
- REST 계약 무변경: 편집 결과는 기존 `PUT /mapping`(전체 `state.mapping` 본문) 경로로만 영속화되며, 본 변경은 새 엔드포인트·본문 스키마를 추가하지 않는다.

## 6. 사용법 · 수동 동작확인 체크리스트

### 6.1 사용법 (뷰어)

1. 매핑을 로드하고 주차면(슬롯)을 클릭해 선택한다(선택 슬롯은 색상 강조).
2. 차량 ROI 박스의 **변 라인 근처**(상/하/좌/우)를 눌러 드래그 → 그 변만 이동해 크기 조절.
3. **코너 근처**를 드래그 → 기존대로 대각 2변 동시 이동.
4. 원하는 크기가 되면 "저장"(`PUT /mapping`)을 눌러 영속화. 저장 전에는 "편집됨(미저장)" 표시.

### 6.2 수동 동작확인 체크리스트 (브라우저 뷰어 필수)

- [ ] **변 히트**: 슬롯 선택 후 4개 변(n/s/w/e) 라인 근처 드래그 시 **해당 변만** 이동(크기 변경)하는가.
- [ ] **코너 우선순위**: 코너 근처에서는 변이 아닌 **코너**가 잡혀 대각 2변이 동시 이동하는가.
- [ ] **선택 슬롯 한정**: 미선택 슬롯/빈 영역에서 변 드래그가 발동하지 않는가.
- [ ] **얇은 박스**: 폭이 `2·tol` 미만인 박스에서 상/하변 무히트(코너로만 조작)로 동작하는가 — 무해 확인.
- [ ] **저장 반영**: 변 드래그 후 "저장" 시 `PUT /mapping`으로 반영되고, 재로드 후 크기가 유지되는가.
- [ ] **커서 피드백**: 이번 범위 미적용 — 커서 모양(ns/ew-resize) 변화 없음이 정상.

## 7. 참고 — 미적용 항목

- **커서 피드백**(n/s→`ns-resize`, e/w→`ew-resize`): 별도 mousemove hover 핸들러가 필요해 최소 변경 원칙(요구 3)에 따라 이번 범위에서 제외. 기능 완결에는 영향 없음. 후속으로 `overlay.style.cursor` 갱신 6~8줄 추가 가능.
