## 1. 개요 — 마스터 요구 원문과 충족 위치

> "점유판단후 점유 영역을 표시 할때 번호판 중심으로 상하좌우로 사다리꼴 모양의 4점을 만들어줘.
>  단, 1. 점유영역이 서로 겹치면 안된다. 2. 번호판중심으로 위가 좀 길어야 한다.
>  3. 가로/세로 모두 번호판영역의 가로 세로의 수평을 유지해야 한다. 4. 가로크기는 3.5~4배 정도 - 서로 겹치면 안됨.
>  가로크기 잡을때는 loop를 계속 돌아 크기를 자동으로 설정가능하게 만들어줘."

## 2. 신규 클래스/함수 상세 (`web/occupancyRegion.js` / `.d.ts`)

브라우저 순수 ESM 단일 소스 — DOM/fetch/state 비의존. 기하 프리미티브는 신규 발명하지 않고 `web/occupancy.js`(`clipByHalfPlane`·`convexIntersectionArea`·`polygonArea`)와 `web/core.js`(`quadCentroid`)를 재사용한다(`occupancyRegion.js:9-10` import 확인).

### 2-1. `plateAxes(quad)`

```ts
function plateAxes(quad: NormalizedPoint[] | null | undefined): PlateAxes | null
// PlateAxes = { c: NormalizedPoint; u: NormalizedPoint; v: NormalizedPoint; width: number }
```

번호판 quad `[TL, TR, BR, BL]` 에서 중심·단위 가로축(`û`)·단위 세로축(`v̂`)·기준폭(`W`)을 구한다.

**수학 모델(장축 채택, 2차 이터레이션에서 확정)**:

```
a = ((TR−TL) + (BR−BL)) / 2      // TL→TR 축 방향 평균
b = ((BL−TL) + (BR−TR)) / 2      // TL→BL 축 방향 평균
swapped = |b| > |a|              // 장축이 b 쪽이면 라벨이 순환 회전된 입력
û = normalize(swapped ? b : a) × usign
v̂ = normalize(swapped ? a : b) × flip
W = (swapped ? b : a).width      // 채택된 축의 평균 엣지 길이
```

- `flip = v̂raw.y < 0 ? -1 : 1` — v̂ 가 화면 위를 향하면 두 축 동시 반전(180° 반전 입력 대응, 핸디드니스 보존).
- `usign = swapped ? -flip : flip` — 두 기저를 맞바꿀 때 핸디드니스가 뒤집히므로 û 를 추가로 반전해 **입력 quad 자신의 핸디드니스를 복원**한다. 절대 부호(`cross(û,v̂)>0` 등)를 강제하지 않는 이유는 §3-2 참조.
- 퇴화 가드: `quadCentroid` 가 비4점/비수치를 이미 걸러 `null` 반환. `|a|<1e-9` 또는 `|b|<1e-9` 도 `null` — throw 없음(강등 철학).

### 2-2. `buildTrapezoid(axes, scale, cfg?)`

```ts
function buildTrapezoid(axes: PlateAxes, scale: number, cfg?: RegionConfig): NormalizedPoint[] // [TL',TR',BR',BL'] 미클램프
```

```
bw = scale · W                              // 아래 변 전체 폭
tw = topWidthRatio · bw                     // 위 변 전체 폭
Ct = C − upRatio · bw · v̂                  // 위 변 중심(−v̂ = 화면 위)
Cb = C + downRatio · bw · v̂                // 아래 변 중심
TL' = Ct − (tw/2)·û   TR' = Ct + (tw/2)·û
BL' = Cb − (bw/2)·û   BR' = Cb + (bw/2)·û
반환: [TL', TR', BR', BL']
```

위/아래 변이 `û` 에 평행(R3 가로), `Ct`·`Cb` 가 직선 `C + t·v̂` 위에 있어 중심축이 `v̂` 에 평행(R3 세로), `upRatio > downRatio` 로 위가 김(R2).

**현재 기본값(`topWidthRatio=1.0`)에서의 실제 형상**: `tw=bw` 가 되어 좌우 빗변(`TL'-BL'`, `TR'-BR'`)도 `Ct-Cb`(∥`v̂`)와 같은 벡터가 되므로, 이 사다리꼴은 이제 **평행사변형**이다(사다리꼴은 포함적 정의이므로 이름·함수명 `buildTrapezoid` 는 여전히 참). `topWidthRatio` 를 cfg 로 1.0 미만 값을 주면 다시 일반 사다리꼴(위가 좁음)로 복귀한다.

### 2-3. `clampToUnit(poly)`

```ts
function clampToUnit(poly: NormalizedPoint[]): NormalizedPoint[]
```

단위 정사각형(`x∈[0,1], y∈[0,1]`)을 이루는 4개 내향 반평면으로 `clipByHalfPlane` 을 4회 순차 적용한다. 결과는 볼록 0~8각형. 전부 잘리면(`[]`) 해당 인스턴스는 `computeOccupancyRegions` 결과에서 제외된다(면적 0 필터).

### 2-4. `computeOccupancyRegions(items, cfg?)`

```ts
function computeOccupancyRegions(
  items: Array<{ idx: number; quad: NormalizedPoint[] }>,
  cfg?: RegionConfig,
): { regions: OccupancyRegion[]; globalScale: number | null; overlapPairs: Array<[number, number]> }
```

입력 `items` 를 `idx` 오름차순 정렬 후 `plateAxes` 로 축 계산(퇴화분 제외) → §4 의 2단계 탐색으로 겹침 없는 배율을 구해 각 인스턴스의 사다리꼴(`clampToUnit` 적용)을 반환한다. 순수·무상태·무난수 — 같은 입력이면 항상 같은 출력(결정성).

## 4. 자동 배율 탐색 알고리즘 (2단계, `computeOccupancyRegions` L166-206)

### 4-1. 단조성 논거(이진탐색 정당성)

`buildTrapezoid` 의 모든 정점은 `C + s · (상수벡터)` 꼴(`bw`, `tw`, `Ct`, `Cb` 가 전부 `s` 에 선형 비례). 따라서 `s' < s` 이면 배율 `s'` 의 사다리꼴은 배율 `s` 사다리꼴의 **중심 `C` 기준 스타형 부분집합**이다. `clampToUnit`(단위 사각 클립)도 부분집합 관계를 보존한다. 두 볼록 부분집합 쌍의 교차 면적은 각 집합이 작아질수록 단조 비증가하므로, "겹침 존재 여부"라는 술어는 `s` 에 대해 단조(작을수록 안 겹침 방향)이며 이진탐색이 유효하다.

### 4-2. 1단계 — 전역 단일 배율 이진탐색

```
if !anyOverlap(4.0): s* = 4.0                        // 상한에서도 안 겹치면 즉시 채택
else if anyOverlap(3.5): 2단계로
else:
  lo=3.5(비겹침), hi=4.0(겹침)
  12회 반복: mid=(lo+hi)/2; anyOverlap(mid) ? hi=mid : lo=mid
  s* = max(floor(lo/0.05)·0.05, 3.5)                  // 0.05 그리드 내림(축소 방향 → 비겹침 유지)
```

코드 상 `BINARY_ITERS=12`(L28)로 고정. 모든 인스턴스가 같은 전역 배율을 쓰므로 "3.5~4배" 스펙에 균일하게 부합한다.

### 4-3. 2단계 — 인스턴스별 shrink-to-fit 폴백

하한(3.5)에서도 겹치는 경우에만 진입. 겹치는 쌍만 `shrinkFactor=0.9` 로 개별 축소(`minScale=1.0` 클램프)하고, **축소 후 재판정**해 잔존 겹침만 다음 라운드로 넘긴다(최대 20회). 1차 구현 대비 2차 검증에서 확인된 유일한 설계 이탈: 의사코드는 "축소 전 pairs 로 보고"였으나, 구현은 "축소 후 재판정한 잔존 pairs 만 보고"하도록 재배치했다 — "**잔존** 겹침 쌍 보고"라는 설계 의도를 정확히 구현하기 위함이며 반복 상한·축소율·결정성은 설계 그대로다.

### 4-4. 결정성·종료 보장

- **결정성**: 순수 함수(입력 quads+cfg → 출력), 난수·시각·전역 상태 없음. 쌍 순회는 `idx` 오름차순 정렬 고정 → 동일 입력이면 항상 동일 출력(단위테스트 T10 로 봉인).
- **종료 보장**: 1단계는 고정 12회 반복. 2단계는 고정 20회 상한 + 기하급수 축소(`×0.9`) + `minScale` 하한 클램프로 무한루프 불가.
- **복잡도**: 1단계 O(12·N²·k²), 2단계 O(20·N²) — N(프리셋당 plate 점유면) ≤ 약 30, k(클립 후 정점) ≤ 8 → 프레임당 마이크로초 대. 3fps 렌더 루프에서 무시 가능한 수준.

## 7. 알려진 제약

(a) **스트레스(밀착) 시 폭비 3.5 하한 미달 가능** — 합성 밀착 번호판 3개(중심간격 0.045, 판폭 0.030) 스트레스 케이스에서 2단계 shrink 폴백이 발동하면 인스턴스 배율이 `[1.674, 1.000, 1.674]` 까지 축소된다. R1(겹침 금지)을 지키기 위한 설계상 트레이드오프이며, 실검출 3프레임(cam1 p1/p2/p3, 14 인스턴스)에서는 전부 4.00 배율로 발생하지 않았다.

(b) **정규화 좌표 이방성(0.5625)에 의한 장축 채택 역전 임계** — `plateAxes` 는 정규화 좌표에서 동작하므로 픽셀 종횡비가 `정규화비 ≈ 픽셀비 × 0.5625` 로 압축된다. 역전(장단축 오판) 조건은 대략 **픽셀 종횡비 < 약 1.78:1**. 실검출 최악 마진은 p3 idx16 의 **2.94:1**(5차 실측)로 여유가 있으나, 극단 사선뷰에서 판이 더 압축되면 재발 가능 — 관찰 대상.

(c) **근본 원인 미착수** — `src/domain/geometry.ts:92 normalizeQuad` 는 LPD 폴리곤의 점 순서(`[TL,TR,BR,BL]`)를 정규화 없이 그대로 통과시킨다. 이번 수정은 `plateAxes` 국소 강건화(장축 채택)로 흡수했으나, 근본 해결(점 순서 자체를 규약대로 정규화)은 범위 밖 — 미착수.

(d) **bbox 폴백 미표시** — `source==='bbox'`(번호판 미검출) 행은 축의 원천인 plate quad 가 없어 사다리꼴을 생성하지 않는다. 기존 주황 원 + '번호미인식' 배지만 유지. 요구 원문이 "번호판 중심으로"이므로 bbox 에서 유사 사다리꼴을 발명하지 않았다(추측성 코드 배제).

**Goal 범위 밖 잔여 관찰**(5차 보고 §8): 사선뷰(p3)에서 사다리꼴이 화면 세로(`−v̂`)로만 뻗는데 실제 차체는 화면 위-왼쪽 대각으로 뻗어, 덮음률이 정면뷰(p1) 대비 낮고 `upRatio` 를 올리면 이웃 차량 침범이 동반되는 현상이 관측됐다(3차 스윕 §2-5). 이는 축 버그와 무관한 별개 사안이며, 대안(차체 방향 추정·cuboid 결합 등)은 이번 세션 범위 밖으로 남겨둔다.
