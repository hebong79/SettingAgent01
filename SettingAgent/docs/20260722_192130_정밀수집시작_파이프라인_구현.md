## 6. 검증 못 한 것 (성공으로 위장하지 않음)

검증 보고 §5, 구현 보고 §6 "확인 못 한 항목" 인용:

1. **브라우저 실렌더** — 완료 메시지 문자열, 노란 원의 실제 canvas 픽셀, `#roi-db` 체크→`drawDbCentering` 호출까지의 DOM 이벤트 경로는 **정적 문자열 검사**에 그쳤다. U9는 `drawDbCentering`의 좌표·스킵 로직만 실소스로 확정했다.
2. **대기시간의 벽시계 실측** — U4/U5는 fake sleep이라 `sleep(500)`이 실제로 500ms를 소비하는지는 검증하지 않는다(기본 `setTimeout` 구현을 신뢰). 라이브 실측(탐색→센터링 1.003s)이 이를 부분 보완하나, 슬롯 간 0.5s/1.0s 자체는 잡 상태 폴링(2s 주기)으로 직접 재지 못했다(구현 보고: 탐색 23슬롯 44초≈1.9s/슬롯, 센터링 23슬롯 175초≈7.6s/슬롯 — LPD/PTZ 왕복 포함이라 0.5s/1.0s와 직접 대조 불가).
3. **`data/setting.sqlite` 실 파일 DB 상 동작** — 유닛테스트는 전부 `:memory:`로 수행했다. 실 파일 DB의 잠금·트랜잭션 거동은 이번 변경이 건드리지 않은 기존 범위다.
4. **시뮬레이터 라이브 스모크 일부** — U12(`ground.enabled=false` 강등)는 유닛테스트로는 재현했으나(§2.2), 구현 단계 라이브 재현은 설정 변경이 필요해 수행하지 못했다.
5. **U13 부작용 격리의 라이브 실측** — 유닛테스트는 전 컬럼 `toEqual` 비교로 확정했으나, 구현자의 라이브 1회 실행에서는 `lpd/occupyRange/centered` 카운트 수준으로만 확인했다(구현 보고 §6-5).
