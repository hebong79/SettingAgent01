# Unity 서버 A타입 구현 명세 — 카메라/프리셋 목록 조회 API

- 작성일: 2026-06-24
- 대상: Unity 카메라 시뮬레이터 서버(`CWebCamCtrlServer`) 개발자
- 목적: SettingAgent 가 **camerapos.json 을 수동 작성하지 않고** 카메라/프리셋 목록을 런타임에 정확히
  받아오도록(A타입), Unity 서버에 "목록 조회 API" 1개를 추가한다.
- 근거: Unity 서버는 이미 내부에 카메라/프리셋 정의를 보유(`m_Cameras` 배열, 없는 인덱스 접근 시
  `m_Cameras[n]가 null` 오류를 반환). 즉 **데이터는 이미 있고, 노출만 하면 된다.**

---

## 1. 왜 A타입이 필요한가

- 현재 SettingAgent 는 프리셋 목록을 `config/camerapos.json`(수동) 또는 자동탐색(B, probing)으로 얻는다.
- B(probing)는 "없는 프리셋에도 영상을 주는" 서버에서 프리셋 경계가 불명확 → 부정확할 수 있다.
- **A(서버가 정확한 목록을 알려줌)** 가 가장 정확·안전. Unity 는 내부 정의를 가지므로 A 구현이 자연스럽다.

---

## 2. 추가할 엔드포인트 (필수 1개)

### `GET /cameras`
서버가 보유한 **모든 카메라와 각 카메라의 프리셋 목록(+PTZ)** 을 반환.

**요청**: 파라미터 없음. `GET /cameras`

**응답 200** (`application/json`):
```json
{
  "cameras": [
    {
      "camIdx": 1,
      "name": "Camera 1",
      "enabled": true,
      "presets": [
        { "presetIdx": 1, "label": "C1-P1", "pan": 30.0, "tilt": 12.0, "zoom": 2.0 },
        { "presetIdx": 2, "label": "C1-P2", "pan": 95.0, "tilt": 12.0, "zoom": 2.5 }
      ]
    },
    {
      "camIdx": 2,
      "name": "Camera 2",
      "enabled": true,
      "presets": [
        { "presetIdx": 1, "label": "C2-P1", "pan": 200.0, "tilt": 10.0, "zoom": 3.0 }
      ]
    }
  ]
}
```

**필드 의미**

| 필드 | 타입 | 필수 | 의미 |
|------|------|------|------|
| `cameras[]` | 배열 | ✅ | 서버가 보유한 카메라 목록(존재하는 것만) |
| `camIdx` | int | ✅ | 카메라 인덱스. **1-based**(`/req_img`,`/req_move` 의 `cam_idx` 와 동일 기준) |
| `name` | string | ⛳선택 | 카메라 표시명(없으면 SettingAgent 가 기본명 생성) |
| `enabled` | bool | ⛳선택 | 사용 여부. `false` 면 SettingAgent 가 셋업에서 제외(생략 시 true 취급) |
| `presets[]` | 배열 | ✅ | 그 카메라의 프리셋 목록(실제 존재하는 것만) |
| `presetIdx` | int | ✅ | 프리셋 인덱스. **1-based**(`/req_img` 의 `preset_idx` 와 동일) |
| `label` | string | ⛳선택 | 프리셋 표시명(없으면 `"C{cam}-P{preset}"` 자동) |
| `pan` | number | ⛳선택 | 프리셋 PTZ 의 pan(°). 없으면 SettingAgent 가 PTZ 미전송(서버 자체 프리셋 적용) |
| `tilt` | number | ⛳선택 | 프리셋 PTZ 의 tilt(°) |
| `zoom` | number | ⛳선택 | 프리셋 PTZ 의 zoom. **범위 1.0~36.0**(서버 사양과 동일) |

> **인덱스 1-based 통일**, **PTZ 단위는 `/req_move` 와 동일**, **zoom 1.0~36.0** — 기존 엔드포인트와 정합 필수.

**빈/오류 동작**
- 카메라가 없으면 `{"cameras": []}` (200). (오류 아님)
- 서버 내부 오류 시 기존 패턴대로 비-2xx + `{"code","error"}` (SettingAgent 가 예외 처리).

## 4. SettingAgent 소비 방식 (참고 — 서버 구현엔 불필요)

서버가 위 API 를 제공하면, SettingAgent 는 **A 공급자**를 추가해 그대로 사용한다.
(공급자 인터페이스 `PresetProvider` 는 이미 존재 — `src/setup/presetProvider.ts`)

```ts
// 예시: UnityPresetProvider (추가 예정)
class UnityPresetProvider implements PresetProvider {
  readonly name = 'unity-api';
  constructor(private baseUrl: string, private timeoutMs: number) {}
  async listViews(): Promise<CameraView[]> {
    const res = await fetchWithTimeout(`${this.baseUrl}/cameras`, { method: 'GET' }, this.timeoutMs);
    const body = await res.json() as { cameras: Array<{
      camIdx:number; name?:string; enabled?:boolean;
      presets: Array<{ presetIdx:number; label?:string; pan?:number; tilt?:number; zoom?:number }>;
    }> };
    const views: CameraView[] = [];
    for (const c of body.cameras) {
      if (c.enabled === false) continue;
      for (const p of c.presets) {
        views.push({
          camIdx: c.camIdx, presetIdx: p.presetIdx,
          label: p.label ?? `C${c.camIdx}-P${p.presetIdx}`,
          pan: p.pan, tilt: p.tilt, zoom: p.zoom,
        });
      }
    }
    return views;
  }
}
```

- 그 다음은 기존 파이프라인 재사용: `listViews()` → `writeCamerapos()` → `config/camerapos.json` 저장
  → 셋업은 파일(A)로 정확·빠르게 동작. (수동/B 와 동일 포맷 공유)
- SettingAgent 설정안(추가 예정): `tools.config` 에 공급자 선택
  ```json
  "presetProvider": { "type": "unity-api", "url": "http://localhost:13100" }
  ```
  (`type`: `camerapos`(기본) | `discovery`(B) | `unity-api`(A))

## 5. camerapos.json 과의 매핑

`GET /cameras` 응답 → SettingAgent 표준 `camerapos.json`(형식 A) 으로 1:1 변환된다.

| /cameras 응답 | camerapos.json |
|---------------|----------------|
| `camIdx` | `cam_id` |
| `presetIdx` | `preset_id` |
| `label` | `sname` |
| `pan`/`tilt`/`zoom` | `pan`/`tilt`/`zoom` |

> 즉 Unity 가 **camerapos 형식 그대로 반환**해도 무방하다(매핑 0). 단 신규 API 는 위 §2 의 명시적 스키마를 권장.

---

## 6. 검증 체크리스트 (Unity 구현 후)

- [ ] `GET /cameras` 200 + 위 스키마 JSON 반환
- [ ] `camIdx`/`presetIdx` **1-based**, `/req_img` 의 `cam_idx`/`preset_idx` 와 동일 매칭
- [ ] 각 프리셋의 `pan`/`tilt`/`zoom` 이 `/req_move` 와 동일 단위·범위(zoom 1~36)
- [ ] 존재하지 않는 카메라/프리셋은 응답에 **포함하지 않음**(probing 불필요해짐)
- [ ] 카메라 0대면 `{"cameras":[]}` 200
- [ ] (선택) `enabled:false` 카메라 표기

**확인 예시(curl)**
```bash
curl -s http://localhost:13100/cameras | jq .
```

---

## 7. 요약

- Unity 서버에 **`GET /cameras` 1개** 추가 → 카메라/프리셋(+PTZ) 정확한 목록 노출.
- SettingAgent 는 A 공급자로 이를 받아 `camerapos.json` 으로 저장 → 수동/B/A 가 한 포맷으로 통합, 셋업은 파일 기준.
- 인덱스 1-based, PTZ 단위·zoom 범위(1~36)를 기존 엔드포인트와 반드시 일치시킬 것.
