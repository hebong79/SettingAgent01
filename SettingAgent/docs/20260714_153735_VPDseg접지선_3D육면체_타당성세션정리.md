## 2. Q1 — bbox 만으로는 왜 안 되는가 (자유도)

지면에 선 차량 육면체 = 위치(X,Y) 2 + 요각(yaw) 1 + 치수(L,W,H) 3 = **6 DOF**.
축정렬 bbox = 숫자 4개 = **4 제약**. 게다가 bbox 는 **회전 정보를 원리적으로 하나도 담지 않는다** (정면 세단과 45° 돌아선 SUV 가 같은 bbox 를 만들 수 있다).

p/t/z 를 줘도 늘어나는 건 카메라 광선 방향뿐 → **부족분이 메워지지 않는다.**

지면 평면(z=0) 가정을 넣으면 이만큼만 풀린다:
- 접지 위치: 접지 픽셀 광선 ∩ 지면 → `t = −h / d_w.z`, `P = C + t·d_w`
- 높이 H: bbox 상단변 광선 ∩ (P 위 수직선)
- **yaw · L · W 는 여전히 미지수** ← 여기가 핵심 결손

## 3. Q2 — floor ROI 없이도 되는 이유 + 제공 수치 검토

### 3-1. 빠진 3 DOF 의 출처를 옮기면 된다

```
슬롯 폴리곤(정적·1회 실측) ──→ yaw
접지 픽셀 ──(지면 역투영)──→ 위치 (X, Y)
차종 prior / 마스크 폭 ────→ L, W
bbox 상단변 ─────────────→ H
```
주차장은 **"차는 슬롯 방향으로 정렬된다"** 는 강한 사전지식이 공짜다. floor ROI 는 이 정보를 차량마다·프레임마다 재추정하는 셈이고, **슬롯 폴리곤은 정적·1회·물리적으로 정확**하다. 게다가 슬롯 폴리곤은 **인덱스 매핑 때문에 어차피 그리고 있다**(한계비용 0).

### 3-2. 마스터가 준 수치의 오차 검토

**카메라 높이 ±0.25m — 괜찮다.** `d = h/tanθ` 이므로 높이 오차는 깊이에 **그대로 비례 전이**한다. h=5m ±0.25m → 깊이 5%. 게다가 랜덤이 아니라 **바이어스**라 기준점 1개로 통째 보정된다.

**진짜 위험은 tilt 다.** `Δd/d = 2Δθ / sin(2θ)`:

| 부각 θ | tilt 1° 오차 → 깊이 오차 | 40m 지점 |
|---|---|---|
| 7° | **14.4%** | 5.8m |
| 10° | 10.2% | 4.1m |
| 19° | 5.9% | 2.4m |

프리셋 tilt 가 6.8°/7.4°/18.8° 인데 PTZ 리드백 ±1° 는 흔하다 → **차 한 대가 통째로 옆 슬롯에 놓인다.**
→ 그래서 **p/t/z 로 지면 교차를 직접 푸는 경로는 얕은 tilt 에서 구조적으로 불안정**하고, 소실점/호모그래피 계열이 옳다. **(이미 프로젝트가 그렇게 하고 있다 — §6 참조)**

### 3-3. zoom→fov 환산은 탄젠트 기준이어야 한다

36배줌·수평 58° 는 **광각단** 값이다.
```
tan(hfov_z / 2) = tan(58°/2) / z     ← 옳음
hfov_z = 58 / z                       ← 틀림 (저배율에서 특히 벌어짐)
```
> 이 항목은 `20260714_123806_fovBaseV...md` 가 **이미 다뤘다**. 거기서 `camera.fov`(24.017°)를 zoom=1 base 로 오인해 f 가 **+39.7% 과대**했던 버그가 잡혔다.

### 3-4. floor ROI 자동 생성 경로 (실효성 순)

| 순위 | 방법 | 상태 |
|---|---|---|
| ① | **슬롯 폴리곤 역투영** (LLM 0회, 결정형) | 지면모델 있으면 즉시 가능 |
| ② | **VPD 세그멘테이션 마스크** | **이미 존재 — 본 세션에서 실증(§4)** |

### 6-1. 그래서 이 세션의 **진짜** 기여는 이것이다

`20260714_104645` 문서가 명시한 **최우선 잔여 위험**:

> **지면 위 평행이동 2 DOF** — 주차면 **정확히 한 칸**(2.5m, 이미지 70px) 어긋나도 `metricErr`·`tiltErrDeg`·`dDevRel`·`bearingDevDeg` **네 지표가 전부 침묵**한다. 점유가 옆 칸에 찍혀도 경고가 안 뜬다. **"2단계(빈 배경 median)" 없이는 해결 불가.**

**VPD seg 접지선이 이 구멍을 메울 독립 관측이다.**

- 원리: **차는 슬롯 안에 중심을 두고 선다.** 접지선을 지면모델로 역투영하면 슬롯 중앙에 앉아야 한다. ROI 가 한 칸 밀려 있으면 **접지 footprint 가 슬롯 경계를 계통적으로 타넘는다** → 여러 차량·여러 라운드로 누적하면 통계적으로 검출·보정 가능.
- 기존 지표가 전부 **닮음변환 불변량**이라 침묵했던 것과 달리, 접지선은 **지면 위 절대 앵커**다.
- **비용: REST 호출 1개.** "2단계 빈 배경 median"(신규 데이터 수집 필요)보다 압도적으로 싸다.

### 6-2. 로드맵 5단계 "SAM 접지선" → **SAM 불필요**

## 8. 미검증 (정직 목록)

1. **접지선 → 지면 역투영의 정량 오차 미측정.** 육안으로 "바퀴에 붙는다"만 확인했다. 지면모델로 역투영해 **슬롯 중앙과의 거리(m)** 를 재지 않았다. ← **다음 라운드 1순위**
2. **2 DOF 평행이동을 실제로 닫는지 미검증.** §6-1 은 **가설**이다. 인위적으로 ROI 를 한 칸 밀고 접지선 지표가 반응하는지 확인해야 한다(기존 문서 §5-2 가 쓴 것과 같은 반례 구성법).
3. **실카메라 프레임 미검증.** 검증에 쓴 3장은 전부 **시뮬레이터 렌더**다. 실카메라에서 마스크 품질·접지선이 어떨지는 모른다.
4. **가림 배제 후 잔존 접지선(23~40%)이 직선 적합에 충분한지 미검증.** 유효 열 비율만 셌고, 적합 잔차를 재지 않았다.
5. **프로덕션 코드 미수정.** `VpdClient.segment()` · `tools.config.json` 의 `segPath` 배선은 **LPD 세션 커밋 후**로 이월.
