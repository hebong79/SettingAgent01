# 'DB 보기'(#roi-db) 폴백→소스 전환 수정 — 차량/번호판/점유영역 미표시 버그

작성: 2026-07-21 · 문서화(documenter) · 선형 파이프라인(버그 수정)
변경 소스: `web/app.js` · `web/index.html`
신규 테스트: `test/dbViewSourceSwitch.test.ts`

---

## 1. 배경 · 증상

Live View 상단 'DB 보기'(`#roi-db`) 체크박스를 켜면 `slot_setup` 테이블에 데이터가 있는데도 차량(VPD)·번호판(LPD)·점유영역이 화면에 그려지지 않았다. 서버(`GET /capture/slots`, `SqliteStore.getSlotSetup`)는 정상이었고, 문제는 전부 프런트엔드 `web/app.js` 렌더 로직에 있었다.

## 2. 원인(진단 확정)

1. **소스 미로드**: `setTab()`(`app.js:3306`)이 `loadParkingSlots()`를 '정밀 수집' 탭 진입 시에만 호출한다. 기본 탭(제어·모니터링)에서 `#roi-db`를 켜도 `state.parkingSlotsByKey`가 `null`이라 그릴 원본 데이터 자체가 없었다.
2. **LPD 분기 순서 — 폴백이지 소스 전환이 아니었다**: `drawDetectOverlay`의 LPD 렌더가 `if (d) { 라이브 } else if (dbOn) { DB }` 순서였다. 라이브 검출(`d`)이 하나라도 있으면 DB 분기에 도달하지 못해, 'DB 보기'를 켜도 라이브가 있는 프리셋에서는 DB 값이 전혀 그려지지 않았다.
3. **점유영역도 동일 구조**: `drawOccupancyOverlay`가 `hasLive`(라이브 점유 유무)로 먼저 분기해, 라이브 점유가 하나라도 있으면 DB `occupyRange`를 건너뛰었다.

즉 'DB 보기' 체크박스는 이름과 달리 "라이브 없을 때만 보이는 폴백"으로 동작했고, 실제로 서버·DB 데이터가 표시되지 않는 사례가 흔했다.

## 3. 변경 내용

전부 `SettingAgent/web/` 프런트엔드. 서버·DB·타 에이전트 코드는 무변경.

### 3.1 `drawOccupancyOverlay` — `hasLive` 게이트 제거, 소스 전환으로 재구성
`hasLive`로 먼저 갈라 라이브가 없을 때만 DB를 보던 구조를 없애고, `#roi-db` 체크 여부를 최우선으로 검사한다. 체크돼 있으면 라이브 점유 유무와 무관하게 `drawDbOccupancy(ctx, rows)`를 호출하고 `return`한다(소스 전환 — 이중 렌더 없음). 체크 해제 상태에서만 기존 라이브 로직(`occ.spaces` 순회)이 실행된다.

### 3.2 `drawDbOccupancy` 신규 함수
`row.occupyRange`(정규화 폴리곤)를 `toPixelQuad`로 화면 좌표 변환 후 채우기+테두리로 렌더한다. 색상은 `rgba(255, 77, 77, 0.18)` 채움 / `rgba(255, 77, 77, 0.9)` 테두리로, 기존 라이브 점유 렌더와 동일 색상을 사용해 '점유 표시'라는 시각 언어를 유지한다. `occupyRange`가 `null`인 행은 skip(값을 지어내지 않음).

### 3.3 `drawDetectOverlay`의 LPD 분기 순서 전환
`if (d) {...} else if (dbOn) {...}` → `if (dbOn) {...} else if (d) {...}`로 뒤집었다. `#roi-db`가 체크돼 있으면 라이브 검출 유무와 무관하게 DB `row.lpd`를 노랑 quad로 그리고, 체크 해제 시에만 라이브(`d.vehicles`/`d.plates`)를 그린다. 같은 함수 내 VPD 분기(`drawDbVpd`)가 이미 이 순서였으므로, 이번 변경으로 VPD/LPD 두 분기가 동일한 "DB 우선" 규약으로 통일됐다.

### 3.4 `#roi-db` change 핸들러 — 비동기화 + 온디맨드 로드
```js
$('roi-db').addEventListener('change', async (e) => {
  if (e.target.checked && !state.parkingSlotsByKey) await loadParkingSlots();
  drawRoiOverlay();
});
```
체크 시점에 `state.parkingSlotsByKey`가 아직 없으면(정밀 수집 탭을 방문한 적이 없는 경우 등) 그 자리에서 1회 로드한 뒤 재렌더한다. 이제 어느 탭에서 체크박스를 켜도 동작한다.

### 3.5 `web/index.html` 툴팁 갱신
`#roi-db` 라벨의 `title`을 "라이브 검출이 없는 프리셋의 폴백" 설명에서 "차량(VPD)·번호판(LPD)·점유영역의 소스를 DB로 전환해 표시한다. DB에 값이 없는 항목(예: `vpd_bbox=null`)은 아무것도 그려지지 않는다"로 갱신해, 실제 동작(소스 전환 + null 미표시)과 문구를 일치시켰다.

## 4. 입출력 · REST 계약

서버 API(`GET /capture/slots`)와 응답 shape(`SlotSetupRow[]` — `vpd_bbox`/`lpd`/`occupyRange` 등)는 전혀 변경하지 않았다. 이번 수정은 이미 존재하는 응답 데이터를 클라이언트가 어떤 조건에서 캔버스에 그리느냐(렌더 분기)만 바꾼 것이다. REST 계약·DB 스키마 변경 없음.

## 5. 검증 결과 (있는 그대로)

- **신규 유닛테스트** `test/dbViewSourceSwitch.test.ts`: `viewerToggleGating.test.ts` 선례를 따라 DOM/렌더 계층을 소스 텍스트 회귀 가드로 검사(순수함수가 아니라 함수 본문 문자열을 파싱해 단언). 5케이스:
  1. VPD/LPD 모두 `dbOn`을 먼저 분기해 DB 소스를 그리고, 구 규약(`else if (dbOn)`)이 남아있지 않음을 확인.
  2. 점유영역이 `hasLive`와 무관하게 `#roi-db` 체크만으로 `drawDbOccupancy`를 호출함을 확인(구 `hasLive` 게이트 부재 확인 포함).
  3. `drawDbOccupancy`가 `row.occupyRange`를 `toPixelQuad`로 폴리곤 렌더함을 확인.
  4. `#roi-db` 토글 핸들러가 소스 미로드 시 `loadParkingSlots()`를 `await`한 뒤 재렌더함을 확인.
  5. `index.html` 툴팁이 '전환'을 설명하고 '폴백'이라는 단어를 더 이상 포함하지 않음을 확인.

## 6. 한계 · 정직 보고 (은닉 금지)

`slot_setup.vpd_bbox`는 실측된 17행 전부 `null`이다. 이는 "VPD 자동검출 금지" 정책(기하 폴백 없이 명시적 테스트 검출로만 채움)의 결과이며 이번 버그와 무관한 기존 상태다. 따라서 이번 수정 이후에도 **'DB 보기'를 켜도 차량(VPD) 청록 박스는 표시되지 않는다** — 이는 데이터 부재이며 코드 결함이 아니다. 차량 박스를 DB 소스로 보려면 'VPD 검출(테스트)' 버튼으로 검출해 DB에 반영하는 별도 경로가 필요하다(이번 작업 범위 밖).

## 7. 영향도 분석

요약본은 `_workspace/04_doc_impact.md` 참고. 핵심 결론: 서버·DB·ActionAgent/DMAgent에는 영향 없음. `#roi-db` 체크 시 `state.parkingSlotsByKey`가 채워지면서 `renderSlotList()`의 `finalized` 분기(좌측 주차면 목록 패널)가 제어·모니터링 탭에서도 활성화될 수 있는 부수효과가 있음(정밀 수집 탭에서는 원래 그랬음). 'DB 보기' 체크 상태에서는 라이브 검출 박스가 가려지는 것이 의도된 동작(소스 전환).
