## 1. `vpd:false` 인데 화면에 VPD 박스가 뜨는 이유 (모순 아님)

- `GET /health` 의 `vpd` 필드 = [server.ts:110](../src/api/server.ts) `deps.vpd.health()` → **VPD 서버의 `GET /health` 경로를 실시간 핑**한 결과([VpdClient.ts:55](../src/clients/VpdClient.ts)). **검출(detect/seg) 경로와 별개**다.
- 따라서 배지가 false 여도 화면 VPD 박스는 둘 중 하나:
  - **(a) 라이브 아님(캐시/DB)**: 예전 VPD 가동 시 잡힌 `state.detectByKey` 잔상(오버레이 유지 정책으로 미삭제), 또는 이전 최종화의 DB `slot_setup.vpd`(#roi-db 체크 시).
  - **(b) health 오탐**: VPD 서버가 `GET /health` 미구현/경로 상이로 핑만 실패하고 **검출은 정상**. (진단 플레이북의 "health 경로 ≠ 기능 경로" 패턴.) 박스가 현재 차 위치에 잘 맞으면 (b) 가능성 높음.
- **확인법**: `#roi-db` 체크 상태 / 라이브 검출 1회(`POST /capture/detect`) → 차량 나오면 검출 정상(=b, 배지 오탐), 502면 진짜 다운(=a) / VPD 서버가 실제 `GET /health` 구현하는지 점검.
- **결론**: 코드 모순 아님(배지=실시간 핑, 오버레이=캐시·DB). 단 배지가 detect 실태와 어긋날 수 있어 health 판정을 detect 경로 기준으로 보정하는 게 정확.

## 2. 슬롯 육면체 앞면중심 기준 LPD 박스 산출 — 가능

- 현재 번호판 예상([predictPlateRect](../src/capture/floorRoi.ts))은 **차량 bbox 앵커**(검출 의존): bbox 하단 72%·폭 40%·높이 18% 밴드. 차 미검출 슬롯(마지막 슬롯)은 이것도 못 만든다.
- **슬롯 육면체 앞면중심 앵커 = 검출 무관 산출 가능.** `project.ts` 기하가 전부 존재:
  - 슬롯 앞면 하단변 = 주차공간 앞선(지면, 슬롯 폴리곤+지면모델).
  - 번호판 = 앞-중앙, 번호판 높이(~0.4m), 번호판 실측 크기(한국 번호판 ≈0.52m×0.11m).
  - `backprojectToGround`(앞선 지면점) → `projectPointAtHeight`(번호판 높이) → `projectToPixel`(번호판 크기 코너) → **예상 LPD 박스(OBB, 방향·기울기 포함)**.
- **용도**: (1) 기하 폴백 LPD 박스(미검출 슬롯 구제) (2) 재검출 크롭 시드(예상 박스 주변 타이트 크롭→LPD 재실행→회복) (3) A2 센터링 prior.
- **정직한 한계**: 가정 박스(prior)이지 측정 아님. 앞-중앙·번호판높이·**전면주차·앞번호판 카메라향** 전제. **후면주차는 뒷번호판이 슬롯 뒤·다른 높이**라 앞면중심 가정이 틀림 → 앞/뒤 후보 2개 또는 주차방향 정보 필요. 번호판 높이 차종별 상이 → 크롭 여유 필요.
