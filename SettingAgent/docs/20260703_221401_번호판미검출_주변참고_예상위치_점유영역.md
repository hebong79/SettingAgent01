## 1. 배경 / 목적

정밀수집 파이프라인에서 주차면 점유영역(floor ROI)은 **번호판(LPD) 각도·위치를 권위 기준**으로
회전 사변형을 만든다(전제 문서). 그런데 다음 조합이 문제였다.

- **VPD(차량)는 검출**됐으나 **LPD(번호판)가 완전 실패**한 슬롯은 `plateQuad=null`.
- 이 경우 `buildPlateAnchoredQuad(vehicle, undefined)` 가 호출되어 **각도 0(축정렬)** +
  `predictPlateRect` **상수 위치**로 floor ROI 를 만든다.
- 한 줄 주차에서 이웃 차량들의 번호판은 원근에 따라 기울어 있는데, 이 슬롯만 **수평 사변형**이
  되어 행 정합이 깨진다(참조: `etc/주차면점유영역_01~03.jpg`).

**목적**: 같은 프리셋 그룹에서 **번호판을 가진 이웃 슬롯**의 각도·상대오프셋·상대크기를 산출해,
target 차량 bbox 에 적용한 **예상 번호판 quad** 를 합성하고 그것을 기준으로 floor ROI 를 생성한다.

핵심 설계 통찰: 예상 quad 를 **미리 합성해 기존 `plateQuad` 인자로 주입**하면
`buildPlateAnchoredQuad`/`resolveFloorPolygon`/`recognizeFloorRoi` **시그니처가 전부 무변경**이다.
신규 코드는 **순수 헬퍼 1개 + 두 호출부 배선 + 프롬프트 1문장**으로 국한된다.

## 2. 신규 헬퍼 — `estimatePlateQuadFromNeighbors`

파일: `src/capture/floorRoi.ts` (`predictPlateRect` 바로 아래에 배치, 신규 import 없음 —
기존 `plateAngleRad`·`projectedSpan`·로컬 `clamp01` 재사용).

### 2-1. 시그니처

```ts
/** 이웃(같은 프리셋 그룹·번호판 보유) 슬롯 1개. estimatePlateQuadFromNeighbors 입력. */
export interface PlateNeighbor {
  vehicle: NormalizedRect;
  plateQuad: NormalizedQuad;
}

export function estimatePlateQuadFromNeighbors(
  vehicle: NormalizedRect,
  neighbors: readonly PlateNeighbor[],
): NormalizedQuad | undefined
```

### 2-2. 알고리즘 (전부 결정형, 정규화 0~1 좌표 일관)

1. **최근접 이웃 1개 선택** (중앙값 아님):
   - `vehicle` 중심과 각 이웃 중심의 정규화 유클리드 **거리 제곱 argmin** 1개 채택.
   - `n.vehicle.w < 1e-6 || n.vehicle.h < 1e-6` 인 degenerate bbox 이웃은 스킵.
   - 유효 이웃 0개면 **`undefined` 반환**(호출측이 `predictPlateRect` 상수 폴백).
2. **이웃에서 산출** (최근접 이웃 `nv`, 그 번호판 quad `qN`):
   - 각도 `θ = plateAngleRad(qN)` (하단변 TL→TR 방향).
   - 번호판 중심 `pcN`(4점 평균)의 bbox 내 **상대오프셋**: `rx=(pcN.x−nv.x)/nv.w`, `ry=(pcN.y−nv.y)/nv.h`.
   - 로컬 단위축: `right=(cosθ, sinθ)`, `down=(−sinθ, cosθ)`.
   - **상대크기**: `rw=projectedSpan(qN, right)/nv.w`, `rh=projectedSpan(qN, down)/nv.h`.
3. **target 적용**: `pcT={vehicle.x+rx·w, vehicle.y+ry·h}`, 반폭 `hw=rw·w/2`, 반높이 `hh=rh·h/2`.
4. **quad 합성** (각 점 `clamp01`): `corner(sw,sh) = pcT + sw·hw·right + sh·hh·down`.
   - `TL=corner(−1,−1)`, `TR=corner(+1,−1)`, `BR=corner(+1,+1)`, `BL=corner(−1,+1)` → **TL,TR,BR,BL 규약**.

### 2-3. 라운드트립 정합 (무변경 근거)

합성 quad 를 `buildPlateAnchoredQuad`/`resolveFloorPolygon` 에 넘기면 내부의
`plateAngleRad`·`projectedSpan`·centroid 가 `θ·wp·hp·pc` 를 **정확히 복원**한다.
즉 합성 quad 는 "실측 번호판이 그 각도·위치·크기로 존재하는 것"과 동일하게 다뤄지며,
**빌더/리졸버는 무변경으로 각도를 추종**한다. (검증: `plateAngleRad(estimate) ≈ θ`, QA §3-1.)

## 3. 두 호출부 배선 (시그니처 무변경)

두 경로 모두 **트리거 = 번호판 완전 부재**로 좁힌다. 실측 plateQuad/rect 가 있으면 추정 자체를
실행하지 않는다(실측 1순위 자동 보장). rect-only 구DB 슬롯은 "번호판 rect 검출됨"이므로 제외.

### 3-1. `FloorRoiReviewer.review()` (`src/capture/FloorRoiReviewer.ts`)

- 트리거: `plate === undefined && plateQuad === undefined`.
- `candidates` 에서 `n !== s && n.presetKey === s.presetKey && n.plateQuad` 인 이웃을
  `PlateNeighbor[]` 로 수집 → `estimated = estimatePlateQuadFromNeighbors(vehicle, neighbors)`.
- `const effQuad = plateQuad ?? estimated;` 를 **양쪽에 동일 주입**:
  - `recognizeFloorRoi({..., ...(effQuad ? {plateQuad: effQuad} : {})})` (LLM 힌트)
  - `resolveFloorPolygon(polyRaw, vehicle, effQuad, plate)` (결정형 빌더/리졸버)
- `plate`(rect) 인자는 **실측만** — 예상은 rect 로 위조하지 않는다. 리졸버 `containTarget` 은
  `plate ?? quadBoundingRect(effQuad)` 로 예상 quad 를 포함강제(기존 로직).

### 3-2. `Finalizer.assemble()` (`src/capture/Finalizer.ts`)

- 트리거: `m.plateQuad == null && plateRect === undefined`.
- 같은 프리셋 그룹 `members`(byPreset 그룹이라 presetKey 필터 불필요)에서 `n !== m && n.plateQuad`
  이웃 수집 → `estimated = estimatePlateQuadFromNeighbors(rect, neighbors)`.
- 폴백: `base = floorByRef.get(clusterRef(m)) ?? buildPlateAnchoredQuad(rect, m.plateQuad ?? estimated ?? undefined)`.

## 4. 폴백 우선순위

두 호출부 공통 (코드로 보장):

```
effectivePlateQuad = 실측 plateQuad          // 1순위(있으면 즉시, 추정 미실행)
                  ?? 이웃추정 plateQuad        // 2순위(번호판 완전 부재 + 유효 이웃 ≥1)
                  ?? undefined                 // 3순위(이웃 0 또는 전부 degenerate)
// 빌더 내부: plateQuad ?? rectToQuad(predictPlateRect), θ = plateQuad ? angle : 0
```

- **실측 > 이웃추정 > `predictPlateRect(vehicle)` 상수(θ=0)**.
- 이웃 0개면 `estimated=undefined` → 기존 코드 경로와 **바이트 동일** 폴백(신규 분기 무진입).
- Reviewer: `plateQuad ?? estimated`. Finalizer: `m.plateQuad ?? estimated ?? undefined`.

## 5. 게이트(설계 결정) 3종 — 명시 기록

리더가 기본값 전부 승인. 각 결정의 근거를 문서에 남긴다.

1. **게이트1 — 이웃 선택 = 위치 최근접 1개** (median 아님).
   근거: 참조 이미지에서 번호판 각도는 행을 따라 **원근 그래디언트**로 점진 변화 →
   중앙값은 그래디언트를 뭉개 행 양끝 target 에 체계적 편향. 최근접은 그래디언트를 추종.
   각 슬롯 plateQuad 는 이미 라운드 집계 대표라 단일 이웃도 탈노이즈됨.
   (튜닝 여지: 리더가 "행 전체 평활 우선"이면 각도 median 으로 교체 가능 — §8 후속.)
2. **게이트2 — 트리거 = 번호판 완전 부재**(plateQuad·plate rect 둘 다 없음).
   rect-only 구DB 슬롯은 실측 rect 보유로 간주해 제외(현행 rectToQuad 경로 유지).
3. **게이트3 — 예상 quad 는 floor ROI 생성 전용**.
   `plateRoiByPreset`(아티팩트 번호판 ROI) 저장·`deconflictPolygons` 의 `plate` 보호 인자에는
   **미주입**. 실측 plateRect(`p.plateRect`, `m.plateX/Y/W/H` 유래)만 흘러간다 →
   **번호판 데이터 위조 방지**. Finalizer `plateRoiByPreset` 은 `p.plateRect` 존재 시에만 저장,
   `deconflictPolygons` 의 `plate` 인자도 `p.plateRect` 그대로.

> 7. 번호판이 주어지지 않으면(`(없음)`), **주변 차량들의 번호판 각도·위치를 시각적으로 참고**해
>    이 차량의 예상 번호판 위치를 잡고, 그 각도·좌우중앙 기준으로 점유영역을 그린다.

**이웃 0개(quad 도 `(없음)`)** 인 잔여 케이스에서 LLM 이 비전으로 주변을 참고하게 하는
소프트 가드레일 — 결정형이 못 채우는 유일한 빈틈만 커버.

## 8. 누락 명시 (통과 위장 금지)

  확인(실제 LLM 응답 품질 아님).

  (구조적으로 `plate` 인자 소스가 `p.plateRect` 단일이라 도달 경로 부재).
