## 1. 배경·목표

정밀수집 페이지의 검출 관련 버튼은 현재 `cap-actions` 툴바(index.html:178-185)에 **'LPD 검출'(`#cap-detect-run`)**·**'VPD 검출(테스트)'(`#cap-vpd-test`)** 두 개가 시작/정지/최종화/초기화 버튼과 뒤섞여 있다. 여기에 이미 백엔드가 완성됐지만 **UI에 미연결된** 세 번째 검지 방식(육면체 앞면중심 기준 LOOP 크롭줌 discovery)을 더해야 한다.

목표는 검출부를 **별도 라인(콤보박스 + 실행 버튼)** 으로 분리하고, 콤보박스로 **3가지 LPD 검지 모드**를 선택해 한 개의 실행 버튼으로 디스패치하는 것이다.

| 모드 | 라벨 | 동작 | 기존 자산 |
|---|---|---|---|
| (a) | 순수 LPD | 비-LPD 오버레이 off + `runLiveDetect(false)` (1회 진단) | `#cap-detect-run` 핸들러(app.js:3215) |
| (b) | 앞면중심 LOOP | discovery 배치 잡 실행·폴링(크롭줌 역계산) | `/discover/*` + `PlateDiscoveryJob` (백엔드 완성, **UI 미연결**) |
| (c) | VPD→LPD | `runLiveDetect(true)` (차량+육면체 복원 후 1회 검출) | `#cap-vpd-test` 핸들러(app.js:3225) |

**MCP 두뇌/도구 경계:** 세 모드 모두 **결정형(도구) 경로**다. (a)(c)는 단발 REST 검출, (b)는 순수 기하 크롭줌·아핀 역계산(LLM 미사용, 선행 문서 §1 확정). 이 작업은 UI 배선이라 새 판단 로직·LLM 개입이 전혀 없다.

## 2. 확정된 설계 결정

### 2-1. 기존 두 버튼 → 콤보박스로 **대체**(병존 아님)

**결정: `#cap-detect-run`·`#cap-vpd-test` 두 버튼을 제거하고 콤보박스 3모드로 흡수한다.**

근거:
- 모드 (a) ≡ 'LPD 검출' 버튼과 **기능 동일**(같은 토글 off + `runLiveDetect(false)`), 모드 (c) ≡ 'VPD 검출(테스트)' 버튼과 **기능 동일**(`runLiveDetect(true)`). 병존하면 같은 동작 진입점이 2개가 되어 중복(CLAUDE.md 규칙 2 최소주의 위배).
- 두 버튼 ID는 **바인딩(app.js:3215·3225) 외 참조가 0** (grep 확인). 핸들러 본문을 모드 디스패처로 이동하면 회귀 위험이 없다.
- 두 핸들러의 오버레이 토글 로직은 **그대로 보존**(삭제·개선 금지, CLAUDE.md 규칙 3) — 모드 함수 안으로 이사만 한다.

### 2-2. 레이아웃: `cap-actions` 툴바 아래 별도 라인

- `cap-actions` 툴바(index.html:178-185)에서 `#cap-detect-run`·`#cap-vpd-test` 두 `<button>` **줄만 삭제**. 시작/정지/최종화/검출·센터링초기화 4버튼은 **유지**.
- 툴바 닫는 `</div>`(185) 직후에 **신규 검지 라인 `<div>`** 삽입. 구성:
  - `<select id="lpd-mode">` — 3옵션(value=`lpd` / `discover` / `vpd`, 기본 선택 `lpd`).
  - `<button id="lpd-run">` — 실행.
  - `<progress id="disc-bar">` + `<span id="disc-label">` — 모드 (b) 진행 표시(다른 모드에선 idle 유지).
  - `<div id="disc-msg">` — 모드 (b) 결과·오류 메시지(found/total 요약).
- 스타일: 기존 `<select>`(index.html:93·95 등)가 쓰는 `.field` 라벨 패턴과 `cap-actions toolbar`·`cap-progress` 클래스를 그대로 재사용. **신규 CSS 규칙은 추가하지 않음**(기존 클래스로 충분).

### 2-3. discovery 대상: 전체 배치(`slotIds` 미전달)

- 실행 시 `POST /discover/ptz` body를 **`{}`(빈 객체)** 로 보낸다 → `PlateDiscoveryJob.start(undefined)` → `expandDiscoveryTargets`가 **`slot_setup` 중 `slot3d_front_center` 보유 전 슬롯**을 대상으로 펼침(PlateDiscoveryJob.ts:113, 선행 문서 §2-3). 현재 프리셋 한정하지 않음.
- 근거: discovery는 원거리·소형 번호판 미검출 슬롯을 **사후 일괄 구제**하는 배치 잡으로 설계됨(선행 문서 §1). 센터라이징(`calStart`, app.js:2195)도 동일하게 `body: '{}'` 전체 배치로 실행 → **일관성 유지**. 프리셋 필터 UI는 요청 범위 밖(최소주의).

### 2-4. 상호배타·버튼 비활성

- **프레임 폴 상호배타(불변식3)**: 라이브뷰(`frame` 엘리먼트)는 capture·centering·discovery가 공유한다. 신규 `startDiscFramePolling()`은 기존 패턴대로 **`stopCapFramePolling()`+`stopCalFramePolling()`+`stopLive()`** 를 호출하고, 역으로 `startCapFramePolling`/`startCalFramePolling`에 **`stopDiscFramePolling()` 한 줄씩 추가**해 3자 배타를 완성한다.
- **잡 중복 시작**: discovery가 이미 running이면 백엔드가 **409**(discoverRoutes.ts:30)로 거부 → `disc-msg`에 "이미 실행 중" 표기. (centering `calStart`와 동일한 백엔드-가드 의존 패턴, 프론트 선제 락 불필요.)
- **실행 버튼 disable**: 모드 (b) 폴링 중(`state==='running'`)에는 `#lpd-run`을 `disabled` 처리, 종료(done/error)에 복구. 순수 헬퍼 `discoverView(status)`가 반환(§3-3). (a)(c)는 단발 await이라 별도 disable 불필요.

### 3-3. 모드 (b) discovery 폴링 (web/app.js) — calPoll/calFrameTick 미러

**신규 상태 변수**(cal 대칭): `discPollTimer`, `prevDiscState='idle'`, `discFrameTimer`, `discFrameUrl`.

```js
async function discStart() {
  $('disc-msg').textContent = '';
  const res = await fetch('/discover/ptz', {
    method: 'POST', headers: { 'content-type': 'application/json' }, body: '{}',
  });
  const data = await res.json().catch(() => ({}));
  if (!res.ok) { $('disc-msg').textContent = `시작 실패: ${data.error ?? res.status}`; return; }
  $('disc-msg').textContent = data.total === 0
    ? '대상 0 — 앞면중심 보유 슬롯 없음(최종화 필요)'
    : `시작됨 (대상 ${data.total} 슬롯)`;
  discPoll();
}

async function discFrameTick() {           // calFrameTick 미러: /discover/frame, X-Disc-Cam 헤더
  try {
    const res = await fetch('/discover/frame', { cache: 'no-store' });
    if (!res.ok) return;
    const url = URL.createObjectURL(await res.blob());
    frame.src = url;
    if (frame.decode) await frame.decode().catch(() => {});
    if (discFrameUrl) URL.revokeObjectURL(discFrameUrl);
    discFrameUrl = url;
  } catch { /* ignore */ }
}
function startDiscFramePolling() {
  if (discFrameTimer) return;
  stopCapFramePolling(); stopCalFramePolling(); stopLive();   // 3자 상호배타
  discFrameTimer = setInterval(discFrameTick, 500);
  discFrameTick();
}
function stopDiscFramePolling() {
  if (discFrameTimer) { clearInterval(discFrameTimer); discFrameTimer = null; }
}

async function discPoll() {
  let status = null;
  try {
    const res = await fetch('/discover/status', { cache: 'no-store' });
    status = res.ok ? await res.json() : null;
  } catch { status = null; }
  const view = discoverView(status ?? {});      // 순수 헬퍼(core.js) — vitest 대상
  $('disc-bar').value = view.percent;
  $('disc-label').textContent = view.label;
  $('lpd-run').disabled = view.runDisabled;

  if (view.polling) startDiscFramePolling(); else stopDiscFramePolling();

  const st = status?.state ?? 'idle';
  if (prevDiscState === 'running' && st !== 'running') {   // running → 종료 전환 1회
    if (st === 'done') await renderDiscResult();
    else $('disc-msg').textContent = `종료(${st})`;
    startLive();
  }
  prevDiscState = st;

  const plan = pollPlan(st);                     // 'running'에서만 poll:true(core.js:124)
  if (discPollTimer) { clearTimeout(discPollTimer); discPollTimer = null; }
  if (plan.poll) discPollTimer = setTimeout(discPoll, plan.intervalMs);
}

async function renderDiscResult() {
  try {
    const res = await fetch('/discover/status', { cache: 'no-store' });   // 최종 카운트
    const s = res.ok ? await res.json() : {};
    $('disc-msg').textContent = `완료 — 발견 ${s.found ?? 0}/${s.total ?? 0} 슬롯 (slot_setup.lpd 갱신)`;
  } catch { /* ignore */ }
}
```

- 상호배타 완성: 기존 `startCapFramePolling`(app.js:1884)과 `startCalFramePolling`(app.js:2177)에 각각 `stopDiscFramePolling();` **한 줄 추가**.
- `pollPlan`은 discovery의 `'running'`에도 이미 대응(core.js:125). `'stopping'/'finalizing'` 상태가 discovery엔 없어 부작용 없음.

### 3-4. 순수 헬퍼 (web/core.js) — vitest 대상

`captureUiState`(core.js:136)·`pollPlan`(124)과 같은 자리에 순수 함수 1개 추가:

```js
/** /discover/status → UI 뷰(순수). 진행바·라벨·실행버튼 disable·프레임폴 여부. */
export function discoverView(status) {
  const st = status?.state ?? 'idle';
  const done = status?.done ?? 0;
  const total = status?.total ?? 0;
  const found = status?.found ?? 0;
  const percent = total > 0 ? Math.round((done / total) * 100) : 0;
  const running = st === 'running';
  return {
    percent,
    label: `${st} ${done}/${total} (found ${found})`,
    runDisabled: running,
    polling: running,
  };
}
```
- `web/core.d.ts`에 시그니처 1줄 추가(선언 파일 동기화, 기존 관례).

## 6. 미해결/가정

- **가정 A(대체)**: 두 기존 버튼을 콤보로 대체한다(§2-1). 병존을 원하면 리더 확인 필요 — 단 중복이라 대체 권장.
- **가정 B(전체 배치)**: discovery는 `slotIds` 미전달 전체 배치(§2-3). 현재 프리셋/선택 슬롯 한정을 원하면 별도 요청(범위 밖).
- **가정 C(결과 요약 소스)**: 완료 요약은 `/discover/status`의 `found/total`로 충분(§3-3 renderDiscResult). `/discover/result`(JSON 전문)까지 파싱해 슬롯별 상세를 보일지는 요청에 없어 미포함(최소주의). 필요 시 후속.
