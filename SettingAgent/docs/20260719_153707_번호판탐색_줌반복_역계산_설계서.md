## 0. 목표(관찰형 성공기준) 재확인

각 주차면 슬롯에 대해, **육면체 앞면중심(검출 무관 기하)을 조준 기준점**으로 번호판을 탐색해 **원본(와이드) 프레임에서의 LPD 박스(정규화 OBB quad)** 를 확보한다.

1. 원본 프레임에서 앞면중심 주변 번호판 탐지 시도.
2. 못 찾으면 앞면중심 기준 **확대(줌)** 하여 재탐지 — 반복(줌 스텝·상한).
3. 줌 상태에서 찾으면 **역계산**하여 원본 프레임 LPD 박스로 환원.
4. 검지율 100% 도전. 못 찾은 슬롯은 **정직 리포트**(위장 성공 금지).

마스터 확정 제약: **셋팅 전용(실시간 불필요)** · **결정론 우선(LLM 미사용)** · **실 PTZ 전이 예정** · **기존 자산 최대 재사용**.

## 1. 핵심 결정 — 2단(tier) 확대 구조

### 결정 1. 디지털 크롭-줌을 1차 코어로, 광학 PTZ 줌을 2차 에스컬레이션으로 확정

| | (a) 디지털 크롭-줌 **[1차 코어 — 이번 구현 범위]** | (b) 광학 PTZ 줌 **[2차 에스컬레이션 — 이번엔 설계만·구현 후속]** |
|---|---|---|
| 방식 | 원본 프레임에서 앞면중심 주변을 크롭·업스케일(sharp) → 그 크롭 JPEG 에 LPD | 카메라를 앞면중심으로 pan/tilt + zoom → 새 프레임 캡처 → LPD (PlatePtz 폐루프 재사용) |
| 카메라 이동 | **없음**(원본 프레임 재사용) | 있음(requestImage PTZ override) |
| 역계산 | **아핀**(크롭 offset·scale) — 정확·무손실·저비용 | 지면모델 backproject→project — G_i(줌 지면모델) 유도 필요·복잡 |
| 획득 해상도 | 원본에 담긴 픽셀이 상한(업스케일은 보간) | **실 해상도 추가 확보** |
| 검증 난도 | vitest 왕복 파리티(순수 아핀) — 쉬움 | sharp 파리티 + 지면모델 왕복 — 어려움 |
| 전이성 | 순수 이미지 연산 — 실카 그대로 | PTZ 명령·게인 필요(카메라별 상수) |

**근거(코드 확인):**
- `sharp ^0.35.3` 이 이미 의존성에 존재([package.json]) → 크롭·업스케일 즉시 가능. 신규 라이브러리 0.
- `LpdClient.detect(image)` 는 내부에서 `readJpegSize(image)` 로 **입력 이미지 자체의 해상도로 정규화**한다([src/clients/LpdClient.ts:57,80]). 즉 크롭 JPEG 를 넣으면 반환 quad 는 **크롭 정규화 좌표**로 나온다 → 아핀 역매핑만 하면 원본 좌표 복원. 업스케일 배율은 **역매핑 식에 등장하지 않는다**(정규화가 흡수) — 배율은 오직 "검출 가능성"에만 영향.
- `slot3dFrontCenter`(정규화 {x,y})가 **검출 무관·전 슬롯 기하 산출**로 이미 DB 저장([src/capture/Finalizer.ts:271], `SlotSetupView.slot3dFrontCenter`) → 조준점이 준비돼 있음.

**2단 구조의 논리(마스터 지시 반영):**
- 원본에 번호판이 **작게라도 있으면** 크롭-업스케일이 **정확히 구제**(back-calc 은 오차 0 아핀). 대부분의 원거리 슬롯이 여기 해당.
- **진짜 저해상도**(원본에 번호판 픽셀이 소멸)면 업스케일은 정보를 못 만든다 → 광학 PTZ 로만 실해상도 확보 가능. 이 경우만 2차로 넘긴다.
- ∴ **1차 디지털만으로 Goal 1차 달성**(원본에 존재하는 번호판 전량 회복). 2차 광학은 잔여 꼬리 전용.

### 결정 2. 조준·크롭은 지면모델 재계산 없이 저장된 앞면중심만 사용

디지털 tier 는 `slot3dFrontCenter`(정규화 점) + 원본 프레임 + LPD + sharp 만 쓴다. **discovery 시점에 지면모델을 다시 풀지 않는다**(finalize 때 이미 산출·저장). → 결정론·저의존·실카 전이 그대로. 지면모델은 광학 tier(2차)에서만 필요.

## 2. 역계산(back-calculation) 수식 — 최대 난제

### 2-1. 디지털 크롭(1차, 정확·이번 구현) — 아핀

기호: 원본 프레임 정규화 좌표계 `[0,1]²`. 크롭 창 `W = {x0, y0, w0, h0}`(원본 정규화). 크롭 JPEG 에서 LPD 가 돌려준 코너 `q = {qx, qy}`(크롭 정규화 `[0,1]²`).

```
orig_x = x0 + qx · w0
orig_y = y0 + qy · h0        (각 코너에 독립 적용 → 원본 OBB quad 복원)
```

- **업스케일 배율(scale) 부재 근거**: 크롭을 `outLongPx` 로 확대해도 LPD 는 확대된 픽셀 크기로 정규화하므로 `q` 는 크롭 정규화 그대로다. 역매핑은 창 offset·size(`x0,y0,w0,h0`)만 필요. (배율은 검출 성패에만 영향 — 식 불변.)
- **정확성**: 부동소수 왕복 오차만(≈1e-12). vitest 왕복 파리티로 봉인(§7 T-1).
- **아핀 소유 함수**: `backmapQuad(cropQuad, W)` (신규 `cropZoom.ts`, 순수).

### 2-2. 광학 PTZ(2차, 후속) — 지면모델 왕복 + G_i 유도 **[가정·검증법 명시, 이번엔 구현 안 함]**

```
줌상태 검출픽셀 p_i  --backprojectToGround(·, G_i)-->  지면점 X
지면점 X            --projectToPixel(X, G0)-------->  원본 프레임 픽셀 p0
```
- `G0` = 프리셋 지면모델(finalize 산출, `estimateGroundModels`). `G_i` = 임의 PTZ(pan/tilt/zoom)에서의 지면모델.
- **미해결(정직히 명시)**: `G_i` 를 어떻게 얻는가.
  - **가정 후보 A**: `G0` + PTZ delta 로 유도 — pan/tilt 는 카메라 회전(외부 R), zoom 은 f 스케일(`focalFromZoom(zoom, fovBaseV, …)` 재사용). 지면 평면 `(n,d)` 를 회전 R 로 이동시켜 `G_i` 합성.
  - **가정 후보 B**: 줌 프레임에서 주차면 재검출→`estimateGroundModels` 재추정(느리지만 self-contained).
- **검증법(구현 시)**: 리더 sharp 파리티 — 동일 지면점을 `G0` 로 원본에 투영한 픽셀 vs `G_i` 로 줌 프레임 투영 후 역계산한 픽셀의 좌표 오차(합성 데이터, 목표 오차 < 수 px). 왕복 `projectToPixel(backprojectToGround(p, G_i), G_i) ≈ p` 도 vitest 로 봉인.
- **결정**: `G_i` 유도가 불확실하므로 광학 tier 는 **인터페이스 심(seam)만 남기고 후속으로 미룬다**. 디지털 tier 로 Goal 1차 달성(§1). 광학은 "디지털이 못 구제한 슬롯" 리포트에만 표식(`reason: 'needs_optical'`).

## 3. 탐색 루프 구조

### 3-1. 상태 전이(슬롯 1건, `PlateDiscovery.discoverSlot`)

```
[진입] anchor = slot3dFrontCenter (정규화). anchor 없음 → 즉시 실패(reason:'no_anchor').
        원본 프레임 1회 캡처 = requestImage(cam, preset, resolvePresetPtz)   ← detectPipeline 규약(echo 0/0/1 불신)

Tier 0 (full):  원본 프레임 전체에 LPD → pickNearestPlate(anchor) → matchRadius 이내면
                → found(tier:'full', step:0, box = 검출 quad 그대로, 역계산 불요)

Tier 1 (crop):  step=1..maxSteps 반복:
                  frac_k = frac0 · shrink^(k-1)                     ← 창이 매 스텝 축소(= 실효 줌 상승)
                  W_k = computeCropWindow(anchor, frac_k, aspect)   ← anchor 중심, [0,1] 클램프
                  cropJpeg = cropAndUpscale(원본JPEG, W_k, outLongPx)  ← sharp extract+resize (원본 재사용, 카메라 무이동)
                  plates = LPD(cropJpeg)                            ← 크롭 정규화 좌표
                  pick = pickNearestPlate(plates, anchor_in_crop)   ← anchor 를 크롭 좌표로 환산해 최근접
                  검출 있으면 → box = backmapQuad(pick.quad, W_k)   ← §2-1 아핀 역계산
                             → found(tier:'crop', step:k)
                  없으면 계속. frac_k < minFrac 또는 k=maxSteps 도달 시 종료.

Tier 2 (optical, 후속): 디지털 실패 → reason:'needs_optical' 리포트(이번엔 실제 광학 미수행).
                        [구현 시] PlatePtz.centerOnPlate→zoomToPlateWidth 후 §2-2 역계산.

[실패] found 없음 → not_found(reason: 'no_plate' | 'needs_optical' | 'no_anchor').
```

### 3-2. PlatePtz 재사용 범위 확정

- **디지털 tier(1차)는 PlatePtz 를 호출하지 않는다.** PlatePtz 는 "카메라를 실제로 움직여 폐루프 센터링·줌"하는 광학 도구다. 디지털 크롭은 카메라 무이동이라 성격이 다르다. → 신규 **얇은** 탐색 루프(`PlateDiscovery`)를 만든다.
- **재사용하는 것**: `pickNearestPlate`([controlMath.ts:16], 최근접 후보 선택), `plateCenterError`(불요), `resolvePresetPtz`([detectPipeline.ts:212]), `LpdClient.detect`, `CameraClient.requestImage`, `quadBoundingRect`, `slot3dFrontCenter`.
- **광학 tier(2차)에서만** PlatePtz.centerOnPlate/zoomToPlateWidth 를 **그대로 호출**(수정 없음)하고, 그 최종 PTZ 로 역계산.
- 이 분리는 코드베이스 컨벤션과 정합: PlatePtz(루프) ↔ PtzCalibrator(잡) 분리를 그대로 미러 → `PlateDiscovery`(루프) ↔ `PlateDiscoveryJob`(잡).

### 3-3. 상수(초기값, config 스키마 확장 없이 opts 기본값)

| 상수 | 초기값 | 근거 |
|---|---|---|
| `frac0`(초기 창 폭 비율) | 0.40 | 앞면중심 주변 넉넉히 포함(번호판은 앞면중심보다 약간 아래 — §6-2). QA 미세조정. |
| `shrink`(스텝 축소비) | 0.6 | 매 스텝 실효 줌 ≈1.67배. 큰 점프로 대상을 창 밖으로 밀지 않게. |
| `minFrac`(최소 창 폭) | 0.05 | 실효 줌 ≈20배 상한(원본 대비). 그 이하는 업스케일 이득 소멸. |
| `maxSteps` | 5 | 0.40·0.6^4 ≈ 0.05 도달. |
| `outLongPx`(업스케일 목표 장변) | 원본 장변(예 1920) | 크롭을 원본 해상도로 되돌려 LPD 입력 크기 일관. |
| `matchRadiusNorm`(full tier 게이트) | 0.15 | 번호판이 앞면중심보다 약간 아래(≈0.05~0.1) + 여유. 이웃 슬롯(간격 ~0.15) 오선정 차단. |

## 4. 번호판 후보 선택

- `pickNearestPlate(plates, priorRect)`([controlMath.ts:16]) 재사용. prior = anchor(앞면중심) 중심의 0폭 rect.
- **Tier 0(full)**: 원본 좌표에서 anchor 로부터 `matchRadiusNorm` 초과 후보 기각(이웃 번호판 갈아타기 차단 — PlatePtz `captureAndDetect` 반경 기각과 동일 철학).
- **Tier 1(crop)**: 크롭 창 자체가 공간 게이트다(창 밖 번호판은 애초에 크롭에 안 들어옴). 창 안에서 anchor(크롭 좌표 환산) 최근접 1개 채택. 창이 축소될수록 이웃은 자연 배제.
- anchor 를 크롭 좌표로 환산: `anchor_in_crop = { (ax - x0)/w0, (ay - y0)/h0 }`(§2-1 역).

## 5. 산출물 계약

### 5-1. 슬롯별 결과 `PlateDiscoveryItem`(신규 타입, `calibrate/types.ts` 확장)

```ts
interface PlateDiscoveryItem {
  camIdx: number; presetIdx: number;
  slotId: string; globalIdx: number | null;
  found: boolean;
  /** 원본(프리셋) 프레임 좌표계 LPD OBB(정규화 4점). 못 찾으면 null. */
  lpdOrig: NormalizedQuad | null;
  tier: 'full' | 'crop' | 'optical';
  step: number;                 // full=0, crop=1..maxSteps
  cropWindow?: NormalizedRect;  // crop tier 만(감사·재현용)
  ptz?: Ptz;                    // optical tier 만(후속)
  confidence: number;
  reason?: 'no_anchor' | 'no_plate' | 'needs_optical';
}
```
- JSON 정본: `data/plate_discovery.json`(`{ createdAt, items }`) — writer 는 `writeSlotPtz` 패턴(stringify5, mkdir) 재사용/미러.

### 5-2. DB 반영 — slot_setup.lpd 를 원본 좌표로 채운다(외과적 UPDATE)

- 이 기능의 산출 `lpdOrig` = **원본 프리셋 프레임의 LPD OBB** = 바로 `slot_setup.lpdObb`([types.ts:107])가 담는 값. 즉 discovery 는 **검출 실패로 비어 있던 `slot_setup.lpd` 를 채운다/갱신한다**.
- **센터링(PtzCalibrator)과의 관계 확정**: discovery 는 센터링의 **상류(upstream) 별개 잡**이다. `expandPlateTargetsFromSlotSetup` 은 `v.lpd == null` 슬롯을 **대상에서 누락**([slotPtzWriter.ts:20])한다 → discovery 가 `slot_setup.lpd` 를 채우면 **센터링 대상 열거가 검출 의존에서 독립**된다(과업 A2 해소). 센터링의 확장이 아니라, 센터링을 먹여주는 선행 단계.
- **쓰기 방식(치명적 주의)**: 메모리 노트 "finalize slot_setup wipe fragility" 대로 **절대 DELETE+INSERT 금지**. `upsertSlotCentering`([SqliteStore])과 동일한 **slot_id 키 부분 UPDATE**(신규 `upsertSlotLpd(rows)` — `lpd_obb`, `updated_at`만 SET). best-effort(실패해도 JSON 정본·잡 완료 무방).
- globalIdx(=정수 slot_id) 부재 항목은 매핑 불가 → DB 스킵 + warn(PtzCalibrator.saveCenteringSlots 패턴 동일).

## 6. 명시적 한계(정직 리포트)

1. **후면주차(뒷번호판)**: 앞면중심은 **앞면** 기하다. 후면주차 차량의 뒷번호판은 슬롯 뒤·다른 높이 → 앞면중심 조준이 빗나감. 이번 설계는 **전면주차·앞번호판 카메라향 전제**. 후면 대응은 앞/뒤 후보 2개 또는 주차방향 정보 필요 — **범위 밖(리포트만)**.
2. **번호판 세로 오프셋**: 앞면중심(육면체 중앙높이 0.75m)과 번호판(≈0.4m)의 픽셀 위치가 다르다(하향 틸트에서 번호판이 약간 아래=y 큼). → 크롭 창을 앞면중심에 넉넉히(frac0=0.40) 잡고 과도 축소를 피해 흡수. 개선안(옵션): finalize 때 번호판높이 앵커도 함께 저장해 크롭 중심을 하향 보정(이번 범위 밖).
3. **차종별 번호판 높이 상이** → 크롭 여유 필요(위와 동일 완화).
4. **앞면중심 부재 슬롯**: `slot3dFrontCenter` 는 지면모델 필요(cameraposFile+ground). 없으면 null → 조준 불가 → `reason:'no_anchor'` 리포트(위장 성공 금지).
5. **광학 역계산 오차**: 2차 tier `G_i` 유도 가정 오차(§2-2) — 구현 시 파리티로 정량화. 이번엔 미수행.
6. **못 찾은 슬롯**: `found:false` + reason 로 그대로 노출. 100%는 "도전"이며 미달 슬롯은 정직 리포트.

## 9. 미해결 / 가정(리더 확인 요망)

1. **광학 tier `G_i` 유도(§2-2)**: 이번엔 미구현·리포트만으로 확정. 후속에서 가정 A(회전+f스케일) vs B(재추정) 중 택. → **동의 요청**: 디지털 tier 만으로 1차 마감하고 광학은 별도 goal/loop 로 분리하는 안.
2. **크롭 중심 = 앞면중심 그대로 vs 번호판높이 하향보정(§6-2)**: 1차는 앞면중심 + 넉넉한 창으로 흡수. 정합 미흡 관찰되면 finalize 에 번호판앵커 저장 추가(범위 확대) — 이번엔 보류.
3. **배치 잡·라우트까지 이번에 포함? vs 알고리즘+테스트만 먼저?**: 권고 = Phase 1(cropZoom+plateDiscovery+테스트로 하드코어 봉인) → Phase 2(잡·라우트·DB 미러) → Phase 3(광학, 후속). goal/loop 로 Phase 1 우선.
4. **matchRadius/frac0/shrink 초기값**: 실측 없이 눈대중 초기값(§3-3). 라이브·합성에서 QA 미세조정 대상.
