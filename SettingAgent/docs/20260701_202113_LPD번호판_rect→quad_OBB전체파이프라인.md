## 1. 배경 및 목표

기존 SettingAgent 번호판(LPD) ROI 는 **축정렬 사각형(rect, `{x,y,w,h}`)** 로만 저장·표시되었다. 그러나 실제 번호판은 카메라 각도에 따라 기울어져 있어, 축정렬 사각형은 회전 방향 정보를 소실한다. 이번 변경의 목표는 LPD 검출 결과를 **실제 OBB(Oriented Bounding Box) 회전 4점 폴리곤(quad)** 으로 바꾸어, 검출부터 뷰어 표시까지 **방향을 보존**하는 것이다.

### 사용자(마스터) 확정 결정 4항
1. **lpd_api(Python)도 함께 수정한다** — TS 만이 아니라 응답 계약 자체를 폴리곤으로 바꾼다.
2. **폴리곤만 반환한다** — 기존 `bboxes`(rect) 필드는 완전 제거하고 `polygons`(4점)만 남긴다.
3. **전체 파이프라인**에 적용한다 — LPD 응답 → 클라이언트 파싱 → 집계 → DB 저장 → 최종화(setup_artifact) → 뷰어 렌더까지 quad 를 관통시킨다.
4. **방향 보존** — 집계·최종화 단계에서 실제 OBB quad 방향을 소실하지 않는다. 내부 계산(클러스터링·캘리브레이션)은 quad 에서 유도한 rect 를 쓰되, 저장·표시는 실 quad 를 유지한다.

### 발견 버그 (핵심)
- **`routers/yolo.py:30` 의 `task="detect"`** — 로드한 가중치(`yolov11l_obb_lpd.pt`)는 OBB 모델인데, YOLO 를 `task="detect"` 로 로드하면 `results.obb` 가 `None` 이 되어 **OBB 4점이 절대 나오지 않는 상태**였다. 이번에 `task="obb"` 로 수정하여 OBB 를 활성화했다. (검증: `results.obb` 접근 경로가 실제로 채워지는지는 §9 수동확인 항목.)

## 2. 데이터 흐름도 (방향 보존)

```
[LPD OBB 모델]  results.obb.xyxyxyxy  (N,4,2) 픽셀 4점, ultralytics 규약 TL→TR→BR→BL
      │  (Python: detectors/yolov8.py → schemas/yolo.py polygons)
      ▼
[HTTP 응답]  { success, id, polygons:[[[x,y]×4],...], confidences, classes }   ※ bboxes 제거
      │  (TS: LpdClient.detectOnce → normalizeQuad(poly, imgW, imgH))
      ▼
[PlateBox.quad]  NormalizedQuad = [{x,y}×4]  (0~1 정규화, 점순서 보존)
      │  (CaptureJob: quadBoundingRect(quad)→rect  +  quad 동시 push)
      ▼
[detection 행]  rect(x,y,w,h, 집계·캘리브 입력)  +  quad(px0..py3, 보존)     ← 둘 다 저장
      │  (Aggregator: 클러스터 대표 rect + 대표 quad = 중심 최근접 멤버 quad)
      ▼
[AggregatedSlot]  plateX/Y/W/H(rect 대표)  +  plateQuad(NormalizedQuad|null)
      │  (aggregated_slot 행: plate_px0..plate_py3)
      ▼
[Finalizer]  plateRoiByPreset[key] = plateQuad ?? rectToQuad(rect)          ← 실 quad 우선
      │  (Repository.saveArtifact / loadArtifact)
      ▼
[setup_artifact.json]  plateRoiByPreset: Record<preset, NormalizedQuad>
      │  (web/app.js drawRoiOverlay → toPixelQuad)
      ▼
[뷰어]  회전 폴리곤 렌더 (beginPath/lineTo/closePath/stroke, #ffd60a)
```

- **정규화는 LpdClient 에서 1회만**(`readJpegSize` 로 캡처 해상도 획득). 이후 전 구간 0~1. 뷰어에서만 다시 ×W/×H.
- **점 순서 재정렬 없음** — ultralytics 규약(TL→TR→BR→BL)을 그대로 전달. min/max 계산도 폴리곤 렌더도 점 순서에 무관하므로 안전(설계 §8 저위험 판정).
- **clustering / calibration 내부 수치 계산만 rect(boundingRect) 사용** — 이 경로의 zoom/centering math 는 무변경.

### 3.2 TS `SettingAgent` — 파싱·집계·저장·최종화·캘리브·뷰어

#### (a) 순수함수 3종 신규 — `src/domain/geometry.ts`
- `normalizeQuad(pts, imgW, imgH)` — 픽셀 4점 → 정규화 quad. **길이≠4 면 throw**, `clamp01(px/imgW)`, 점 순서 보존.
- `quadBoundingRect(q)` — quad → 축정렬 bounding rect. `x=min(xi), y=min(yi), w=max−min, h=max−min`.
- `rectToQuad(r)` — rect → 축정렬 quad(TL,TR,BR,BL). 하위호환 승격·폴백 전용.
- 항등식: `quadBoundingRect(rectToQuad(r)) === r`(왕복 무손실, 테스트 3에서 검증).

#### (b) 클라이언트 — `src/clients/LpdClient.ts`
- `LpdResponse.bboxes` → `polygons: number[][][]`.
- `PlateBox` 재정의: `VehicleBox` 별칭 폐기 → `{ quad: NormalizedQuad; confidence; cls }`.
- `detectOnce` 가 `body.polygons.map(poly → normalizeQuad(poly, imgW, imgH))`. `confidences ?? confidence ?? []` 로 단수 confidence 방어. `normalizeBox` import 제거.

#### (c) 캡처 적재 — `src/capture/CaptureJob.ts` · `types.ts`
- `DetectionRow.quad?: NormalizedQuad`(plate 만), `AggregatedSlot.plateQuad: NormalizedQuad|null` 추가.
- plate 적재 시 `br=quadBoundingRect(p.quad)` 로 rect(집계용) + `quad`(보존) **동시** push.

#### (d) 저장 — `src/capture/SqliteStore.ts` (스키마 확장 · §5)
- `detection` 에 `px0..py3` 8컬럼, `aggregated_slot` 에 `plate_px0..plate_py3` 8컬럼(REAL, nullable) 추가.
- 기존 파일 DB 마이그레이션: `addColumnsIfMissing`(PRAGMA table_info → 없는 컬럼만 ALTER TABLE ADD COLUMN).

## 4. 순수함수 3종 알고리즘

```ts
// 1) 픽셀 4점 → 정규화 quad. 점 순서 보존, 길이≠4 throw.
normalizeQuad(pts, imgW, imgH):
  if pts.length !== 4: throw
  return pts.map(([px,py]) => ({ x: clamp01(px/imgW), y: clamp01(py/imgH) }))

// 2) quad → 축정렬 bounding rect (캘리브·집계·매칭 유도용)
quadBoundingRect(q):
  x = min(q.x); y = min(q.y)
  return { x, y, w: max(q.x)-x, h: max(q.y)-y }

// 3) rect → 축정렬 quad (하위호환 승격·폴백 전용, TL→TR→BR→BL)
rectToQuad(r):
  return [{x,y}, {x+w,y}, {x+w,y+h}, {x,y+h}]
```
- **대표 quad 선정**(Aggregator)은 순수함수가 아니라 클러스터 멤버 순회: 대표 rect 중심에 최근접인 멤버의 quad 를 그대로 채택. 4점 평균을 쓰지 않는 이유 = 이상치가 섞이면 대각선이 뒤틀려 실제 회전이 왜곡되기 때문.

## 5. 저장 스키마 확장 · 마이그레이션

### 5.1 확장 컬럼
- `detection`: 기존 `x,y,w,h`(rect=boundingRect) **유지** + `px0,py0,px1,py1,px2,py2,px3,py3`(REAL, nullable — vehicle 행은 NULL).
- `aggregated_slot`: 기존 `plate_x/y/w/h` **유지** + `plate_px0..plate_py3`(REAL, nullable — plate 없으면 NULL).
- rect 와 quad 를 **동시 저장** — rect 는 클러스터링·캘리브 입력, quad 는 방향 보존.

### 5.2 마이그레이션 방식 (구DB 호환)
better-sqlite3 는 `ADD COLUMN IF NOT EXISTS` 미지원 → `addColumnsIfMissing`:
```
PRAGMA table_info(<table>)  →  기존 컬럼 집합
없는 컬럼만  →  ALTER TABLE <table> ADD COLUMN <col> REAL
```
- 기존 파일 DB 는 최초 `new SqliteStore(path)` 시 자동 ALTER. `:memory:` 는 CREATE 문에 이미 포함.
- 구 DB 의 quad 컬럼은 NULL → 읽기 시 `quadFromCols` 가 `undefined`/`null` 반환 → Finalizer 가 `rectToQuad(rect)` 폴백.
- 테이블명은 하드코딩(주입 아님) → SQL 인젝션 무관.
- **누락 시 위험**: quad 컬럼 없이 insert 바인딩하면 개수 불일치 런타임 오류 → 설계 §7 테스트9(구스키마 마이그레이션)로 필수 검증.

## 6. 하위호환 (구 setup_artifact rect → quad 승격)

- `setup_artifact.json` 의 `plateRoiByPreset` 이 **구 rect 형태**(`{x,y,w,h}`)일 수 있음.
- **승격 지점 = `Repository.loadArtifact()`** 단 한 곳. `promotePlateRois` 가 `'w' in v` 로 rect 감지 → `rectToQuad` 승격(제자리). 배열(quad)이면 무변경.
- PUT 저장 경로(`server.ts`)도 zod `union(quad, rect)` 허용 후 저장 전 quad 정규화.
- **실측 확인**: 현재 리포지토리의 `SettingAgent/data/setup_artifact.json` 은 **아직 rect 형태**다. 예: `"1:1":{"x":0.088,"y":0.612,"w":0.033,"h":0.019}`. 즉 승격 경로는 가설이 아니라 실제 파일 최초 로드 시 **반드시 실행**된다. (검증자가 `slotPtzWriter.test.ts` 를 이 실 파일 + Repository 로드 경로로 교정하여 이 회귀를 겸함 — §8 참조.)

## 7. 캘리브레이션 rect 유도 정책

- 결정4에 따라 캘리브레이션 prior(`plateRoi`)는 **rect 유지**, quad 에서 `quadBoundingRect` 로 유도해 채운다. zoom/centering/probe 수치 로직은 무변경 → 회귀 없음.
- **정확도 주의(설계 §8 수용)**: 회전 번호판의 boundingRect 는 실제 번호판보다 폭이 넓다 → `targetPlateWidth` 수렴점에 미세 편차 가능. 안전 우선(기존 math 재사용)으로 이번 범위에서 수용. 필요 시 후속에서 quad 실변 길이(edge length) 도입 여지 — 이번 범위 밖(단순함 우선).

### 수동 동작 확인 체크리스트 (미검증 — 통과 위장 금지, 03 §6 그대로)
Python `Sub/lpd_api` 는 pytest 부재·YOLO 실행 환경 필요로 vitest 범위 밖. 아래는 **미수행**:
1. **OBB 응답 polygons 형식** — OBB 모델(`yolov11l_obb_lpd.pt`)에 실이미지 POST → `polygons`(검출별 4×[x,y] 픽셀) 존재·`bboxes` 키 부재. 검출0 → `polygons:[]`, `success:false`. (curl + 스키마 육안)
   ```bash
   curl -X POST 'http://localhost/lpd/api/v1/imgupload' \
     -H 'accept: application/json' -H 'Content-Type: multipart/form-data' \
     -F 'file=@plate.jpg;type=image/jpeg'
   ```
2. **routers task 버그 수정 실효** — `task="obb"` 적용 후 실호출로 4점이 실제로 반환되는지(수정 전엔 `results.obb=None` 으로 항상 빈 polygons).
3. **뷰어 회전 폴리곤 렌더** — 브라우저에서 plate 오버레이(`#ffd60a`)가 축정렬 사각형이 아니라 **회전 폴리곤**으로 그려지는지.
4. **실 회전 표시 정합** — 실 캘리브레이션 후 `plateRoiByPreset` quad 가 실제 번호판 기울기와 시각적으로 일치하는지.
5. **실 da_lpd_api 연동 스모크** — 유닛은 fetch 모킹만. 실 REST 왕복 스모크는 미수행.

## 9. 요약

- LPD 번호판 ROI 를 rect → OBB 회전 quad 로 전환하는 **양쪽 코드베이스 전체 파이프라인** 변경.

- 저장 스키마는 quad 8+8 컬럼을 추가하되 구DB 는 ALTER TABLE 마이그레이션 + NULL 폴백으로 호환. 구 setup_artifact rect 는 로드 시 승격.
