# 클릭지점 중앙조준(mode:'point') 폐기 · 되돌리기 + 개별 center+zoom 재검증

작성: 2026-07-21 / 브랜치: `fix/revert-point-aim`

## 1. 배경

정밀수집 페이지 "개별 센터라이징" 콤보의 **개별 center** 를 *클릭한 지점 자체를 화면중앙으로 보내는*
패치-NCC 폐루프(`PointAimer`)로 재구현했으나, 마스터 실사용에서 **전혀 동작하지 않았고**
그 여파로 잘 동작하던 **개별 center+zoom** 까지 함께 못 쓰게 되었다.
→ 클릭지점 조준 계층 전체를 폐기하고, 개별 center 를 원래 설계(클릭 최근접 **번호판** center)로 되돌린다.

(십자는 조준 확인용 시각 보조로 무해).

## 3. 현재 동작 규약

- **개별 center** = `mode:'plate'` → 클릭 **최근접 번호판** 을 화면중심으로 pan/tilt (줌 없음)
- **개별 center+zoom** = `mode:'plate-zoom'` → 위 + `zoomToPlateWidth`(목표 폭 0.2) 마감
- 두 경로 모두 **저장 없음**(slot_ptz.json · DB `centering_slot` · Setup 스냅샷 미기록)

## 5. 영향도

- 영향 범위는 개별(클릭) 경로 한정. **배치 센터라이징**(`/calibrate/ptz`)·저장 경로·`platePtz.ts`·`controlMath.ts`·DB 스키마 **무접촉**.
- 외부 의존성 추가분 없음(폐기된 patchTrack 은 기존 `sharp` 만 사용했고 함께 삭제).
- 응답 계약: `POST /calibrate/point` 는 항상 `{ok, ptz, plateWidth, reason?}` 로 단일화(과거 point 응답의 `iterations` 필드 소멸 — 사용처는 프론트에 없었음).
- legacy `zoom:boolean` 바디 하위호환 유지.

## 6. 교훈

클릭 **지점** 조준은 합성 검증(6/6 수렴)에서 통과했어도 실사용에서 실패했다.
관찰형 기능은 **마스터의 실제 조작 경로(브라우저 클릭)** 로 확인되기 전에는 완료로 보지 않는다.
개별 센터라이징의 정본은 **번호판 기준**(LPD 측정 폐루프)이다.
