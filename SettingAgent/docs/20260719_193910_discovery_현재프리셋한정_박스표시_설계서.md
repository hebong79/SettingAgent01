## 1. 배경·목표

콤보박스 3모드 중 (b) "앞면중심 LOOP"(discovery)만 두 지점에서 다른 모드와 어긋나 있다.

1. **범위 불일치**: (a)순수 LPD·(c)VPD→LPD 는 `runLiveDetect`가 **현재 프리셋 1회**만 검출한다(app.js:928-929, `capFrameKey2?.cam ?? state.cam`). 반면 (b)discovery 는 `POST /discover/ptz` body `{}` → 카메라 전체·전 슬롯 순회(선행 콤보 설계서 §2-3, 명시적 "전체 배치"). 사용자가 보는 화면(현재 프리셋)과 잡의 작업 범위가 불일치.
2. **박스 미표시**: (a)(c)는 검출 결과를 `state.detectByKey`에 넣어 `drawDetectOverlay`가 quad 를 그린다. (b)는 진행바·프레임·"found N/total" 요약만 갱신하고 **검지 박스를 화면에 그리지 않는다**(renderDiscResult:2374 는 텍스트만).

**목표**: (b)를 (a)(c)와 정합하게 — **현재 프리셋만 검사**하고, **검지된 LPD 박스를 현재 프리셋 오버레이로 표시**한다. 그 외 동작(폴링·상호배타·slot_setup.lpd 부분 UPDATE)은 불변.

**MCP 두뇌/도구 경계**: 두 변경 모두 **결정형(도구) 경로**다. ① 현재 프리셋 필터는 `camIdx===cam && presetIdx===preset` 순수 비교(LLM 무관), ② 박스 표시는 이미 산출된 `lpdOrig`(원본 프레임 정규화 OBB, 결정형 아핀 역계산 결과)를 캔버스에 그리는 순수 렌더. 새 판단·LLM 개입 없음.

## 2. 확정된 설계 결정

### 2-1. 현재 프리셋 한정 — 방식 (a): 라우트 body 에 `{cam, preset}` 추가

두 방식을 검토했다.

| 방식 | 내용 | 트레이드오프 |
|---|---|---|
| **(a) 라우트 필터 (채택)** | `/discover/ptz` body 에 `cam`/`preset` 추가 → `PlateDiscoveryJob.start`가 `camIdx===cam && presetIdx===preset` 대상만 남김 | 자기완결(백엔드가 필터 소유), 프론트는 현재 cam/preset 만 전달(slotIds 계산 불요), `detect`/`frame`이 cam/preset 을 쓰는 것과 개념 일관. **채택.** |
| (b) 프론트 slotIds 계산 | 프론트가 `parkingSlotsByKey[현재키]`에서 slotId 배열을 뽑아 기존 `slotIds` 필드로 전달 | 백엔드 스키마 무변경이나, 프론트가 `parkingSlotsByKey` 로드·정합에 의존(최종화 전이면 비어 실패), 필터 책임이 프론트로 누수 |

**채택: (a).** 변경 범위 최소(스키마 2필드·`start` 시그니처 1건·프론트 body 1줄)이고, 필터 책임이 잡에 응집된다.

**필터 위치**: `expandDiscoveryTargets`(순수 펼침 함수)는 **불변**으로 두고, 기존 `slotIds` 필터와 **같은 자리**(`PlateDiscoveryJob.start`)에서 cam/preset 을 거른다(CLAUDE.md 규칙3 외과적 — 기존 필터 패턴 미러, 로직 분산 회피).

**옵셔널 유지**: `cam`/`preset` 둘 다 있을 때만 필터. 미전달(빈 `{}`)이면 기존 전체 배치 동작 보존 → 기존 라우트 테스트(`payload: {}`)·백엔드 호출부 회귀 0.

### 2-2. 박스 표시 — 완료 후 일괄, `/discover/result` → `drawDetectOverlay` 재사용

**소스**: `GET /discover/result`(plate_discovery.json)의 `items[]` 중 `found && lpdOrig!=null`. `lpdOrig`는 **원본(프리셋) 프레임 정규화 4점 OBB**(plateDiscovery.ts:117·133) — `drawPlateQuad(ctx, quad, recovered)`(app.js:851)와 `state.detectByKey.plates[].quad`가 쓰는 좌표계와 **동일**. 즉 별도 변환 없이 기존 LPD 렌더 경로에 그대로 태울 수 있다.

**타이밍: 완료 후 일괄(채택)**. "진행 중 실시간(프레임마다 발견분)"도 가능하나, discovery 진행 중엔 `/discover/frame`(잡이 방금 크롭·재명령 없이 찍은 원본 JPEG)을 라이브뷰로 추종하며 표시 크기·프레임이 순환한다. 그 위에 부분 결과를 실시간 오버레이하면 좌표 정합·깜빡임 관리가 늘어난다(최소주의 위배). **완료 → `startLive` 복귀 → 현재 프리셋 정지 프레임 위에 확정 박스 일괄 표시**가 최소·안정. (선행 콤보 설계서 §3-3 의 done 전환 흐름을 그대로 활용.)

**토글 정합**: discovery 결과는 LPD 박스이므로 **기존 `#roi-plate` 토글**로 가드(모드 a·c·DB LPD 와 동일 게이트). 새 체크박스·CSS 없음(최소주의).

**저장·키잉**: `state.discoverByKey`(신규, `detectByKey` 대칭) = `presetKey(item.camIdx, item.presetIdx) → NormalizedQuad[]`. 렌더는 `currentFrameKey()` 로 조회 → 현재 화면 프리셋 결과만. 프리셋 전환 시 자동 은닉·복귀(detectByKey 와 동일 시맨틱, MEMORY "overlay retain policy" 부합 — 삭제 안 함·유지).

**색상**: `drawPlateQuad(ctx, quad, false)` 재사용(노랑 실선, DB/라이브 LPD 와 동색). 라이브 검출과 시각 구분은 요청 범위 밖 → 도입 안 함(§6 가정 C).

```ts
start(filter: { slotIds?: string[]; cam?: number; preset?: number } = {}): { total: number } {
  if (this.state === 'running') throw new Error('discover already running');
  let targets = expandDiscoveryTargets(this.store.getSlotSetup());
  if (filter.cam != null && filter.preset != null) {                       // 신규: 현재 프리셋 한정

    const set = new Set(filter.slotIds);

  }
  // ... 이하 불변(state/total/run 발화)
}
```

**renderDiscResult(2374)** — 상태 요약 뒤에 result 파싱·저장 추가(기존 요약 문구 보존):
```js
async function renderDiscResult() {
  try {
    const res = await fetch('/discover/status', { cache: 'no-store' });
    const s = res.ok ? await res.json() : {};
    $('disc-msg').textContent = `완료 — 발견 ${s.found ?? 0}/${s.total ?? 0} 슬롯 (slot_setup.lpd 갱신)`;
  } catch { /* ignore */ }
  // 검지 박스 표시(§2-2): result 의 found+lpdOrig 를 프리셋키별로 저장. 뒤이은 startLive→drawRoiOverlay 가 렌더.
  try {
    const res = await fetch('/discover/result', { cache: 'no-store' });
    if (!res.ok) return;
    const data = await res.json();
    const byKey = {};
    for (const it of data.items ?? []) {
      if (!it.found || !it.lpdOrig) continue;
      (byKey[presetKey(it.camIdx, it.presetIdx)] ??= []).push(it.lpdOrig);
    }
    state.discoverByKey = byKey;   // 새 결과로 대체(현재 프리셋 한정이라 단일 키가 정상)
  } catch { /* ignore */ }
}
```

## 6. 미해결/가정

- **가정 A(현재 프리셋 정의)**: "현재 프리셋" = `state.capFrameKey2?.cam/preset ?? state.cam/preset`(runLiveDetect 와 동일 해석). 정밀수집 폴 중이면 표시 프레임 프리셋, 아니면 라이브 선택 프리셋. 다른 정의를 원하면 리더 확인.
- **가정 B(완료 후 일괄)**: 박스는 잡 완료 후 표시(§2-2). "진행 중 발견분 실시간 표시"가 필요하면 별도 요청(범위·안정성상 완료 후 권장).
- **가정 C(색 구분 없음)**: discovery 박스는 라이브/DB LPD 와 동색(노랑). 라이브 검출과 시각 구분이 필요하면 `drawPlateQuad` 에 색 인자 추가(사소) — 요청에 없어 미도입.
- **가정 D(단일 프리셋 키)**: 현재 프리셋 한정이라 `discoverByKey` 는 실무상 단일 키. `renderDiscResult`가 매 완료 시 `discoverByKey`를 **대체**(누적 아님)해 이전 프리셋 잔여를 정리 — 이 정책이 맞는지(여러 프리셋 결과 누적 표시 원하면 병합으로 변경) 확인 여지.
