# 분석 탭 '프리셋별 요약' 표에 PTZ 값 열 추가

- 작업일: 2026-07-24
- 브랜치: `worktree-preset-ptz-column` (워크트리 `.claude/worktrees/preset-ptz-column`)
- 요청: "프리셋 리스트에 ptz 값 필드를 추가해줘"

## 1. 설계

`분석` 탭 → `프리셋별 요약` 표(`#an-presets`)는 지금까지
`프리셋 키 / 카메라 / 프리셋 / 라벨 / 주차면 수` 5열만 표시했다. 여기에 **PTZ(pan, tilt, zoom)** 열을 추가한다.

### 데이터 소스 결정

| 후보 | 상태 | 채택 |
|------|------|------|
| 산출물(`GET /viewer/api/mapping` = `setup_artifact.json` 또는 slot_setup 즉석 조립)의 `presets[].pan/tilt/zoom` | 타입(`Preset`)에는 optional 로 존재하지만 **현재 어떤 생성 경로도 PTZ 를 싣지 않는다**(`Finalizer`, `buildArtifactFromSlotSetup` 모두 미기록) | 1순위(있으면 사용) |
| 라이브 카메라 목록 `GET /viewer/api/cameras` (`state.cameras[].presets[].pan/tilt/zoom`) | 항상 제공. `프리셋 이동` 버튼이 쓰는 것과 **같은 정본**(`findPresetPtz`) | 2순위(폴백, 실질 표시원) |
| DB `preset_info.pan/tilt/zoom` | 존재하나 이를 읽어 산출물/뷰어로 내보내는 경로가 없음 → 신규 백엔드 작업 필요 | 이번 범위 밖 |

→ **산출물 보관값 우선 · 없으면 라이브 카메라 목록 폴백 · 둘 다 없으면 `-`**.
백엔드 변경 0으로 화면이 실제 값을 표시하고, 향후 산출물이 PTZ 를 싣게 되면 자동으로 그 값이 우선한다.

### 성공 기준

1. `analyzeArtifact().perPreset[].ptz` 가 pan/tilt/zoom 3개 모두 유한수일 때만 객체, 아니면 `null` — 유닛테스트 통과.
2. 실데이터(현재 구동 중인 SettingAgent 13020)로 5개 프리셋 행에 실제 PTZ 문자열이 찍힌다.
3. 기존 테스트 전량 green, `tsc --noEmit` 0.

## 2. 변경 내용

### `SettingAgent/web/core.js` (순수 로직)

- **신규 내부 함수 `presetPtzOf(p)`**: `pan/tilt/zoom` 이 **모두** `typeof === 'number' && Number.isFinite` 일 때만 `{ pan, tilt, zoom }`, 하나라도 누락·비수치면 `null`.
  - `0` 은 유효값으로 통과(falsy 로 버리지 않음).
  - 부분값(`pan`만 있음)은 표에 반쪽 PTZ 를 그리지 않도록 전부 `null` 로 강등.
- `analyzeArtifact()` 의 `perPreset` 원소에 `ptz` 필드 추가.

### `SettingAgent/web/core.d.ts` (타입 선언)

- `ArtifactAnalysis.perPreset` 원소에 `ptz: Ptz | null` 추가.
- `PresetLike` 에 `pan?/tilt?/zoom?: number` 추가(도메인 `Preset` 과 정합).

### `SettingAgent/web/app.js` (렌더)

- 신규 표시 헬퍼 `fmtPtz(ptz)` — `fmtRoi` 와 동일 규약(소수점 3자리, 없으면 `-`).
- `renderAnalysis()` 의 프리셋 표:
  - 헤더에 `PTZ (pan, tilt, zoom)` 열 추가.
  - 셀 값 = `fmtPtz(p.ptz ?? findPresetPtz(state.cameras, p.camIdx, p.presetIdx))`.
  - `findPresetPtz` 는 이미 import 되어 있어 추가 import 없음.

### `SettingAgent/test/analyzeArtifact.test.ts` (검증)

- 기존 `perPreset` 동등비교 케이스를 `ptz: null` 포함으로 갱신(필드 추가에 따른 필수 수정).
- 신규 2케이스:
  - `pan/tilt/zoom` 전부 보유 → `perPreset[].ptz` 로 노출(`0`/`1` 같은 falsy 값 포함 정상 통과).
  - `zoom` 누락 / 문자열 `'2'` / `NaN` → 전부 `ptz=null`(부분 표기 금지).

## 4. 영향도 분석

| 대상 | 영향 | 비고 |
|------|------|------|
| `analyzeArtifact` 소비처 | `app.js renderAnalysis` **1곳**뿐(+ 유닛테스트) | 필드 **추가**만 — 기존 필드 형태·순서 불변, 하위호환 |
| 백엔드(`src/**`) | **변경 없음** | 신규 라우트·DB 조회 없음. 이미 폴링 중인 `/viewer/api/cameras` 재사용 |

| 다른 표(주차면 목록·점유율) | 영향 없음 | `buildTable` 헬퍼는 헤더/행 배열만 받는 범용 |

### 남은 선택지(요청 시 별도 작업)

- 산출물(`setup_artifact.json` / slot_setup 즉석 조립)에 **프리셋 PTZ 를 영속화**하려면 `Finalizer` + `buildArtifactFromSlotSetup` 에서 `preset_info` 의 pan/tilt/zoom 을 읽어 `Preset` 에 실어야 한다. 이번 변경은 그 값이 생기면 **우선 사용**하도록 이미 열려 있다.
