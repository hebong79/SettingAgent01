## 1. 배경 / 목적

정밀수집(장기 관측)은 라운드×프리셋마다 차량/번호판을 검출해 SQLite `detection` 에 원시 누적한다. `Aggregator.aggregate` 가 이 원시 검출을 클러스터링해 주차면 대표 좌표를 산출한다.

**기존 방식의 취약점**(변경 전 기준선):
- 대표 bbox = 좌표별 **단순 median**(`medianRect`). `detection.conf`(검출 신뢰도) 미반영, 이상치 제거 없음.
- 대표 번호판 quad = `representativeQuad` = **클러스터 대표 중심에 최근접한 단일 멤버 quad**를 그대로 채택. 즉 **단일 프레임 표본 1장**의 각도·크기에 전적으로 의존 → LPD OBB 점순서 뒤바뀜(θ↔θ+π)·단발 오검출·지터에 취약.

**목적**(사용자·현자 합의 핵심 3가지): 원시 검출의 대표값 산출을, 단일 표본 선택이나 단순 평균이 아니라 **강건 추정(robust estimation)** 으로 고도화한다. median/MAD 기반으로 지터·이상치에 견디게 하고, 검출 신뢰도(conf)를 가중으로 반영하며, 슬롯별 종합 신뢰도 스칼라를 산출해 저장한다.

이 로직은 전부 **결정형 도구 영역**(고빈도 수치 반복·재현성 요구, LLM 미개입, 좌표 불변식 유지)이다. LLM 두뇌는 기존대로 채택/거부/라벨 메타만 담당하며 본 변경과 무관하다.

### 평균(mean)을 채택하지 않은 근거
산술평균은 붕괴점(breakdown point)이 0 이라 gross outlier(오검출·점순서 뒤바뀜) 1개만으로 대표값이 끌려간다. median/가중 median 은 붕괴점 ≈50% 로 소수 이상치에 강건하다. 각도는 특히 π-반전 오검출이 산술평균을 정반대로 오도한다(예: {+170°, −170°} 산술평균 0° → 오류). 따라서 **평균 채택은 강건성 후퇴**이므로 median/robust 로 통일했다.

## 2. 개선 ① — 번호판 각도 통계화 (단일 최근접 폐기 → 축 순환 median 합성)

### 변경 요지
`representativeQuad`(단일 최근접 멤버 선택)를 **제거**하고, 클러스터 quad 멤버 전체를 통계 집계해 대표 quad 를 **합성**한다(`robustPlatePose`).

### 알고리즘 (`Aggregator.robustPlatePose`, quad 멤버 ≥2)
1. 멤버별 각도 `θᵢ = plateAngleRad(quadᵢ)`, conf `wᵢ`.
2. 피벗 각도 `θ₀ = circularMedianAngle(θ, w)`.
3. (N≥`ROBUST_MIN_MEMBERS`=4) 각도 MAD 컷 + plate center 좌표 MAD 컷으로 이상치 멤버 제외(생존 0 이면 컷 취소).
4. 생존 멤버로 대표 각도 `θ_rep = circularMedianAngle(생존θ, 생존w)`.
5. 강건 중심 `c = medianPoint(생존 quad centroid)`.
6. 로컬 축 `u=(cosθ_rep,sinθ_rep)`(폭·하변), `v=(−sinθ_rep,cosθ_rep)`(높이). `W = weightedMedian(projectedSpan(qᵢ,u), w)`, `H = weightedMedian(projectedSpan(qᵢ,v), w)`.
7. `quad = synthesizePlateQuad(c, θ_rep, W, H)`, `angleSpread = circularMad(생존θ, θ_rep)`.

### 핵심: 축(axial, 주기 π) 순환 median — `circularMedianAngle`
번호판 방향은 **선(line) 방향**이라 π-반전(θ↔θ+π)이 같은 자세다. LPD 점순서 뒤바뀜으로 `plateAngleRad` 가 θ↔θ+π 로 튈 수 있으므로:
1. **각도 배증** `φᵢ = 2θᵢ` — π-반전을 2π-반전(동일)으로 흡수.
2. **가중 벡터평균 피벗** `φ₀ = atan2(Σwᵢsinφᵢ, Σwᵢcosφᵢ)` — 이상치 1개가 무한대로 끌지 못하는 안정 피벗.
3. **피벗 기준 언랩** `φ′ᵢ = φ₀ + wrapToPi(φᵢ − φ₀)` — 브랜치컷(±180° 경계) 제거, 선형 윈도우로 정렬.
4. `(가중) median` → `wrapToPi(m)/2 ∈ (−π/2, π/2]` 복원.

검증: {+170°, −170°}(±180° straddle)에서 산술평균 오류(0°)를 회피하고 올바른 대표(≈±90°)로 수렴함을 QA 가 유닛으로 실증(§6).

### 합성 규약 정합 (파이프라인 무결성 보장)
`synthesizePlateQuad(c, θ, W, H)` 는 점순서 **TL,TR,BR,BL** 로 합성하며 하변(TL→TR) 방향 = θ 이므로 **`plateAngleRad(합성 quad) == θ_rep`** (경계 clamp01 미발생 시). 하위 소비측(`Finalizer.buildPlateAnchoredQuad` / `estimatePlateQuadFromNeighbors` / floorRoi)은 각도·centroid·projectedSpan(모두 점순서 무관)만 소비하므로 **합성 quad 규약만 지키면 소비측 코드 변경 불필요**하다.

## 3. 개선 ② — conf 가중 + MAD 이상치 컷 (`robustRect`)

### 변경 요지
대표 bbox 산출을 `medianRect`(단순 median)에서 **conf 가중 median + MAD 이상치 컷**(`robustRect`)으로 교체.

### 알고리즘 (`Aggregator.robustRect`)
1. **이상치 컷**(멤버 수 ≥ `ROBUST_MIN_MEMBERS`=4 일 때만): 멤버 center `(cx,cy)` 의 축별 median·MAD 로 `scaled = MAD_SCALE·mad`. `|cᵢ − med| > MAD_K·scaled`(단, `scaled ≥ MAD_FLOOR` 인 축만)이면 이상치 → 제외. **생존 0 이면 컷 취소**(전체 사용).
2. **가중 median**: 생존 멤버로 `x/y/w/h = weightedMedian(값, conf)`.
3. **posSpread** = `hypot(MAD_SCALE·mad(cx), MAD_SCALE·mad(cy))`(생존 center 공간 퍼짐, 정규화 스칼라 ≥0).

같은 강건화를 번호판 자세(`robustPlatePose`)의 좌표·각도 잔차에도 적용한다.

### 상수 (모듈 named const — G1: config 미승격, floorRoi 선례)
| 상수 | 값 | 근거 |
|---|---|---|
| `ROBUST_MIN_MEMBERS` | 4 | MAD 붕괴점상 이상치 1개가 <25% 이려면 ≥4. 미만은 컷 불안정 → 생략(현행 보존). |
| `MAD_SCALE` | 1.4826 | MAD→σ 정규분포 일치 표준상수. |
| `MAD_K` | 3.0 | ≈3σ 컷(정규분포 오제거 ~0.3%). 지터 보존·gross outlier만 제거. |
| `MAD_FLOOR` | 1e-4 | 정규화좌표 하한. 동일좌표(MAD≈0) 과민제거 방지. |
| `SUPPORT_SAT` | 10 | 지지 10 관측 이상 위치 신뢰 포화(경험적). |
| `POS_SPREAD_MAX` | 0.06 | = clusterDist 규모. 병합거리만큼 퍼지면 위치신뢰 0. |
| `ANGLE_SPREAD_MAX` | 0.35 rad(~20°) | 그 이상 각도분산이면 자세신뢰 0. |
| `W_OCC/W_SUP/W_POS/W_ANG` | 0.35/0.25/0.25/0.15 | 점유지속·지지 우선, 기하일관 보조. 합=1. |

### 소표본 폴백 (= 현행 동작 보존)
| 상황 | 신규 동작 | 현행 대비 |
|---|---|---|
| vehicle 멤버 1 | 그 멤버 | median(1)=자신, 동일 |
| vehicle 멤버 2~3 | 등가중 weightedMedian(짝수 평균 포함) | conf 등가중 → median 동일값 |
| vehicle 멤버 ≥4 | MAD 컷 후 가중 median | **신규**(이상치 제거·conf 반영) |
| plate quad 0 | `{quad:null, angleSpread:null}` | 동일 |
| plate quad 1 | 원본 quad 그대로 | 최근접(=유일)과 동일, 왜곡 없음 |
| plate quad ≥2 | 순환 median + 합성 | **신규**(단일 최근접 폐기) |

`support`/`occupancyRate`/`status`/클러스터링(`clusterDetections` 는 여전히 `clusterRect`=median 사용)·번호판 귀속 규칙은 **전부 보존**. 이상치 컷은 대표 좌표 산출에만 적용된다(지지수·점유율은 관측 사실이라 보존).

## 4. 개선 ③ — 슬롯 종합 신뢰도 (`slotConfidence`)

`AggregatedSlot` 에 3필드 신설: `confidence`(0~1) · `posSpread`(≥0) · `angleSpread`(rad|null).

### 알고리즘 (`Aggregator.slotConfidence`)
서브스코어(전부 0~1, 단조):
- `s_sup = min(1, support / SUPPORT_SAT)` — 지지 많을수록 ↑
- `s_occ = clamp01(occupancyRate)` — 점유 지속 ↑
- `s_pos = 1 − clamp01(posSpread / POS_SPREAD_MAX)` — 퍼짐 작을수록 ↑
- `s_ang = 1 − clamp01(angleSpread / ANGLE_SPREAD_MAX)` — 각도 일관 ↑

`confidence = Σ(wₖ·sₖ) / Σwₖ`(가용항만). **plate 부재(angleSpread=null) 시 각도항 제외 후 잔여 가중 재정규화** → 무번호판 슬롯이 각도 결손으로 불이익받지 않는다(분모가 W_ANG 만큼 줄어 나머지 3항이 재정규화됨).

**단조성 불변식**(QA 검증): 다른 입력 고정 시 support↑·occupancy↑·posSpread↓·angleSpread↓ 각각에 대해 confidence 비감소, 범위 0~1.

## 5. 입출력 · 저장 계약 (스키마 · 마이그레이션)

### 5.1 신규/변경 순수 헬퍼 (`src/domain/geometry.ts`, 모두 신규 export, 기존 함수 불변)
| 시그니처 | 역할 |
|---|---|
| `axialWrap(a): number` | 축(주기 π) 래핑 `wrapToPi(2a)/2 ∈ (−π/2,π/2]`. **계획은 비수출이었으나 수출로 승격**(Aggregator 각도 MAD 잔차가 필요 — §7 편차). |
| `mad(values, center?): number` | 중앙절대편차(raw). center 미지정 시 median 기준. 길이<2 → 0. |
| `weightedMedian(values, weights): number` | 누적 가중이 총합 절반을 처음 넘는 값. 경계(==절반)는 다음 유효값과 평균(짝수 median 규약 정합, 등가중=median). 가중 전부 0/음수·빈배열 → median 폴백. |
| `medianPoint(pts): NormalizedPoint` | 좌표별 median 점. 빈 배열 → {0,0}. |
| `circularMedianAngle(angles, weights?): number` | 축 순환 median(§2). |
| `circularMad(angles, center): number` | `median(|axialWrap(θᵢ−center)|)`. 길이<2 → 0. |
| `synthesizePlateQuad(c, θ, w, h): NormalizedQuad` | 중심·각도·크기 → TL,TR,BR,BL 합성(각 점 clamp01). `plateAngleRad(결과)==θ` 보장. |
> 내부 비수출: `wrapToPi(a) → (−π,π]`.

### 5.2 `AggregatedSlot` 신규 필드 (`src/capture/types.ts`)
```ts
confidence: number;         // 슬롯 종합 신뢰도 0~1
posSpread: number;          // vehicle center 공간 퍼짐(정규화 스칼라, MAD 기반 ≥0)
angleSpread: number | null; // 번호판 각도 분산(rad, circularMad). plate quad<2 → null
```
`plateQuad` 타입 불변(`NormalizedQuad|null`) — **의미만** "최근접 원본"→"강건 합성 대표"로 변경.

### 5.3 SQLite 스키마 마이그레이션 (`src/capture/SqliteStore.ts`, 하위호환)
- `ensureSchema` CREATE `aggregated_slot` 에 `confidence REAL, pos_spread REAL, angle_spread REAL` 추가(신규 DB용, status 앞).
- `addColumnsIfMissing('aggregated_slot', ['confidence','pos_spread','angle_spread'])` 추가 → **구DB 는 `PRAGMA table_info` 검사 후 ALTER TABLE ADD COLUMN**(값 NULL). better-sqlite3 는 `IF NOT EXISTS` 미지원이라 존재검사 방식.

- `getAggregatedSlots` SELECT: `confidence, pos_spread AS posSpread, angle_spread AS angleSpread`. **구DB NULL 폴백**: `confidence ?? 0`, `posSpread ?? 0`, `angleSpread ?? null`(AggregatedSlot 타입 충족). 구 `aggregated_slot` 은 finalize 시 recompute 되는 임시데이터라 실질 영향은 prior status 조회에 한정.
- plate quad 8컬럼·기존 컬럼·인덱스 **불변**(멱등 delete+insert 트랜잭션 유지).
