## 1. 배경 / 원칙 (마스터 승인 아키텍처)

기존 뷰어 소스(`RpcCameraSource.listCameras`)는 카메라/프리셋 목록을 `cam.list`(device 목록) + `preset.list`(주차면 프리셋)로 구성했다. 이는 **두 종류의 "프리셋"을 혼동**한 것이다.

- **(A) 카메라 PTZ 프리셋** — "카메라가 어디를 바라보는가"(pan/tilt/zoom). 셋팅에이전트의 산출물. 뷰어 카메라 드롭다운·`gotoPreset`의 소스여야 한다.
- **(B) 주차면 프리셋** — Unity RPC `preset.*`(faceCount 등). 시뮬 3D 씬 전용. 카메라 PTZ 값이 없다.

확정된 소유 원칙:

| 관심사 | 소유 주체 | 비고 |
|---|---|---|
| 카메라 갯수 · URL · 위치(리얼=고정) | `config`(tools.config.json) | 단일 Unity RPC(13110)가 camId로 다중 카메라 주소지정 |
| **카메라 PTZ 프리셋 (A)** | **`config/camerapos.json` + 웹 편집** | 뷰어 드롭다운·gotoPreset 소스 |
| 주차면 프리셋 (B) `preset.*` | 시뮬 Unity RPC | 뷰어 카메라 드롭다운에서 **분리** |
| device 실시간 제어(setPTZ/getPTZ/stream/snapshot) | Unity RPC(순수 device) | 기존 경로 재사용 |

즉 **카메라(시뮬레이터/리얼) = 순수 device**(연결·현재 PTZ·setPTZ·stream)로 남기고, **"카메라가 무엇을 볼지"(PTZ 프리셋 A)는 셋팅에이전트가 camerapos.json으로 소유·웹에서 편집**한다. 두 프리셋(A ≠ B)은 완전히 분리된다.

이 작업은 전부 **결정형(파일/라우트/수치 조작)** 으로, LLM 두뇌 개입이 없다(목록 구성·CRUD·PTZ 이동은 파일·수치 조작). 센터라이징 등 반복 보정 루프는 이번 범위 밖(`PtzCalibrator` 유지).

## 2. 변경/신규 클래스·함수

### 2.1 신규 `src/viewer/cameraposCatalog.ts` — `buildCameraList` (순수)

```ts
export function buildCameraList(
  views: CameraView[],
  devices?: Array<{ camId: number; name?: string }>,
): CameraList
```

- `camerapos.json` 파싱 결과(`CameraView[]`, `parseCameraViews` 재사용)를 뷰어 드롭다운용 `CameraList`로 변환하는 **순수 함수**.
- camIdx로 그룹핑 → `cameras[{ camIdx, name, enabled, presets:[{presetIdx,label,pan,tilt,zoom}] }]`. camIdx·presetIdx 오름차순 정렬, PTZ 보존.
- device 병합: `cam.list`에 해당 camId 존재 → `name`=device.name(없으면 `C{camIdx}`)·`enabled=true`; 미존재 → `C{camIdx}`·`enabled=false`(드롭다운 `[off]`).
- **카메라 집합은 파일(views) 기준.** device에만 있고 파일에 없는 camId는 표시하지 않는다(설계 가정 A2). 입력 불변.

### 2.2 신규 `src/viewer/CameraposSource.ts` — `CameraSource` 구현(합성)

`RpcCameraSource`를 **합성(composition)** 해 device 제어를 위임하고, 목록/프리셋 스냅샷만 camerapos 기반으로 재정의한다. `kind='rpc'`(라우트·스트림 계약 동일).

- 생성자 `(cameraposFile, inner: RpcCameraSource, rpc: CRpcClient)`.
- `listCameras()`:
  1. camerapos.json을 **매 호출 fresh read**(`readFileSync`, 캐시 없음) → `parseCameraViews`. 파일 없음/파싱 실패 → 빈 배열(graceful).
  2. `rpc.callRpc('cam.list', {})`로 device 연결/이름 조회. **실패 시 throw** → 라우트 502 → 프론트 `loadCameras()=false` → `badge-camera` off. ("연결 = cam.list 성공" 폴 시맨틱 보존.)
  3. `buildCameraList(views, devices)` 반환. `preset.list`는 **호출하지 않는다**(뷰어 목록이 카메라 PTZ 프리셋 전용).
- `snapshot(cam, opt)`: **preset 모드**는 파일에서 (cam,presetIdx)의 PTZ를 찾아 `inner.snapshot(cam,{mode:'manual', ptz})`로 변환 → 주차면 `preset.select`(B) 경로를 타지 않는다. PTZ 미발견 시에만 원본 opt로 inner 위임(폴백). manual 모드는 그대로 위임.
- `move / streamMjpeg / toNativePtz / fromNativePtz`: 전부 inner 위임.

**매 호출 fresh read가 핵심**이다 — 웹 편집으로 camerapos.json이 바뀌면 4초 연결 폴(`loadCameras`)이 캐시 없이 즉시 반영한다.

### 2.3 `src/viewer/sourceRegistry.ts` (수정 소)

`buildSourceRegistry`의 **폴백**(`cameraSources` 미설정 시)을 `RpcCameraSource` → `CameraposSource`로 교체.
- 시그니처 Pick에 `map` 추가: `Pick<ToolsConfig,'camera'|'cameraSources'|'unityRpc'|'map'>`.
- `new CameraposSource(cfg.map.cameraposFile, new RpcCameraSource(rpc, new CameraClient(cfg.camera)), rpc)`(id=`'rpc'`).
- `cameraSources[]` 명시 경로(sim/hucoms)는 **무변경**(하위호환).

### 2.4 `src/viewer/routes.ts` — camerapos CRUD 라우트 (수정)

`cameraposFile` 주입 시에만 등록(가산·graceful, 미주입 시 라우트 없음).

- `GET /viewer/api/camerapos` (무게이트, 읽기): 파일 → `parseCameraViews` → `{ views: CameraView[] }`. 파일 없음/파싱 실패 → `{ views: [] }`.
- `PUT /viewer/api/camerapos` (controlToken 게이트, `/move`와 동일 선택 게이트):
  - body zod 검증 `CameraposBody`: `{ views:[{ camIdx:int≥1, presetIdx:int≥1, label:string, pan:number, tilt:number, zoom:number }] }`.
  - 서버측 **zoom clamp(1~36)** 후 `writeCamerapos`로 전체 파일 기록(중첩 datas 포맷 생성, 왕복 호환).
  - 응답 `{ ok:true, count }`. 잘못된 body → 400, 토큰 불일치 → 403.
- `ViewerDeps.cameraposFile?: string` 신설.

### 2.6 프론트 (`web/core.js` · `app.js` · `index.html` · `app.css`)

`web/core.js` 순수 함수(불변, 유닛테스트 대상):

## 3. 수정 로직의 의도와 동작 — 프론트 저장 흐름

`savePreset(asNew)` / `deletePreset()` → `persistCamerapos(views, keepPreset?)`:
1. `PUT /viewer/api/camerapos`(x-viewer-token 준용).
2. `await loadCameras()`(state.cameras 갱신).
3. `renderCamSelect()` **명시적 강제 재렌더**.

**강제 재렌더가 필요한 이유:** `camerasChanged()`는 presetIdx 집합만 비교(라벨/PTZ 무시) → PTZ만 수정하면 4초 폴이 재렌더하지 않는다. 저장 직후 즉시 반영을 위해 저장 경로에서 명시적으로 재렌더한다. 새 프리셋 저장 시 `keepPreset=presetIdx`로 재렌더 전 선택을 고정해 드롭다운에 즉시 추가·선택된다.

"현재 PTZ를 프리셋으로 저장"의 PTZ 소스 = **`state.ptz`(명령 기준)** — 프리셋 이동·PTZ 제어 버튼이 갱신하는 신뢰 원천(설계 가정 A3). 별도 `cam.getPTZ` 왕복은 도입하지 않았다(단순화). `gotoPreset`은 무변경 — camerapos 프리셋은 항상 PTZ 보유 → `findPresetPtz`가 값 반환 → 기존 `/move`(→`cam.setPTZ`)로 물리 이동.

## 4. 입출력 · REST 계약

| 라우트 | 게이트 | 요청 | 응답 |
|---|---|---|---|
| `GET /viewer/api/camerapos` | 없음(읽기) | — | `{ views: [{camIdx,presetIdx,label,pan,tilt,zoom}] }`(파일 없음/파싱 실패 시 `{views:[]}`) |
| `PUT /viewer/api/camerapos` | controlToken(설정 시 `x-viewer-token`) | `{ views:[{camIdx≥1,presetIdx≥1,label,pan,tilt,zoom}] }` | `{ ok:true, count }` · 400(body) · 403(token) |
| `GET /viewer/api/cameras` | 없음 | `?source=` | `CameraposSource.listCameras()` = camerapos 기반 목록(presets에 PTZ). cam.list 실패 → 502 |

**데이터 흐름:**
```
camerapos.json ──fresh read──▶ CameraposSource.listCameras ──▶ /viewer/api/cameras ──▶ 프론트 드롭다운
      ▲                                                                                        │
      │  PUT(writeCamerapos, zoom clamp)                                    편집(upsert/remove) │
      └───────────────── /viewer/api/camerapos ◀── persistCamerapos ◀───────────────────────┘

프리셋 이동: gotoPreset ─findPresetPtz(파일 PTZ)─▶ /viewer/api/move ─▶ cam.setPTZ(device 물리 이동)
```

`writeCamerapos`(PUT 기록) ↔ `parseCameraViews`(GET 읽기)는 왕복 호환(중첩 `datas` 포맷 ↔ 정규화 `views[]`).

## 5. 영향도 분석 (요약)

상세는 `_workspace/04_doc_impact.md` 참조. 핵심만:

- **4초 연결 폴(`loadCameras`)**: 유지. `CameraposSource.listCameras`가 파일 read + cam.list ping을 함께 수행 → 파일 변경·연결 상태 모두 반영. PTZ-only 편집은 `camerasChanged` 미감지 → 저장 흐름의 **명시적 `renderCamSelect()`** 로 대응.
- **setup / capture / calibrate**: 동일 camerapos 파일 사용 → 무변경. 웹 편집이 셋업 입력으로 자연 반영. `loadSetupTargets(camerapos)`로 소스 일원화.
- **presetProvider(unity-api/discovery)**: export 부트스트랩용으로 존치. `presetProvider.type=camerapos` + `refreshOnRun=false`로 **run 시 camerapos.json 자동 덮어쓰기 중단**(수동 편집 보존).
- **RPC `preset.*`(주차면 B)**: 뷰어 카메라 드롭다운에서 분리. RPC 콘솔(`/viewer/api/rpc`)·시뮬 3D 씬 용도는 무관·유지.
- **하위호환**: `cameraSources[]`(sim/hucoms)·`RpcCameraSource`(합성 대상) 무변경. `RpcCameraSource.listCameras`의 `preset.list`는 뷰어 폴백 경로에서 더 이상 호출되지 않으나 **삭제하지 않음**(요청 범위 밖, CLAUDE.md 규칙3 — RPC 콘솔/시뮬 3D 용도 존치).
