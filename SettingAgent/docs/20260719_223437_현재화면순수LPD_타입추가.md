# LPD 검지 "현재화면 그대로 순수 LPD"(lpd-live) 타입 추가

## 0. 배경 — 기존 "순수 LPD"의 프리셋 스냅 문제

`web/index.html`의 검지 방식 콤보(`#lpd-mode`)에는 기존에 `순수 LPD`/`앞면중심 LOOP`/`VPD→LPD` 3가지 타입이 있었다.
이 중 `순수 LPD`(`runModeLpd` → `runLiveDetect(false)`)는 서버 `runDetect`가 `resolvePresetPtz`로 **DB/카메라에 저장된 프리셋 PTZ**를 조회해 그 값으로 base 프레임을 재렌더한다.

문제: 사용자가 뷰어에서 팬/틸트/줌을 수동으로 조정한 뒤 `순수 LPD`를 눌러도, 검출은 항상 저장된 프리셋 PTZ로 스냅되어 렌더된 프레임을 기준으로 수행된다. 즉 **화면에 보이는 현재 앵글과 실제 검출에 쓰인 프레임이 다를 수 있다.**

마스터 확정 방식: "현재 PTZ로 재렌더 후 LPD" — 브라우저 캔버스 픽셀을 그대로 업로드하는 방식이 아니라, **뷰어가 들고 있는 현재 PTZ 값(`state.ptz`)을 서버에 전달해 그 PTZ로 카메라 프레임을 다시 렌더한 뒤 LPD-only 검출**을 수행하는 방식이다. 프리셋 스냅이 없다.

## 1. 변경/신규 함수 설명

| 파일 | 대상 | 변경 |
|---|---|---|

| `web/index.html` | `#lpd-mode` 콤보 | `<option value="lpd-live">현재화면 순수 LPD</option>` 신규(맨 끝 추가, 기존 3옵션 순서 불변). |

| `web/app.js` | 검지 방식 디스패처(`$('run-detect')` 클릭 핸들러 내부) | `else if (mode === 'lpd-live') runModeLpdLive();` 분기 추가. |

## 2. 수정 로직의 의도와 동작

### 2.1 서버 — `runDetect`의 base PTZ 결정 분기 (`src/capture/detectPipeline.ts`)

기존:
```ts
const presetPtz = await resolvePresetPtz(deps.camera, cam, preset);
const base = await deps.camera.requestImage(cam, preset, presetPtz ?? undefined);
const basePtz = presetPtz ?? { pan: base.pan, tilt: base.tilt, zoom: base.zoom };
```

변경 후:
```ts
const presetPtz = args.ptz
  ? { pan: args.ptz.pan ?? 0, tilt: args.ptz.tilt ?? 0, zoom: args.ptz.zoom ?? 1 }
  : await resolvePresetPtz(deps.camera, cam, preset);
const base = await deps.camera.requestImage(cam, preset, presetPtz ?? undefined);
const basePtz = presetPtz ?? { pan: base.pan, tilt: base.tilt, zoom: base.zoom };
```

핵심 의도: `args.ptz`가 제공되면 `resolvePresetPtz` 호출(DB/카메라 프리셋 조회) 자체를 **건너뛰고** 오버라이드 값을 `presetPtz`에 대입한다. 이렇게 하면:
- `requestImage(cam, preset, presetPtz)` — base 프레임이 오버라이드 PTZ로 렌더된다.
- `basePtz` — 이후 `inverseProjectQuad`(역투영)와 줌 재시도 루프(미검출 번호판 복구 시 `requestImage(cam, preset, { pan: basePtz.pan, tilt: basePtz.tilt, zoom })` 호출)가 참조하는 기준값이 되어, **오버라이드 PTZ가 파이프라인 전체에 일관되게 전파**된다.
- `vpdEnabled`, `onPlace` 필터, 육면체(cuboid) 로직 등 나머지 부분은 완전히 불변.

부분 필드(예: `{zoom: 8}`만 전달) 방어: `pan`/`tilt`/`zoom`을 각각 `?? 0 / 0 / 1`로 기본값 처리한다. 실사용에서는 프론트가 항상 `state.ptz`의 3필드 완전체를 전송하므로 이 기본값 경로는 방어적 장치다.

### 2.2 프론트 — `runLiveDetect`/`runModeLpdLive`/디스패처 (`web/app.js`)

- `runLiveDetect(vpdEnabled = false, ptz)`: 기존 호출부(`runLiveDetect()`, `runModeLpd()`의 `runLiveDetect(false)`, `runModeVpd()`의 `runLiveDetect(true)`)는 `ptz` 인자를 넘기지 않으므로 `if (ptz) body.ptz = ...` 분기를 타지 않는다 — body에 `ptz` 키 자체가 없어 서버 옵셔널 파싱이 기존과 동일하게 프리셋 경로로 흐른다.
- `state.ptz`는 프리셋/시뮬레이터 경로에서는 number이지만 실카메라(`/ptz` 응답) 경로에서는 문자열일 수 있어, body 구성 시 `Number(ptz.pan)` 등으로 명시 변환한다. 서버 zod 스키마가 `z.number()`를 요구하므로 문자열이 그대로 전달되면 400이 난다 — 이 변환이 그 실패를 방지한다.
- `runModeLpdLive()`는 `runModeLpd()`와 동일한 오버레이 체크박스 상태(차량/점유/육면체/마스크 숨김, 번호판만 표시)를 설정한 뒤 `runLiveDetect(false, state.ptz)`를 호출한다 — VPD는 실행하지 않고(LPD 전용), 현재 뷰어 PTZ를 오버라이드로 전달한다.
- 검지 방식 디스패처에 `lpd-live` 분기를 추가해 콤보에서 신규 옵션 선택 시 `runModeLpdLive()`가 실행되도록 배선했다.

## 3. 입출력·REST 계약

### `POST /capture/detect`

**요청 body 확장(옵셔널 필드만 추가, 하위호환)**:
```jsonc
{
  "cam": 1,
  "preset": 1,
  "vpdOnParkingOnly": true,
  "vpdEnabled": false,
  "ptz": { "pan": 42.5, "tilt": -3.2, "zoom": 2 }   // 신규 옵셔널. 미지정 시 기존 프리셋 경로.
}
```
- `ptz`는 세 필드 모두 옵셔널(`z.number().optional()`). 미지정 필드는 서버에서 `pan/tilt/zoom` 각각 `0/0/1`로 기본 처리.
- `ptz` 자체를 생략하면 body shape·동작이 기존과 완전히 동일(회귀 0).
- 잘못된 타입(`ptz.pan: "x"`, `ptz` 자체가 숫자 등)은 zod가 거부 → `400 { error: 'invalid body' }`.

**응답(`DetectResult`)**: 기존 shape 불변. `basePtz` 필드가 `ptz` 오버라이드 제공 시 그 값과 정확히 일치(프리셋 PTZ 아님)한다는 점이 이번 변경으로 보장되는 계약이다.

### 프론트 → 서버 실제 호출 (`runModeLpdLive`)

```
POST /capture/detect
{ cam, preset, vpdOnParkingOnly: <#cap-vpd-onplace 체크값>, vpdEnabled: false,
  ptz: { pan: Number(state.ptz.pan), tilt: Number(state.ptz.tilt), zoom: Number(state.ptz.zoom) } }
```

## 4. 영향도 분석

`_workspace/04_doc_impact.md` 참조(요약은 아래와 동일, 상세는 그쪽 문서가 정본).

- **하위호환**: `runDetect` args, `DetectBodySchema`, `runLiveDetect` 시그니처 모두 옵셔널 필드/파라미터 추가이며 기존 3모드(`순수 LPD`/`앞면중심 LOOP`/`VPD→LPD`)와 자동 검출 경로(프리셋 순환 중 1회 자동 `runLiveDetect()`)는 `ptz`를 전달하지 않아 완전히 기존 동작으로 회귀한다.
- **VPD 정책 미접촉**: `runModeLpdLive`는 `vpdEnabled=false`로 고정 호출 — VPD 자동검출 금지 정책(기본 OFF)을 그대로 준수하며, 이 변경이 그 정책에 손을 대지 않는다.
- **미접촉 모듈**: `resolvePresetPtz`, `CaptureJob`, `CameraClient`/`RpcCameraClient`, MCP 서버, 다른 라우트(`/capture/start` 등)는 변경 없음.
- **확인 필요(미검증 항목)**: `requestImage(cam, preset, ptz)`가 실제로 카메라를 물리적으로 그 PTZ 위치로 이동시키는지 여부는 **카메라 소스 구현에 의존**한다. 시뮬레이터 소스는 `/req_img` 렌더 요청만으로 즉시 해당 각도 프레임을 반환하지만, 실카메라(Hucoms RTSP/PTZ) 소스는 `req_img`가 이동을 트리거하지 않고 별도 `req_move` 후 안정화 대기가 필요할 수 있다. 이번 세션에서는 활성 카메라 소스가 세션 도중 실카메라로 전환되어 이 물리이동 여부를 종단 관찰하지 못했다(§5 한계 참조) — 실카메라 소스에서 `lpd-live`를 실사용하기 전에 별도 확인이 필요하다.
