# 정밀수집 센터라이징 종합 정리 — 슬롯순서·저장·검출확실성 완결

> 2026-07-20. 정밀수집 페이지 센터라이징을 5차 반복(goal/loop)으로 개편. 최종 **17/17 검출·완주**.
> 세부 이력은 개별 문서(130900·173000·182500·210000) 참조. 본 문서는 전체 요약.

---

## 1. 개요

정밀수집 센터라이징(주차면 번호판을 화면 중앙 정렬 + 폭 20%까지 줌)에서 발생한
**중복/건너뛰기 · 저장 누락 · 검출 실패**를 순차 해결해, 최종적으로 **전 슬롯(17/17)이
번호판을 정확히 찾아 20~23% 폭에 수렴**하고 결과가 3중 저장되도록 완성했다.

**최종 라이브 결과(카메라 13110 simulator-1, 2회 재현):**
- 검출 17/17 · 완주 17/17 · 폭 0.202~0.222(전부 [0.20,0.23]) · 중복 0 · 저장 3중 정상.

---

## 2. 증상 (마스터 신고)

1. 한 주차면을 여러 번 센터라이징 / 다른 주차면 건너뛰기.
2. 최종 결과 JSON 미저장 · DB(slot_setup) 미저장.
3. 줌이 먼저 되고 팬/틸트가 나중 조정되는 것처럼 보임(순서).
4. 번호판 위치를 못 찾아 센터라이징 실패(정확한 검출이 최우선).

---

## 3. 반복 개발 여정 (근본원인 → 해결)

| 이터 | 근본원인(실측 확정) | 해결 |
|------|--------------------|------|
| **1. 선조준+저장** | 저장 게이트가 `centered && converged`만 저장 → 미수렴 슬롯 DB 누락. 최종 셋업 스냅샷 부재. | 슬롯별 **pre-aim**(LPD 박스 중심 선조준) + 저장 게이트 `centered`로 완화 + **`save/Setup_*.json`**(완전 셋업 스냅샷) 신규. 저장 3중(파일·DB·slot_ptz.json). |
| **2. 소유권 배타성** | `PtzCalibrator`가 모든 슬롯에 공유 시작 PTZ → 재검출 nearest가 **이웃 판에 latch** → 중복 수렴/건너뛰기. | 재검출 대상선정을 **소유권(Voronoi) 게이트**(`pickOwnedPlate` 재사용)로. 같은 프리셋 타 슬롯 lpd 중심을 peer로 주입, 이웃 판 기각. 좌표계는 강체평행이동 불변 + zoom 사전스케일. |
| **3. 순서 + config 노브** | 2단계 구조(pan/tilt→zoom)는 이미 순서 준수. "줌 먼저"는 zoom 단계 가드 재중심(물리 필수). | `maxZoomStepRatio`(점진 줌)·`centerZoom` config 노브 추가. **centerZoom=1 실험 실패**(판 너무 작아 미검)→프리셋 zoom 기본. |
| **4. 줌인 acquire + 사다리** | 넓은 시야에서 판이 작아 LPD 미검(넓은시야 센터가 실패근원). | **방안 2**: 게인무관 목표 zoom 직접산출(`Zt=presetZoom×targetW/lpdW`) → **먼저 확대해 찾기**. **방안 3**: acquire 미검 시 줌아웃 사다리 재포착. → 17/17 검출, 15/17 완주. |
| **4c. 게인 실측** | 남은 plate_lost 가설 = 게인 불일치? | **diagSweep 실측(3프리셋)** → gainPan -62~-64 / gainTilt -35.2~-35.8 = **config(-62/-35.5)와 ±3% 일치. 게인은 정확 = 범인 아님.** |
| **4d. 검출 사각지대(최종)** | **zoomDetect 실측**: slot5 등은 **zoom 6~10에 LPD 검출 사각지대**, acquireWidth 0.12가 계산한 zoom(~8)이 사각지대 정중앙에 떨어져 miss. | **`acquirePlateWidth` 0.12→0.18** — 게인이 정확해 pre-aim이 판을 중앙에 정확히 놓으므로 FOV 여유용 중간 zoom 불필요 → acquire를 목표 근처(zoom~12)로 올려 **사각지대 회피**. → **17/17 완주.** |

**핵심 교훈:** 세 가설(게인·매칭반경·검출)을 **추측 대신 라이브 실측**으로 하나씩 검증해 진짜 원인(중간 zoom LPD 사각지대)에 도달했다.

## 4. 최종 동작 흐름 (calibrateSlot)

```
슬롯 순서대로(카메라→프리셋→주차면번호):
  pre-aim(프리셋 시야에서 lpd 중심을 화면중앙으로 조준, 게인)
   → acquire(목표 근처 zoom~12로 줌인해 큰 판 검출·센터. 미검이면 줌아웃 사다리로 재포착)
      · 소유권 게이트로 이웃 판 배제(peerOffsets, 각 zoom 사전스케일)
   → width(zoomToPlateWidth로 20~23% 폭 마감. plate_lost 시 줌아웃 재포착+고운스텝 복구)
   → 저장 3중(slot_ptz.json + DB slot_setup 부분UPDATE + save/Setup_*.json 스냅샷)
```
