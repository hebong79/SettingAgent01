## 1. 세션 개요 — 방향성

이 세션은 **"VPD(차량 검출) 이탈 → LPD(번호판) 중심 셋업"** 으로 SettingAgent 를 재편했다.

- **핵심 산출물** = 주차장관리 에이전트용 **setup 파일**: 각 주차면 슬롯의 **번호판 인식 최적 PTZ**(`slot_ptz.json` + DB `centering_slot`/`slot_setup`).
- **확립된 원칙**(마스터 확정): 결정론 우선·LLM 최후 / 셋팅 전용(누적·저속 허용) / VPD 자동검출 금지·LPD 중심 / 실 PTZ 카메라로 전이 예정(3D 트윈).
- 관련 메모리: `vpd-auto-detect-forbidden`, `settingagent-llm-minimized`, `finalize-slotsetup-wipe-fragility`, `settingagent-overlay-retain-policy`, `prefers-goal-loop-mode`, `autonomous-execution-no-approval`.

## 3. 확립된 정책·제약 (새 세션이 반드시 지킬 것)

1. **VPD 자동검출 금지**: 정밀수집 라운드·`/capture/detect`는 기본 LPD-only(`vpdEnabled` 라우트/UI 기본 false). VPD 는 "VPD 검출(테스트)" 경로에서만. 새 검출 경로 만들 때 VPD 기본 포함 금지. (메모리 `vpd-auto-detect-forbidden`)
2. **결정론 우선·LLM 최후**: 점유/검지 로직은 기하+검출로. LLM 은 잔여 애매 처리에만(현재 discovery·센터링·점유 전부 LLM 미사용).
3. **slot_setup DB 쓰기**: `replaceSlotSetup`(DELETE+INSERT)은 검출 없는 finalize 시 데이터 파괴 위험. 신규 writer 는 **slot_id 부분 UPDATE** 사용(예: `upsertSlotLpd`). (메모리 `finalize-slotsetup-wipe-fragility`)
4. **영속화 수치 5자리**: slot_setup TEXT 등 writer 는 `stringify5` 필수. (메모리 `settingagent-persist-5decimals`)
5. **오버레이 유지**: 정밀수집 종료 후 오버레이 삭제 금지. DB 소스는 `#roi-db` 체크 시. VPD 는 #roi-db 시 소스 전환(replace). (메모리 `settingagent-overlay-retain-policy`)
6. **개발 방식**: 코드 변경은 parkagent-dev 하네스(goal/loop), 중간 승인 요청 없이 끝까지. 관찰형은 리더 sharp 렌더/라이브 라우트로 검증.

## 4. 핵심 아키텍처·데이터 흐름

### 셋업 파이프라인
```
정밀수집(인메모리) → 최종화(finalize) → 센터라이징(calibrate) → setup 저장
  · 최종화: setup_artifact.json + slot_setup DB(slot_roi·slot3d_front_center 기하 / vpd·lpd·occupy 검출)
  · 센터라이징: slot_ptz.json + centering_slot + slot_setup PTZ(pan/tilt/zoom/centered/img1)
  · autoChain=true 면 3단계 백엔드 자동 연쇄(SetupPipeline)
```

### LPD 검지 3모드(콤보박스 `#lpd-mode` + 실행 `#lpd-run`)
- `lpd` 순수 LPD: 비-LPD 오버레이 off + `runLiveDetect(false)`(현재 프리셋 1회).
- `discover` 앞면중심 LOOP: `POST /discover/ptz {cam,preset}` → 현재 프리셋 슬롯 순회, 앞면중심 크롭-줌 LPD → 완료 후 찾은 박스 표시. **검출 무관 앵커**(slot3d_front_center 기하).
- `vpd` VPD→LPD: `runLiveDetect(true)`.

### discovery 역계산 (검출 미스 구제의 핵심)
- 앞면중심 기준 크롭(frac0=0.40→shrink 0.6→minFrac 0.05, maxSteps 5) → 장변 1920px 업스케일 → LPD → 아핀 역계산 `orig = W.xy + q·W.wh`(업스케일 배율 무관, 오차 0) → 원본 프레임 LPD OBB(`lpdOrig`) → slot_setup.lpd.
- **광학 PTZ tier(진짜 저해상도)는 설계만 있고 미구현**(G_i 유도 미확정).

### LPD API 입력 이미지 크기
- 순수 LPD / VPD→LPD: **1920×1080 캡처 원본 그대로**(리사이즈 없음).
- discovery: Tier0 원본 1920×1080 → 실패 시 Tier1 크롭을 **장변 1920px로 업스케일**(다운스케일 없음).

## 5. "100% 검지" 논의 결론 (재개 시 참고)

- **순수 비전으로 증명된 100%는 불가**(통계적). 앞면중심 discovery 는 검출 미스를 크게 구제하나 100% 보장 아님.
- **100% 경로**: (A) **시뮬 정답 오라클** — 결정형 100%이나 **현재 Unity 인터페이스에 점유/차량위치 엔드포인트 없음**(추가 요청 필요, 실카 전이 불가라 오프라인 검증용으로만). (B) **자동초안 + 사람 확정 게이트**(셋팅이라 실효 100%).
- **실 PTZ 전이**: 기하(앞면중심)·VLM/검지는 전이됨, 시뮬 정답은 오라클(튜닝·검증)로만. 실카 신뢰도는 도메인갭으로 sim 성적이 보장 못 함 → 실데이터 eval 필요.
- 상세: `docs/20260719_113423_VPD점유판정_*.md`, `docs/20260719_123634_셋업PTZ_목표재정의_*.md`, `docs/20260719_153316_VPD배지_LPD기하산출_*.md`.

## 6. 미해결 / 다음 단계 (우선순위 제안)

1. **discovery 실동작 라이브 검증** — 실 LPD(응답 중)·카메라로 `앞면중심 LOOP` 실행 → 검지율·박스 정합·slot_ptz 홀 감소 실측. (지금까지 순수 로직·역계산은 검증, 실 검지율 미관찰)
2. **discovery → 센터라이징 자동 연결**(과업 A2): discovery 가 채운 lpd 를 센터링 대상으로 → slot_ptz 홀 제거. 원버튼 파이프라인에 discovery 단계 삽입 검토.
3. **광학 PTZ tier**(discovery Phase 3): 진짜 저해상도 슬롯용. 역계산 G_i(임의 PTZ 지면모델) 유도 확정 필요.
4. **후면주차 대응**: 앞면중심 prior 는 전면주차 가정 → 앞/뒤 후보 박스 또는 주차방향 정보.
5. **수동 finalize 경로 stale/wipe 가드**: autoChain 경로만 F10 가드 있음. 수동 [최종화] 는 dets 0 finalize 가능(센터링 컬럼 리셋 취약).
6. **VPD off finalize "슬롯 0" UX 갭**: vehicle 앵커 없어 setup_artifact slots 빈 배열 → "최종화 완료: 슬롯 0" 이 실패처럼 보임(정본 slot_setup 기하는 정상 기록). 메시지·검수탭 정리.
7. **health.vpd 배지 오탐**: `deps.vpd.health()`는 `/health` 핑(detect 경로와 별개) → detect 는 되는데 배지 false 가능. detect 기준 판정으로 보정 검토.
8. **백엔드 카메라소스 테스트 flakiness**(범위 밖): 전량 병렬 실행 시 공유 `config/tools.config.json`·`data/` 경합으로 7건 flaky(직렬은 통과). 별도 안정화 대상.
