## 1. Goal / Requirements

### Goal
정밀수집 프레임에서 **캡처 파이프라인과 무관하게** on-demand 로 차량 점유 판단에 쓸 정보를 획득·보관·표시한다.
- **VPD**(차량 검출)로 차량 bbox(정규화 rect),
- **LPD**(번호판 검출)로 번호판 OBB(정규화 4점 quad)를 얻어
- 차량↔번호판을 귀속시키고, 번호판을 못 찾은 차량은 앞쪽을 당겨 zoom 확대 후 재검출하여 복원한다.

### Requirements (요구 → 대응)
| 요구 | 대응 |
|---|---|
| **R1** VPD·LPD 모두 화면 표시 | `drawDetectOverlay` — VPD rect(청록), 귀속/복원 번호판 quad, **매칭 안 된 base LPD 전체**까지 렌더. `#roi-detect` 토글. |
| **R3** 번호판 미검출 차량 → 앞쪽 확대 후 재검출, 실패 시 최대 4회 | 미귀속 차량마다 `vehicleCenterZoomPtz`로 재중심+zoom → LPD 재검출. `ZOOM_FACTORS=[2,3,4,5]`(4회) 소진 시 종결. |
| **R4** zoom 검출 번호판을 base 좌표로 역산(중심 근사) | `inverseProjectQuad`로 뷰 정규화 → base 정규화 역투영. `recovered:true`. |
| **R2** 프리셋마다 체크·정상까지 반복 후 다음(프리셋당 10회) | **라우트는 1프리셋 1회 검출(멱등)**만 수행. 반복·10회 상한은 리더/프론트가 라우트를 재호출하여 소유(라우트 밖). |

## 2. 파이프라인 (요청 → 반환)

```
POST /capture/detect { cam, preset }
  → resolvePresetPtz(GET /cameras) 로 프리셋 실제 PTZ 조회
  → camera.requestImage(cam, preset, presetPtz) 로 base 프레임 렌더  ← [핵심] echo 0/0/1 불신
  → basePtz = presetPtz (조회 실패 시에만 echo 폴백)
  → VPD.detect(base) + LPD.detect(base)
  → matchPlatesToSlots(VPD박스 어댑트, platesBase) 로 차량↔번호판 귀속
  → 귀속 차량: plate {quad, conf, recovered:false, attempts:0}
  → 미귀속 차량마다 zoomFactors [2,3,4,5] 순회:
       vehicleCenterZoomPtz(rect, basePtz, factor) → clampZoom
       → requestImage(재중심 PTZ) → 실적용 viewPtz
       → LPD.detect(view) → pickNearestPlate(뷰 중심 최근접)
       → pick 있으면 inverseProjectQuad(pick, viewPtz, basePtz)
                    → clampQuadCenterToRect(recQuad, rect)  ← 차량 밖 이탈 보정
                    → recovered:true, attempts:a+1, break
       → 4회 실패 시 plate:undefined
  → 반환
```

### 반환 shape (`DetectResult`)
```ts
{
  imageSize: { w, h },
  cam, preset,
  basePtz: { pan, tilt, zoom },              // 프리셋 실제 PTZ(echo 아님)
  vehicles: [{ rect, confidence, cls,
               plate?: { quad, confidence, recovered, attempts } }],
  plates:  [{ quad, confidence }],           // base 프레임 LPD 전체(매칭 무관 표시용)
  summary: { vpdCount, lpdCount, recovered }
}
```

- 라우트: `POST /capture/detect`, body `{cam:int+, preset:int+}`. zod 실패 → 400, `runDetect` 예외 → 502.
- **camera·vpd·lpd 3개 모두 주입 시에만 라우트 등록**(가산 가드). 미주입 시 라우트 자체가 없어 404.
- 검출 결과는 **DB 미기록** — 표시·보관(프론트 `state.detect`)만.

## 3. detectMath 공식 (`src/calibrate/detectMath.ts`, 신규 순수 모듈)

정규화 좌표 `(u,v)∈[0,1]`, **좌상단 원점**(u→우 / v→하). pan/tilt 는 도(°), zoom 은 배율.

### FOV ↔ zoom (수직 fov 기준, Unity `Camera.fieldOfView`=vertical 가정)
```
fovV(z) = 2·atan( tan(fovBaseV/2) · zoomRef / z )
fovH(z) = 2·atan( tan(fovV(z)/2) · aspect )
```
- `fovBaseV` = `PtzCamRoi.json camera.fov`(=24.017), `zoomRef=1`, `aspect=imageWidth/imageHeight`.
- 성질: `fovV(1)===fovBaseV`(atan∘tan 항등), zoom↑ → fov↓ 단조, `fovH>fovV`(aspect>1).

### 재중심 PTZ — `vehicleCenterZoomPtz(rect, base, opts)`
rect 는 **top-left 규약** `{x,y,w,h}`. 앞쪽(번호판=차량 하단) 타깃점:
```
cx = rect.x + rect.w/2
cy = rect.y + rect.h·frontBias         (frontBias=0.62, >0.5=하단=앞)
pan  = base.pan  + (cx−0.5)·fovH(base.zoom)     // panSign = +
tilt = base.tilt + (cy−0.5)·fovV(base.zoom)     // tiltSign = +  ★리더 실측 확정
zoom = base.zoom · zoomFactor                   // clampZoom 은 호출측(runDetect)
```

### 역투영 — `inverseProjectPoint(view, viewPtz, base, opts)`
```
wp = viewPtz.pan  + (view.x−0.5)·fovH(viewPtz.zoom)
wt = viewPtz.tilt + (view.y−0.5)·fovV(viewPtz.zoom)     // tilt항 부호 +
x  = 0.5 + (wp − base.pan)/fovH(base.zoom)
y  = 0.5 + (wt − base.tilt)/fovV(base.zoom)             // tilt항 부호 +
```
- `inverseProjectQuad` = 4점 각각 `inverseProjectPoint`. `projectBaseToView`(역방향)은 라운드트립 테스트 전용 export.
- 순수함수 자기일관성: `projectBaseToView → inverseProjectPoint` 왕복 ±1e-9 복원(vitest).

### 차량 밖 보정 — `clampQuadCenterToRect(quad, rect, frontBias=0.62)`
- quad 중심(cx,cy)이 rect(`[x,x+w]×[y,y+h]`) 안이면 **무변경**(원 참조 그대로 — 양호한 역산 유지).
- 밖이면: x 는 nearest-edge 클램프. y 는 **상단 이탈 시 rect top 이 아니라 앞쪽(전면=하단) `rect.y+rect.h·frontBias` 근방으로 당김**, 하단 이탈은 rect 하단 경계.
- 클램프 델타만큼 **4점 전체 평행이동**(모양·크기·각도 보존).
- rect **top-left 규약** 명시: 지시문 예시식 `[rect.x−w/2, rect.x+w/2]`(중심 오인)이 아니라 코드베이스 실 데이터 규약 `[rect.x, rect.x+rect.w]`로 구현(로직 의도 동일).

## 5. Goal/Loop 경험적 검증 서사 (핵심 — 리더 라이브 3프리셋 순회로 발견·교정한 버그 3건)

설계 §04 는 tilt 부호(+)를 실측으로 이미 확정한 채 착수했으나, 실제 시뮬레이터(13100) 라이브 검출을 프리셋 1·2·3 순회하며 **초기 구현이 방치했던 좌표계 3대 결함**이 드러났고 loop 재수정으로 교정했다.

### 버그 1 — tilt 부호 반전 (설계 착수 전 실측 확정)
- **증상**: 재중심 뷰가 지면이 아니라 **하늘을 향해** zoom-in → 번호판 검출 실패.
- **원인**: 초안의 tilt `−` 부호. tilt↑ 시 지면 특징점이 화면 **위**로 이동(=카메라가 아래를 봄)이므로, 하단 차량(cy>0.5)을 중심에 두려면 tilt 를 **증가**시켜야 함.
- **교정/근거**: `tiltSign=+`로 확정(A2/A3 전체 반영). 이 부호로 zoom 뷰 번호판 검출 성공 실증. pan(+, pan↑=우)·frontBias(0.62)는 정상.

### 버그 2 — basePtz 가 항상 0/0/1 (프리셋 PTZ 미전달로 preset 2·3 차량 0) — §02-F
- **증상**: preset 1 은 정상 검출하나 **preset 2·3 은 차량 0**, 응답 `basePtz` 가 항상 `{0,0,1}`.
- **원인**: `runDetect` 가 base 프레임을 `requestImage(cam, preset)`로 **PTZ 미지정** 요청 → 시뮬 `/req_img` 는 ptz 생략 시 기본 0/0/1 로 렌더(`preset_idx`만으론 시야를 안 잡음) → preset 2·3 은 빈 시야, echo 도 0/0/1 이라 재중심·역투영 기준 전체가 오염.
- **실측 근거**: 프리셋 실제 PTZ 를 `/req_img`에 넘기면 preset1(pan22,tilt6.8,zoom1.6)→**8대**, preset2(pan56.6,tilt7.4,zoom1.9)→**8대**, preset3(pan43.5,tilt18.8,zoom1.4)→**4대** 정상.
- **교정**: `resolvePresetPtz(camera.listCameras())`(뷰어·`findPresetPtz`와 동일 `GET /cameras` 소스 재사용, 신규 의존 0)로 프리셋 PTZ 조회 → base 요청에 명시 전달. `basePtz=presetPtz`(신뢰 원천 우선, 조회 실패 시에만 echo 폴백). `DetectDeps.camera` Pick 에 `listCameras` 추가.

### 버그 3 — recovered 역산이 차량 밖으로 이탈 (지면 원근) → clamp — §02-G
- **증상**: §02-F 수정 후 basePtz 는 정상(p1 recovered 2, p2 recovered 3, p3 0)이나, 복원 번호판의 **역산 표시 위치가 일부 차량(box트럭·먼 차량)에서 차량 밖(빈 주차공간)으로 크게 이탈**.
- **원인**: 설계 §04-A3 명시 한계 — 지면 원근으로 인한 per-axis fov 역산의 **y축 계통 오차 ~10~12%**(근본 해소 불가, 정밀 역산 미추구가 설계 확정 사항).
- **교정**: `clampQuadCenterToRect`(신규 순수 헬퍼)로 복원 quad 중심을 zoom-in 대상 차량 rect 안으로 클램프(모양 보존 평행이동). "어느 차량에서 zoom-in 했는지 정확히 안다"는 사실을 활용해 표시 위치만 차량 위로 고정. base 매칭(recovered:false)은 무변경.

## 7. 한계 정직 기록 (통과로 위장하지 않음)

- **역산 y축 계통오차 ~10~12%** (지면 원근): 선형 fov·tan 핀홀 **둘 다** 동일 ~11% 왕복 오차(실측). 주차장 지면 원근이 y축을 지배 → per-axis fov 모델로 완전 보정 불가(fovBaseV/zoomRef 튜닝 무효). **근본 미해소·설계 확정** — `clampQuadCenterToRect`로 **표시 위치만** 차량 위 고정. 복원 번호판은 정확히 아는 zoom-in 차량에 귀속되므로 점유 판단엔 충분(R4 "중심 근사면 됨").
- **스퍼리어스 초대형 VPD 박스 1건**(conf 0.69): 신뢰도 필터는 정상 트럭이 conf 0.25 로 탈락할 위험이라 미적용, 크기 휴리스틱은 추측성이라 미적용(CLAUDE.md 2항) — **검출기(VPD) 자체 한계**로 남김.
- **가림(occlusion) 예외 LLM 판단은 이번 스코프 밖**(요구 명시). 전 과정 결정형, 새 두뇌/도구 경계 없음.
- **LLM 미사용**: VPD/LPD 검출·zoom 재시도·fov↔zoom·역산·매칭 모두 실시간·수치반복 → MCP 결정형 경계.
- **실 프레임 정합**: vitest 는 외부 서비스 모킹(계약·결정형 로직만). 실 VPD/LPD/시뮬(9081/9082/13100) 연동은 리더 경험적 스모크(sharp 합성 렌더)로 확인 — Canvas DOM 결선(`drawDetectOverlay`/토글)은 vitest 비대상.
