## 1. Goal / Requirements

**Goal.** 정밀수집 **최종화(finalize)** 시, 파일 바닥ROI(`PtzCamRoi.json`)에 미리 정의된 주차면을 기준으로 각 주차면의 VPD(차량 bbox)·LPD(번호판 quad)·ROI(정규화 폴리곤)·점유 여부를 **카메라·프리셋별로 DB에 저장**하고, 그 리스트를 뷰어 UI에 노출한다.

| ID | 요구사항 |
|----|----------|
| R1 | 기존 VPD/LPD 화면 표시(라이브 검출 오버레이·§04 레이어) 유지 — 저장 기능이 표시를 훼손하지 않는다. |
| R2 | 최종화 시 VPD·LPD·ROI·점유를 **카메라·프리셋별로 DB 저장**(신규 `parking_slots` 테이블). |

## 2. 리더 확정 해석 (A안)

여러 해석이 가능한 지점(트리거 시점·데이터 소스·저장 위치)을 리더가 **A안**으로 확정:

- **트리거 = 최종화 1회**: 라이브 검출(§04)이나 체크포인트가 아니라, `finalize` 호출 시 1회 조립·저장. (수집 중 매 라운드 저장은 범위 밖 — 최종 산출물의 스냅샷 성격.)
- **소스 = 파일 바닥ROI(`PtzCamRoi.json`)**: 주차면의 개수·idx·폴리곤은 **파일**이 정의한다(LLM floor·집계 클러스터가 아님). 파일이 "몇 개의 면이 어디에 있는가"의 정본.
- **저장 위치 = 신규 `parking_slots` 테이블**: 기존 `setup_artifact.json`(정본 artifact)·`aggregated_slot`(집계)와 **분리된 별도 테이블**. 부수 산출물이라 artifact 정본 흐름을 건드리지 않는다.

## 3. `parking_slots` 스키마 (H1)

`SqliteStore.ts` L91–100:

```sql
CREATE TABLE IF NOT EXISTS parking_slots (
  run_id INTEGER, cam_idx INTEGER, preset_idx INTEGER, preset_key TEXT,
  slot_idx INTEGER,
  roi_json TEXT,          -- 파일 폴리곤 정규화 점 [{x,y}...]
  vpd_json TEXT,          -- 차량 bbox {x,y,w,h} (nullable)
  lpd_json TEXT,          -- 번호판 quad [{x,y}×4] (nullable)
  occupied INTEGER,       -- 0/1
  occupancy_rate REAL,    -- nullable
  updated_at TEXT,
  PRIMARY KEY (run_id, preset_key, slot_idx)
);
```

- PK `(run_id, preset_key, slot_idx)` → 한 run 안에서 프리셋키+파일 idx 조합이 유일. `preset_key`는 `${cam_idx}:${preset_idx}`(프론트 `presetKey(cam,preset)`와 동일 형식).
- `roi_json`은 **항상** 저장(파일 폴리곤 정규화값). `vpd_json`/`lpd_json`/`occupancy_rate`는 배정 성공 시에만 값, 실패 시 `null`.
- 저장/조회 메서드(`SqliteStore.ts`):
  - `replaceParkingSlots(runId, rows)` — run 기준 **delete+insert 트랜잭션**(멱등, `replaceAggregatedSlots` 미러). `*_json`은 이미 직렬화된 문자열.
  - `getParkingSlots(runId)` → `ParkingSlotView[]` — `roi_json`/`vpd_json`/`lpd_json`을 `parseJsonOrNull`로 복원, `occupied` 0/1 → boolean, `ORDER BY cam_idx, preset_idx, slot_idx`.
- 타입(`types.ts`): 저장 행 `ParkingSlotRow`(L80, `*Json` 문자열·`occupied` 0/1), 조회 shape `ParkingSlotView`(L94, `roi`/`vpd`/`lpd` 파싱·`occupied` boolean).

## 4. 데이터 출처 (D1~D3)

파일은 "면의 위치(idx·폴리곤)"만 정의한다. 각 면의 VPD/LPD/점유는 다음으로 채운다(`Finalizer.ts` L207–240):

- **D1 — VPD/LPD = accepted 집계 클러스터 재사용**: 최종화가 이미 만든 `accepted`(채택 집계 클러스터, 프리셋별)를 재사용. 새 프레임·새 LLM 호출 없음. 각 면에 배정된 클러스터의 대표 bbox(`{x,y,w,h}`)가 VPD, `plateQuad`가 LPD.
- **D2 — idx↔클러스터 배정 = `pointInPolygon`**: 각 파일 폴리곤 `sp.points` 안에 클러스터의 **번호판 중심(`quadCentroid(plateQuad)`) 우선, 없으면 차량 중심(`x+w/2, y+h/2`) 폴백**이 들어오면 그 클러스터를 그 면에 배정(`clusters.find(...)`). 즉 "번호판/차량 중심 ∈ 파일 폴리곤".
- **D3 — 점유 = 배정 유무(백엔드 재계산)**: 배정된 클러스터가 있으면 `occupied=1`+VPD/LPD/rate, 없으면 `occupied=0`+null. 점유는 프론트 로직(`computeOccupancy`, §05)이 아니라 **최종화 백엔드가 파일 폴리곤 기준으로 독립 재계산**. `slot_idx`는 파일 원본 idx 단독(slotId 미연동).

## 8. 한계 (정직 기록)

- **VPD/LPD가 accepted 클러스터 재사용**: `clusterMinSupport`(기본 3) 미만인 짧은 런에선 채택 클러스터가 없어 면에 배정이 안 될 수 있다 → 그 면은 `occupied=0`·vpd/lpd null. **단 ROI는 항상 저장**(파일 폴리곤은 검출과 무관).
- **최종화 best-effort**: 파일ROI 없음/파싱 실패/저장 예외 시 **graceful skip** — parking_slots 미저장, **정본 artifact 흐름은 불변**(이미 저장 완료 후 실행). 저장 실패는 `logger.warn`만.
- **H7 UI 렌더는 vitest 밖**: `renderSlotList` DB 분기·`loadParkingSlots` DOM 결선은 브라우저 통합이라 유닛 비대상 → 라우트 데이터·리더 라이브 E2E(§6)로 확인.
- **idx↔기존 slotId 미연동**: `slot_idx`는 파일 원본 idx 단독. 최종화 artifact의 `slotId`(`c{cam}p{preset}s{pos}`)와 매핑되지 않아 목록 클릭 선택·하이라이트 없음(표시 전용).
- **`parking_slots`는 기존 산출물과 분리**: `setup_artifact.json`(정본 artifact)·`aggregated_slot`(집계)와 별개 테이블. 서로 참조·동기화하지 않는다(부수 스냅샷).
