# 센터라이징 순서 엄수 — Pan/Tilt 중앙정렬(넓은시야) 완료 후 Zoom 확대

> 2026-07-20 goal/loop 이터3. 카메라 13110(simulator-1) 라이브 실증.

## 마스터 요구
① 먼저 Pan/Tilt로 번호판을 화면 중앙에 완전 정렬(넓은 시야) → ② 그 후 zoom만 점진 확대해 번호판 폭이 화면 20%(targetPlateWidth=0.2)가 될 때까지. 반드시 이 순서. ("줌이 먼저 되고 팬/틸트를 나중 조정"으로 보임)

## 진단 (로그 실증)
- 구조는 이미 2단계: `calibrateSlot` = `centerOnPlate`(pan/tilt, zoom 고정) → `zoomToPlateWidth`(zoom 확대). 즉 pan/tilt 먼저, zoom 나중이 구조적으로 성립.
- "줌 먼저" 관측의 정체 = **`zoomToPlateWidth`의 가드-선행 재중심**: zoom-in 은 중심오차를 배율만큼 확대하므로, 매 반복 중심이 tol 밖이면 zoom 을 보류하고 1스텝 재중심한다. 스텝이 크면(기본 1.5×) 드리프트가 커 재중심이 잦음 → "줌 뒤 팬틸트"로 보임(물리적으로 필요한 안전장치).

## 수정 (config 노브 2개, 코어 무접촉)
`calibrate` config:
| 노브 | 의미 | 기본 |
|------|------|------|
| `centerZoom` | 1단계 pan/tilt 센터링을 수행할 넓은시야 zoom override | 미지정=프리셋 zoom |
| `maxZoomStepRatio` | 2단계 zoom 1스텝 최대 증배 — 작을수록 드리프트↓ → 줌 중 재중심(pan/tilt) 빈도↓ = "줌 우세" | 1.3 (기존 1.5) |

- `PtzCalibrator.calibrateSlot`: 1단계 시작 zoom = `cfg.centerZoom ?? baseStart.zoom`(넓은시야). `baseOpts`가 `maxZoomStepRatio` 전달.
- **centerZoom=1 실험 실패**: 라이브 결과 판이 너무 작아 LPD 미검(2/17) → 프리셋 zoom(판이 검출되는 넓은시야) 기본으로 철회. centerZoom 은 실카메라 광각 튜닝용 옵션으로 존치.

## 라이브 검증 (13110 simulator-1)
- **순서 확인**: center 단계 프리셋 넓은시야(zoom~2.03)에서 수렴 → zoom 8~17 확대. `phase:center` → `phase:zoom` 순.
- **중복 0**: 같은 프리셋 내 근접 PTZ 쌍 없음(이터2 소유권 게이트 정상).
- **폭**: converged 슬롯 plateWidth ~0.18~0.20(목표 0.20 충족).
- **검출 10~13/17**: no_plate/plate_lost 7건 잔존.

## 남은 한계 (은닉 금지)
- 검출 실패 7건(no_plate: slot2·3·13 / plate_lost: slot1·5·11·17)은 **순서와 무관**. 원인 = fallback 게인 −62/−35.5 가 **cam1 실측값**이라 이 시뮬 인스턴스의 pre-aim·줌중 재중심과 안 맞아 판을 못 맞춤(방향/배율 오차). 완전 해결은 **이 카메라의 게인 diagSweep 재실측**이 필요(별개 과업·이월).
- 실 PTZ 물리 이동은 라이브로 봤으나, 실카메라(Hucoms) 및 타 카메라 게인 정확도는 미검증.
