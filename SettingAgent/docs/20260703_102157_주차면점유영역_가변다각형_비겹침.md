## 1. 배경 / 요구사항

기존 차량 바닥 점유영역(floor ROI)은 **고정 4점 사변형**(`NormalizedQuad`, `[FL,FR,RR,RL]`)으로 표현되었다. 실 주차장 CCTV에서 차량 접지 footprint 는 원근·차량 형상에 따라 4점으로는 부정확하고, 인접 주차면과 겹쳐 표시되는 문제가 있었다. 마스터 요구 4개(첨부 목표 이미지: 초록 폴리곤이 노란 번호판 박스를 포함하며 인접 슬롯과 밀착하되 겹치지 않음)를 만족하도록 확장한다.

| # | 요구 | 성격 | 구현 담당 |
|---|------|------|----------|
| R1 | 여유있는 크기(마진) — 번호판 박스가 항상 ROI 안 | 결정형 | `applyMargin` + `expandPolygonToContainRect` |
| R2 | 가변 정점 4~10점 | LLM 생성 + 결정형 정규화 | 프롬프트 + `normalizePolygon` |
| R3 | 번호판이 없어도 예상 위치까지 포함 | 결정형 | `predictPlateRect` + 포함강제 |
| R4 | 같은 프리셋 내 슬롯 간 비겹침(hard 보장) | 결정형(신규 핵심) | `deconflictPolygons`(Finalizer) |

**MCP 두뇌 vs 결정형 경계**: 좌표를 "생성"하는 것만 LLM(가변 정점 판단). 검증·클램프·정렬·마진·번호판 포함강제·비겹침은 전부 결정형(순수 기하). 특히 R4(비겹침)는 프롬프트로 보장 불가하므로 반드시 후처리(Finalizer)에서 수학적으로 보장한다.

## 2. 설계 결정(리더 확정 5)

| # | 결정 | 근거 |
|---|------|------|
| 1 | **우선순위 R3(번호판) > R4(비겹침) > R1(마진)** | 서로 다른 차량의 번호판 박스는 현실적으로 겹치지 않으므로, 비겹침 분리선을 두 번호판 사이 창으로 이동시키면 R3·R4 동시 만족. 마진은 겹침 구역에서만 희생. 병리적으로 두 번호판이 겹치면 R4 우선 + `logger.warn` 강등. |
| 2 | **볼록껍질 정렬** | 반평면 클리핑(Sutherland–Hodgman)이 볼록에서 견고. 주차 footprint 는 실제로 볼록 사다리꼴이라 오목 손실 무해. 4점 입력 시 기존 `[FL,FR,RR,RL]` 순서 재현(하위호환). |
| 3 | **`polygon_json TEXT` DB 컬럼 병행** | `floor_roi` 테이블에 컬럼 추가(신 테이블 없음). 구 `x0..y3` 존치 → 마이그레이션 최소·구 DB 읽기 유지. |
| 4 | **비겹침 = Finalizer 최종화에서만 보장** | 프리셋 전체 폴리곤이 한자리에 모이는 유일 지점. 라이브(FloorRoiReviewer)는 슬롯 단위·상한이라 프리셋 전체 미보장 → 비겹침 미적용(관측용, 일시 겹침 허용). |
| 5 | **정점 증감 UI 범위 제외(defer)** | 산출물은 LLM+결정형으로 생성. 수동 정점 추가/삭제 UI 는 이번 범위 밖. 기존 정점 드래그만 N정점 일반화. |

## 3. 신규/변경 구조 설명

### 3-1. 공유 타입 — `packages/types/src/index.ts`
- **신규** `NormalizedPolygon = NormalizedPoint[]`(런타임 불변식 4~10점, 정규화 0~1, 볼록·시계방향).
- **변경(폭 넓힘)** `ParkingSlot.floorRoiByPreset?: Record<string, NormalizedQuad>` → `Record<string, NormalizedPolygon>`. `NormalizedQuad`(4점 튜플)는 구조적으로 `NormalizedPolygon`(점 배열)에 할당 가능 → **구 4점 데이터 하위호환**.
- `NormalizedQuad` **존치**: 번호판 LPD OBB(`plateRoiByPreset`)·detection 은 계속 고정 4점. floor 만 polygon 화.

### 3-2. 순수 기하 모듈 — `src/domain/polygon.ts`(신규, 외부 의존 0)
테스트 용이한 순수 함수 9종(전부 export). 좌표계 0~1, 소형 n(≤10)이라 자작 순수 TS로 충분(라이브러리 미추가).

| 함수 | 역할 |
|------|------|
| `convexHull` | Andrew monotone chain. 중복·공선 정점 제거, NaN/Inf 필터. 점≤2면 그대로. |
| `polygonArea` / `polygonSignedArea` | shoelace 절댓값 / 부호면적(감김 방향 판정). |
| `polygonCentroid` | 면적가중 무게중심(퇴화 시 정점 평균 폴백). |
| `pointInPolygon` | ray casting(뷰어 `pointInQuad` 의 TS판). |
| `rectCorners` | 사각형 4모서리(TL,TR,BR,BL). |
| `clipByHalfPlane` | 단일 반평면 S–H 클립(`n·(x−p0)≥0` 유지). 비겹침 분리의 코어. |
| `convexIntersectionArea` | A를 B 각 변으로 클립 후 면적 → 겹침 판정·측정. |
| `perpBisector` | 두 무게중심의 수직이등분선(비겹침 분리선 기본값). |

### 3-3. 결정형 floor 모듈 — `src/capture/floorRoi.ts`(전면 재설계)
LLM 은 좌표 "생성"만, 검증·정렬·마진·포함강제·비겹침은 전부 이 모듈이 결정형으로 수행.

- **`predictPlateRect(vehicle)`**(R3): 번호판 예상 박스 = 차량 bbox 하단 중앙 밴드(`x+0.30·w, w=0.40·w, y+0.72·h, h=0.18·h`, 0~1 클램프). plate 인자 부재 시 이 rect 를 포함강제 대상으로 사용.
- **`fallbackPolygon(vehicle, plate?)`**: bbox 하단부 원근 사다리꼴(FALLBACK_BAND/inset 상수 재사용) → `applyMargin`(R1 방사 확장) → `expandPolygonToContainRect`(plate 또는 예상 번호판 포함강제, R3). **항상 4점 이상 존재 보장**(폴백 안전망).
- **`normalizePolygon(raw)`**: LLM raw → 유효 폴리곤. 점 수 4~10 밖·NaN·전범위초과면 null. 0~1 클램프 + `convexHull` + `orderConvexCanonical`(캐노니컬 정렬) + 마진. **껍질이 4정점 미만이면 null**(D1 수정 지점, §4).
- **`expandPolygonToContainRect(poly, rect)`**: rect 4모서리 중 폴리곤 밖 점이 있으면 `convexHull(poly ∪ 모서리)` → 볼록 유지 + 포함 보장, 정점>10이면 `simplifyToMax`. 전부 내부면 원본 반환(**멱등, 축소 없음**).
- **`resolveFloorPolygon(llm, vehicle, plate?)`**: 최종 진입점. `normalizePolygon(llm) ?? fallbackPolygon(vehicle, plate)` → `expandPolygonToContainRect(base, plate ?? predictPlateRect(vehicle))`. **어떤 입력에도 항상 유효 폴리곤(≥4점)** 반환.
- **`deconflictPolygons(items)`**(R4 핵심): 같은 프리셋 그룹 다각형들에 상호 수직이등분선 반평면 클리핑을 **단일 패스**(O(n²))로 적용해 **겹침=0 수학적 보장**.
  - 겹침 쌍(i,j)에 대해 `perpBisector(centroid_i, centroid_j)` → `adjustForPlates`(R3 우선 이동) → i측/j측 각각 `clipByHalfPlane`.
  - **보장 근거**: 클리핑은 오직 축소만 하고 쌍을 동일 직선 양쪽으로 분리. 이후 다른 쌍이 더 클립해도 축소일 뿐이라 분리 유지 → 1패스로 모든 쌍 겹침 0(결정적, 반복 불필요).
- **`adjustForPlates`**(R3 우선): 분리선을 법선 방향 평행이동해 plateI 는 i측, plateJ 는 j측이 되게 함. 양립 창 있으면 그 중앙(둘 다 있을 때)/창 안 클램프(한쪽만). 창 없음(번호판 겹치는 병리적 케이스)이면 수직이등분선 유지 + `logger.warn` 1회(비겹침 우선, 마진 희생).

### 3-4. LLM 스키마·프롬프트
- **`src/brain/SetupBrain.ts`**: `FloorRoiResultSchema` 의 `quad: array.length(4)` → `polygon: z.array({x,y}).min(4).max(10)`. 필드/타입 rename.
- **`src/brain/AgentRuntime.ts`**: 코드 변경 없음(스키마 파싱만 따라감).
- **`config/prompts/floor_roi.yaml`**: 출력 스키마 `{"polygon":[…4~10점],"confidence"}` 교체 + 정점 수 판단 기준·여유마진(R1)·번호판 예상 포함(R3)·비겹침 지향(R4, 최종은 결정형 보장) 지침 개정.

### 3-5. DB — `src/capture/SqliteStore.ts`
- `floor_roi` 테이블에 `polygon_json TEXT` 컬럼(CREATE + `addColumnsIfMissing('floor_roi', ['polygon_json'], 'TEXT')` 존재 가드 마이그레이션). `addColumnsIfMissing` 를 타입 인자 지원(`type='REAL'` 기본)으로 일반화.
- **쓰기** `upsertFloorRoi(runId, presetKey, clusterId, polygon, updatedAt)`: `polygon_json = JSON.stringify(polygon)` authoritative + `x0..y3` 에 앞 4점 병행 기록(구 뷰어/구 코드 하위호환).
- **읽기** `getFloorRois(runId)`: `polygon_json` 우선 parse(`length>=4` 가드), 없으면(구 런) `x0..y3` → 4점 폴리곤 폴백. 반환 필드 `polygon`.

### 3-6. Finalizer 2-pass — `src/capture/Finalizer.ts`
`assemble()` 의 프리셋 그룹(`byPreset`) 루프를 2-pass로:
1. **base 수집**: 위치순(`orderByPosition`)으로 각 슬롯의 바닥 다각형 수집 — 체크포인트 LLM 산출(`floorByRef`)이 있으면 그것, 없으면 `fallbackPolygon(rect, plateRect)`.
2. **비겹침**: `deconflictPolygons(positioned…)` 단일 패스로 프리셋 전체 겹침 제거.
3. **할당**: 슬롯별 `floorRoiByPreset[key]` 에 deconflict 결과 할당.
`roiByPreset`/`plateRoiByPreset` 로직은 불변.

### 3-7. API·뷰어
- **`src/api/server.ts`**: `FloorPolygonSchema = z.array(NormalizedPointSchema).min(4).max(10)`, `floorRoiByPreset` 값 스키마 교체. plate 는 quad(4점) 유지. 구 4점 저장 데이터 통과.
- **`web/core.js`**: `hitTestQuadVertex`/`moveQuadVertex` 의 4점 하드코딩 제거(가드 `<3`, `index>=quad.length`). 렌더(`toPixelQuad`/`drawQuadHandles`)는 기존 N정점 지원 그대로 → 변경 불필요.
- **`web/core.d.ts`**: `hitTestQuadVertex` 반환 `0|1|2|3|null` → `number|null`.

## 4. 발견·수정 결함 D1 (구현↔검증 루프 사례)

**D1: `normalizePolygon` 근사 공선 4점 → 3정점 붕괴 → `upsertFloorRoi` 크래시** (심각도 높음, 프로덕션 경로).

- **증상**: LLM 이 스키마상 유효한 4점을 반환해도, 한 점이 나머지 3점의 변 위(공선)에 있으면 `convexHull` 이 그 점을 제거해 **정점 3개**로 붕괴. 당시 가드가 `if (hull.length < 3) return null;` 이라 3정점이 그대로 통과(min-4 불변식 위반). `resolveFloorPolygon` 에서 번호판이 이미 삼각형 내부면 확장도 안 일어나 3정점이 최종 반환 → `FloorRoiReviewer.review()` → `SqliteStore.upsertFloorRoi` 의 고정 인덱스 `polygon[3].x` 접근에서 `TypeError` 크래시.
- **재현 입력**: `raw=[{0.2,0.2},{0.8,0.2},{0.8,0.8},{0.5,0.5}]`((0.5,0.5)가 대각선 위 공선).
- **수정**(`src/capture/floorRoi.ts` L130): `if (hull.length < 3) return null;` → **`if (hull.length < 4) return null;`**. 근사 공선 붕괴 시 null 반환 → `resolveFloorPolygon` 이 `fallbackPolygon`(항상 ≥4점)으로 강등 → 불변식·DB·API·뷰어 전 계층 정합. 기존 폴백 안전망과 일관되어 최소·외과적.
- **min-4 보장 경로 재확인**: `normalizePolygon`(≥4 또는 null) `?? fallbackPolygon`(≥4) → `expandPolygonToContainRect`(축소 없음) → **어떤 입력에도 ≥4 반환**. 크래시 단일 공급원(`upsertFloorRoi`)이 더 이상 undefined 를 만나지 않음.
- **교훈**: 스키마 검증(min 4) 통과 ≠ 기하 불변식(볼록껍질 정점 4) 충족. 공선은 실 LLM 출력에서 흔하므로, 순수 기하 변환 이후에도 불변식을 재확인해야 한다. 구현자 단독으로는 놓쳤고, 검증자가 근사 공선 엣지 케이스를 `it.fails` 회귀 테스트로 문서화 → 구현자 1줄 수정 → 검증자 승격·재검증으로 닫힌 전형적 루프.

- **비겹침은 최종화에서만 보장**: `deconflictPolygons` 는 Finalizer(`assemble`)에서만 적용된다. **라이브 뷰(FloorRoiReviewer 체크포인트 시각화)는 최종화 전이라 슬롯 간 겹쳐 보일 수 있다**(설계 결정4, 무해·의도된 동작). 최종 artifact(뷰어가 렌더하는 저장본)는 비겹침 보장.
- **수동 편집**: 뷰어 정점 드래그(N정점 지원)는 비겹침을 깰 수 있으나 사용자 권한 존중으로 재클립하지 않음(정점 증감 UI 는 이번 범위 밖).
