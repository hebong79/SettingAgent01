## 1. 기능 개요 · 목적

뷰어 오버레이에 **"마스크"** 토글(`#roi-mask`, 기본 off)을 추가했다. 체크하면 VPD 세그멘테이션(seg)이 반환한
마스크 중 **on-place det(현재 프리셋 ROI 통과분)에 정합된 차량 마스크(실루엣)만**을 보라 반투명
(`rgba(175,82,222,0.28)`, 외곽선 `#af52de`)으로 화면에 겹쳐 그린다. VPD det bbox 오버레이(`#roi-detect`)가
on-place 차량만 표시하는 것과 **동일한 차량 집합**을 그리도록 맞췄다(§4.3).

### 왜 필요한가 (육면체 오류 육안 검증 맥락)
차량 3D 육면체(`#roi-vcuboid`)는 **det 검출(권위) + seg 마스크 접지선**으로 산출되며, 배치(X,Y)는 원리적으로
정량 검증이 불가능하다(`estimateUnverified: true` — 자기참조 잔차뿐, D-1). 따라서 육면체가 실제 차량과 얼마나
어긋나는지는 **사람 눈으로** 볼 수밖에 없다. 마스크 show는 그 육안 검증의 근거 레이어다:

- **실루엣 정합** — seg 마스크가 실제 차량 실루엣과 얼마나 맞는지(오검·번짐·인접차 침범).
- **접지 하단 뜸** — 마스크 하단이 바퀴 접지선에서 얼마나 떠 있는지(육면체 바닥 오차의 직접 원인).

즉 이 기능은 **육면체 튜닝과 같은 화면에서** 쓰이는 진단용 표시 레이어다. 점유 판정·육면체 산출 로직에는
전혀 관여하지 않는 **순수 표시(read-only)** 이다.

## 2. 변경 클래스 / 함수

### 2.1 `FrameCuboids.masks?` — 산출 경계의 가산 필드
- 위치: `src/ground/frameCuboids.ts` L97 (interface `FrameCuboids`)
- 시그니처: `masks?: NormalizedPolygon[];`
- 역할: **on-place det(keptIdx)에 정합된 seg 마스크만**(정규화 폴리곤 배열). VPD det bbox와 동일한 차량 집합
  기준 — seg-only(ROI 밖/미정합) 마스크와 off-place det(정합됐어도 현재 프리셋 ROI 밖)의 마스크는 제외한다.
- 채움 위치: **성공 반환에만**(L333~336)
  ```ts
  const keptSet = new Set(keptIdx);
  ...
  masks: a.pairs
    .filter((p) => keptSet.has(p.detIdx))          // on-place det(현재 프리셋 ROI 통과분)만
    .map((p) => segBoxes[p.segIdx].mask)            // segBoxes = 압축 배열 위치(내부 계산 기준)
    .filter((m): m is NormalizedPolygon => !!m),    // 옵셔널 타입 방어
  ```
  - `keptIdx`는 `runDetect`가 넘기는 **on-place 필터 통과 det 인덱스**(`DetectResult.vehicles`와 동일 집합,
    detectPipeline.ts L271 — §3 경계면 참조).
  - `p.segIdx`는 `segBoxes`(마스크 drop 후 **압축** 배열)의 위치다. `assoc` export용으로 원문 인덱스(`vpdIdx`)로
    되돌린 값(DEFECT-1, L312)과 **혼용하지 않는다** — masks 산출은 내부 계산과 같은 압축 위치 기준을 그대로 쓴다.
  - `SegBox.mask`가 옵셔널 타입(`VehicleBox.mask?`)이라 `b.mask!` 단정 대신 **타입가드 `.filter`** 로 방어(tsc strict 통과).
  - **강등(degraded) 경로에는 넣지 않는다.** seg 호출 전/실패 강등 시 `masks` 키 자체가 없다(옵셔널 가산 규약 — 기존 shape 불변).
  - `seg.segDegraded`(검출 0대 · S-1)면 `a.pairs = []` → `masks: []`(빈 배열, 성공 반환).
  - **on-place인데 seg 정합이 없는 경우(unmatched det)**: 그 det의 마스크는 없다(`masks`에서 빠짐) — bbox만
    표시되고 실루엣은 뜨지 않는다(§6 QA 케이스 ⑤로 확인).

## 3. 데이터 흐름 (경계면 · 좌표계 명시)

```
VPD seg 서버 ── VpdClient.segment() ──▶ SegBox[] (mask-less drop 완료)
                                          │  각 SegBox.mask: NormalizedPolygon (정규화 [0,1])
                                          ▼
runDetect() ── det 전량 ──▶ onPlaceFilter ──▶ keptIdx(on-place det 인덱스) ──┐
                                                                              │
buildFrameCuboids({ detBoxes, keptDetIdx: keptIdx, ... })                    │
   │  a.pairs = det↔seg 정합 결과(전량, det/seg 원문 인덱스 보유)             │
   │  keptSet = new Set(keptIdx)  ◀───────────────────────────────────────────┘
   ▼
성공 반환 FrameCuboids.masks
   = a.pairs.filter(p => keptSet.has(p.detIdx))     // on-place det만
            .map(p => segBoxes[p.segIdx].mask)       // 정합된 seg 마스크(압축 배열 위치)
            .filter(유효)
   (seg-only·off-place det 마스크는 제외 — 좌표는 무변환 보존)
                                          │
   ┌──────────────────────────────────────┼──────────────────────────────────────┐
   ▼(라이브 검출)                          ▼(정밀수집 잡)                          ▼(수동 토글)
runDetect() → DetectResult.cuboids   CaptureJob.cuboidsByPreset            /capture/vehicle-cuboids
   (detectPipeline.ts L89,L343)      (JobCuboids = FrameCuboids & {...})    (라이브 촬영 1회)
   app.js runLiveDetect L789         GET /capture/job-cuboids               app.js L650
        │                            app.js L613                                 │
        └──────────────┬─────────────────────┴────────────────────────────────────┘
                       ▼
             state.vcuboidByKey[key]   ← 세 경로 공통 목적지 (masks 동승)
                       ▼
             drawMaskOverlay(ctx)  ── toPixelQuad(정규화→overlay.width/height 픽셀) ── 캔버스 렌더
```

- **경계 필드**: `SegBox.mask`(정규화) → `FrameCuboids.masks`(on-place 필터링 후) → `DetectResult.cuboids.masks` →
  `state.vcuboidByKey[key].masks`.
- **좌표계**: 마스크는 det bbox와 동일한 캡처 해상도 기준 정규화([0,1]). 변환 없이 `overlay.width/height` 곱만으로
  뷰어 오버레이에 정합한다(육면체·bbox와 동일 정규화 기준).
- **kept-det 인덱스 공유 (반전의 핵심 경계면 사실)**: `masks` 필터의 `keptSet`은 `runDetect`가 `buildFrameCuboids`에
  넘기는 바로 그 `keptIdx`이며, 이는 `DetectResult.vehicles`(= on-place VPD bbox 출력)를 만드는 필터와 **동일
  인덱스 집합**이다(detectPipeline.ts L271). 즉 **`masks`와 `DetectResult.vehicles`는 이제 같은 kept-det 인덱스
  집합을 공유**한다 — bbox 오버레이(`#roi-detect`)에 뜨는 차량과 마스크 오버레이(`#roi-mask`)에 뜨는 차량이 항상
  같은 집합이 된다(QA 케이스 ④가 이를 직접 증명, §6).
- **인덱싱 안전성**: `p.segIdx`는 `segBoxes`(압축 배열, mask-less drop 후) 위치를 그대로 쓴다. `assoc[].segIdx`가
  출력 경계에서 원문 인덱스(`SegBox.vpdIdx`)로 되돌려지는 것(DEFECT-1, L312)과는 **다른 인덱스 공간**이므로
  masks 산출부는 이 재매핑값을 쓰지 않는다(frameCuboids.ts L297 주석).

## 4. M1 설계 결정 근거 (+ 결정 반전 이력)

### 4.1 [원안, 폐기됨] 왜 assoc 게이팅 없이 seg 마스크 전량인가
> ⚠️ 아래는 **최초 설계(M1 원안)의 근거이며 §4.3 반전으로 폐기됐다.** 경위 추적을 위해 삭제하지 않고 남긴다.

원안의 논리: 마스크 show의 목적은 "seg가 무엇을 봤는지"를 육안 검증하는 것이므로, det↔seg 정합(assoc)으로
필터링하면 정합 실패분·seg-only 검출이 화면에서 사라져 오히려 진단 대상(오검·미정합)을 숨긴다 — 따라서
마스크 유효분 **전량**(정합 실패·seg-only 포함)을 그대로 내보낸다는 것이었다.

### 4.2 왜 지면모델 커플링(M1)을 수용했나 (반전 이후에도 유지)
대안 M2("지면모델 없을 때만 seg를 한 번 더 부르는 폴백")는 신규 seg 호출 사이트·정합·중복가드를 더해야 하는
과설계다. 반면 실사용 경로(`/capture/detect`, 정밀수집)는 **지면모델이 배선된 워크플로**에서만 돌고, 마스크 show의
두 목적(실루엣 정합·접지 하단 뜸)은 육면체 튜닝과 같은 화면에서 쓰인다 — 접지 검증은 지면모델이 있을 때만
의미가 있다. 따라서 M1의 유일한 한계(지면모델 부재 시 마스크 미표시)는 이 기능의 실제 사용 맥락과 충돌하지 않는다.
구현은 `buildFrameCuboids` 성공 반환에 `masks` 1필드 가산 + 뷰어 렌더 1블록으로 끝난다(이중 호출 위험 0).
이 결론은 masks 의미 반전(§4.3)과 무관하게 그대로 유지된다.

### 4.3 [반전, 마스터 요구] on-place det 정합분만으로 좁힌 이유
**변경:** `masks` 산출을 `seg.boxes 전량`에서 `a.pairs 중 on-place det(keptSet)에 정합된 것만`으로 교체했다
(`frameCuboids.ts` L295~336, 마스터 요구로 §4.1의 원안 결정을 반전).

**사유:** VPD det bbox 오버레이(`#roi-detect`)는 애초에 **on-place 차량만** 표시한다(주차면 필터 통과분).
그런데 마스크는 seg 전량이라 통로 등 ROI 밖 차량 실루엣까지 그려져, 두 오버레이가 **서로 다른 차량 집합**을
그리는 불일치가 발생했다. on-place 기준을 bbox와 마스크에 동일하게 적용해 이 불일치를 제거한다 —
"마스크를 VPD와 같은 방식으로 맞춘다."

**§4.1 논리와의 긴장(솔직히 기록):** 원안은 "정합 실패·seg-only도 보여야 진단이 된다"는 논리였고, 반전은
"VPD와 같은 집합만 보여야 일관된다"는 논리다. 두 목적이 상충하므로 마스터가 후자를 택했다 — seg가 무엇을
"더" 봤는지의 진단력은 이번 변경으로 낮아지지만(통로 차량 실루엣은 이제 안 보임), 화면 일관성(bbox=마스크
차량 집합)을 얻는다. on-place인데 seg 정합이 없는 경우(unmatched det)는 여전히 마스크 없이 bbox만 뜬다
(§2.1 마지막 항목, §6 케이스 ⑤).

## 5. 알려진 한계 (은닉 없이 명시)

1. **지면모델(cuboidCtx) 배선 시에만 표시.** `buildFrameCuboids`가 seg 호출 **전** 지면모델 부재로 강등되면
   `masks` 필드가 없어 마스크가 뜨지 않는다. 이때 사유는 `issues[]`에 남으므로 조용한 실패는 아니다.
   지면모델 없는 프리셋에서도 마스크가 필요해지면 M2 폴백을 별도 작업으로 추가한다.

2. **[해결됨] `#roi-mask` change 리스너.** 이전 판(2026-07-15 17:41)에서는 `#roi-mask`에 전용 `change` 리스너가
   없어 토글 직후 즉시 재렌더가 보장되지 않는다고 기록했다. 이후 후속 처리로 다른 순수 렌더 토글 6종
   (`#roi-vehicle`·`#roi-plate`·`#roi-floor`·`#roi-occupancy`·`#roi-detect`·`#roi-cuboid`)과 동일한 패턴으로
   `app.js` L2856에 `$('roi-mask').addEventListener('change', drawRoiOverlay);`가 추가되어 **해결됐다**
   (실측 확인: app.js L2856). `#roi-vcuboid`식 자동 데이터 로드(`loadVehicleCuboids()`)는 추가하지 않았다 —
   masks는 detect 응답에 이미 동승하므로 별도 로드가 불필요해 과설계로 보고 제외했다.

3. **on-place 필터링으로 진단 범위가 좁아짐(§4.3 트레이드오프).** seg-only(ROI 밖 통행차)·off-place det의
   마스크는 이제 화면에 뜨지 않는다. "seg가 무엇을 오검했는지" 전반을 보고 싶다면 이 기능만으로는 부족하다
   (원안의 진단력 일부 상실 — 화면 일관성과 교환됨, §4.3).

4. **실 VPD 스모크 미수행(모킹만).** 검증은 vitest 유닛테스트(seg 스텁 주입)로만 이뤄졌다. 실제 VPD seg 서버를
   통한 라이브 렌더 육안 확인(보라 실루엣이 실제 차량 위에 정합하는지, on-place만 그려지고 통로 차량은 안
   뜨는지)은 리더 육안 검증 단계로 남아 있다(QA도 이 항목을 스코프 밖으로 명시, §6).

## 7. 사용법

1. 뷰어 상단 `.roi-toggles`에서 **"마스크"** 체크박스(기본 꺼짐)를 켠다. 위치는 "육면체"(`#roi-vcuboid`) 토글 바로 뒤.
2. 검출을 실행하면(정밀수집·라이브 검출) **on-place 차량(현재 프리셋 ROI 통과분, VPD bbox가 뜨는 차량과 동일
   집합) 위에만** 보라 반투명 실루엣이 겹쳐 그려진다. 통로 등 ROI 밖 차량은 seg가 봤더라도 그려지지 않는다(§4.3).
   토글은 다른 렌더 토글과 동일하게 `change` 시 즉시 재렌더된다(§5-2).
3. on-place인데 seg 정합이 안 된 차량은 bbox만 뜨고 마스크(실루엣)는 뜨지 않는다(정상, §2.1).
4. 마스크 하단이 바퀴 접지선에서 얼마나 뜨는지, 실루엣이 실제 차량과 얼마나 맞는지 육안으로 확인한다.
5. 지면모델이 배선되지 않은 프리셋에서는 마스크가 뜨지 않는다(정상 — `issues[]` 확인).

색상은 초록(바닥)·청록(bbox)·주황(육면체)과 구분되는 보라(`rgba(175,82,222,0.28)`)다.
