## 2. `@parkagent/types` 전파 (공유 도메인 타입 — 핵심)

`@parkagent/types` 는 3개 에이전트(Setting/Action/DM)의 **계약**이다. 이번 변경은 **가산만**:
- 신규 export `NormalizedPoint`/`NormalizedQuad` (기존 심볼 불변).
- `ParkingSlot.floorRoiByPreset?` **옵셔널 필드** 추가 (`roiByPreset`/`plateRoiByPreset` shape·좌표 불변).

**전파 대상별 영향**

| 패키지 | 영향 | 근거 |
|--------|------|------|
| SettingAgent | **생산자**. floor quad 를 만들어 산출물에 포함. 본 변경의 전부. | — |
| ActionAgent | **런타임 무영향**. `floorRoiByPreset?` 옵셔널 → 기존 코드가 참조·소비하지 않음. 타입 인지만(빌드 시 노출). | 옵셔널 가산 필드라 타입 불일치·런타임 깨짐 없음 |
| DMAgent | **런타임 무영향**. 동일. | 동일 |

- `@parkagent/types` 는 `exports`/`types` 가 `src/index.ts` 를 직접 가리키는 **소스-소비 패키지**(별도 build/typecheck npm 스크립트 없음). 검증은 패키지 디렉터리에서 `npx tsc --noEmit` 로 수행했고 통과. SettingAgent `typecheck` 가 이 타입을 실제 해석하므로 통합 타입 인식도 입증됨.
- **확인 필요(미검증)**: Action/DM 의 실 빌드는 본 작업 범위에서 돌리지 않았다. 옵셔널 가산이라 타입상 깨질 수 없으나, 두 에이전트의 독립 빌드 그린 여부는 각 에이전트 작업 시 확인 권장.

## 3. LLM 역할 확대의 안전성

이번 변경은 SettingAgent 두뇌가 **좌표를 생성**하는 첫 단계다(기존 stage/checkpoint reviewer 는 좌표 불변, 병합/라벨/거부만 판정). 위험을 다음으로 격리:
- **생성=LLM, 검증·강등·폴백=결정형**. `resolveFloorQuad`/`normalizeQuad`/`fallbackQuadFromRect` 는 순수 함수(외부 의존 0)로 클램프·순서 강제·폴백을 담당. LLM 출력이 무엇이든 floor ROI 는 항상 유효한 4점이며 항상 존재.
- **신규 HTTP 경로 0**. 기존 `chatJson`+`image_url`(data:image/jpeg;base64) 멀티모달 경로 100% 재사용 → 네트워크/인증/타임아웃 표면 불변.
- **별 클래스 분리**(`FloorRoiReviewer`)로 기존 `CheckpointReviewer`(좌표 불변 규약)와 책임 충돌 없음.

## 5. 저장 / 비용 영향

- **신규 테이블 `floor_roi`**(`IF NOT EXISTS`, 마이그레이션 불필요). PK=(run_id, preset_key, cluster_id), upsert 멱등. `aggregated_slot` 의 run 단위 delete+insert(집계 멱등)와 **수명주기 분리** → 집계 재실행에도 floor quad 소실 없음.
- **토큰·시간 비용**: 체크포인트마다 채택 슬롯당 멀티모달 호출 1회. `maxPerCheckpoint`(기본 12) 로 상한. 초과·프레임부재 슬롯은 이번 주기 skip(폴백이 항상 있어 누락 무해).
- **메모리**: `lastFrameByPreset` = 프리셋 수 × JPEG 1장(수십 프리셋 가정 수 MB). run 시작 시 clear. 수백 프리셋 규모면 재검토(현재 규모 아님).

## 8. 결론

이번 변경은 **전 구간 가산·옵셔널**이라 기존 계약·경로·테스트에 회귀를 일으키지 않는다(278/278 통과, 기존 248 회귀 0). `@parkagent/types` 전파는 Action/DM 에 **런타임 무영향**(옵셔널 필드, 미소비). LLM 의 좌표 생성 위험은 결정형 검증·폴백으로 격리되어 floor ROI 는 항상 유효·존재. 신규 HTTP 0, 신규 테이블 1개(멱등·수명주기 분리). **유일한 실측 잔여는 실 gemma 좌표 정확도** — 라이브 수집 후 뷰어 육안 검증으로 마무리해야 한다.
