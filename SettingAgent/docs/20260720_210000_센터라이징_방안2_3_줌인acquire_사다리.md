# 센터라이징 검출확실성 재설계 — 방안2(줌인 acquire) + 방안3(줌아웃 사다리)

> 2026-07-20 goal/loop 이터4. 카메라 13110(simulator-1) 라이브 실증. 마스터 확정: "순서보다 정확히 찾기가 우선."

## 문제
넓은 시야(낮은 zoom)에서 첫 검출/센터링 → 번호판이 작아 LPD 미검(no_plate)·줌중 소실(plate_lost). 라이브 converged 10/17.

## 핵심 통찰
- `slot_setup.lpd` 는 이미 discovery(디지털 크롭-줌, 무이동)가 확실히 찾은 산출물. "찾기"는 검증됨.
- **게인 무관 목표 zoom 직접산출**: 폭 ∝ zoom → `Zt = presetZoom × targetPlateWidth / lpdWidth`. pan/tilt 조준만 게인 필요.
- 따라서 **먼저 목표 근처로 줌인(판이 커짐)→확실히 검출→센터→폭 마감**, 미검 시 **줌아웃 사다리로 재포착**.

- **크럭스 — 소유권 오프셋 zoom 스케일**: peerOffsets(이터2 소유권)는 프리셋 프레임 상수라, acquire 큰 zoom 에선 화면상 이웃 간격이 `zoom/presetZoom` 배로 확대됨 → PtzCalibrator 가 각 단계 호출 zoom 으로 오프셋을 **사전스케일**해 주입(centerOnPlate 는 zoom 고정 → 정적 사전스케일 정확). **PlatePtz 무변경**.

## 후속: width 복구 사다리(방안3 확장) — 구현·라이브·결론
- `PtzCalibrator.zoomToWidthWithRecovery`: 최종 폭확대 중 `plate_lost` 시 줌아웃 재포착(centerOnPlate)+고운스텝(maxZoomStepRatio 절반씩 축소) 재시도(WIDTH_RECOVERY_MAX=2 바운드, best-effort 반환). **plate_lost 일 때만 발동** → 정상 슬롯 무영향.
- **라이브 3회: 17/17 검출 일관, 완주 14~16/17 변동**(실패 슬롯 slot3·5·11 사이 매 실행 바뀜). 복구는 재포착 성공(로그 reAcquired:true, 한 실행 slot11 회복)하나 **순 완주수 미상승**.
- **결론 — 근본 벽은 게인**: cam1 fallback 게인(-62/-35.5)이 이 시뮬 인스턴스와 안 맞아 경계 슬롯이 폭확대 중 재드리프트 → 복구가 물러나 재포착해도 같은 잘못된 게인으로 재소실. 복구는 게인이 대략 맞는 카메라(실 Hucoms diagSweep 후)에선 값싼 안전망이나, 지금 게인 불일치는 못 넘는다.
- **정본 다음 단계 = 이 카메라 diagSweep 게인 재실측**(경계 슬롯 근본 해결 + 전 슬롯 정밀도↑).

## 최종 해결 — 17/17 완주 (중간 zoom LPD 검출 사각지대 회피)
plate_lost 근본원인을 실측으로 확정(추측 배제):
- **게인 실측(diagSweep, 3프리셋)**: gainPan -62.0/-64.0/-62.6, gainTilt -35.4/-35.8/-35.2 → **config(-62/-35.5)와 ±3% 일치. 게인은 정확 = 범인 아님.** (큰 스텝 pan+2/+3 이상값은 최근접매칭 aliasing 허상 — 작은 스텝만 신뢰.)
- **검출 사각지대(zoomDetect, pre-aim 후 zoom 스윕)**: slot5 는 **zoom 6~10 에서 LPD 검출 0개**(중간 사각지대), 그 위 12~16 에선 정상 검출(w 0.19~0.28). slot4(성공)는 전 zoom 정상.
- **원인 확정**: acquirePlateWidth=0.12 → slot5 acquire zoom ≈ 8.1 = **사각지대 정중앙** → miss. 줌아웃 사다리는 아래로만 가 사각지대를 못 넘음.
- **해결: `acquirePlateWidth` 0.12 → 0.18**(config). 게인이 정확해 pre-aim 이 판을 중앙에 정확히 놓으므로 FOV 여유용 중간 zoom 불필요 → acquire 를 목표 근처(zoom≈12)로 올려 사각지대를 건너뛴다.
- **폭 밴드(마스터 확정)**: targetPlateWidth 0.2→0.215, widthTol 0.02→0.015 = 수렴 **0.20 ≤ w ≤ 0.23**.
- **라이브 2회 재현: 17/17 검출·17/17 완주, 폭 0.202~0.222(전부 밴드 내), 중복 0.**
- 진단 자산 보존: `_workspace/_live/diagSweep.mts`(게인), `_workspace/_live/zoomDetect.mts`(검출 사각지대) — 실 Hucoms·타 카메라 배포 시 재실행.
