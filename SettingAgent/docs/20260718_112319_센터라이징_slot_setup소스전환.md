## 1. 배경 · 목표

센터라이징 잡(`POST /calibrate/ptz`)은 기존에 `setup_artifact.json`(`repo.loadArtifact()`)을 대상 소스로 삼았다. 그러나 DB 스키마 개편(6테이블 전면 개편, 2026-07-16) 이후 `setup_artifact.json`은 사실상 비어 있어(`data/setup_artifact.EMPTY_BACKUP_20260716.json` 존재), 센터라이징 대상이 항상 `total 0`으로 강등되는 문제가 있었다.

이번 작업은 센터라이징 대상 소스를 **`slot_setup` 테이블**로 전환한다. `slot_setup`에서 `lpd_obb`(LPD로 검출된 번호판 OBB, `NormalizedQuad`)가 채워진 모든 슬롯을 대상으로 번호판 중심 PTZ 센터라이징을 수행하고, 그 결과(pan/tilt/zoom, centered=1)를 해당 `slot_id` 행에 부분 UPDATE 한다.

**전체 성공 기준**(설계서 §0): `store.getSlotSetup()`이 `lpd != null` 슬롯 N개를 반환할 때 `POST /calibrate/ptz` → `total=N`, 잡 완료 후 각 성공 슬롯의 `slot_setup` 행이 `pan/tilt/zoom/centered=1`로 갱신되고 `slot_ptz.json`도 병행 저장된다.

**MCP 경계**: 본 경로는 전 구간 **결정형 도구**다 — 기하 변환(`quadBoundingRect`) + DB 조회/부분 UPDATE + `PlatePtz` 수치 폐루프. LLM 두뇌 개입 없음(메모리 "셋업 좌표 정본은 전부 기하 / LLM 최소 보조" 원칙에 정합). 이번 전환은 대상 소스만 교체했고 신규 LLM 호출은 추가하지 않았다.

## 2. 변경 상세 (파일별)

### 2.1 `src/calibrate/slotPtzWriter.ts` — 신설 함수

```ts
export function expandPlateTargetsFromSlotSetup(views: SlotSetupView[]): PlateTarget[]
```

`slot_setup` 조회 결과(`SlotSetupView[]`)를 캘리브레이션 대상(`PlateTarget[]`)으로 펼친다. `v.lpd == null`(번호판 미검출 슬롯, 즉 빈 주차면)인 행은 제외한다.

| `PlateTarget` 필드 | 소스 | 비고 |
|---|---|---|
| `camIdx` | `v.camId` | |
| `presetIdx` | `v.presetId` | |
| `slotId`(string) | `String(v.slotId)` | REST 계약상 문자열 — 구 소스의 `'c1p1s1'` 형식에서 `'1'`(정수 문자열)로 의미 변경 |
| `globalIdx`(number\|null) | `v.slotId` | 정수, `slot_setup`이 소스이므로 항상 존재(null 케이스 신규 소스에서 발생 불가) |
| `plateRoi`(NormalizedRect) | `quadBoundingRect(v.lpd)` | LPD OBB(quad) → 축정렬 rect 유도 |
| `presetSlotIdx`(number\|null) | `v.presetSlotIdx` | DB 값 그대로, 재계산 없음(1-based) |

기존 `expandPlateTargets(artifact)`(setup_artifact 소스)는 **삭제하지 않고 존치**하며 `@deprecated` JSDoc을 부착했다("센터라이징 소스가 slot_setup으로 전환됨. artifact 경로 잔존."). 이유는 §4 참조.

### 2.2 `src/calibrate/PtzCalibrator.ts` — 소스 전환 + store 필수화 + repo 제거

- **`store` 의존성 확대·필수화**: `store?: Pick<SqliteStore, 'upsertSlotCentering'>`(옵셔널) → `store: Pick<SqliteStore, 'upsertSlotCentering' | 'getSlotSetup'>`(필수). `PtzCalibratorDeps`와 클래스 필드 양쪽 모두 변경.
  - 판단 근거(설계서 §2-1): `store`가 잡의 유일한 데이터 소스가 된 이상, 미주입 시 `total 0`으로 "조용히 강등"시키는 것은 프로젝트 CLAUDE.md 원칙(모호한 실패를 숨기지 않는다)에 위배된다. 타입 레벨 필수화로 오구성을 컴파일 타임에 차단했다.
- **`repo`(Repository) 제거**: `PtzCalibratorDeps.repo`, `private readonly repo` 필드, 생성자 대입, `import type { Repository }` 전부 삭제. `PtzCalibrator.start()`에서만 쓰이던 의존이 이번 전환으로 완전히 고아화되어 제거 대상이 됐다(설계서 §3 grep 확인 — `Repository`는 `server.ts`/`index.ts`/`Finalizer` 등 타 모듈에서 계속 광범위 사용되므로, **PtzCalibrator 국소 import만** 제거하고 전역 삭제는 하지 않았다).
- **`start()` 본문 교체**:
  ```ts
  // 변경 전: this.repo.loadArtifact() → 없으면 throw → expandPlateTargets(artifact)
  // 변경 후:
  let targets = expandPlateTargetsFromSlotSetup(this.store.getSlotSetup());
  ```
  `slotIds`(문자열) 필터, 상태 세팅(`state='running'`, `done=0`, `total=targets.length`), `void this.run(targets)`, 리턴 shape(`{ total }`)은 불변. `'no setup artifact'` throw 경로는 사라졌다 — `slot_setup`이 비었거나 lpd 슬롯이 없으면 이제 예외가 아니라 `total 0`으로 정상 완료된다(빈 필터 결과와 동일 취급).
- `calibrateSlot`/`saveCenteringSlots`/`startPtzFor`, PlatePtz 위임 순서(center→zoom)는 전부 불변.

## 3. 데이터 흐름

```
slot_setup(lpd_obb 보유 행)
  → SqliteStore.getSlotSetup()               [SlotSetupView[] 조회, lpd_obb JSON → NormalizedQuad 파싱]
  → expandPlateTargetsFromSlotSetup(views)    [lpd==null 제외, PlateTarget[] 펼침]
  → PtzCalibrator.start()                     [slotIds 필터(옵션) → run(targets)]
  → PlatePtz.centerOnPlate → zoomToPlateWidth [번호판 중심정렬 + 줌, 결정형 수치 폐루프]
  → saveCenteringSlots(items)                 [converged&&centered 항목만 rows 구성, slotId=globalIdx]
  → SqliteStore.upsertSlotCentering(rows)     [slot_setup.slot_id 키로 pan/tilt/zoom/centered=1 부분 UPDATE]
  → writeSlotPtz(...)                         [slot_ptz.json 병행 저장, 기존 writer 그대로]
```

REST 계약(`/calibrate/ptz`(start)·`/calibrate/status`·`/calibrate/result`)의 응답 shape·상태 코드는 불변이다. 변한 것은 `total`이 의미하는 바 — "artifact 펼침 수"에서 "lpd 보유 slot_setup 수"로 바뀌었다.

## 4. 설계 판단 및 그 근거

1. **`store` 필수화 채택**(옵셔널 유지 + 런타임 throw 대안 대신): 소스가 DB로 완전히 이동한 이상 store 없이는 잡이 성립하지 않는다. 타입 레벨 강제가 런타임 방어보다 단순하고 실수를 더 이르게(컴파일 타임) 잡는다. `index.ts`가 이미 항상 `store: sqlite`를 주입하므로 프로덕션 영향은 없다.
2. **`repo` 제거, `expandPlateTargets` 존치(@deprecated)**: `repo`는 PtzCalibrator 내부에서 완전히 고아화된 비공개 필드라 제거가 안전하고 명확하다. 반면 `expandPlateTargets`는 exported 공개 함수라 삭제 시 `SetupArtifact`/`logger` import 정리까지 변경 범위가 번진다(외과적 변경 원칙 위배). 존치 + deprecated 표기로 최소 변경을 유지했다.
3. **`saveCenteringSlots`의 `if (!this.store) return;`은 그대로 둠**: `store` 필수화로 이 분기는 이제 항상 `false`가 되는 사실상 죽은 코드다. 설계서가 `saveCenteringSlots`를 "불변"으로 명시했기 때문에 구현자가 손대지 않았다(외과적 변경 원칙 — 요청 범위 밖 리팩토링 금지). 런타임에 무해하며, 정리는 별도 작업 범위로 남겼다.
