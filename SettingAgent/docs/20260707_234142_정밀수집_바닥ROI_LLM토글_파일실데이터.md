## 1. 목적 / Goal / 요구사항

### Goal
정밀수집('precise') 탭에서 바닥(floor) ROI의 **소스를 이원화**한다.
1. **"바닥 LLM 사용" 모드 토글**(`#cap-floor-llm`)을 신설한다.
2. 선행 작업(§02)이 만든 파일 ROI(`PtzCamRoi.json`)를 별도 파랑 preview가 아니라 **실제 바닥(floor) 레이어로 승격**해 표시한다.

### Requirements → 설계 대응
| ID | 요구 | 대응 |
|----|------|------|
| **R1** | 파일 JSON을 바닥 실데이터로 연결(preview → 실 바닥) | 파일 모드(LLM OFF)에서 `state.placeRoi`를 **초록 바닥 레이어**(`roi-floor` 토글 관장)로 프리셋별 렌더. 별도 `roi-placeroi` preview **제거** |
| **R2** | LLM 모드는 **배선만**, capture 실행 테스트 안 함 | `/capture/start` 바디 `floorRoiUseLlm?` → `CaptureJob` → `floorReviewer` 게이트. 시뮬 미가동으로 capture 파이프라인 실행 테스트 없음 |
| **R4** | 프리셋 단위 출력 | 파일 ROI를 `byPreset["cam:preset"]` 폴리곤 리스트로 현재 프리셋만 렌더 |
| **R5** | 프리셋마다 검수 | `normalizePtzCamRoi`의 프리셋별 `report[].issues`를 `state.placeRoiReport`에 보관, 파일 모드 렌더 시 프리셋당 1회 `console.warn` advisory |

> R3(3회 반복 실패 시 정지)은 리더 루프 제어 항목으로 본 구현과 무관.

## 2. "실데이터 연결"의 해석 (가장 단순·테스트 가능한 정의)

§02는 파일 ROI를 **별도 파랑 preview 레이어**(`roi-placeroi`, `rgba(120,170,255,0.16)`)로만 그렸다 — 실 바닥(floor) 데이터와 무관한 참고 표시였다. 본 작업의 "실데이터 연결"은 다음을 의미한다.

- **파일 모드(LLM OFF)**: 정규화된 파일 ROI(`state.placeRoi`)를 **바닥 ROI 실데이터로 취급** → §02의 파랑 preview가 아니라 실제 **floor 레이어(초록, `rgba(57,255,20,0.22)` / `#39ff14`)**로 프리셋별 렌더한다. **캡처 없이 즉시 반영**되며, 표시 관장 토글이 `roi-placeroi`(전용)에서 **`roi-floor`(바닥 공통)**으로 이관된다. 즉 `roi-placeroi` preview는 폐기·통합된다.
- **LLM 모드(ON, 기본값)**: 기존 슬롯별 LLM floor 경로를 그대로 유지하고, 파일 바닥은 숨긴다.

**핵심**: 이것은 "바닥을 LLM으로 생성할지, 파일로 공급할지"의 **소스 선택**이지 새 두뇌/도구가 아니다. 파랑 preview → 초록 바닥 승격으로 레이어가 **하나로 통합**된다.

## 3. 변경/신규 요소 설명

### 3.1 `selectFloorRoi` 순수 함수 (신규 · `web/core.js:419`)

토글 상태에 따라 현재 프리셋의 바닥 폴리곤 소스를 선택하는 결정형 순수 함수(브라우저 API 미참조 → vitest 직접 import 가능).

```
selectFloorRoi({ useLlm, slots, placeRoi, key })
  → { source: 'llm' | 'file', polygons: [{ quad, label, slotId? }] }
```
- `useLlm===true` → `source:'llm'`. `slots[].floorRoiByPreset[key]` 보유 슬롯만 `{ quad, label:'', slotId }`로 수집(기존 슬롯별 floor와 대칭 — 회귀 검증용).
- `useLlm` falsy → `source:'file'`. `placeRoi[key]`의 각 `{ idx, points }` → `{ quad:points, label:String(idx) }`(프리셋 단위, R4).
- `slots`/`placeRoi` 누락 → `polygons:[]`(throw 금지, graceful).

타입 시그니처는 `web/core.d.ts:171-181`에 `FloorRoiPolygon` 인터페이스 + `selectFloorRoi` 선언으로 추가(`SlotLike`/`PlaceRoiSpace` 재사용).

### 3.2 UI "바닥 LLM 사용" 토글 (신규 · `index.html:145`)

수집 파라미터 그리드(capture-grid)에 `<input id="cap-floor-llm" type="checkbox" checked>` 추가. **기본 ON=LLM**(기존 동작 보존). ROI **표시** 토글 행(`roi-vehicle/plate/floor/...`)이 아니라 **수집/소스 모드** 제어이므로 파라미터 그리드에 배치해 역할을 구분한다. 동시에 표시 토글 행의 `roi-placeroi` label은 **제거**(§02 preview 폐기).

### 3.3 파일 바닥 렌더: `drawPlaceRoiOverlay` → `drawFileFloorRoi` (용도 변경·승격 · `app.js:280`)

§02의 파랑 preview 함수를 **재활용**(신규 렌더 코드 최소). 변경점:
- **게이트 이원화**: `if (!$('roi-floor').checked) return;`(파일 ROI가 바닥 레이어이므로 `roi-floor` 토글이 관장) **AND** `if ($('cap-floor-llm').checked) return;`(LLM 모드에선 파일 바닥 숨김).
- **데이터**: `selectFloorRoi({ useLlm:false, placeRoi: state.placeRoi, key: currentFrameKey() })`.
- **색상**: 파랑 → **초록 floor 스타일**(`rgba(57,255,20,0.22)` fill, `#39ff14` stroke) = 실 바닥 데이터로 승격.
- **검수(R5)**: `state.placeRoiReport[key]`에 issue가 있으면 프리셋당 1회 `console.warn('[FloorRoi:file] …')`. 모듈 스코프 `floorRoiFileWarned` Set 가드로 500ms 렌더 반복 스팸을 방지.
- 호출부(`drawRoiOverlay` 내 occupancy 직후·mapping 가드 이전, `app.js:190`)는 동일 위치 유지.

### 3.4 LLM 슬롯 floor 렌더 모드 게이트 (`app.js:222`)

mapping 루프의 floor quad 블록 조건에 `&& $('cap-floor-llm').checked` 추가 → LLM 모드에서만 슬롯별 초록 floor + 편집 핸들 렌더. 파일 모드에선 `drawFileFloorRoi`가 대신 그린다. **vehicle/plate 블록은 무변경**(모드 무관 표시).

### 3.5 floor 편집 히트테스트 게이트 (`app.js:395`)

`hitTestFloorVertex` 조건에 `!$('cap-floor-llm').checked` 추가 → 파일 모드(안 보이는 LLM floor 정점)에서 드래그 편집 차단. 파일 ROI는 읽기 전용(§02 스코프 준수), 편집은 LLM 슬롯 floor 전용 유지.

### 3.6 결선 (`app.js`)

- `state.placeRoiReport = {}` 신설, `loadPlaceRoi` 확장 — report 순회로 `placeRoiReport[presetKey(camId,presetIdx)] = issues` 채움.
- `wire()`: `roi-placeroi` change 리스너 제거 → `$('cap-floor-llm').addEventListener('change', drawRoiOverlay)`(app.js:1607)로 교체(모드 전환 즉시 재렌더).
- `capStart` body에 `floorRoiUseLlm: $('cap-floor-llm').checked` 추가(app.js:912).

### 3.7 백엔드 배선 (R2 — 배선만, 테스트 없음)

- `CaptureJob.ts` — `CaptureStartParams.floorRoiUseLlm?`(:55), 멤버 `private floorRoiUseLlm = true`(:87), `start()`에서 `this.floorRoiUseLlm = p.floorRoiUseLlm ?? true`(:153, **기본 true**).
- `CaptureJob.ts:349` — checkpoint의 floorReviewer 게이트: `if (this.deps.floorReviewer)` → `if (this.deps.floorReviewer && this.floorRoiUseLlm !== false)`. 파일 모드면 LLM floor 생성 스킵(occupancy/advisory/집계는 무변경).

## 4. 입출력 / REST 계약

**`POST /capture/start` 바디에 옵셔널 필드 1개 가산(하위호환).**

| 필드 | 타입 | 의미 |
|------|------|------|
| `floorRoiUseLlm` | `boolean?` (옵셔널) | `true`/미지정 → LLM으로 바닥 생성(기본). `false` → 파일 모드(LLM floor 스킵) |

- 미지정 시 `?? true`로 기본 LLM → **기존 클라이언트·호출자 회귀 0**.
- 잘못된 타입(예: 문자열 `'yes'`)은 zod가 400으로 거부.
- 그 외 `/capture/*` 라우트 계약(status·frame·stop·finalize·place-roi 등)은 **불변**.

## 5. 영향도 분석 (요약)

> 상세는 `_workspace/04_doc_impact.md` **# 04-D** 참조.

- **`drawFileFloorRoi` 승격**: `state.placeRoi`(별도 state)만 읽는 독립 데이터 소스. vehicle(`state.mapping.slots`)·plate·occupancy(`state.occByKey`)와 데이터 분리 → 오버레이 충돌 없음. 슬롯 floor 편집(핸들·드래그)은 `cap-floor-llm` 게이트로 모드 배타되어 파일 모드에서 안전하게 비활성.

- **`roi-placeroi` 제거**: §02(20260707_184524) 파랑 preview를 승격 대체. `roi-floor`(바닥 공통 토글)가 파일/LLM 표시를 겸한다.

## 7. 한계 / 스코프 경계 (통과 위장 없이 명시)

- **스코프 제외 — 파일 ROI의 백엔드 floor_roi DB 주입**: 파일 폴리곤을 캡처 파이프라인 `floor_roi` 테이블(`upsertFloorRoi`, idx↔cluster 매핑)에 주입하는 백엔드 승격은 **본 스코프에서 제외**. 시뮬레이터 미가동·idx↔cluster 매핑 정책 미정의(cluster 미존재 상태에서의 매핑 결정 선행 필요). 파일 모드 백엔드는 **LLM floor를 스킵만** 하고, 뷰어가 파일에서 직접 바닥을 표시한다. "실 DB에도 반영" 요구 시 별도 작업으로 리더 확인 필요.
- **LLM capture 실동작 미검증(R2)**: 시뮬 미가동 → 배선/게이트 호출 여부만 확인. LLM 응답 소비·실 floor 생성은 미검증(별도 실연동 스모크 필요).
- **DOM/Canvas 결선 수동 스모크**: `drawFileFloorRoi` 승격·모드 게이트·`hitTestFloorVertex`·토글 결선·index.html은 브라우저 통합이라 vitest 비대상. 로직은 순수함수(`selectFloorRoi`)로 이관 커버, 실제 렌더는 리더 경험적 스모크로 확인됨.
- **가정(§02 승계)**: `presetKey(cam_id, preset_idx)` ↔ `currentFrameKey()` 1-based 정합(파일 cam_id=1/preset 1·2·3이 라이브 키와 일치). 다중 카메라·다중 Place는 미검증(현 파일 1대·단일 Place 전제).
- **읽기 전용**: 파일 ROI 편집/저장/핫리로드 없음(§02 동일). `cap-floor-llm` 단일 토글이 뷰어 소스 선택 + 백엔드 capture 모드를 겸한다.
