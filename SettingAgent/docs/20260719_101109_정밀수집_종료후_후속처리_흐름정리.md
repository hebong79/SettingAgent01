# 정밀수집 종료 후 후속 처리 흐름 정리

> 작성일: 2026-07-19 10:11:09
> 대상: SettingAgent 정밀수집(capture) → 최종화(finalize) → 센터라이징(calibrate) 파이프라인
> 목적: "정밀수집이 끝난 뒤 무엇을 하는가"를 코드 근거와 함께 단계별로 정리(조회/문서화 요청).

---

## 0. 요약 (한 줄)

**정밀수집(인메모리)** → **최종화**로 `setup_artifact.json` + `slot_setup` DB 확정(공간배정·3D 앞면중심까지) → **센터라이징**으로 각 슬롯의 PTZ 값(`slot_ptz.json` · `centering_slot`)을 채운다.

```
[정밀수집 종료]  →  [1. 최종화 finalize]  →  [2. 센터라이징 calibrate]
 인메모리 스냅샷      DB 정본 확정             슬롯별 PTZ 정렬값 채움
 (오버레이 잔존)      setup_artifact.json      slot_ptz.json
                     slot_setup(vpd/lpd/         slot_setup(pan/tilt/
                       occupy/front_center)       zoom/centered/img1)
```

---

## 1. 정밀수집 종료 직후 상태

잡(`CaptureJob`)이 멈추면 마지막 프레임 위에 오버레이가 **그대로 유지된다**(삭제하지 않는 것이 정책).

| 오버레이 | 의미 | 소스 |
|----------|------|------|
| 청록 박스 | VPD(차량 검출 bbox) | 검출 파이프라인 |
| 노랑 사변형 | LPD(번호판 OBB quad) | 번호판 검출 |
| 보라/빨강 육면체 | 차량 3D 큐보이드 | 지면모델 기반 `job-cuboids` |
| 초록 영역 | 점유영역(발자국, occupy_range) | 번호판 앵커 사변형 |
| 14·15·16·17 등 숫자 | 전역 슬롯번호 | globalIndex |
| 분홍 점 | 번호판 중심 | LPD |

> ⚠️ **이 시점 데이터는 아직 인메모리 스냅샷**이다. `getSnapshot()`으로 들고 있을 뿐, DB 정본(`setup_artifact.json`/`slot_setup`)에는 아직 확정 저장되지 않았다. 화면에 보이는 것과 DB 정본을 일치시키는 것이 다음 단계(최종화)의 역할이다.

---

## 2. 1단계 — 최종화 (finalize)

- **트리거**: 웹 `최종화` 버튼 → `capFinalize()` → `POST /capture/finalize`
- **핵심 로직**: `Finalizer.finalize(snapshot, { logicOccupancy })`
- **관련 파일**: `web/app.js`(capFinalize), `src/api/captureRoutes.ts`(라우트), `src/capture/Finalizer.ts`(로직)

정밀수집 결과를 **DB 정본으로 확정**하는 단계. 세부 순서:

1. **결정형 집계 재실행(멱등)** — 스냅샷의 검출/라운드로 클러스터를 다시 산출한다. 단, 체크포인트에서 사용자가 바꾼 `merged`/`rejected` status는 보존 병합한다(덮어쓰지 않음).
2. **(LLM 활성 시) 최소 보조 판정** — 중복(duplicates)·거부(rejects)·존 라벨(zoneLabels)만 받는다. **좌표 생성·수정은 금지**(기하가 정본, LLM은 메타만). LLM 비활성/실패 시 결정형 강등.
3. **채택 클러스터 조립** — 채택분을 프리셋별 `positionIdx`로 정렬 → `ParkingSlot`/`Preset` 조립 → 바닥 발자국(floorRoi) 결정형 생성.
4. **전역 인덱스 빌드 + 커버리지 검증** — `buildGlobalIndex` → 1..N 전역번호 부여, `validateCoverage`로 누락/초과 검사(불일치 시 throw).
5. **`setup_artifact.json` 저장** — `repo.saveArtifact(artifact)`. **정본 산출물**.
6. **`slot_setup` DB 전량 교체** — 단일 트랜잭션(`replaceSlotSetup`), 실패 시 자동 롤백 → 직전 확정본 보존. 소스는 `PtzCamRoi.json`(파일 정의 바닥 ROI):
   - **VPD/LPD/점유영역**: accepted 클러스터를 `pointInPolygon` 기반 **공간배정**으로 슬롯에 배정. 선착순 find가 아니라 프리셋 단위 **상호배타 1:1 최대매칭**(`assignClustersToSpaces`).
   - **`slot3d_front_center`**: 슬롯 quad + 프리셋 지면모델 + 캐노니컬 높이(H_CONST = 1.5m) → 앞면 중심(정규화). 지면모델 부재/퇴화 시 `null` 강등.
   - **`pan/tilt/zoom/centered/img1`**: 이 시점엔 **null/0**(센터라이징 전이라 미보유). 이후 센터라이징이 채운다.
   - **검출 hit 없는 슬롯 보존**: 기존 `slot_setup`의 vpd/lpd/occupy를 조회해, hit이 없으면 이전 값을 유지한다(빈 finalize가 정상 데이터를 null로 파괴하는 것 방지).
7. **save/ 스냅샷 자동 저장** — `saveStore.save(defaultSaveName(), artifact)`(실패는 격리).
8. **응답** — `{ ok, slots, globalCount, (occupancyAgreement?) }`.
9. **프론트 후처리** — `loadParkingSlots()`(slot_setup 재조회 → `parkingSlotsByKey`) → `loadMapping()` → `drawRoiOverlay()` → `renderSlotList()`.

### 최종화 산출물 요약

| 산출물 | 위치 | 내용 |
|--------|------|------|
| `setup_artifact.json` | data/ | presets, slots, globalIndex, (report) — 정본 |
| `slot_setup`(DB) | data/setting.sqlite | slot_roi, vpd_bbox, lpd_obb, occupy_range, slot3d_front_center (pan/tilt/zoom/centered/img1은 null/0) |
| save/ 스냅샷 | save/ | finalize 결과 자동 백업 |

## 3. 2단계 — 센터라이징 / PTZ 보정 (calibrate)

- **트리거**: 웹 `센터라이징` 버튼 → `calStart()` → `POST /calibrate/ptz`
- **핵심 로직**: `PtzCalibrator` → `PlatePtz`(번호판 중심정렬·줌) 위임
- **관련 파일**: `web/app.js`(calStart/calPoll), `src/api/calibrateRoutes.ts`, `src/calibrate/PtzCalibrator.ts` / `platePtz.ts`

**최종화 이후**에 도는 별도 단계. 대상은 `setup_artifact`에 번호판 ROI가 있는 슬롯이므로, **최종화를 하지 않으면 대상 0**이 된다(웹 메시지: "대상 0 — 셋업 산출물에 번호판 ROI 슬롯이 없습니다(최종화 필요)").

세부:

1. 주차면(슬롯)별로 카메라 PTZ를 돌려 **번호판을 화면 중심에 정렬하고 줌**을 맞춘다(반복 보정).
2. 결과를 **`slot_ptz.json`** + DB **`centering_slot`**에 저장하고, `slot_setup`의 `pan/tilt/zoom/centered/img1`을 `upsertSlotCentering`으로 채운다.
3. 진행 중에는 `/calibrate/frame` 폴링(500ms)으로 라이브뷰를 실시간 갱신(카메라 재명령 없이 잡이 찍은 최신 JPEG 추종).
4. 완료 시 **수렴 N/전체** 요약 + 미수렴 슬롯 목록 렌더(`renderCalResult`).

### 센터라이징 산출물 요약

| 산출물 | 위치 | 내용 |
|--------|------|------|
| `slot_ptz.json` | data/ | 슬롯별 pan/tilt/zoom/centered/converged |
| `centering_slot`(DB) | data/setting.sqlite | 센터링 결과 |
| `slot_setup`(DB) 갱신 | data/setting.sqlite | pan/tilt/zoom/centered/img1 채움 |

## 4. 보조/선택 동작 (필요 시)

| 동작 | 트리거 | 효과 |
|------|--------|------|
| **검출·센터링 초기화** | `POST /capture/slots/reset` (resetSlotSetupDb) | `slot_setup`의 vpd/lpd/occupy/ptz/centered/img1만 비움. **slot_roi·행 자체는 보존**. |
| **결과 저장** | `POST /capture/save` (saveResult) | 현재 상태를 이름 지정 스냅샷(save/)으로 저장. |
| **결과 열기** | `GET /capture/saves/:name` (openResult) | 저장된 스냅샷을 다시 불러와 검수. |
| **DB 보기 토글** | `#roi-db` 체크 | 라이브 검출이 없을 때 `slot_setup` 저장분을 폴백 오버레이로 표시. |

## 5. 데이터 흐름 한눈에

```
정밀수집(CaptureJob)
   └─ 인메모리 스냅샷: dets, presetRounds, aggregated, occByPreset, cuboids
        │
        ▼  POST /capture/finalize
[최종화 Finalizer]
   ├─ aggregate 재실행(멱등, status 보존)
   ├─ (LLM) 중복/거부/존라벨 메타만
   ├─ 채택 → 프리셋별 슬롯 조립 → globalIndex(1..N) + 커버리지 검증
   ├─ setup_artifact.json  ← 정본
   └─ slot_setup 전량 교체(트랜잭션)
        · vpd/lpd/occupy = 공간배정(pointInPolygon 1:1 최대매칭)
        · slot3d_front_center = quad+지면모델+H(1.5m)
        · pan/tilt/zoom/centered/img1 = null/0  ← 센터라이징 전
        │
        ▼  POST /calibrate/ptz
[센터라이징 PtzCalibrator → PlatePtz]
   ├─ 슬롯별 번호판 중심정렬·줌 반복 보정
   ├─ slot_ptz.json + centering_slot
   └─ slot_setup.pan/tilt/zoom/centered/img1 채움(upsertSlotCentering)
```

## 6. 주의점 (회귀 방지)

- **최종화 전에는 센터라이징 대상 0**: 번호판 ROI 슬롯은 finalize 산출물에서 나온다. 순서(정밀수집 → 최종화 → 센터라이징)를 지켜야 한다.
- **빈 finalize의 데이터 파괴 취약성**: `replaceSlotSetup`은 DELETE+INSERT 방식이라, 검출이 전혀 없는 finalize가 직전 정상 vpd/lpd/occupy를 null로 덮을 수 있다. 현재는 검출 컬럼에 대해 "hit 없으면 기존값 보존" 가드가 있으나, 센터링 컬럼(pan/tilt/zoom)은 finalize 시 항상 null로 리셋되므로 **finalize를 다시 돌리면 센터라이징 결과가 초기화**된다 → 센터라이징은 finalize 이후에 다시 수행해야 한다.
- **오버레이 유지 정책**: 정밀수집 종료 후 VPD/LPD/점유/육면체 오버레이는 유지(삭제 금지). DB 소스(`slot_setup`) 표시는 `#roi-db` 체크 시에만.

---

## 7. 영향도 (참고)

- 이 문서는 **조회/정리 목적**이며 코드 변경은 없다.
- 다루는 정본 파일: `data/setup_artifact.json`, `data/slot_ptz.json`, `data/Place01/PtzCamRoi.json`, `data/setting.sqlite`(slot_setup / centering_slot 테이블).
- 소비자(Action/DM/뷰어)는 위 정본을 읽어 슬롯 위치·PTZ를 사용한다 — 최종화·센터라이징이 모두 끝나야 PTZ까지 채워진 완전한 슬롯 셋업이 된다.
