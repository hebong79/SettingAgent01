입력 산출물: `_workspace/01_architect_plan.md`(설계) · `02_developer_changes.md`(구현) · `03_qa_report.md`(검증)
근거 코드: `web/core.js`, `web/app.js`, `web/index.html`, `web/app.css`, `src/api/server.ts`

> 한 줄 요약: 최종화로 산출된 주차면(ROI)을 **사람이 직접 선택·삭제·크기조정**하고, **전역 슬롯 번호를 수동 재정렬**한 뒤,
> **명시적 "저장"(PUT)** 으로 `setup_artifact.json` 에 영속화한다. 저장은 서버의 정합 게이트(`validateCoverage`)를 통과해야만 파일에 기록된다.

## 1. 기능 7가지

### #1 주차면 선택 (캔버스/목록 클릭, 하이라이트)
- 검수 탭 오버레이 캔버스를 클릭하면 클릭 지점(정규화 좌표)이 어느 슬롯 ROI 안인지 히트테스트하여 선택한다.
- 차량 ROI(사각형)가 1순위, 바닥 ROI(4점 사변형)가 2순위. 겹친 슬롯은 **나중에 그려진(=상단)** 슬롯이 우선.
- 레이어 토글(차량/바닥)이 꺼진 레이어는 히트 대상에서 제외 — 화면에 보이는 것과 클릭 판정이 일치한다.
- 슬롯 목록 행 클릭으로도 동일하게 선택되며, 선택 슬롯은 캔버스에서 굵은 대비색(빨강, 선폭 4) 테두리 + 라벨로 강조된다.
- 순수 함수: `pointInRect`, `pointInQuad`, `hitTestSlots`(`web/core.js`). 캔버스→정규화 변환은 `app.js eventToNorm`(환경 의존).

### #2 선택 슬롯 삭제 (+ globalIndex / coveredSlotIds 재구성)
- "삭제" 버튼은 선택 시 활성화. 삭제 1건이 **3곳을 동시에** 바꾼다:
  1. `slots[]` 에서 해당 slot 제거
  2. 모든 `preset.coveredSlotIds[]` 에서 해당 slotId 제거(순서 보존)
  3. `globalIndex[]` 를 `rebuildGlobalIndex` 로 재생성(globalIdx 가 1..N-1 로 연속 유지)
- 재번호의 position 진실은 **`coveredSlotIds` 배열 순서**다. slotId 문자열(`c1p3s7`)의 `sN` 을 파싱하지 않는다(수동 삭제 후 sN 이 불연속될 수 있으므로 배열 순서가 진실 — 설계 §positionIdx).
- 순수 함수: `removeSlot`, `rebuildGlobalIndex`. 삭제는 **메모리에만 반영(미저장)**, "저장"을 눌러야 PUT 으로 영속화.

### #3 크기 조정 (4모서리 핸들 드래그, 0~1 클램프)
- 선택 슬롯에 4개 모서리 핸들(nw/ne/sw/se)을 렌더. 핸들을 드래그하면 정규화 델타가 사각형에 적용된다.
- `resizeRect` 가 좌우/상하 뒤집힘을 정규화하고, `clamp01Rect` 가 `x,y∈[0,1]`, `w,h≥0.001`, `x+w≤1`, `y+h≤1` 로 클램프한다(경계 밖 이탈·음수폭 붕괴 방지).
- ROI 좌표만 갱신(`updateSlotRoi`) — 슬롯 집합은 불변이므로 globalIndex·coveredSlotIds 는 그대로다.
- **floor(바닥) 4점 사변형 편집은 1차 범위 제외**(사각형 ROI만). 설계 확정 C.

### #4 프리셋3 미표시 원인 + 수정 + 경고 진단
- **원인(코드/데이터 분석)**: 산출물(`setup_artifact.json`)에는 `1:3`(cam1·preset3) 슬롯 14개가 정상 존재하고 키 매칭도 정상. 즉 Finalizer/오버레이/키매칭에 버그 없음.
  잔여 원인 후보는 두 가지이며 둘 다 처리했다:
  1. **(수정)** 프리셋/카메라 전환 시 `drawRoiOverlay()` 가 호출되지 않아 오버레이가 갱신되지 않을 수 있었다. → `sel-cam`·`sel-preset` change 핸들러에 `drawRoiOverlay()` 호출을 추가(app.js L937·L945). 전환 시 `state.selectedSlotId=null`(선택 해제)도 함께.
  2. **(진단)** 카메라 드롭다운(`/cameras`)에 프리셋3가 없으면 사용자가 `1:3` 을 선택할 길이 없어 영구 미표시. → `diffArtifactVsCameras(artifact, cameras)` 로 "산출물엔 있으나 드롭다운에 없는 프리셋 키"를 분석 탭 경고로 노출 + 검수 탭 빈 상태 안내 1줄.
- 순수 함수: `diffArtifactVsCameras`. `state.roiHidden` 가드(아래 #5)는 그대로 유지된다.

### #5 표시 초기화 버튼 (roiHidden, 데이터 보존)
- "표시 초기화"는 `clearRoiDisplay()` → `state.roiHidden=true`, `selectedSlotId=null` 로 화면의 ROI·선택 오버레이를 끈다.
- **데이터는 보존**한다(표시만 끔). `state.mapping` 은 그대로이며 파일도 건드리지 않는다. 다시 그릴 때(프리셋 전환·최종화 등)는 `drawRoiOverlay()` 의 `roiHidden` 가드로 일관 처리.

### #6 정밀수집 시작 시 화면 정리, 최종화 시 다시 표시
- 정밀 수집을 시작하면 화면의 기존 ROI 표시를 정리(`roiHidden`)하여 수집 중 혼동을 막는다.
- 최종화(`/capture/finalize`) 성공 시 `state.roiHidden=false` 로 복귀하고, `loadMapping()` 으로 누적 결과를 다시 불러와 검수 탭에 ROI 를 다시 그린다(app.js `capFinalize` L573~576).

### #7 전역 인덱스 수동 매핑 (분석 탭 ▲▼ 재정렬 · 정합 검증)
- 분석 탭의 "전역 인덱스 수동 매핑" 섹션에서 slotId↔globalIdx 목록을 ▲▼ 버튼으로 재정렬한다(신규 탭 없이 분석 탭에 가산 — 설계 확정 A).
- `validateManualIndex(globalIndex)` 로 globalIdx 의 1..N 연속·중복 여부를 실시간 검사(`{ ok, duplicates, gaps }`).
- 저장 시 `reorderGlobalIndex(artifact, orderedSlotIds)` 로 지정 순서대로 globalIdx 1..N 재부여 — slots 집합과 1:1(누락/초과/중복 입력)이 아니면 `null`(거부). camIdx/presetIdx 는 기존 globalIndex 값을 보존.
- 저장 경로는 #2/#3 과 동일한 PUT(`saveMapping`)을 재사용한다.

## 2. 영속화 — PUT /mapping (+ /viewer/api/mapping)

**저장 모델: 명시적 "저장" 버튼(편집은 메모리, 저장 시 1회 전체 PUT).** 자동저장 아님(동시성·오조작 위험↓ — 설계 확정 B).

| 항목 | 값 |
|---|---|
| 경로 | `PUT /mapping`(헤드리스) · `PUT /viewer/api/mapping`(뷰어). 동일 핸들러 `saveMappingHandler` |
| 본문 | 완전한 SetupArtifact JSON `{ presets, slots, globalIndex, createdAt, warnings?, report? }` |
| 200 | `{ ok:true, slots, globalCount }` + `repo.saveArtifact` 1회 |
| 400 (shape) | `{ error:'invalid artifact', detail }` — **미저장** |
| 400 (정합) | `{ error:'coverage mismatch', missing, extra }` — **미저장(파일 보호)** |
| 게이트 순서 | ① zod `SetupArtifactSchema` 검증 → ② `validateCoverage(globalIndex, slots)`(기존 `GlobalIndexer` 재사용) → ③ `repo.saveArtifact` |

- **GET /mapping · /viewer/api/mapping 은 불변**(읽기 직접 반환, artifact 없으면 404).
- **PATCH(슬롯 단위) 기각**: 삭제 1건이 3곳을 동시에 바꾸므로 부분 PATCH 는 정합 책임이 클라이언트·서버로 갈라진다. 산출물이 작은 단일 JSON 이라 전체 교체가 멱등·단순. 정합 책임은 "클라이언트(core.js 순수함수)가 재구성 → 서버가 `validateCoverage` 로 게이트"의 단일 지점으로 모은다.

## 3. 순수 함수 목록 (web/core.js — 전부 vitest 대상, DOM/fetch 미참조)

| 함수 | 용도 | 기능 |
|---|---|---|
| `diffArtifactVsCameras(artifact, cameras)` | #4 | 산출물에만 있는/카메라에만 있는 프리셋 키 산출 |
| `pointInRect(nx, ny, rect)` | #1 | 정규화 점이 사각형 내부인지(경계 포함) |
| `pointInQuad(nx, ny, quad)` | #1 | 정규화 점이 4점 다각형 내부인지(ray casting) |
| `hitTestSlots({nx,ny,slots,key,layers})` | #1 | 클릭 지점의 최상위 slotId(레이어 토글 반영) |
| `rebuildGlobalIndex(slots, presets)` | #2 | coveredSlotIds 순서 기반 globalIndex 재생성 |
| `removeSlot(artifact, slotId)` | #2 | 삭제 후 정합 재구성(불변) |
| `clamp01Rect(rect)` | #3 | 0~1 클램프(최소폭 보장) |
| `resizeRect(rect, handle, ndx, ndy)` | #3 | 모서리 드래그 적용 + 뒤집힘 정규화 + 클램프 |
| `updateSlotRoi(artifact, slotId, key, rect)` | #3 | 해당 slot ROI만 교체(불변) |
| `validateManualIndex(globalIndex)` | #7 | 1..N 연속·중복 검사 |
| `reorderGlobalIndex(artifact, orderedSlotIds)` | #7 | 지정 순서로 globalIdx 재부여(불일치 null) |

서버 측 정합 게이트는 기존 `validateCoverage`(GlobalIndexer)를 그대로 재사용한다(신규 정합 로직 없음).

## 5. 사용 흐름

**검수 탭(편집):**
```
정밀 수집 → 최종화(누적 결과로 ROI 표시)
  → 슬롯 선택(캔버스/목록 클릭)
  → 삭제 또는 모서리 핸들 드래그로 크기조정
  → "저장"(PUT) → 서버 정합 게이트 통과 시 파일 기록 + 재로드
```

**분석 탭(전역 인덱스):**
```
"전역 인덱스 수동 매핑"에서 ▲▼로 재정렬 → 정합 표시(중복/누락) 확인
  → "저장"(PUT) → 통과 시 globalIndex 1..N 재부여 영속화
```

**표시 제어:** "표시 초기화"로 화면만 끄기(데이터 보존) / 정밀수집 시작 시 자동 정리 / 최종화 시 자동 복귀.
