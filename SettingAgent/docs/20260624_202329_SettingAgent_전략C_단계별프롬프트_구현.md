# SettingAgent 전략 C 구현 — 단계별 프롬프트 파일 + 단계 실행 코드 + config 설정

- 설계 근거: `20260624_195315_SettingAgent_LLM셋업프롬프트_설계와최적안.md`(전략 C 선정)
- 요구: ① 단계별 프롬프트를 각각 파일로 ② 단계별로 사용하는 코드 ③ config에서 단계별 프롬프트 설정

## 1. 단계별 프롬프트 파일 (각각 파일로)

`SettingAgent/config/prompts/` 에 단계별 **system / user** 프롬프트를 분리 저장.

| 단계 | system | user(템플릿) |
|------|--------|--------------|
| 1. 프리셋별 비전 판정 | `stage1_preset_judge.system.md` | `stage1_preset_judge.user.md` |
| 2. 프리셋 간 중복제거+라벨 | `stage2_dedupe_label.system.md` | `stage2_dedupe_label.user.md` |
| 3. 최종 검증+설치 리포트 | `stage3_final_report.system.md` | `stage3_final_report.user.md` |

- user 템플릿은 `{{placeholder}}` 치환식(예: `{{camIdx}}`, `{{boxes}}`, `{{slotsByPreset}}`).
- 각 system 프롬프트는 **"좌표를 만들지 말고 번호/slotId 로만 지칭"** 원칙과 **출력 JSON 스키마**를 명시.
- 프롬프트는 코드 수정 없이 파일만 편집해 교체 가능.

## 2. config 에서 단계별 설정 (llm.config.json)

`llm.config.json` 에 `setupPrompts` 섹션 추가(도구 config 와 분리된 LLM 영역).

```json
"setupPrompts": {
  "stage1Enabled": true, "stage2Enabled": true, "stage3Enabled": true,
  "stage1": { "system": "config/prompts/stage1_preset_judge.system.md",
              "user":   "config/prompts/stage1_preset_judge.user.md" },
  "stage2": { "system": "...stage2...system.md", "user": "...stage2...user.md" },
  "stage3": { "system": "...stage3...system.md", "user": "...stage3...user.md" }
}
```
- 단계별 **on/off**(`stageNEnabled`)와 **프롬프트 파일 경로**를 config에서 지정.
- 스키마/검증: `src/config/llmConfig.ts`(zod). 로더가 기본값과 병합.
- 비전 모델 권장: `llm.model` 기본값을 `qwen3-vl` 로 설정(1·2단계 이미지 판단).

## 3. 단계별 실행 코드

### 3.1 두뇌 인터페이스 — `src/brain/SetupBrain.ts`
`SetupBrain` 인터페이스 + 단계별 입력 타입 + **zod 결과 스키마**(Stage1/2/3Result).
좌표는 만들지 않고 판정만 반환(번호/slotId).

### 3.2 구현 — `src/brain/AgentRuntime.ts` (`implements SetupBrain`)
- `judgePreset()`(1단계): 설정된 system+user 로드 → `{{...}}` 치환 → **이미지(base64) 첨부** chat 호출
  → 응답 JSON 추출·zod 검증 → `Stage1Result`.
- `dedupeAndLabel()`(2단계), `finalReport()`(3단계): 동일 패턴(텍스트).
- `src/brain/prompts.ts`: `loadPrompt`, `renderTemplate`, `extractJson`(코드펜스/잡텍스트 안전 파싱).
- `llm.enabled=false` 또는 해당 `stageNEnabled=false` → 각 메서드 **null 반환(결정형 폴백)**.
- model-agnostic: OpenAI 호환 엔드포인트 → Qwen3-VL/Gemma3/Claude 교체 가능.

### 3.3 게이트 통합 — `src/setup/SetupOrchestrator.ts`
결정형 흐름에 단계 게이트를 삽입(브레인 주입·활성 시에만):
```
프리셋 순회:
  캡처 → VPD → ROI(built)
  └ [게이트1] judgePreset → validBoxes 필터 + 순서보정 + 재촬영/제외 경고 (positionIdx 재부여)
전체 수집 후:
  └ [게이트2] dedupeAndLabel → 중복 slotId 병합(대표만) + zone 라벨
  전역 인덱스 생성 + 커버리지 검증
  └ [게이트3] finalReport → mismatches 경고 병합 + 한글 리포트 → artifact.report
```
- 좌표는 항상 검출 결과(built.roi) 사용 → LLM 좌표 환각 차단.
- 각 게이트는 try/catch 로 실패 시 경고만 남기고 **결정형 결과 유지**(셋업 비중단).
- `SetupArtifact` 에 `report?: string` 추가(게이트3 리포트).

  - `setupOrchestratorBrain.test.ts`(2): fake brain 으로 게이트1 제외 → 게이트2 병합/라벨 → 게이트3 리포트,
    brain 미주입 시 결정형 그대로.

## 5. 영향도

- 기존 결정형 경로 **불변**(brain 미주입/비활성 시 동작 동일 — 테스트로 보증).
- `SetupArtifact.report?` 는 선택 필드 → Action/DM 계약 하위호환.
- `llm.config` 에 `setupPrompts` 추가 → 로더 병합으로 기존 파일 하위호환.
- 비전 입력(base64 이미지)은 1단계에서만 사용. 텍스트 모델이면 1·2단계 품질이 제한되나 동작은 유지.

## 6. 사용법

1. 로컬 비전 LLM(Qwen3-VL 등, OpenAI 호환) 기동.
2. `config/llm.config.json` → `llm.enabled=true`, `model`/`baseUrl` 설정. 단계별 on/off·프롬프트 경로 조정.
3. 프롬프트 수정이 필요하면 `config/prompts/*.md` 파일만 편집(코드 변경 불필요).
4. 셋업 실행(`/setup/run-from-map` 또는 `npm run e2e`) → 게이트 적용 결과·경고·`report` 산출.
