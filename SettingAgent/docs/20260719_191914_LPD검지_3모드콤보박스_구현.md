## 1. 배경

정밀수집 페이지의 검출 관련 버튼이 `cap-actions` 툴바(`web/index.html:178`)에 시작/정지/최종화/초기화 4버튼과 함께 **'LPD 검출'(`#cap-detect-run`)**·**'VPD 검출(테스트)'(`#cap-vpd-test`)** 2개가 뒤섞여 있었다. 여기에 이미 백엔드가 완성됐지만 UI에 연결되지 않았던 세 번째 검지 방식(육면체 앞면중심 기준 LOOP 크롭줌 discovery, `PlateDiscoveryJob` + `/discover/*` 라우트)이 있었다.

이번 작업은 검출부를 **별도 라인(콤보박스 + 실행 버튼)** 으로 분리하고, 콤보박스 3옵션으로 세 검지 모드를 한 개의 실행 버튼에서 디스패치하도록 배선했다.

## 2. 사용법

`web/index.html:184-199`에 `cap-actions` 툴바(시작/정지/최종화/초기화) 바로 아래 신설된 라인:

1. **`#lpd-mode` 콤보박스**에서 모드 선택(기본값 `순수 LPD`).
2. **`#lpd-run`(실행)** 클릭 → 선택값에 따라 디스패치(`web/app.js:3338-3342`).

| 콤보 옵션(value) | 동작 | 내부 함수 |
|---|---|---|
| 순수 LPD(`lpd`) | 차량·점유·육면체·마스크 오버레이 off, 번호판 오버레이만 on 후 `runLiveDetect(false)` 1회 진단 | `runModeLpd()` (app.js:2370) |
| 앞면중심 LOOP(`discover`) | `POST /discover/ptz`(body `{}`, 전체 배치)로 discovery 배치 잡 시작 → `/discover/status` 폴링으로 진행바·라벨 갱신, `/discover/frame`으로 라이브뷰 추종, 종료 시 `/discover/status`의 `found/total`로 완료 요약 | `discStart()`(2306) / `discPoll()`(2325) / `discFrameTick()`(2275) / `renderDiscResult()`(2359) |
| VPD→LPD(`vpd`) | 차량·점유 오버레이 on 후 `runLiveDetect(true)`(차량+육면체 복원 후 검출) | `runModeVpd()` (app.js:2381) |

모드 (b) 진행 중에는 `#disc-bar`(progress)·`#disc-label`(텍스트)이 `idle→running N/M(found F)→done/error`로 갱신되고, `#lpd-run`이 자동 비활성화됐다가 종료 시 복구된다(`discPoll` → `discoverView()` 반영). `#disc-msg`에 시작/완료/오류 메시지가 표시된다.

## 3. 구조

### 3-1. HTML 배선 (`web/index.html`)
- `cap-actions` 툴바(178-183)에서 기존 `#cap-detect-run`·`#cap-vpd-test` **2버튼 삭제**, 시작/정지/최종화/초기화 4버튼 유지.
- 툴바 아래 신규 라인(184-199): `<select id="lpd-mode">`(3옵션) + `<button id="lpd-run">` + `<progress id="disc-bar">`/`<span id="disc-label">` + `<div id="disc-msg">`. 신규 CSS 없이 기존 `cap-actions toolbar`·`cap-progress`·`cap-msg`·`.field` 클래스를 재사용했다.

### 3-2. 모드 디스패처 (`web/app.js`)
- 기존 두 버튼의 클릭 핸들러 본문을 **그대로 이동**해 `runModeLpd()`(2369-2379)·`runModeVpd()`(2380-2385) 함수로 만들고, `#lpd-run` 단일 바인딩(3338-3342)에서 `$('lpd-mode').value`로 `lpd`→`runModeLpd`/`discover`→`discStart`/`vpd`→`runModeVpd` 분기한다. 오버레이 토글 로직은 수정 없이 이사만 했다.

### 3-3. discovery 폴링 (`web/app.js`) — `calPoll`/`calFrameTick` 미러
- 신규 상태 변수(`disc` 접두): `discPollTimer`, `prevDiscState`, `discFrameTimer`, `discFrameUrl`.
- `discStart()`(2306): `POST /discover/ptz` body `'{}'`(전체 배치, `slotIds` 미전달) → 실패/대상 0/시작됨 메시지 후 `discPoll()` 개시.
- `discPoll()`(2325): `GET /discover/status` → `discoverView(status ?? {})`(core.js) 결과로 `disc-bar.value`/`disc-label.textContent`/`lpd-run.disabled` 갱신, `view.polling`에 따라 `startDiscFramePolling()`/`stopDiscFramePolling()` 스위칭, `running→종료` 전환 1회에 `renderDiscResult()` 또는 종료 메시지 + `startLive()` 복귀, `pollPlan(st)`로 `running`에서만 재폴링.
- `discFrameTick()`(2275): `GET /discover/frame` → blob → `frame.src` 교체(`calFrameTick`과 동일 패턴, 이전 objectURL revoke).
- `renderDiscResult()`(2359): 최종 `GET /discover/status`의 `found`/`total`로 완료 요약 텍스트 작성.

### 3-4. 3자 프레임폴 상호배타 (불변식3)
- `startDiscFramePolling()`(2290)이 `stopCapFramePolling()`+`stopCalFramePolling()`+`stopLive()`를 호출.
- 역방향 완성: 기존 `startCapFramePolling`(app.js:1888)·`startCalFramePolling`(app.js:2182)에 `stopDiscFramePolling();` **각 1줄 추가** → capture·centering·discovery 3자가 서로를 정지시킨다.

### 3-5. 순수 헬퍼 `discoverView` (`web/core.js:152` / `web/core.d.ts:72`)
```js
export function discoverView(status) {
  const st = status?.state ?? 'idle';
  const done = status?.done ?? 0;
  const total = status?.total ?? 0;
  const found = status?.found ?? 0;
  const percent = total > 0 ? Math.round((done / total) * 100) : 0;
  const running = st === 'running';
  return { percent, label: `${st} ${done}/${total} (found ${found})`, runDisabled: running, polling: running };
}
```
`status`가 `null`/`undefined`거나 필드가 누락돼도 `idle 0/0(found 0)`로 안전 폴백하며, `total===0`일 때 0으로 나누지 않도록 방어한다. `runDisabled===polling===(state==='running')` 불변식을 지킨다.

### 3-6. REST 계약 (백엔드 무수정, 재사용만)
| 엔드포인트 | 메서드 | 요청 | 응답(성공) | 비고 |
|---|---|---|---|---|
| `/discover/ptz` | POST | body `{}`(전체 배치) | `{ ok, started, total }` | 이미 running이면 **409**(already running) |
| `/discover/status` | GET | - | `DiscoverStatus { state: 'idle'\|'running'\|'done'\|'error', done, total, found }` | `discoverView` 입력과 필드·타입 1:1 |
| `/discover/frame` | GET | - | image blob | `X-Disc-Cam`/`X-Disc-Preset` 헤더 존재하나 라벨 미사용(최소주의) |

## 5. 한계 (정직 리포트)

- **discovery 실 검지율 미관찰**: 실 LPD(da_lpd_api)·실 카메라 가동 시 "실제 몇 % 슬롯이 구제되는가"는 이번 검증 범위 밖(선행 문서 `docs/20260719_163729_번호판탐색_줌반복_역계산_구현.md` §5와 동일한 한계).
- **DOM 디스패치·폴 전이는 vitest 밖**: `discoverView` 순수 로직만 자동 커버되며, 실제 브라우저 이벤트 배선·타이머 전이는 리더 라이브 관찰에 의존한다.
- **백엔드 카메라소스 테스트 flakiness**: `cameraMode`/`sourceRegistry`/`settingsFormErrors` 계열이 전량 동시 실행 시 공유 설정파일 경합으로 불안정한 것으로 확인됐으나, 이는 이 작업 범위 밖의 별개 안정화 대상이다.
