## 2. 계약 불변 (파급 차단 지점)

- **`setup_artifact.json` 읽기 전용**: 캘리브레이션 코드는 `repo.loadArtifact()` 만 호출하고 `saveArtifact` 를 **호출하지 않는다**. setup_artifact 스키마·내용 무변경. → Action/DM 무영향(QA §2.6 검증).
- **`@parkagent/types` 무변경**: `slot_ptz.json` 타입은 `src/calibrate/types.ts` 로컬에만 존재. 공유 패키지 미수정 → 다른 에이전트 재컴파일·재배포 불필요.
- **기존 라우트 불변**: `/setup/*`·`/capture/*`·`/mapping`·`/brain/*`·뷰어(`/viewer/*`) 라우트 코드 미변경. `/calibrate/*` 는 `deps.calibrator && deps.calibrate` 주입 시에만 등록(가산). 미주입 시 미등록 → 기존 경로 무영향(QA: calibrator 미주입 시 `/calibrate/status` 404 + `/setup/status` 200 확인).
- **`slot_ptz.json` 은 별도 파일**: `slotPtzWriter.writeSlotPtz` 가 독립 파일 I/O. `Repository`·`Finalizer` 코드 비오염.
- **CameraClient/LpdClient 무변경**: 기존 시그니처 그대로 사용. `clampZoom`(1~36) 보존 — `zoomCorrection`·`applyZoomAdvice` 가 `camera.clampZoom` 경유.
- **`SetupBrain` 확장은 옵셔널 메서드 가산**: `adviseCentering?` 가 옵셔널 → 기존 구현체(AgentRuntime 외)·기존 호출부 무영향. `AgentRuntime` 의 다른 메서드 미변경.

## 3. 의존성 그래프 (이번 변경의 전파 범위)

```
config/tools.config.json (calibrate 섹션)
  └ toolsConfig.ts (CalibrateSchema)
      └ index.ts ──┬─ PtzCalibrator (cfg: tools.calibrate)
                   └ server.ts (calibrate?, calibrator?) ─ calibrateRoutes.ts ─ /calibrate/*

PtzCalibrator
  ├ controlMath.ts (순수)        ← 외부로 전파 없음
  ├ slotPtzWriter.ts → slot_ptz.json (신규 파일)
  ├ CameraClient/LpdClient (기존, 미변경)
  ├ Repository.loadArtifact (읽기 전용)
  └ SetupBrain.adviseCentering? ─ AgentRuntime (옵셔널)

web/app.js ── /calibrate/* (절대경로 폴링)
```

전파 경계는 **SettingAgent 내부에서 닫힘**. 모노레포 다른 패키지(`@parkagent/types`·ActionAgent·DMAgent·SettingViewer)로의 컴파일·계약 전파 없음.
