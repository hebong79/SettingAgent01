# 웹 뷰어 "표시 초기화" · "검출 삭제" 버튼 정상화

- 작성: 2026-07-15
- 대상: `SettingAgent/web/app.js`, `SettingAgent/web/index.html`
- 실행 모드: parkagent-dev **goal/loop(B)** — 관찰형(시각 동작) 성공기준

## 1. 요청

웹 UI의 **검출 삭제**(`#det-delete`)·**표시 초기화**(`#roi-clear`) 버튼이 정상 동작하게 수정.

## 2. 원인 분석 (사실 확인 후 판정)

두 버튼 모두 핸들러·이벤트 결선은 존재했다. 각각 사실로 검증한 결과:

### 2-1. 표시 초기화 — **실제 결함** (수정 대상)

`clearRoiDisplay()`는 `state.roiHidden = true`만 세팅했다. 그런데 `drawRoiOverlay()`는
`roiHidden` 가드([app.js:279](../web/app.js#L279)) **이전**에 다음 6개 레이어를 그린다:

| 레이어 | draw 함수 | 가드 |
|--------|-----------|------|
| 점유 | `drawOccupancyOverlay` | `#roi-occupancy` 토글만 |
| 파일 바닥 ROI | `drawFileFloorRoi` | `#roi-floor` 토글만 |
| 라이브 검출(VPD/LPD) | `drawDetectOverlay` | `#roi-detect` 토글만 |
| 주차면 육면체 | `drawCuboidOverlay` | `#roi-cuboid` 토글만 |
| 차량 육면체 | `drawVehicleCuboidOverlay` | `#roi-vcuboid` 토글만 |
| seg 마스크 | `drawMaskOverlay` | `#roi-mask` 토글만 |

즉 `roiHidden`은 **슬롯 ROI(가드 이후)** 만 숨기고, 위 6개는 각자 토글로만 가드되어 **화면에 그대로 남았다.**
→ 사용자가 "표시 초기화"를 눌러도 검출/점유/육면체/마스크가 남아 "아무것도 안 지워짐"으로 보였다.

**제약(중요):** `clearRoiDisplay()`는 수집 시작 `capStart()`([app.js:1869](../web/app.js#L1869))에서도 호출된다.
수집 중에는 검출·점유·육면체가 **라이브로 계속 표시돼야** 하므로(그래서 가드 이전에 그림), `clearRoiDisplay`를
전역으로 바꾸면 라이브 수집 표시가 깨진다. → 버튼 전용 함수를 **분리**해야 한다.

### 2-2. 검출 삭제 — **로직 정상** (코드 변경 없음)

아래를 모두 사실로 확인했고 결함이 없었다:

- 핸들러 `deleteSelectedDetect`([app.js](../web/app.js)) · 이벤트 결선(`#det-delete` click, Delete/Backspace/Esc) 정상.
- 순수 로직 `hitTestDetections` / `removeDetection`([core.js:1255,1289](../web/core.js#L1255)) 정상(불변 splice).
- 버튼 활성화 `renderDetectSelection`(선택 시 `disabled=false`) 정상.
- 좌표 변환 `eventToNorm`(getBoundingClientRect 정규화) 정상.
- 캔버스 `#overlay { pointer-events: auto }` — 클릭 수신 정상.
- 그리기/히트테스트 모두 `vehicles[i].rect` · `plates[i].quad` 동일 shape 사용 — 불일치 없음.
- `drawDetectOverlay`·선택·삭제 모두 `currentFrameKey()` 동일 키 사용 — 키 어긋남 없음.

**결론:** 검출 삭제는 **설계대로 동작**한다. 사용법은 2단계다 —
① 검출 박스를 클릭해 선택(선택 시 빨강 강조 + 버튼 활성) → ② "검출 삭제"(또는 Delete/Backspace).
선택 전에는 버튼이 비활성(`disabled`)이라 "안 눌린다"고 보일 수 있으나 의도된 상태다.
동작하는 코드를 함부로 고치지 않는다(CLAUDE.md 3원칙).

## 3. 수정 내용 (최소·외과적)

> **정정(2026-07-16, 마스터 요청):** 표시 초기화는 **Hide(토글 off)가 아니라 데이터 삭제**여야 하며, **바닥 ROI는 제외**(유지)한다. 아래는 최종(삭제) 방식 기준.

### app.js
- `clearRoiDisplay()`는 **capStart 전용**으로 주석 명확화(동작 불변 — 슬롯 ROI만 숨김, 라이브 레이어 보존).
- **신규 `resetOverlayDisplay()`** 추가 — "표시 초기화" 버튼 전용, **오버레이 데이터 삭제**(Hide 아님):
  - `state.detectByKey = {}` (검출 차량/번호판), `state.occComputeByKey = {}`·`state.occByKey = {}` (점유),
    `state.vcuboidByKey = {}` (차량 육면체 + seg 마스크) — 실제 삭제라 재토글로 복원 안 됨.
  - `selectedSlotId/selectedPlaceIdx/selectedDetect=null` + `renderDetectSelection()` + `drawRoiOverlay()` + `renderSlotList()`.
  - **바닥 ROI는 보존**: 파일 소스 `state.placeRoi` 와 `#roi-floor` 토글을 건드리지 않음 → 그대로 표시.
- `#roi-clear` 결선을 `clearRoiDisplay` → `resetOverlayDisplay` 로 변경.

### index.html
- `#roi-clear` 툴팁을 실제 동작(바닥 제외 삭제·Hide 아님)에 맞게 갱신.

삭제 후 `drawRoiOverlay`는 남은 소스(placeRoi)만 그리므로 화면엔 **바닥 ROI만** 남는다. 검출/점유/육면체/마스크는
데이터가 사라져 토글을 다시 켜도 나오지 않고, 다음 검출/수집 때 새로 채워진다.

## 5. 영향도 분석

- **직접 변경**: `web/app.js`(함수 1개 추가 + 주석 + 결선 1줄), `web/index.html`(툴팁 1줄), 신규 테스트 1개.
- **`clearRoiDisplay` 호출부**: `capStart`(수집 시작) 1곳 — **동작 불변**(라이브 검출/점유/육면체 표시 보존). 회귀 없음.
- **`DISPLAY_TOGGLE_IDS`**: `wire()`의 각 토글 change 리스너와 동일 id 집합. 토글 자체 로직/점유 판정과 무관(순수 표시).
- **검출 삭제 경로**: 무변경 → 기존 동작·테스트 영향 없음.
- **정적 자산 서빙**: `@fastify/static`이 요청마다 최신 파일 서빙(nodemon `src/` 감시 밖이라도 새로고침이면 즉시 반영). 서버 재시작 불필요.
- **점유/최종화/DB 등 백엔드**: 표시 레이어 토글만 조작 → 데이터·판정 파이프라인에 영향 없음.

## 6. 사용법 요약(사용자 안내)

- **표시 초기화**: 누르면 **바닥 ROI만 남기고** 검출·점유·차량육면체·마스크·선택이 **삭제**된다(Hide 아님 — 재토글로 복원 안 됨). 다시 보려면 검출 실행/정밀수집으로 새로 채운다. (mapping 슬롯 ROI는 sim-less 워크플로에선 비어 있어 영향 없음 — 로드된 경우는 미삭제, 필요 시 후속.)
- **검출 삭제**: 먼저 화면의 검출 박스를 **클릭해 선택**(빨강 강조 + 버튼 활성) → "검출 삭제" 클릭 또는 Delete/Backspace. Esc 로 선택 해제. (임시·메모리 편집이라 다음 재검출 시 원복.)

## 7. 후속 수정 (같은 세션 — 뷰어 토글)

마스터 추가 요청(2026-07-15, Unity sim 13100 미사용 전제):

### 7-1. 차량 · 번호판 토글 무동작 → 검출 오버레이 서브필터로 수정
**원인:** `#roi-vehicle`·`#roi-plate` 토글은 그동안 **슬롯 매핑 ROI**(`state.mapping` 필요)만 가드했다.
sim/최종화 없이는 mapping 이 비어(`slots:[]`) 있어 두 토글이 무동작이었다. 반면 **바닥**은 파일 ROI(PtzCamRoi),
**검출**은 `detectByKey` 로 mapping 없이도 그려져 동작했다. 화면에 실제 보이는 차량/번호판 시각물은 **검출 오버레이**다.

**수정:** `drawDetectOverlay` 를 `검출`=마스터, `차량`/`번호판`=서브필터 구조로 변경.
- `roi-detect` off → 검출 전체 숨김(마스터, 기존과 동일).
- `roi-detect` on + `roi-vehicle` off → 차량 청록 박스만 숨김.
- `roi-detect` on + `roi-plate` off → 번호판 quad(차량 부속 + base LPD)만 숨김.

이제 mapping 유무와 무관하게 두 토글이 화면에서 즉시 동작한다. (기존 슬롯 ROI 가드 용도도 그대로 유지 — 동일 토글이 차량/번호판 계열 시각물 전부를 관장.)

### 7-2. 차량육면체 토글 시작 시 체크 해제
`index.html` 의 `#roi-vcuboid` 에서 `checked` 속성 제거 → **시작 시 off**. (인접 "기본 on" 주석도 갱신.)
켤 때 해당 프리셋 캐시가 없으면 그때 라이브 촬영 1회 하는 change 핸들러는 그대로라 동작 변화 없음.

### 7-3. 검증(후속)
- 신규 `test/viewerToggleGating.test.ts` — 차량/번호판 독립 가드 + 시작 토글 상태 가드.
- 전체 회귀 통과. 서버(13020)가 수정 자산 서빙 확인.
- 한계: 브라우저 클릭까진 미구동(상동) — 각 draw 함수 토글 early-return 경로로 논증.

## 8. 후속 수정 — 검출 체크박스 제거 + 차량/번호판 완전 독립 (2026-07-16)

마스터 요청: "검출 체크박스는 불필요. 제거하고, 차량/번호판이 검출 체크박스와 **무관하게** 동작하도록."

§7-1 에서 `검출`=마스터 게이트 구조로 만들었으나(차량/번호판이 검출 ON 일 때만 동작), 이를 **폐기**하고 검출 토글 자체를 제거한다.

**변경:**
- `index.html`: `#roi-detect` 라벨·체크박스 **삭제**.
- `drawDetectOverlay`: 마스터 게이트 `if (!$('roi-detect').checked || !d) return;` → `if (!d) return;`.
  차량 박스는 `#roi-vehicle`, 번호판 quad(차량 부속 + base LPD)는 `#roi-plate` 로 **각각 독립 가드**. 두 토글이 검출 토글 없이 즉시 동작.
- `wireOverlayEditing` mousedown: 검출 박스 편집/선택 진입 조건을 `roi-detect ON` → `roi-vehicle OR roi-plate 표시 중` 으로 변경(보이는 레이어에서만 선택·삭제).
- `wire()`: `#roi-detect` change 리스너 제거.

**영향도:** `#roi-detect` 활성 참조 전부 제거(코드/주석/테스트 — `grep roi-detect web/` = 0). 검출 데이터(`detectByKey`)·검출 삭제·순수로직 무변경. 전체 회귀 **150 파일 · 1641 테스트 통과**. 서버 서빙 확인.
(과거 문서의 `#roi-detect` 언급은 히스토리로 보존.)
