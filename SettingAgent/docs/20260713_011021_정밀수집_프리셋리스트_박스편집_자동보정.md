## ① 카메라 PTZ 프리셋 리스트 CRUD

정밀수집 탭 하단에 camerapos 프리셋 리스트를 두고, 기존 제어패널의 프리셋 편집 로직을 **공유 함수화**해 중복 없이 리스트에서도 선택/추가/수정/삭제/저장/열기를 수행한다.

### 동작
- **선택**: 리스트 행 클릭 → `selectCameraposItem(cam,preset,label)` → `state.cam/preset` 설정 → `renderCamSelect()` 동기화 → `gotoPreset()`(카메라 물리 이동) → `reconnectLiveIfActive()`(라이브 시 스트림 재연결).
- **추가**: 현재 PTZ를 새 프리셋으로. `savePreset(true, labelOverride)` 재사용(즉시 PUT).
- **수정**: 선택 프리셋을 현재 PTZ로 갱신. `savePreset(false, labelOverride)` 재사용.
- **삭제**: `deletePreset()` 재사용.
- **저장**: 열기 버퍼(`state.cameraposEdit`)를 `persistCamerapos`(PUT `/viewer/api/camerapos`)로 확정 후 버퍼 폐기·서버 뷰 복귀.
- **열기**: `pickAndReadJsonFile`(네이티브 파일 대화상자) + `parseLoadedCamerapos` → `state.cameraposEdit` 버퍼에 **메모리 표시만**(즉시 PUT 아님). 리더 확정 = **(b) 표시 후 별도 '저장'**(실수 저장 방지, 결과 열기 패턴과 동일).

추가/수정/삭제는 `cpresetAction(fn)` 래퍼가 열기 버퍼 폐기 + 실행 + 리스트 재렌더를 묶는다.

### 순수 함수(`web/core.js`)
- `cameraposListRows(views)` → `camIdx ASC → presetIdx ASC` 정렬(불변, 새 배열). 리스트 렌더 순서 결정.
- `parseLoadedCamerapos(text)` → `{ok:true, views}` | `{ok:false, error}`. 허용 형태 = **최상위 배열** 또는 `{views:[...]}`. 각 항목 `camIdx/presetIdx/pan/tilt/zoom` 수치 검증(`parseLoadedArtifact` 미러). `label` 없으면 `Preset {n}` 폴백. 깨진 JSON·배열아님·필드누락(1-based 항목번호 메시지)·문자열 zoom 각각 거부.

### 공유 함수화(중복 제거, 외과적)
- `savePreset(asNew, labelOverride?)` — **`labelOverride` 선택 인자만 추가**. 미지정 시 기존대로 제어패널 `#preset-label`, 지정 시 정밀수집 `#cpreset-name`. 제어패널/정밀리스트가 동일 함수를 공유한다.

### state
- `state.cameraposEdit` — '열기'로 불러온 **미저장 views 버퍼**. 있으면 리스트가 이것을 표시, '저장'으로 서버 확정.

### UI(`web/index.html` `#cpreset-box`)
`#cpreset-name`(이름) + 버튼 `#cpreset-add`·`#cpreset-update`·`#cpreset-delete`·`#cpreset-open`·`#cpreset-save` + `#cpreset-list` + `#cpreset-msg`.

### 라우트
**신규 없음.** 기존 GET/PUT `/viewer/api/camerapos` 재사용(PUT은 `tokenHeaders` controlToken 준용).

### 한계·가정
- '열기' 대상 형식은 **정규화 views**(GET/PUT과 동일 shape). Unity 원본 `camerapos.json`의 중첩 `datas` 포맷은 백엔드 `parseCameraViews` 소관 — 프론트 열기는 정규화 형식만.
- 버퍼에만 있고 서버에 없는 프리셋 행 클릭 시, `renderCamSelect`/`gotoPreset`이 서버 목록 기준이라 선택이 서버 첫 프리셋으로 폴백될 수 있음(저장 후엔 정합). '표시 후 저장' 흐름과 정합.

## ② VPD/LPD 검출 박스 편집(임시·메모리만)

overlay 캔버스에서 검출 박스를 클릭 선택 → 하이라이트 → 모서리 리사이즈/이동 → 삭제. `state.detectByKey`를 **메모리에서만** 편집하며 **저장 경로가 없다**(다음 검출/프레임 순환 시 덮여 사라짐 = 사용자 확정 '임시').

### 데이터 형태
- VPD = `vehicles[i].rect{x,y,w,h}`(정규화) → rect 8핸들 리사이즈(`hitTestRectHandle`+`resizeRect`+`moveRect` 재사용).
- LPD = `plates[i].quad[4]{x,y}` → quad 정점 드래그(`hitTestQuadVertex`+`moveQuadVertex` 재사용).

### 순수 함수(`web/core.js`)
- `hitTestDetections({nx,ny,detect,tolX,tolY,selected})` → vehicle `{kind,index,handle}`(`'nw'..'se'`|`'in'`) / plate `{kind:'plate',index,vertex?}` | `null`.
  - 우선순위: **선택 대상(selected)의 핸들/정점 > vehicle rect 내부 > plate quad 내부**. 같은 종류 겹침은 마지막(=위) 항목 우선(그리는 순서와 정합).
- `removeDetection(detect, sel)` → sel.kind/index 항목 splice한 새 `{...detect, vehicles, plates}`(불변, 여타 필드 보존). sel 없음/범위밖이면 얕은복사만.

### app.js 결선
- `state.selectedDetect = {kind,index} | null`(신규).
- `wireOverlayEditing()` mousedown **최상단(기존 가드 이전)** 검출 분기: `roi-detect ON && !Ctrl && detect 존재` → `hitTestDetections` → 히트 시 선택 + dragState(`detResize`/`detMove`/`detVertex`), 빈 곳 클릭 시 선택 해제 후 기존 슬롯편집으로 낙하.
- **슬롯편집(Ctrl+드래그)과 물리 배타**: 검출 편집은 plain click(Ctrl 아님) + detect 토글 ON + 가드 이전 분기.
- mousemove: `detResize=resizeRect`, `detMove=moveRect`(vehicle rect), `detVertex=moveQuadVertex`(plate quad) → `state.detectByKey[key]` 제자리 변형 후 `drawRoiOverlay()`. mouseup은 `markDirty` 생략(mapping 미저장 표시 아님).
- `drawDetectOverlay` 확장: 선택 vehicle=굵은 대비색 rect + `drawHandles`, 선택 plate=`drawQuadHandles`(기존 렌더 재사용).
- 삭제: `deleteSelectedDetect()` + `renderDetectSelection()`(`#det-delete` disabled 토글). `runLiveDetect` 도착 시 `selectedDetect=null`(인덱스 무효화).
- 단축키(`document` keydown): **Delete/Backspace=삭제, Esc=해제**(입력/텍스트영역 포커스 시 무시).

### UI(`web/index.html`)
Live View 툴바(`.roi-toggles`)에 `#det-delete` 버튼 1개(최소 UI, 단축키 title 안내).

### 라우트
**없음**(메모리 편집만).

### 임시 특성(반드시 인지)
정밀수집 중 프레임 순환(`capFrameTick`) 또는 `runLiveDetect` 재검출 시 편집분이 덮여 사라진다. **단일 프리셋 정지 상태**에서 편집이 유의미. 영속화는 후속 과제.

## ③ 주차면 자동보정(이동+스케일, sharp 상호상관)

기준 프레임을 프리셋별로 저장해두고, 현재 프레임과 상호상관으로 이동(dx,dy)+스케일(s)을 추정해 주차면 ROI 폴리곤을 아핀 이동/스케일한 뒤 검토·저장한다. **이동+스케일만 보정하며 회전·원근은 미보정**(리더 확정).

### 데이터 흐름
```
[기준 저장] #align-save-ref → POST /capture/refframe {cam,preset}
            → camera.requestImage(cam,preset).jpg → data/refframes/cam{c}_p{p}.jpg

[자동보정]  #align-run → POST /capture/autocorrect {cam,preset}
            → 기준jpg + 현재캡처 → sharp greyscale+resize(128×72)+raw → estimateAlign
            → {dx:dxPx/128, dy:dyPx/72, scale, peak, confidence:peak}
            → transformPlaceRoiPreset 로 state.placeRoi[key] 변환 → drawRoiOverlay()
            → #align-msg 에 peak/오프셋 + "이동/스케일만(회전·원근 미보정)" 표기

[되돌리기]  #align-undo → state.placeRoiBackup(직전 깊은복사) 복원

[저장]      #align-apply → PUT /capture/place-roi {camId,presetIdx,spaces}
            → applyPlaceRoiUpdate(정규화→픽셀 역변환) → PtzCamRoi.json write
```
- 대상 프리셋 = `alignTarget()`(수집 중이면 `capFrameKey2`, 아니면 `state.cam/preset` — runLiveDetect 패턴).
- `state.placeRoiBackup = {key, spaces}`(되돌리기용 직전 깊은복사).

### 순수 수학(`src/capture/frameAlign.ts`, 신규 · vitest 대상)
- `normalizedCrossCorrelation(ref, cur, w, h, maxShift)` → `{dx,dy,peak}`. 제로평균 정규화 상호상관(NCC). **부호 규약: peak인 (dx,dy)는 `ref[p] ≈ cur[p+(dx,dy)]`** → ref 좌표계 ROI를 (dx,dy)만큼 이동하면 cur와 정합. 분산 0(featureless)이면 `peak=0` 방어. 반환 (dx,dy)는 항상 `[-maxShift, maxShift]` 경계 내.
- `scaleGray(src, w, h, s)` → `Uint8Array`. 중심 기준 바이리니어 리샘플(동일 w×h). s>1 확대. 경계 clamp.
- `estimateAlign(ref, cur, w, h, {scales, maxShift})` → `{dx,dy,scale,peak}`. scale 후보마다 cur을 `1/scale`로 되돌려(zoom 상쇄) NCC → peak 최대 선택. 반환 scale = cur이 ref 대비 확대된 배율. scales 미지정 시 `[1]` 폴백.

### 순수 아핀(`web/core.js`)
- `applyTranslateScale(point, {dx=0,dy=0,scale=1,cx=0.5,cy=0.5})` → `{x,y}`: `new = center + scale*(old-center) + (dx,dy)`(정규화 좌표, 중심=이미지 중앙).
- `transformPlaceRoiPreset(spaces, transform)` → 각 `space.points`에 적용(불변, idx 보존).

### 백엔드 역변환(`src/capture/placeRoi.ts`)
- `applyPlaceRoiUpdate(json, {camId,presetIdx,spaces})` → 새 json. 정규화 spaces(`[{idx,points:[{x,y}×4]}]`)를 대상 카메라(cam_id) `imageWidth/imageHeight`로 **픽셀 역변환**(`[x*W, y*H]`)해 그 프리셋 `parking_spaces`만 교체. 타 카메라·프리셋·메타 보존, 불변. 대상 부재·크기 오류 시 원본 그대로 반환(**throw 금지**, 강등 철학).

### 라우트(`src/api/captureRoutes.ts`, 3종 가산)

| 메서드·경로 | 본문 | 성공 응답 | 오류 코드 |
|---|---|---|---|
| POST `/capture/refframe` | `{cam,preset}` | `{ok,path}` | 캡처 실패 502 |
| POST `/capture/autocorrect` | `{cam,preset}` | `{ok, dx, dy, scale, peak, confidence}` | 기준 없음 404, sharp/캡처 실패 502 |
| PUT `/capture/place-roi` | `{camId,presetIdx,spaces}` | `{ok, spaceCount}` | 미설정/파일없음 404, body 400, 쓰기실패 500 |

- **autocorrect 파라미터**(고정): `ALIGN_W=128, ALIGN_H=72, ALIGN_MAX_SHIFT=12`(±≈9.4% 이동), `ALIGN_SCALES=0.9~1.1 step0.02`(11개). 정규화 오프셋은 다운스케일 불변.
- **sharp 경계**: `jpegToGray()`가 `sharp(jpg).greyscale().resize(w,h,{fit:'fill'}).raw()`로 **픽셀 추출만**. 상호상관·스케일탐색·아핀은 전부 순수 함수.
- **place-roi PUT zod**: `camId/presetIdx = int positive`, `spaces = [{idx:int, points:[{x,y}]}]`. **무토큰**(GET place-roi와 대칭, `/capture/*` 관례).
- GET `/capture/place-roi`는 raw JSON 그대로 반환(정규화는 클라이언트 core.js `normalizePtzCamRoi`).

### dep 배선
`CaptureRouteDeps.refFrameDir?` → `server.ts ApiDeps.refFrameDir?` → `index.ts`에서 `join(tools.store.dataDir,'refframes')`. **미주입 시 refframe/autocorrect 미등록**(가산, 회귀 0). PUT place-roi는 `placeRoiFile` 런타임 게이트(GET 대칭).

### UI(`web/index.html` `#align-box`)
버튼 `#align-save-ref`(기준 저장)·`#align-run`(자동보정)·`#align-undo`(되돌리기)·`#align-apply`(저장) + `#align-msg`.

### 한계(UI·문서 명시)
**이동+스케일만**(회전·원근 미보정). 특징 부족·큰 변화 시 부정확 → **peak(신뢰도) 표기 + 되돌리기** 제공.
