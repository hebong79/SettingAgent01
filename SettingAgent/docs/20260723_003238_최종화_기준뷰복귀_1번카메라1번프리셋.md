# 최종화 → 기준 뷰(1번 카메라·1번 프리셋) 복귀

- 작성: 2026-07-23 00:32:38
- 요청(마스터): "시작을 한 후에 모두 마무리 후 최종화를 누르면 1번 카메라 1번 프리셋으로 이동해줘."
- 대상: SettingAgent 뷰어(`web/app.js`, `web/index.html`)

## 1. 배경

정밀수집 시작 → LPD 탐색 → 점유영역 → 센터라이징 파이프라인이 끝나면 카메라는 **마지막으로 조준한 슬롯의 PTZ**에 그대로 멈춰 있다. 이 상태에서 "최종화"를 누르면 DB(slot_setup) 결과는 표시되지만, 화면은 마지막 센터라이징 지점(임의의 클로즈업)이라 결과 확인의 기준 시야가 되지 못했다.

→ 최종화를 **마감 동작**으로 정의하고, 누르면 기준 뷰(1번 카메라·1번 프리셋)로 카메라를 물리 이동한 뒤 그 프리셋 기준으로 DB 결과를 그린다.

## 2. 변경 내용

### 2-1. `gotoHomePreset()` 신규 (web/app.js, capFinalize 바로 위)

```js
const HOME_CAM = 1;    // 기준 뷰 카메라(1-based) — 최종화 마감 시 복귀 대상.
const HOME_PRESET = 1; // 기준 뷰 프리셋(1-based).

async function gotoHomePreset() {
  const cam = state.cameras.find((c) => c.camIdx === HOME_CAM);
  if (!cam || !(cam.presets ?? []).some((p) => p.presetIdx === HOME_PRESET)) return false;
  state.cam = HOME_CAM;
  state.preset = HOME_PRESET;
  state.selectedSlotId = null;  // 프리셋 컨텍스트 전환 → 선택 해제(수동 전환과 동일).
  state.selectedDetect = null;
  renderDetectSelection();
  renderCamSelect();            // 드롭다운 값 동기화(프리셋 셀렉트·PTZ 표시·목록 갱신 동반).
  renderSelectionInfo();
  await gotoPreset();           // 프리셋 PTZ 로 물리 이동(/move → /req_move).
  reconnectLiveIfActive();      // 라이브 중이면 새 cam:preset 으로 스트림 재연결.
  return true;
}
```

- **새 알고리즘 0건**: 수동 드롭다운 전환(`#sel-cam`/`#sel-preset` change 핸들러)이 하던 절차를 그대로 재사용한다(선택 해제 → 셀렉트 재렌더 → `gotoPreset()` → `reconnectLiveIfActive()`).
- **물리 이동 경로**: `gotoPreset()` → 프리셋 PTZ 가 있으면 `move()`(검증된 `/move` → Unity `/req_move`), 없으면 preset 모드 스냅샷 폴백. 기존 함수 그대로.
- **위장 금지 가드**: `/cameras` 목록에 `cam1:preset1` 이 없는 현장 구성이면 이동하지 않고 `false` 반환(없는 프리셋으로 이동한 척하지 않는다). 최종화의 나머지 표시 동작은 정상 진행.
- 1-based 인덱스 규약(ParkSimMgr) 준수 — `camIdx === 1`, `presetIdx === 1`.

### 2-2. `capFinalize()` 에 결선

```js
async function capFinalize() {
  $('roi-db').checked = true;
  state.roiHidden = false;
  await gotoHomePreset(); // 마스터 요청 2026-07-23: 마감 후 기준 뷰(1번 카메라·1번 프리셋)로 물리 복귀.
  await loadParkingSlots();
  await loadMapping();
  drawRoiOverlay();
  renderSlotList();
  ...
}
```

**순서가 중요하다** — 이동을 `loadParkingSlots()`/`drawRoiOverlay()` **앞**에 두어야 오버레이가 이동 후 프리셋(`currentFrameKey()` = cam1:preset1) 기준으로 그려진다. 이동을 뒤에 두면 이전 프리셋 키로 한 번 그린 뒤 이동해 한 프레임 어긋난다.

### 2-3. 툴팁 (web/index.html)

`#cap-finalize` title 에 "1번 카메라·1번 프리셋으로 이동한 뒤" 문구 추가.

## 3. 적용 범위

| 경로 | 이동 여부 | 근거 |
|------|----------|------|
| 정밀수집 탭 "최종화"(`#cap-finalize`) | O | `capFinalize` 직결 |
| 완료 팝업 "최종화"(`#cap-result-finalize`) | O | 팝업 닫고 `capFinalize()` 호출 — "모두 마무리 후" 시나리오의 주 경로 |
| 자동 파이프라인 `pollPipeline()` 의 finalize 완료 전환(`finalizing → calibrating\|done`) | X | 마스터 요청은 "**최종화를 누르면**" — 버튼 전용. 자동 전환 중 카메라를 뺏으면 이어지는 센터라이징을 방해한다 |
| 백엔드 `POST /capture/finalize`·`Finalizer` | X (무변경) | 뷰어 표시 계층만 변경 |

## 4. 검증

- 신규 회귀 가드 `test/finalizeHomePreset.test.ts` (7 케이스): 상수 1/1, `capFinalize` 가 DB 조회 **전에** `await gotoHomePreset()`, 선택 전환+물리이동+스트림 재연결 포함, 미존재 시 `return false` 가드, 두 버튼 동일 경로, `pollPipeline` 미개입, 툴팁 문구.
- **전체 스위트: 211 파일 / 2513 테스트 통과**(`npx vitest run`).
- 경험적 확인:
  - 구동 중 서버(13020)가 수정된 `app.js`(`gotoHomePreset` 정의·`capFinalize` 결선)를 실제 서빙함을 HTTP 로 확인.
  - `GET /viewer/api/cameras` 응답에 `camIdx=1`, `presets=1,2,3` 실재 → 현재 구성에서 가드를 통과해 실제 이동 경로를 탄다.
- **한계**: 브라우저에서 최종화를 눌러 Unity 창이 실제로 cam1:preset1 로 움직이는지는 육안 확인하지 않았다(현재 진행 중일 수 있는 수집을 방해하지 않기 위해 `/move` 실호출을 하지 않음). 이동 자체는 기존 검증 경로(`gotoPreset` → `/move` → `/req_move`)를 그대로 쓴다.

## 5. 영향도 분석

- **`gotoPreset()`/`move()`/`reconnectLiveIfActive()`/`renderCamSelect()`**: 호출만 추가, 함수 무변경 → 기존 드롭다운·PTZ 버튼 경로 회귀 0.
- **`state.cam`/`state.preset` 변경 파급**: `currentFrameKey()` 가 바뀌므로 최종화 후 오버레이·슬롯 목록·검출 캐시 조회가 cam1:preset1 기준이 된다. 이는 의도된 동작이며, 프리셋별 캐시(`detectByKey` 등)는 키 조회라 데이터 손실 없음.
- **선택 해제**: 최종화 시 슬롯/검출 선택이 해제된다(프리셋이 바뀌므로 교차 프리셋 오선택 방지 — 수동 전환과 동일 규약).
- **라이브 스트리밍**: 라이브 중이면 새 cam:preset 으로 재연결. 정지 상태면 `reconnectLiveIfActive()` 가 no-op.
- **`pollPipeline` 자동 경로 무개입** → 원버튼 파이프라인(finalize→센터라이징 자동 연쇄)의 카메라 소유권 규약 회귀 0.
- **DB·파일 산출물 무영향**: 최종화는 여전히 표시 전용(`POST /capture/finalize` 미호출) — `slot_setup` 을 쓰지 않는다.
- **다중 카메라 현장**: cam1:preset1 이 없으면 조용히 skip. 기준 뷰를 바꿔야 하면 `HOME_CAM`/`HOME_PRESET` 상수 2줄만 수정하면 된다.
