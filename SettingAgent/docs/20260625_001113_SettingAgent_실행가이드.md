# SettingAgent 실행 가이드 (설치 → 설정 → 실행 → 확인)

- 작성일: 2026-06-25
- 대상: SettingAgent 를 처음 띄우고 셋업 데이터를 생성하려는 사용자
- 한 줄 요약: **외부 서버(카메라/VPD/LLM) 띄우기 → config 2개 맞추기 → 설치 실행 → `setup_artifact.json` 확인**

---

## 0. SettingAgent 는 무엇을 하나

주차장 **설치 시 1회** 동작하는 에이전트. 카메라 프리셋을 돌며 차량을 검지(VPD)하고 주차면 ROI 와
**전역 슬롯 인덱스**를 만들어 `data/setup_artifact.json` 으로 저장한다(이후 Action/DM 이 사용).
LLM(gemma4:12b 등)이 켜져 있으면 비전 게이트로 품질을 보강한다.

---

## 1. 사전 준비

### 1-1. 런타임
- Node.js 20+ (확인: `node -v`)

### 1-2. 의존성 설치 (워크스페이스 루트에서 1회)
```bash
cd d:/Work/Parking3D/AgentVLA/ParkAgent
npm install
```
> 루트가 npm 워크스페이스라 `@parkagent/types`(공유 타입)가 자동 링크된다. 이후 명령은 `SettingAgent/` 에서 실행.

### 1-3. 외부 서버 (필요한 것만)
| 서버 | 기본 주소 | 필수? | 비고 |
|------|-----------|-------|------|
| 카메라(Unity/실 PTZ) | `http://localhost:13100` | ✅ | 캡처/이동. A타입이면 `GET /cameras` 도 제공 |
| VPD(차량검출) | `http://127.0.0.1:9081` | ✅ | `Sub/da_vpd_api` |
| LPD(번호판검출) | `http://127.0.0.1:9082` | 선택 | `setup.lpdEnabled=true` 일 때만 |
| LLM(두뇌) | `http://192.168.0.210:11434/v1` | 선택 | gemma4:12b(Ollama). `llm.enabled=true` 일 때만 |

> 다른 PC 로 옮기면 아래 config 의 **호스트만** 바꾸면 된다(코드 수정 불필요).

## 2. 설정 파일 (config 2개)

`SettingAgent/config/` 에 **역할이 분리된 2개** 파일이 있다. (값은 `.example.json` 참고)

### 2-1. `tools.config.json` — 도구(능력)·셋업 파라미터
| 섹션 | 핵심 키 | 초기값/의미 |
|------|---------|-------------|
| `camera` | `baseUrl` | 카메라 서버 주소(`http://localhost:13100`) |
| `vpd` | `endpoint`,`detPath` | `http://127.0.0.1:9081`, `/vpd/api/v2/det/imgupload` |
| `lpd` | `endpoint`,`detPath` | `http://127.0.0.1:9082`, `/lpd/api/v1/imgupload` |
| `setup` | `lpdEnabled` | 번호판 위치 저장 여부(true/false) |
| `setup` | `accumFrames` | 1=단일프레임(시뮬), >1=다프레임 누적(실 PTZ) |
| `map` | `cameraposFile`,`presetFile` | 프리셋 목록/기대치 파일 경로 |
| `presetProvider` | `type` | 프리셋 출처: `unity-api`(A) / `discovery`(B) / `camerapos`(수동) |
| `discovery` | `enabled` | 셋업 시 즉석 자동탐색(B) 사용 여부 |
| `server` | `port` | SettingAgent REST 포트(`13020`) |
| `store` | `dataDir` | 산출물 저장 폴더(`data`) |

### 2-2. `llm.config.json` — LLM 두뇌 + MCP + 단계별 프롬프트
| 키 | 초기값/의미 |
|----|-------------|
| `llm.enabled` | LLM 게이트 사용 여부(true 면 셋업 품질↑, false 면 결정형만) |
| `llm.model` / `llm.baseUrl` | `gemma4:12b` / `http://192.168.0.210:11434/v1` |
| `setupPrompts.stageNEnabled` | 단계별(1 비전판정/2 중복·라벨/3 리포트) on/off |
| `setupPrompts.stageN.{system,user}` | 단계별 프롬프트 파일 경로(`config/prompts/*.md`) — 파일만 수정해 교체 |

> **권장 초기값**: 1회성 설치라 `llm.enabled=true` 로 품질을 높이고, 평상시엔 꺼도 된다.
> LLM 서버가 없으면 `llm.enabled=false` → 결정형 경로로 정상 동작.

## 3. 초기 셋업 절차 (시나리오별)

### 시나리오 A — 카메라가 `/cameras` 제공(권장, 가장 정확)
```bash
cd SettingAgent
# 1) (선택) 연결 점검
npm run e2e            # 헬스+셋업 한 번에 (서버 모두 떠 있어야 함)

# 2) 프리셋 목록을 카메라에서 받아 camerapos.json 생성  (presetProvider.type=unity-api)
npm run export:camerapos
#   → config/camerapos.json 갱신(실제 프리셋·PTZ). 필요 시 파일 수동 보정.

# 3) 서버 기동
npm run start          # http://localhost:13020

# 4) 셋업 실행 (camerapos 기준)
curl -X POST http://localhost:13020/setup/run-from-map
```

### 시나리오 B — `/cameras` 없음 + 자동탐색(probing)
- `tools.config.json`: `discovery.enabled=true` (또는 `presetProvider.type=discovery`)
```bash
npm run start
curl -X POST http://localhost:13020/setup/run-from-map   # discovery 로 즉석 목록
```
> 주의: 일부 서버는 없는 프리셋에도 영상을 줘 **과다 탐색**될 수 있다 → 결과 검토 필요.

### 시나리오 C — 수동(휴컴스 등 프리셋 API 없음)
- `tools.config.json`: `presetProvider.type=camerapos`
- `config/camerapos.json` 을 직접 작성(형식은 `config/camerapos.json` 예시 참고: `cam_id/preset_id/sname/pan/tilt/zoom`).
```bash
npm run start
curl -X POST http://localhost:13020/setup/run-from-map
```

## 4. REST API (포트 13020)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/health` | 카메라/VPD/두뇌 상태 |
| POST | `/setup/run-from-map` | camerapos(또는 discovery)로 셋업 실행 |
| POST | `/setup/run` `{targets:[...]}` | 대상 프리셋을 직접 지정해 실행 |
| POST | `/setup/export-camerapos` | 공급자(A/B)로 camerapos.json 생성 |
| GET | `/setup/status` | 진행 상태(state/진행률) |
| GET | `/mapping` | 최종 산출물(`setup_artifact.json`) 조회 |
| GET | `/brain/ping` | LLM 연결 점검 |
| POST | `/brain/review` | 저장된 산출물 LLM 검토 |

**예시**
```bash
curl http://localhost:13020/health
curl -X POST http://localhost:13020/setup/run-from-map
curl http://localhost:13020/mapping | jq .
```

## 8. 트러블슈팅 (실제 사례)

| 증상 | 원인 | 해결 |
|------|------|------|
| `/health` 에서 vpd/lpd 가 `false`(404) | da_vpd/lpd_api 는 `/health` 미구현 | 정상. 검출 경로는 동작. (헬스 표기만 false) |
| `m_Cameras[2]가 null` (500) | camerapos 가 없는 카메라 참조 | camerapos 를 실제 카메라 수에 맞추거나 A(`/cameras`)/export 사용 |
| 자동탐색이 프리셋을 과다 검출 | 서버가 없는 프리셋에도 영상 반환 | `presetProvider.type=unity-api`(A) 사용 권장 |
| `기대 슬롯 N ≠ 검출 M` 경고 | preset.json 기대치와 실제 불일치 | preset.json 보정 또는 무시(검증용 경고, 비차단) |
| LLM 게이트가 느림/불필요 | 1회성 외 상시 구동 시 | `llm.enabled=false` 로 결정형만 사용 |
| 다른 PC 의 서버 | 호스트 다름 | `tools.config`/`llm.config` 의 **호스트만** 변경 |
