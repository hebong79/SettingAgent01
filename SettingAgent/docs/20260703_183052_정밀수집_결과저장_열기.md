## 1. 기능 개요

정밀수집(finalize)으로 산출된 `SetupArtifact`(주차면 매핑 결과)를 `SettingAgent/save/` 폴더에 **명명된 JSON 스냅샷**으로 저장하고 다시 열어 뷰어에 표시하는 기능이다. 세 가지 흐름을 제공한다.

1. **자동 스냅샷**: finalize 완료 시 서버가 `result_YYYYMMDD_HHMMSS.json` 이름으로 자동 저장.
2. **수동 저장**: 뷰어의 "결과 저장" 버튼 → 이름 입력(prompt) → 현재 화면 상태(편집 반영본)를 저장.
3. **열기**: "결과 열기" 버튼 → 저장 목록 모달 → 선택 → 오버레이에 3종 ROI 표시.

저장 파일의 내용은 `data/setup_artifact.json` 과 **완전히 동일한 순수 `SetupArtifact` 구조**(래퍼 없음)이므로 열기 시 별도 변환 없이 기존 렌더/편집 로직을 100% 재사용한다.

## 3. 신규/수정 클래스 · 함수

### 3.1 신규: `src/store/SaveStore.ts`

`save/` 폴더 파일 IO 전담(Repository 미러). 파일명 안전화의 권위를 소유하며 라우트는 위임만 한다.

| 멤버 | 설명 |
|------|------|
| `constructor(saveDir: string)` | 저장 폴더 경로 주입(cwd 상대, `tools.store.saveDir`). |
| `sanitizeName(input): string \| null` | 허용 문자 `[A-Za-z0-9가-힣_-]`(+ 공백→`_`), 그 외 제거. 빈값·`.`·`..`·경로 구분자(`/ \`) 차단(null). 입력의 `.json`은 사전 제거(중복 확장자 방지). 반환은 **확장자 없는 안전 base 이름**. |
| `save(name, artifact): string` | `save/{safe}.json`에 `JSON.stringify(artifact, null, 2)` 기록. 디렉터리 자동 생성(`mkdirSync recursive`), 동명 **덮어쓰기**(결정 C). 안전화 실패 시 throw. |
| `list(): {name, savedAt}[]` | `.json` 파일 목록을 파일 **mtime 내림차순**으로 반환. `name`=확장자 없는 base, `savedAt`=mtime ISO 문자열. 폴더 없으면 `[]`. |
| `load(name): SetupArtifact \| null` | 안전화 실패·파일 없음 → null. 존재 시 파싱하여 반환. |
| `defaultSaveName(date?)` (export 함수) | finalize 자동 스냅샷 기본 이름 `result_YYYYMMDD_HHMMSS`(로컬 시각). |

**설계 결정**: `savedAt`은 파일에 넣지 않고 mtime에서 도출한다. 이로써 저장 파일 내용은 순수 `SetupArtifact`로 유지되어 열기 시 언랩이 불필요하다.

### 3.2 신규: `src/api/artifactSchema.ts` (server.ts에서 추출)

`SetupArtifactSchema`(zod) + `validateArtifactBody(body)` 헬퍼를 담은 **중립 모듈**. server.ts(PUT `/mapping`)와 captureRoutes.ts(POST `/capture/save`)가 공유하며, 두 모듈이 스키마를 직접 주고받지 않고 여기서 import함으로써 **순환참조를 회피**한다.

`validateArtifactBody(body)` 3단계:
1. zod shape 검증 실패 → `{ok:false, code:400, body:{error:'invalid artifact', detail}}`.
2. `plateRoiByPreset` 구데이터(rect) → `rectToQuad`로 quad 승격(저장은 항상 quad).
3. `validateCoverage(globalIndex↔slots)` 불일치 → `{ok:false, code:400, body:{error:'coverage mismatch', missing, extra}}`.
4. 통과 시 `{ok:true, artifact}`(정규화본) 반환.

스키마가 커버하는 3종 ROI: `roiByPreset`(차량 rect) / `plateRoiByPreset`(번호판 quad 4점 또는 rect 하위호환) / `floorRoiByPreset`(바닥 가변 다각형 4~10점).

## 4. 신규 REST 계약

경로 프리픽스 `/capture/*`. **saveStore 미주입 시 3라우트 모두 미등록(가산, 하위호환)**.

### (a) POST /capture/save — 결과 저장
- 요청: `{ name: string, artifact: SetupArtifact }`
- 검증: `sanitizeName(name)` 통과 + `validateArtifactBody(artifact)`.
- 200: `{ ok:true, name, slots, globalCount }`
- 400: `{ error:"invalid name" }` / `{ error:"invalid artifact", detail }` / `{ error:"coverage mismatch", missing, extra }`
- 저장 파일: `save/{safeName}.json`(덮어쓰기).

### (b) GET /capture/saves — 저장 목록
- 200: `{ saves: [{ name, savedAt }] }` (mtime 내림차순, 없으면 `[]`)

### (c) GET /capture/saves/:name — 특정 결과 열기
- `:name`은 클라이언트가 `encodeURIComponent`, 서버가 `sanitizeName`으로 재검증.
- 200: `SetupArtifact`(GET `/mapping`과 동일 shape)
- 400: `{ error:"invalid name" }` / 404: `{ error:"not found" }`

## 5. 저장 JSON 스키마

`save/{name}.json` = **`SetupArtifact` 그대로**(`data/setup_artifact.json`과 동일 구조, 래퍼 없음).

```json
{
  "presets":     [{ "camIdx", "presetIdx", "label", "coveredSlotIds": [], "pan?", "tilt?", "zoom?" }],
  "slots":       [{ "slotId", "zone",
                    "roiByPreset":      { "c:p": {x,y,w,h} },
                    "plateRoiByPreset?":{ "c:p": [ {x,y}×4 ] },
                    "floorRoiByPreset?":{ "c:p": [ {x,y}×4~10 ] } }],
  "globalIndex": [{ "globalIdx", "slotId", "camIdx", "presetIdx" }],
  "createdAt":   "ISO8601",
  "warnings?":   [],
  "report?":     "..."
}
```

`globalIndex`는 1-based(`globalIdx:1`부터). `savedAt`은 파일에 포함하지 않고 파일 mtime에서 도출한다.

## 6. 프론트 UI / 상태 흐름

### 버튼 · 모달 (index.html)
- `roi-edit-bar`: `#result-save`("결과 저장"), `#result-open`("결과 열기").
- `#open-result-modal`: 목록 컨테이너 `#open-result-list` + 닫기.
- 이름 입력은 모달이 아닌 `window.prompt`(리더 지시 "이름 입력 대화창").

### 상태 흐름 (app.js)
1. **로드**: `init()`이 `loadMapping()`을 호출하지 않음 → `state.mapping=null` 유지 → 오버레이 빈 화면(요구사항 5). 잔여 `loadMapping` 호출부(map-save 재동기화 393행 / finalize 반영 749행 / 검수탭 동기화 1077행)는 모두 사용자·finalize 트리거로만 실행되어 시작 시 미표시 보장.
2. **결과 저장** (`saveResult`, 413행): `state.mapping` 없으면 "표시된 결과 없음" 안내. 있으면 `window.prompt` 이름 → 빈값 방지 → `POST /capture/save {name, artifact: state.mapping}` → 성공 시 `map-msg`에 `저장됨: {name} (슬롯 N, 전역 M)`. coverage mismatch는 누락·초과 목록 별도 표기.
3. **결과 열기** (`openResult`→`renderSaveList`→`openSaved`, 446~): `GET /capture/saves` → 모달 목록(`이름 — 저장시각`) → 항목 클릭 → `GET /capture/saves/:name` → `state.mapping` 주입, `roiHidden=false`, `selectedSlotId=null` → `drawRoiOverlay`/`renderSlotList`/`renderSelectionInfo` → 모달 닫기.
4. **finalize**: 기존 표시 동작 유지. 서버가 자동 스냅샷 저장(요구사항 1) — 프론트 추가 동작 없음.
5. **편집 후 저장**(요구사항 2): floor 정점 드래그·삭제로 `state.mapping` 변경 후 "결과 저장" → 편집 반영본 저장. 기존 `map-save`(PUT `/mapping`, 정본 반영)와 독립.

## 7. 사용법

1. 서버 기동 후 뷰어 접속 → 오버레이는 **빈 화면**(요구사항 5).
2. 정밀수집 → 최종화(finalize) → 결과가 화면에 표시되고, 서버가 `save/result_YYYYMMDD_HHMMSS.json` 자동 저장.
3. 필요 시 floor ROI 등 편집.
4. "결과 저장" → 이름 입력 → 편집 반영본이 `save/{이름}.json`으로 저장.
5. "결과 열기" → 목록에서 선택 → 3종 ROI가 오버레이에 표시.
