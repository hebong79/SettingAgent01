## 2. 계약 불변 — 외부 의존자에게 무영향

이번 작업은 **REST·정적 서빙·설정 계층의 통합**이다. 도메인 타입·외부 REST 계약은 한 줄도 바뀌지 않았다.

| 계약/타입 | 영향 | 근거 |
|-----------|------|------|
| `@parkagent/types`(SetupArtifact·SlotState·ParkingEvent 등) | **무영향** | 타입 정의 미수정. `/mapping` 응답 shape = `repo.loadArtifact()` 그대로 |
| `/setup/*`(run-from-map·run·export-camerapos·status) | **무영향** | 핸들러·경로·메서드 미수정 |
| `/capture/*`(start·stop·finalize·status·runs·aggregate) | **무영향** | 서버측 라우트·핸들러 불변. SPA 가 직접 호출하도록만 변경(클라이언트측) |
| 루트 `/mapping` | **무영향** | 기존 `/mapping`(repo 직접) 유지. 신설은 별 경로 `/viewer/api/mapping` |
| 루트 `/health` | **무영향** | 뷰어는 `/viewer/api/health`(별 경로) → 충돌 없음 |
| `/brain/*` | **무영향** | 미수정 |

### 의존성 그래프 전파 — 외부 에이전트

```
@parkagent/types (도메인 타입)
   ├─ SettingAgent  ── 이번 변경(타입 미사용 변경) ── 전파 없음
   ├─ ActionAgent   ── /mapping·SetupArtifact 소비 ── 영향 없음(shape 불변)
   └─ DMAgent       ── ParkingEvent/SlotState 소비 ── 영향 없음(타입 불변)
```

- **ActionAgent / DMAgent**: 별 워크스페이스. SettingAgent 외부 계약(`/setup`·`/mapping`·`/capture`)만 의존하며 그 셋이 모두 불변이므로 **무영향**. 재빌드·재배포 불요.
- **MCP 도구 / LLM 두뇌 경계**: `src/mcp`·`src/brain` 미수정. CameraSource 추상화는 기존 결정형 카메라 제어의 어댑터일 뿐 — 경계 변동 없음. 이번 작업은 결정형 도구도 LLM 판단도 신설하지 않는 **순수 인프라 변경**.

> 검증 근거: 기존 라우트 테스트(apiRefresh·captureRoutes 19개·setupOrchestrator 등) 전부 통과(회귀 0). 외부 계약을 깨는 변경이 있었다면 이들이 실패했을 것.

## 3. 내부 영향 — 가산/병합의 파급

### 3-1. CameraClient (가산)

`listCameras()` 추가는 **순수 가산**. 기존 메서드를 호출하는 SettingOrchestrator·CaptureJob·Finalizer·setup 경로는 시그니처가 그대로라 무영향. 신규 의존: `CameraClient → ../viewer/CameraSource.js`(CameraList 타입). 순환 의존 없음(CameraSource 는 CameraClient 를 타입으로 import 하지 않음 — SimulatorSource 가 런타임 import).

### 3-2. buildServer register 래핑 (가산)

뷰어 블록은 `if (deps.viewer?.enabled && deps.sources)` 이중 가드 안에서만 `app.register` 된다. 두 dep 모두 **옵셔널**이라 기존 `buildServer({...})` 호출처(테스트 포함)는 인자를 늘리지 않으면 자동으로 헤드리스 경로 → **기존 호출 무변경**. async 화 대신 register 래핑을 택해 호출처 `await` 보정 0건.

### 3-3. toolsConfig 확장 (가산)

`viewer` 기본값 + `cameraSources` 옵셔널. `config.test.ts`(기본값/병합 비교)는 viewer 기본값 포함 상태로 통과. `cameraSources` 패스스루 1줄이 없으면 배열이 조용히 누락되는 함정이 있었으나 보정됨(검증 통과).

## 7. 결론

- **외부 영향 0**: `@parkagent/types`·`/setup`·`/capture`·루트 `/mapping`·`/health` 전부 불변 → ActionAgent·DMAgent·MCP·LLM 두뇌 무영향. 재배포 불요.
