## 1. 완료 작업 (시간순, 전부 parkagent-dev 하네스)

### 작업 A — 번호판 discovery 검지율 근본개선 (goal/loop, 설계=Fable)
- **문제**: cam1:preset2 6슬롯 중 2개만 LPD 검지. 마스터 지시 루프(중심점→확대→사방이동→반복)대로 안 움직임 확인.
- **근본원인 2겹**: ① 앵커=`slot3d_front_center`(앞면 4모서리 평균=지상 0.75m 중간높이)가 번호판(지상 0.3~0.5m 하단)보다 위 ② 크롭이 앵커 고정 확대만 → 확대할수록 번호판 크롭 밖 탈락.
- **해결 A+B**: 앵커 하향(`lowerFrontAnchor`, 픽셀 h선형 항등보간) + 2D 격자 탐색 **30칸**(5줌×6오프셋: 중심/하/하좌/하우/좌/우, maxSteps 30).
- **라이브 발견(중요)**: `found:6/6`이 **위장** — matchRadius(0.15)>슬롯간격(~0.11)이라 자기판 미검 시 **옆 슬롯 판을 훔쳐** found 위장·격자 줌 미실행. 실제 고유판 3개.
- **배타성 게이트(§9)**: `pickOwnedPlate`(Voronoi 소유권 — 검출중심에 자기 앵커 최근접일 때만 채택), discoverSlot에 peerAnchors 주입, PlateDiscoveryJob.run 프리셋별 그룹핑. Tier0·crop 양 tier.
- **최종 라이브(정직)**: **5/6, 중복점유 0**. slot8/12 자기판을 격자줌으로 회수. **slot10 미검 = 외부 LPD 서비스 한계**(판 육안 선명한데 15+ 크롭구성·전처리 전부 미검 — 파이프라인 결함 아님). **마스터 결정: 정직 5/6 수용**(기하폴백 없음, LPD팀 이관). 핸드오프 §5 "순수 비전 100% 불가" 구체사례.
- 변경 파일: `src/calibrate/{plateDiscovery,cropZoom,plateDiscoveryWriter,PlateDiscoveryJob}.ts`. 문서 `docs/20260719_220248_*`.

### 작업 B — "현재화면 순수 LPD"(lpd-live) 타입 추가 (선형)
- LPD 콤보(`#lpd-mode`)에 4번째 타입. 기존 순수 LPD는 서버가 **저장된 프리셋 PTZ로 재렌더**(수동 조정 무시). 신규는 **현재 뷰어 PTZ(`state.ptz` 수동 팬/틸트/줌)로 재렌더 후 LPD-only**.
- 구현: `/capture/detect`에 옵셔널 `ptz` 추가 → `runDetect`가 제공 시 `resolvePresetPtz` 스킵하고 그 값을 base(역투영·zoom재시도 기준)로. `runLiveDetect(vpdEnabled, ptz)`·`runModeLpdLive()`.
- 라이브 종단 검증(시뮬): basePtz=보낸 PTZ 그대로, 수동 줌 뷰(zoom4)가 프리셋(1개)보다 판 더 검출(4개). 문서 `docs/20260719_223437_*`.
- **한계**: 실카메라(Hucoms)에서 requestImage ptz 오버라이드의 물리이동 반영은 미검증(별도 스트림 소관).

### 작업 C — "DB에 추가" 버튼 (선형, v1→v2 반복)
- LPD 검지 패널에 `#lpd-db-add` 버튼. **현재 프리셋의 라이브 LPD 박스**(`state.detectByKey.plates`)를 각 슬롯 공간배정해 `slot_setup.lpd_obb` 저장(POST `/capture/slots/lpd`). discovery는 이미 자동저장이라 이 버튼은 나머지 라이브 모드용.
- **v1 오배정 발견(라이브)**: `matchPlatesToSlots`(bbox 중심 포함판정) 재사용했으나, **원근 왜곡 주차면 폴리곤의 bbox가 인접슬롯과 ~60% 겹쳐** plate가 왼쪽으로 한 칸 오배정(0.325→slot9 오답).
- **v2 수정**: 배정을 **nearest `lowerFrontAnchor` 전역 1:1 그리디 + MATCH_RADIUS 0.15**(discovery와 동일 앵커·게이트)로 교체. `matchPlatesToSlots` 불변.
- **라이브 재검증**: 0.326→slot10, 0.463→slot11, 0.620→slot12, 0.765→slot13 — **4/4 discovery 앵커와 일치, 밀림 제거, unassigned 0**.
- 변경: `src/setup/plateMatch.ts`(assignPlatesToSlotViews), `src/api/captureRoutes.ts`(라우트), `web/{index.html,app.js}`. 문서 `docs/20260719_234602_*`.
- **한계**: 프리셋 정합 검출(순수 LPD/VPD→LPD)에 유효. **현재화면 LPD(수동 줌)는 좌표계 불일치로 배정 부정확 가능**(역투영 범위 밖).

## 3. 핵심 아키텍처·API (이번 세션 추가)
- **LPD 3+1 모드 콤보**(`#lpd-mode`): `lpd`(순수, 프리셋PTZ 재렌더) / `discover`(앞면중심 LOOP, 격자30+배타성, 자동 DB저장) / `vpd`(VPD→LPD) / `lpd-live`(현재화면, state.ptz 재렌더).
- **POST /capture/detect** `{cam,preset,vpdEnabled,ptz?}` — ptz 제공 시 그 PTZ로 base 렌더(lpd-live).
- **POST /capture/slots/lpd** `{cam,preset,plates}` — nearest 하향앵커 배정 → upsertSlotLpd. 반환 `{ok,updated,assigned:[{slotId,confidence}],unassigned}`.
- **POST /discover/ptz** `{cam,preset}` — discovery(격자30+배타성), slot_setup.lpd 자동저장.
- 배정 핵심함수: `assignPlatesToSlotViews`(plateMatch.ts), `lowerFrontAnchor`(plateDiscoveryWriter.ts), `pickOwnedPlate`(plateDiscovery.ts).
