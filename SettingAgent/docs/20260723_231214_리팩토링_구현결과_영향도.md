## 4. 의도적 예외 2건

### 4.1 `centerAndZoomByLadder` 289줄 유지(150줄 초과 유일 예외)

**사유**: rung 루프가 환원불가 상태기계다. 6개 반환지점이 루프 지역상태(`ptz`/`plate`/`latched`/`best`/`zLo`/`zHi`/`rung`)와 강결합되어 있고, 3개 장비한계 종료블록(zoom-stall / bracket 미형성 / saturation)의 진단 로그 필드가 서로 상이해 공용 추출을 시도하면 매개변수가 폭증한다. 안전하게 값을 되돌릴 수 있는 부분(수치 결정부)만 순수함수(`detectZoomStall`/`nextLadderZoom`)로, 결과 생성은 빌더 3종으로 추출해 302→289줄로 축소했다.

설계서 §4.3/§8.2의 "과분해 절대 금지" 원칙이 이 예외를 사전 승인했다 — line-golf를 위해 동작 드리프트 위험과 가독성을 맞바꾸지 않는다는 판단.

### 4.2 `captureWithDither` 통합 보류

`captureTrack`/`captureTrackZoom`(둘 다 이미 150줄 이하)은 dither-retry 루프 구조가 유사하지만, `recovered`/`retry`/`exhausted`/`skip` 4개 로깅 지점의 필드구조·한글 메시지가 축(tilt/zoom)마다 전부 달라 안전한 콜백 통합이 불가능했다. 로그 바이트는 vitest가 검증하지 않으므로 통합해도 green으로 안전을 확인할 방법이 없다 — 로깅 구조 통일을 선행하는 별도 설계가 필요하다는 판단으로 이번 범위에서 제외했다(§8 잔여 과제).

`CaptureJob.maybeUpdateCuboids` 분리도 보류되었다(주석분산 리스크, 설계서상 선택 항목이었으므로 필수 미달성 아님).

라우트 응답 스키마·상태코드·에러 메시지 문자열은 §5(설계서 불변식 4)에 따라 바이트 단위로 보존했다. `test/captureRoutesShape.test.ts`가 라우트 등록 순서·메서드·경로 스냅샷을 봉인해 클라이언트(웹 UI `web/app.js`, 외부 REST 클라이언트) 관점에서 계약 변경이 없음을 확인했다. `web/app.js`는 이번 변경에서 미접촉(제외 파일)이므로 클라이언트 코드 자체의 수정 필요성도 없다.

### 6.4 신규 `@internal` export의 공개 API 표면 영향

테스트 검증을 위해 노출한 `/** @internal */` export(`unmatchedReason`, `lineFitRejectReason`, `crossPresetSimilarityChecks`, `parseMultipartFrame`, `compareOccupancyAgreement` 등)는 테스트 전용 노출이며, 각 모듈의 기존 공개 export 시그니처와는 별개다. 이들은 다른 에이전트(ActionAgent/DMAgent)나 외부 클라이언트가 참조할 계약이 아니므로 공개 API 표면에 대한 실질적 영향은 없다고 판단한다. 다만 향후 이 내부 함수들을 신뢰해 외부에서 임포트하는 코드가 생기면 "테스트 전용" 취지가 흔들릴 수 있다는 점은 **확인 필요** 항목으로 남긴다.

## 8. 잔여 과제

1. **`captureWithDither` 로깅 구조 통일**(§4.2) — `captureTrack`/`captureTrackZoom`의 dither-retry 로깅 필드·메시지를 축(tilt/zoom) 간 통일하는 선행 설계 후, 콜백 통합을 재시도할 수 있다.
2. **`centerAndZoomByLadder` 289줄**(§4.1) — 현재는 설계서가 승인한 의도적 예외. 상태전이(latch/bracket) 로직 자체를 재설계하지 않는 한 추가 축소는 동작 드리프트 위험을 동반한다.
3. **`replaceSlotSetup` 센터링컬럼 가드 보강** — `persistSlotSetupFromPlace`(구 `Finalizer.finalize` 내부 로직) 상단에 이전·보존된 주석이 가리키는 취약점(검출 없는 finalize 시 DELETE+INSERT로 센터링 데이터 파괴 가능, 검출컬럼은 가드됨·센터링컬럼은 미가드)은 이번 리팩토링 범위에서 **동작을 바꾸지 않았으므로 미해결 상태 그대로**다. 별도 설계·구현 작업이 필요하다.
4. **`test/buildTouringPlan.test.ts` 환경 실패**(§7.1) — 이번 리팩토링과 무관하지만, `save/setup_result.json` fixture 부재로 인한 red 상태가 지속되고 있다. 리팩토링 범위 밖이므로 본 작업에서는 손대지 않았다.
