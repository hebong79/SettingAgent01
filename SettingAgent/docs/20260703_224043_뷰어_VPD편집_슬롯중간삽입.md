## 1. 배경 / 목적

SettingViewer(주차면 매핑 편집 뷰어)에서 두 가지 편집 요구를 충족한다.

- **요구 A — VPD(차량 rect) 미세조정**: 캡처·자동 검출로 생성된 차량 bounding box(VPD ROI)를 사용자가 화면에서 직접 리사이즈/이동해 미세조정한다. 기존에는 floor(바닥) 다각형 정점 편집만 가능했고 차량 rect 편집 배선은 코드에 **휴면 상태**(`[미사용]` 주석)로만 존재했다.
- **요구 B — 누락 슬롯 수동 추가(중간삽입)**: 자동 파이프라인이 놓친 주차면을 사용자가 수동으로 추가하되, **전역 인덱스(globalIdx)의 임의 중간 위치**에 꽂을 수 있어야 한다(맨 끝 append 뿐 아니라 1..N+1 임의 위치).

두 요구 모두 좌표 수학·인덱스 재부여는 전량 결정형(순수함수)이며 LLM 두뇌 개입은 없다.

## 2. 요구 A — VPD 차량 rect Ctrl+드래그 편집

### 2.1 조작 모델 (핵심 결정: Ctrl 배타)

`overlay.mousedown` 콜백 최상단에 **Ctrl 분기**를 삽입해 기존 floor 정점편집·슬롯 선택 분기보다 **우선**시킨다(`web/app.js:1158`).

```
mousedown:
  가드(roiHidden/!mapping) → return
  {nx,ny}=eventToNorm ; key=presetKey(cam,preset)
  ── [요구 A] if (e.ctrlKey && #roi-vehicle 체크) ──          ← 최우선(물리 배타)
     h = hitTestVpd(nx,ny)                                    // 8핸들 | 'in' | null
     h ∈ 8핸들  → dragState={kind:'vpdResize', handle:h, ...}; preventDefault; return
     h === 'in' → dragState={kind:'vpdMove', ...};            preventDefault; return
     (vrect 밖/미선택) hitTestSlots(vehicle-only) hit → 선택+재렌더 + {kind:'vpdMove', slotId:hit}; return
     Ctrl+빈곳 → 낙하(아래 기존 분기 = 선택 해제)
  ── [기존] hitTestFloorVertex → floorVertex 드래그 ──         ← Ctrl 아닐 때만 실질 도달
  ── [기존] hitTestSlots 선택/해제 ──
```

- **Ctrl 누름 = VPD 편집 / Ctrl 없음 = 기존(floor 정점·선택)**. 물리적으로 배타이므로 기존 인터랙션에 회귀가 없다.
- 수식키는 `e.ctrlKey`만 사용한다(게이트 G-4). macOS `metaKey`는 채택하지 않음(Windows 주 타깃).

### 2.2 드래그 진행/확정 (mousemove/mouseup)

- **mousemove**(`app.js:1199`): `dragState.kind` 스위치.
  - `floorVertex` → 기존 유지(`moveQuadVertex` → `updateSlotFloorRoi`).
  - `vpdResize` → `resizeRect(cur, handle, ndx, ndy)` → `updateSlotRoi`.
  - `vpdMove` → `moveRect(cur, ndx, ndy)` → `updateSlotRoi`.
  - slot lookup은 kind 분기 이전에 1회만 수행해 공유.
- **mouseup**(`app.js:1223~`): **무변경**. kind 무관 공통(`dragState=null; markDirty(); drawRoiOverlay()`).

### 2.3 8핸들 시각화

- `drawHandles`(`app.js:259`)를 4모서리 → **8핸들(4코너 nw/ne/sw/se + 4변 중점 n/s/e/w)** 로 확장하고 `[미사용]` 주석을 제거(재활성). 색은 차량 강조색 `#ff4d4d`(테두리) + 흰색 채움.
- `drawRoiOverlay`(`app.js:190`)가 선택 슬롯 + `#roi-vehicle` on 일 때 vrect 위에 `drawHandles`를 호출. floor quad의 선택-핸들 UX와 대칭.

### 2.4 저장 흐름 (디바운스 없음 — 명시적 정정)

- 편집은 `markDirty()`만 호출 → `mappingDirty=true` + 안내 메시지. **자동저장/디바운스 타이머는 코드에 존재하지 않는다**(작업지시서의 "디바운스" 표현은 실제와 다름).
- 실제 영속화는 **오직** `저장` 버튼(`#map-save`) → `saveMapping()` → `PUT /viewer/api/mapping` → 서버 `saveMappingHandler` → `validateArtifactBody`(zod + `validateCoverage`) → `repo.saveArtifact`. VPD·슬롯추가 모두 이 기존 패턴을 그대로 재사용한다(신규 저장 경로 없음).

## 3. 요구 B — 주차면 슬롯 전역 중간삽입

### 3.1 "중간삽입 위치" = 전역 globalIdx 기준 (게이트 G-1)

- 시스템에 이미 수동 전역인덱스 개념(`applyManualGlobalIds`/`reorderGlobalIndex`)이 존재하므로, `atGlobalIdx`(1-based) 위치에 신규 슬롯을 꽂고 이후 globalIdx를 +1 미는 것이 정합적이다.
- preset 내 `positionIdx`(coveredSlotIds 순서)는 별개 축. 신규 슬롯은 해당 preset의 `coveredSlotIds` **말미에 append**한다(전역 위치와 preset-내 위치는 독립 — 수동 매핑 체계와 일관).

### 3.2 신규 순수함수 (`web/core.js`)

**`nextSlotId(artifact, camIdx, presetIdx) → 'c{cam}p{preset}s{N}'`** (`core.js:365`)
- 해당 preset의 기존 `sN` 최대치+1부터 시작, 전체 slotId 집합과 충돌 없을 때까지 bump. 결번 회피가 목적이다: s2 삭제로 결번이 생겼을 때 `length+1`은 기존 s3와 충돌하므로, 최대 sN 파싱 후 +1 → 집합 검사로 재증가.

**`insertSlotAt(artifact, atGlobalIdx, newSlot) → artifact`** (`core.js:387`, 불변)
- ① 중복 slotId면 **no-op**(원본 참조 반환, 방어).
- ② `slots=[...slots, newSlot]`(배열 순서는 전역순서를 결정하지 않음).
- ③ `newSlot.roiByPreset`의 첫 key(`"cam:preset"`)를 파싱해 (camIdx,presetIdx) 도출 → 해당 preset의 `coveredSlotIds` 말미 append, 부재 시 `{camIdx,presetIdx,label:"cam:preset",coveredSlotIds:[slotId]}` 신규 push.
- ④ `globalIndex`를 정렬 후 `splice(clamp(at,1,N+1)-1, 0, {...})` 삽입 → 전체 `globalIdx` 1..N+1 재부여.
- **`rebuildGlobalIndex` 미사용 근거**: rebuild는 coveredSlotIds 정규순서(cam→preset→position)로 재생성하므로 사용자가 지정한 전역 중간위치를 무시한다. 명시적 splice만이 수동 위치를 보존한다.

**`moveRect(rect, ndx, ndy) → NormalizedRect`** (`core.js:473`)
- 평행이동(w,h 유지), `x∈[0,1−w]`, `y∈[0,1−h]` 클램프. `clamp01Rect`는 경계에서 w/h를 축소하므로 이동엔 부적합 → 별도 함수.

**`hitTestRectHandle(rect, nx, ny, tolX, tolY) → RectHandle|'in'|null`** (`core.js:486`)
- 우선순위 **코너 > 변 > 내부 > 외부(null)**. tol 주입식(DOM 미참조, `hitTestQuadVertex` 패턴). 반환 핸들 문자열은 `resizeRect`의 handle 인자와 1:1(`nw/ne/sw/se/n/s/e/w`), 내부는 `'in'`.

### 3.3 신규 슬롯 기본값 (게이트 G-2)

- `slotId`: `nextSlotId(...)`.
- `zone`: `cam{camIdx}`(기존 컨벤션).
- `roiByPreset`: `{ [key]: {x:0.45, y:0.45, w:0.1, h:0.1} }` — 화면 중앙 소형 rect. 추가 직후 선택되어 8핸들 표시 → 사용자가 Ctrl+드래그로 배치.
- `plateRoiByPreset`/`floorRoiByPreset`: 미포함(optional, floor min4 제약 무관).

### 3.4 UI 배선

- **`web/index.html:115-116`**: `.roi-edit-bar`에 `#slot-insert-idx`(number, min=1, placeholder "위치", 비우면 맨 끝) + `#slot-add`("추가") 2요소 가산.
- **`web/app.css:571`**: `.slot-insert-idx { width: … }` 입력창 폭 소량.
- **`web/app.js:339` `addSlot()`**: 산출물 가드 → `nextSlotId` → 기본 rect/zone → `#slot-insert-idx`(1..N+1 clamp, 비우거나 <1이면 N+1) → `insertSlotAt` → 신규 슬롯 선택 → `markDirty()` → 재렌더. `wire()`(`app.js:1291`)에서 `#slot-add` click 배선.

## 8. 알려진 한계 (본 작업 범위 밖 — 후속 고려)

- `insertSlotAt`로 넣은 수동 전역위치는 이후 `removeSlot`(→ `rebuildGlobalIndex` 정규정렬) 호출 시 cam→preset→position 정규순서로 **리셋될 수 있다**. 이는 기존 `removeSlot` 동작에서 비롯한 기존 한계이며 본 작업 범위 밖이다. 수동 전역위치의 영속 보존이 필요하면 별도 설계가 필요하다(후속 고려사항).
