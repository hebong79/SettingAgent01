### L4-5. 🔴 **z 오염을 볼 계기가 하나도 없다** — 리더 판단 요청

| | D | `depthDevM` |
|---|---|---|
| p1 | 41.3 m | **+0.501** |
| p3 | 14.6 m | **−0.029** |

### L4-6. 🔴 후속 실측 — **위 가설은 현 데이터로 결론이 나지 않는다(음성 결과). 상수를 코드에 넣지 마라.**

검증자가 교락(z 오염 vs 주차 습관 setback)을 지적하며 *"같은 차량을 두 프리셋에서 보면 결정적"* 이라고 제안했다. **확인 결과 현 데이터로는 불가능하다 — 프리셋 간 공유 슬롯이 0개다**(p1:7 / p2:6 / p3:4, 겹침 없음).

**그러나 그 확인 과정에서 내 원래 가설의 오류를 발견했다: `push = D·z/(d−z)` 의 `D` 는 프리셋이 아니라 *차량마다* 다르다**(D = 나디르 ↔ **그 차량 접지점**의 수평거리). ∴ 프리셋 겹침 없이도, **한 프리셋 안에서** 분리가 원리적으로 가능하다:

```
y_i = (차량 접지앞선 − 슬롯 앞선) = setback_i + [z/(d−z)] · D_i
→ D_i 에 회귀:  기울기 = z/(d−z) (z 오염)  /  절편 = 평균 setback (주차 습관)
```

**실측(실 seg 3프리셋, 유효 육면체 n=11):**

| preset | D 범위 | y 범위 |
|---|---|---|
| p1 | 24.0 ~ 27.0 m | +0.945 → −0.074 m (**D 증가 시 y 감소 — z 모델과 반대 부호!**) |
| p2 | 24.2 ~ 31.1 m | +0.92 / +0.89 |
| p3 | 10.2 ~ 16.3 m | −0.049 ~ +0.095 |

회귀: `y = −0.533 + 0.0434·D`, **R² = 0.484** → 형식상 **z ≈ 0.21 m**(설계 §A-2 ② 예측 0.15~0.25m 와 일치).

**🔴 그러나 이 값을 믿으면 안 된다:**
1. **D 지렛대가 전부 프리셋 간에서 나온다.** 프리셋 **내부** D 범위는 3~6m 뿐이라 예측 push 변화가 0.13~0.26m 인데, setback 산포는 ±0.5m 다 → **프리셋 내부 신호는 잡음에 묻힌다.**
2. **p1 내부 기울기는 오히려 음수**다(D 증가 → y 감소). z 모델은 양수를 예측한다. 잡음 지배의 증거.
3. ∴ 회귀의 R²=0.484 는 사실상 **"p3 는 D 도 y 도 작고, p1/p2 는 둘 다 크다"** 를 재는 것 — **검증자가 지적한 교락 그 자체**다. 옷만 갈아입었다.

**결론: z 오염은 여전히 미확정이다. `CONTACT_Z_OFFSET_M` 같은 상수를 지금 넣으면 그것이 다음 Loop 의 순환논법이 된다. 넣지 않았다.**

**결정적 실험이 되려면** — 둘 중 하나:
- **한 프리셋 안에서 D 가 넓게 퍼진 차량들**(같은 슬롯열 = 같은 습관 분포). 예: 얕은 tilt 로 근·원 슬롯을 함께 담는 프리셋. 현 3프리셋은 전부 D 산포가 좁다.
- **프리셋 시야를 겹치게** 재조준해 **같은 차량**을 두 D 에서 관측(같은 차 = 같은 setback → 차이는 push 뿐. **n=1 로도 결정적**). 현재는 슬롯 겹침 0이라 불가 — **리더가 프리셋을 하나 추가/재조준해야 한다.**

**반증 ②: 앞선은 관측 접지선 위에 정확히 놓여 있다.** `frontPxDev` = **0.9~3.4 px**(p1), 3.1~7.2 px(p3). 재투영 앞선이 관측 접지열과 픽셀 단위로 일치한다. **footprint 는 1~5m 뒤로 밀려 있지 않다.**

**★ 진짜 원인 ③ — `solveHeight` 의 "차 = 직육면체" 전제** (계측으로 특정)

`solveHeight` 는 마스크 최상단 y 를 **육면체 8점 중 최상단 코너**에 맞춘다. 계측 결과 그 코너는 **전 차량·전 프리셋에서 예외 없이 idx=6 = 뒤-상단 코너**(= `tFront + PRIOR_L` = 4.7m 뒤)였다.
**그러나 차의 최상단 실루엣 점(지붕)은 4.7m 뒤에 있지 않다.** 지붕은 **짧고 안쪽으로 물러난 슬래브**다. 모델의 뒤-상단 코너는 실제 지붕보다 **멀리** 있으므로 이미지에서 **더 위로** 투영되고, 이를 마스크 상단에 맞추려면 `h` 를 **낮춰야** 한다 → **계통적 H 과소.**

**증거 (동일 앞선·동일 지면모델, `L` 만 바꿔 `h` 재해):**

| | `h(L=4.7)` | `h(L=3.5)` | `h(L=2.0)` | GT | `dh/dL` |
|---|---|---|---|---|---|
| p1 veh#1 | 1.09 | 1.25 | **1.44** | 1.445 | 0.13 m/m |
| p1 veh#3 | 1.14 | 1.29 | **1.46** | 1.445 | 0.13 m/m |

### F-3 → **처리: 정정 + 봉인** ✅

설계 §6-5 의 "폭축 정수배 → `unmatchedRate` 약한 반응" 은 **틀렸다**. 재실측에서도 **0.00**.
→ 코드 주석(`contactTypes.AnchorMetrics.unmatchedRate`, `anchor.ts` 헤더)과 뷰어 tooltip 을
**"폭축 정수배 = 3지표 전부 침묵 가능"** 으로 정정했다. `unmatchedRate` 는 advisory 유지 · **임계 게이팅 없음**(설계대로).
**QA: 이 침묵을 통과 테스트로 명시 봉인할 것**(`roiAlignmentBlindSpot.test.ts` 전례).

### F-4. 재실측 앵커 반응표 (수정 후 — **이것이 최종본**)

픽스처 A: 차량 3대, 앞범퍼가 슬롯 앞선에서 **0.5m 물러남**(현실적). 슬롯만 지면에서 밀고 차량 마스크는 고정.

| 인위 변환 | `depthDevM` | Δ(기준 대비) | `phaseDevM` | `unmatchedRate` |
|---|---|---|---|---|
| 없음(기준) | **+0.500** (= 실제 setback) | — | −0.000 | 0.00 |
| **폭축 +2.5m**(정수배) | +0.500 (불변) | 0 | **0.000 침묵** | **0.00 침묵** ⚠️F-3 |
| **폭축 +1.25m** | +0.500 (불변) | 0 | **−1.250** ✅ | 0.00 |
| **깊이축 +2.5m** | **−2.000** | **−2.500** ✅ | −0.000 | 0.00 |
| **대각 +1.5/+1.5m** | **−1.000** | **−1.500** ✅ | **+1.000** (= −1.5 mod 2.5) ✅ | 0.00 |

픽스처 B: setback 0 → 기준 **−0.0000** / 깊이축 +2.5m → **−2.5000** / 대각 → **−1.5000**, `phaseDevM` **+1.000**.

**결론: 1.5 DOF 를 닫는다 — 깊이축 완전(선형·정확·prior 무관) + 폭축 비정수배. 폭축 정수배는 원리적 침묵(봉인 대상).**

## 4-이력. 최초 보고 원문 (리더 결정의 근거 — 보존)

### F-1. **관측 폭 W 에 상한 게이트가 없다** (설계 §8 #10 은 하한만 정의)

합성 실측: **비정면 차량에서 W 가 +0.33m(18%) 과대추정**된다(2.185 vs 1.85).
원인: 실루엣 하단 윤곽이 차량 **바닥면 모서리만** 지나는 게 아니다. 비스듬한 시야에서는 하단 윤곽 일부가 **상단 모서리 → 하단 모서리를 잇는 실루엣 현(chord)** 위를 지나고, 그 점들은 지면점이 아닌데 지면으로 역투영된다. 그중 `|b − tFront| < 0.25m` 인 것이 inlier 에 섞여 폭축 스팬을 부풀린다. (같은 이유로 비정면 차량의 `inlierRatio` 가 94% → 72% 로 떨어졌다 — 게이트 0.6 은 통과.)

**실차에서는 더 심할 수 있다** — 범퍼·사이드미러 오버행이 접지선보다 바깥으로 나온다.

- 조치 **안 했다**(설계 범위 밖 + 임의 판단 금지).
- 선택지: ① `W > priorW × 1.2` → `W:'prior'` 강등 + issue ② W 를 항상 prior 로(관측 포기, `CuboidSource.W` 를 `'prior'` 리터럴로) ③ 그대로 두고 G6 실측으로 결정.
- **리더 결정 요청.** 참고: 이 오차는 **앵커 지표에 영향을 주지 않는다**(centerGround 의 `a` 는 `median(a)` 이고, 스팬 확장은 좌우 대칭이라 중심을 옮기지 않는다 — 실측 `phaseDevM ≈ 1e−16` 이 이를 뒷받침).

### F-2. **`depthDevM` 에 상수 바이어스 +0.35m 가 있다**(정상 상태에서 0 이 아니다)

정의상 `C_vehicle` = footprint **중심**(= tFront + L/2), `C_slot` = 슬롯 **중심**(= 슬롯앞 + depth/2).
→ 바이어스 = (차량 앞범퍼가 슬롯 앞선에서 뒤로 물러난 거리) + (slotDepth − PRIOR_L)/2 = 0.5 + (5.0−4.7)/2 − ... 합성값 **정확히 +0.35m**.
`DEPTH_DEV_M = 0.5` 임계에 대해 **이미 70% 를 소진**한다. PRIOR_L 이 틀리면 바이어스가 그만큼 더 움직인다.

- 조치 **안 했다**(설계가 정의한 수식 그대로 구현).
- 선택지: ① 임계를 **바이어스 대비 편차**로 재정의(기준 프레임에서 baseline 을 1회 측정해 빼기) ② `DEPTH_DEV_M` 을 실측 후 상향 ③ `C_vehicle` 을 중심이 아닌 **tFront(접지선)** 로 바꾸고 `C_slot` 도 슬롯 앞선으로 — **L prior 의존이 사라져 더 깨끗하다**(설계자 검토 요청).
- **③ 을 권한다**: 깊이축 지표가 `PRIOR_L` 에 의존하는 현재 구조는, prior 가 틀리면 지표가 조용히 틀린다.

### F-3. **폭축 정수배 밀림에서 `unmatchedRate` 도 침묵했다**(설계 §6-5 는 "약한 반응" 예측)

합성(슬롯 5칸 · 중앙 3칸에 차량): 폭축 +2.5m → `unmatchedRate = 0.00`. 스트립 **끝단 차량이 없으면** 신호가 아예 없다.
→ 설계 §6-5 표의 "약한 반응 ⚠️" 는 **"0 일 수 있음"** 으로 정정해야 한다. **폭축 정수배는 3지표 전부 침묵 가능** — 은닉 금지. `roiAlignmentBlindSpot.test.ts` 전례로 봉인할 때 이 사실까지 포함하라.
