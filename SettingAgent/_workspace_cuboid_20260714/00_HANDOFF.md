6. **차량 bbox 중심은 바닥 ROI 밖이다 — 주차면 판정에 중심점 규칙을 쓰지 말 것.** VPD rect 는 **지붕까지 포함한 axis-aligned bbox** 이고 주차면 ROI 는 **바닥 quad** 다. 높이 z 의 점은 nadir 로부터 `H/(H−z)` 배 바깥으로 밀린다(H≈8m, z≈0.75m → +10% ≈ **주차면 한 칸**).

   - → 접지 근사(**bbox 하단 25% 밴드**)를 써라. `src/capture/onPlaceFilter.ts` 의 `isVehicleOnPlace` 가 정답 구현이다.
   - ⚠️ `Finalizer.ts:237-241` 의 **차량중심 폴백**은 이 함정을 그대로 밟고 있다(번호판 중심 우선이라 평소엔 가려짐) → **F-2a**.

## 3. 최상위 미해결 위험 — #0 정합 사각지대

**지면 평행이동 2 DOF 는 원리적으로 검출 불가능하다.** 4개 지표(`metricErr`, `tiltErrDeg`, `dDevRel`, `bearingDevDeg`)가 **전부 침묵**한다.

- **2.5m = 정확히 주차면 한 칸.** 즉 ROI 가 옆 칸에 올라앉아 **점유가 엉뚱한 주차면에 찍혀도 아무 경고가 안 뜬다.**
- 숨기지 않고 **"침묵한다"는 사실 자체를 통과 테스트로 봉인**했고(`test/roiAlignmentBlindSpot.test.ts`), 뷰어 배지 tooltip 에 "경고 없음 ≠ 정합됨"을 상시 표시한다.
- **1단계에서 해결 불가.** 차량이 주차선을 가려 대조할 이미지 증거가 없다. → **2단계 median 배경이 유일한 해법.**

## 5. 마스터가 확정한 설계 제약 (불변)

- **C1. 두 경로 공존**: 기본 = 현행 파일 경로(`PtzCamRoi.json`), 자동 경로는 **옵션 플래그**. UI 에서 어느 소스가 표시 중인지 항상 알 수 있어야 한다. 선례: `selectFloorRoi({useLlm})` / `#cap-floor-llm`.
- **C3. 실카메라 호환 (최상위)**: 프로덕션 경로는 `camera.position`/`eulerAngles`/`fov` 에 **의존 금지**(실카메라가 못 주는 값). 이미지 위의 점 + 알려진/측정된 metric 크기만 사용. → **Unity 와 실카메라가 동일 코드 경로.** (검증에서 ground truth 로 쓰는 건 허용.)
- **PtzCamRoi.json 스키마 무변경**: Unity `CParkingSpace3DTo2D.cs` 가 생성하고, 서버 `applyPlaceRoiUpdate` 가 저장 시 `{idx, points}` 로만 재구성해 **새 필드를 지운다.**
- 육면체 높이 h = **1.5m 상수 + 슬라이더**.
- **주차면 규격이 섞인다**: 표준(2.5×5m) / 도로변 평행주차(2×6m) / 장애인 / 경차 / 전기차충전소. 규격 메타데이터는 `PtzCamRoi.json` **밖**의 신규 파일에, **전역 인덱스를 키로**.

## 6. 다음 작업 — 2단계: 셋업 자동화

**목표**: 사람이 ROI 를 그리는 작업을 없앤다. 에이전트가 관측으로 스스로 주차면을 찾아낸다.

정밀수집이 이미 프리셋을 돌며 수십~수백 라운드 VPD/LPD 를 관측 중이다. **그 데이터가 답을 품고 있다.**

**2단계 = median 배경 합성 (가장 싸고 즉시 가치 있음)**
- 같은 프리셋 프레임들의 **시간축 중앙값** → **차량이 지워진 빈 주차장 바닥** → 흰 주차선이 선명해진다.
- **#0 사각지대를 닫는 유일한 수단**(주차선과 ROI 를 직접 대조할 수 있게 된다).
- ⚠️ **선행 조건**: 현재 정밀수집은 **프레임 원본을 보존하지 않는다**(`CaptureJob.ts:270-271` 이 프리셋당 1장을 메모리 Map 에 덮어씀. `observation` 테이블은 `img_name TEXT` 만). **프레임 영속화가 먼저다.**
  - 밑자락은 있다: `sharp` 는 이미 서버 의존성, `data/refframes/` 에 프레임 쓰는 코드 존재(`captureRoutes.ts:395-396`), `store.captureDir` 설정이 정의돼 있으나 **참조 0건인 데드 설정**.
- **median 배경은 캐시가 아니라 관측 산출물이다 → 반드시 저장해야 한다**(원본 프레임이 사라지면 재생성 불가). §2-3 의 "파생값 저장 금지"와 구분할 것.

**이후 단계 (설계만 됨, 미착수)**
- **SAM 접지선**: VPD 는 axis-aligned rect 라 접지점을 모른다. SAM 마스크 **하단 경계 = 접지선**. VPD/LPD 처럼 **REST 선택 의존**으로 붙인다(미가동 시 rect 하단 근사로 강등).
- **번호판 = 절대 스케일 앵커**: 번호판 규격이 고정이므로 주차면 규격을 **가정하지 않고 측정**할 수 있다.
- **시간 누적 footprint**: 차가 반복적으로 서는 자리 = 주차면.
- **LLM 은 의미 담당**: 기하는 결정형이 하고, LLM 은 후보 **검수** + **규격 타입 분류**(장애인 마크·EV 초록 도색·경차 표시는 기하로 불가). **좌표를 만들게 하지 말 것** — 기하가 만든 후보에 번호를 그려 주고 "몇 번이 주차면인가"를 묻는 **set-of-mark** 방식.
- 상보성: 한 번도 차가 안 선 자리는 **주차선**으로, 선이 지워진 노후 주차장은 **누적**으로 찾는다.

**최상위 위험**: 자동화는 **조용히 틀린다.** 사람이 그린 ROI 는 틀려도 사람이 안다. → **신뢰도 점수 · advisory · 검수 UI 가 기능의 일부**여야 한다(`placeRoiReport` issues 패턴).

- **리더의 경험적 검증이 핵심**이다. 이번 세션에서 리더가 직접 잡은 것들:

  - **런타임 데이터를 픽스처로 삼은 테스트 설계 결함**
  - 마스터의 "프레임 밖 좌표 정리" 지시를 **라이브 검증으로 중단**(데이터가 옳았다)
- **은닉 금지**: 검증 못 한 항목은 성공으로 위장하지 말고 **한계로 명시**.
