[ 시뮬레이터 기준 셋팅 에이전트 ]

* 주차면 바닥면 그리기  
  - LLM 사용여부 필요( UI 추가)
  - 기존 시뮬레이터에서 바닥면(ROI) 정보를 받음.
  - LLM 사용이 체크 되어 있으면 기존대로 LLM 사용함. ( 리얼 카메라 사용시 필요함)
  
* 주차면 점유 판정 
  - LLM 사용 여부 필요( UI 추가 )
  - LLM 없이도 점유 여부 판정 가능하게 처리
  - 예외상황 발생시( 차량을 가리는 오브젝트(차량, 사람) 발생시 LLM이 판단한다. )
  - 먼저 VPD를 통해 차량 검출하고
  - LPD를 통해 번호판을 검출한다.
  - 만약 VPD에서 검출된 차량이 존재하는데 번호판 box가 없으면 그 차량 중심으로 zoom을 하여 
    다시한번더 LPD로 번호판 검출을 시도한다. 번호판이 검출되면 다시 원래 위치로 카메라 zoom을 줄여서 
	이동하고 검출된 번호판 위치를 역 계산하여 번호판 위치를 추정하여 만든다.( 기존 번호판의 크기와 위치도 비교한다.) 
  - 차량 점유영역 그리기 와 주차면의 차량 점유 판단 ( camid, preset id, slot id, is_occupy ) 
  - 차량 점유영역은 작은 원으로 표시만 한다. ( 점유 판정이 중요함 )

* 차량번호판 중심 센터라이징
  - 카메라를 차량 번호판 중심으로 이동 시킨다.( Pan, Tilt만 조정 )
	+ 차량 번호판이 이미지의 중심이 되게 한다.
	+ LLM으로 만 가능하면 LLM 사용, 일반로직으로 가능하면 로직사용 - 두개의 채널 모두 사용가능하게 제작
  - 카메라의 zoom을 점점 확대하여 차량 번호판의 가로크기가 이미지의 20%가 될때까지 확대한다.  

* 웹 UI에서 옵션 페이지 필요
  - LLM 모델 및 유료/무료 CLI 용 ai
  - LLM 모델의 URL   
  - VPD, LPD, LPR 각각의 		
     "endpoint": "http://192.168.0.125:9081",
	 "detPath": "/vpd/api/v2/det/imgupload",

* LPR API을 이용한 차량 번호 검출 ( 확인용 )  
  - LPR을 이용해 차량번호 검출하고 정상적인 정보인지 확인
  - 차량번호가 재대로 인지 되지 않았으면 zoom을 25% 될때까지 확대하여 LPR을 이용해 차량번호 다시 검출한다.
