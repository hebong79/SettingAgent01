# ParkAgent 병렬(동시) 코드 작업 가이드 — git worktree

## 1. 문제

이 리포는 **단일 워킹트리 + 단일 브랜치**로 작업하면, 한 작업(기능 A)이 파일을 점유하는 동안
다른 작업(기능 B)을 같은 폴더에서 진행하면 편집·브랜치가 충돌한다.

동시 작업을 막는 공유 자원(계층별):

| 계층 | 공유 자원 | 충돌 | 이 가이드의 해결 |
|------|-----------|------|------------------|
| **코드** | 단일 워킹트리 / 브랜치 | 같은 파일 동시 편집·브랜치 스위칭 충돌 | **git worktree**(작업별 폴더+브랜치) |
| 서버 | 포트 `13020`, DB `data/observations.sqlite`, `data/`·`config/` 산출물 | 두 인스턴스 포트/DB/파일 다툼 | 워크트리가 tracked 파일을 각자 복사 → **자동 격리**. 서버 동시 실행은 §6 참고 |
| 유니티 | Editor 1개 + REST `13110` + **카메라 PTZ 전역 상태** | 두 작업이 같은 카메라를 동시 제어 | 본 가이드 범위 밖(§6) |

> 본 가이드는 **"여러 코드 작업을 병렬로 편집·테스트"** 를 대상으로 한다.

## 2. 해법 개요 — git worktree

`git worktree` 는 **하나의 리포(.git)** 에 대해 **여러 작업 폴더**를 두는 기능이다.
각 폴더는 서로 다른 브랜치를 체크아웃하므로, 작업 A와 B가 완전히 독립된 파일 트리에서 동시에 진행된다.

```
D:\Work\Parking3D\AgentVLA\
├─ ParkAgent\              ← 메인(현재, feat/vpd-seg-cuboid)
├─ ParkAgent-plate-fix\    ← 작업 A(feat/plate-fix)
└─ ParkAgent-zoom-ui\      ← 작업 B(feat/zoom-ui)
```

### 왜 이 리포에서 안전한가(파일 격리가 자동)
- `config/`·`data/`의 **tracked 파일**(camerapos.json, tools.config.json 등)은 워크트리마다 **각자 복사본** → 서로 안 건드린다.
- 관측 DB(`*.sqlite`)·`save/`·`reports/`·`captures/`는 **.gitignore** → 워크트리마다 **새로 생성**되어 겹치지 않는다.
- 즉 편집·유닛테스트(vitest는 외부 REST를 **모킹**)는 워크트리 간 **완전 격리**된다.

## 3. 헬퍼 스크립트 — `scripts/wt.ps1`

Windows PowerShell 용. 워크트리 생성/목록/제거를 한 줄로.

```powershell
# 생성: feat/<name> 브랜치로 형제 폴더 ..\ParkAgent-<name> 생성 + npm install
scripts\wt.ps1 new plate-fix

# 생성(빠름): npm install 대신 메인 node_modules 를 정션 재사용
scripts\wt.ps1 new plate-fix -LinkModules

# 다른 기준으로 분기(기본 HEAD)
scripts\wt.ps1 new hotfix -Base main

# 목록
scripts\wt.ps1 list

# 제거(브랜치 유지) / 브랜치까지 제거
scripts\wt.ps1 rm plate-fix
scripts\wt.ps1 rm plate-fix -DeleteBranch
```

생성 후 **새 VSCode 창/터미널에서 해당 폴더를 열어** 작업하면 된다. 메인 창은 그대로 둔다.

## 4. node_modules 전략

node_modules 는 gitignore 라 워크트리에 자동 생성되지 않는다. 두 가지 선택:

| 방식 | 명령 | 장점 | 주의 |
|------|------|------|------|
| **정식 설치**(기본) | `wt.ps1 new <name>` | 브랜치별 의존성 차이 안전, `packages/*` 편집 가능 | 설치 시간 소요 |
| **정션 재사용**(빠름) | `wt.ps1 new <name> -LinkModules` | 설치 생략, 즉시 시작 | `@parkagent/types` 등 워크스페이스 심링크가 **메인** packages 를 가리킴 → 워크트리에서 `packages/*` 를 편집하려면 이 방식 금지 |

- 네이티브 모듈(better-sqlite3, sharp)은 동일 OS/arch라 정션 공유해도 로드된다(검증 완료).
- **정션 안전 제거**: `wt.ps1 rm` 은 정션을 **먼저 해제**한 뒤 워크트리를 지운다 → 메인 node_modules 는 보존(검증 완료).
  일반 `git worktree remove` 를 직접 쓰면 정션을 따라가 **공유 node_modules 를 삭제**할 위험이 있으니 항상 `wt.ps1 rm` 을 쓸 것.

## 5. 작업 흐름(병합까지)

```powershell
# 1) 작업 시작
scripts\wt.ps1 new plate-fix -LinkModules

# 2) 새 창에서 ..\ParkAgent-plate-fix 열고 편집 + 테스트
cd ..\ParkAgent-plate-fix\SettingAgent
npm test                     # 워크트리 내부에서 독립 실행

# 3) 커밋(각 워크트리는 자기 브랜치)
git add -A; git commit -m "..."

# 4) 완료 후 메인에서 병합
cd ..\..\ParkAgent
git merge feat/plate-fix     # 또는 PR

# 5) 정리
scripts\wt.ps1 rm plate-fix -DeleteBranch
```

- 서로 **다른 파일**을 만지면 병합 충돌이 없다. **같은 파일**을 만졌다면 병합 시점에 한 번만 충돌 해소(동시 편집 중 실시간 충돌은 없음).

## 6. 런타임 병렬(서버·유니티) — 범위 밖, 주의점

편집·유닛테스트가 아니라 **서버를 두 개 동시에 띄워** 라이브 테스트하려면 추가 격리가 필요하다(현재 미적용):

- **포트**: `SettingAgent` 는 `config/tools.config.json` 의 `server.port(13020)` 고정. 두 인스턴스는 포트 충돌 →
  포트를 env(`process.env.SETTING_PORT`)로 오버라이드하도록 [SettingAgent/src/index.ts](../SettingAgent/src/index.ts) 1줄 수정이 필요(요청 시 반영).
- **유니티**: Editor·REST(`13110`)·카메라 PTZ가 **단일 전역** → 두 작업이 동시에 카메라를 움직이면 서로 덮어씀.
  라이브 정합·정밀수집을 병렬로 하려면 유니티 인스턴스/카메라 분리 설계가 별도로 필요(가장 큰 병목).

> 결론: **코드 편집·유닛테스트 병렬화는 워크트리로 지금 바로 가능**. 라이브 서버/유니티 병렬은 별도 과제.

## 7. AI 에이전트 병렬(참고)

여러 Claude 세션/에이전트로 병렬 개발 시에도 동일하게 워크트리로 격리하면 서로 파일을 덮지 않는다.
Claude Code 는 에이전트 실행 시 워크트리 격리(`isolation: worktree`)를 지원하며, 위 `wt.ps1` 로 만든 폴더를
각 세션이 열어 작업해도 된다. 완료 후 브랜치 병합으로 통합한다.

## 8. 영향도 분석

- **신규 파일**: `scripts/wt.ps1`(헬퍼), 본 문서. **기존 코드 변경 없음**(비침투적).
- **리포 구조**: 워크트리는 리포 **형제 폴더**에 생성 → 메인 트리·빌드·gitignore에 영향 없음. `.git/worktrees/` 메타만 추가.
- **위험**: 낮음. 유일한 주의는 node_modules 정션 삭제인데 `wt.ps1 rm` 이 정션 안전 제거로 처리(검증 완료).
- **검증**: 생성→정션 의존성 해석(vitest·better-sqlite3 네이티브 로드)→정션 안전 제거→브랜치 정리 전 과정 실측 통과.
