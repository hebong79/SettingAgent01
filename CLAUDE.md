# ParkAgent 프로젝트 코딩 규칙
"당신은 REST API 및 VLA AI, NodeJs(TypeScript), 파이썬 전문가 입니다. 반드시 유닛테스트를 진행 해주고, 실제 테스트를 거쳐 정상적인 동작을 확인해줘. 그리고 생성된 클래스 및 수정된 내용의 설명을 md파일로 만들어줘(한글로 생성) , 주변 파일에 영향이 있는지도 분석해줘."

1. **설계 필수**: 모든 코드 변경 및 신규 기능 구현 시 먼저 설계(계획)을 한다.
2. **유닛 테스트 필수**: 모든 코드 변경 및 신규 기능 구현 시 유닛 테스트를 작성하고 실행하여 검증한다.
3. **동작 확인**: 단순히 코드만 작성하는 것이 아니라, 실제 테스트 환경(Edit Mode/Play Mode)에서 정상 동작함을 확인한다.
4. **상세 문서화**: 변경 사항, 신규 클래스, 수정 로직에 대해 한글 `.md` 파일로 상세히 기록한다. ( yyyyMMdd_hhmmss_이름.md 형식의 파일이름) 
5. **영향도 분석**: 수정 사항이 프로젝트의 다른 부분(어셈블리, 의존성, 기존 기능 등)에 미치는 영향을 분석하여 보고한다.

## 하네스: ParkAgent 개발

**목표:** CLAUDE.md 5대 규칙(설계→유닛테스트→동작확인→한글문서화→영향도분석)을 4인 에이전트 팀으로 자동 실행한다.

**트리거:** SettingAgent/ActionAgent/DMAgent 코드 개발(기능 추가·버그 수정·리팩토링·MCP 도구/REST 클라이언트 작성) 요청 시 `parkagent-dev` 스킬을 사용하라. 단순 질문·조회는 직접 응답 가능.
