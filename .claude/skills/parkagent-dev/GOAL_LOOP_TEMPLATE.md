# goal/loop 프롬프트 템플릿

`parkagent-dev` 오케스트레이터를 **B(goal/loop) 모드**로 확정 실행하는 구조화 프롬프트 형식.
아래 세 섹션(`Goal:` / `Loop:` / `Requirements:`)을 채워 그대로 붙여넣으면 된다.

```
Goal:
  <달성할 성공 목표 — 관찰 가능한 결과로. 예: "PtzCamRoi.json을 참조해 바닥 ROI를 그린다">

Loop:
  * <분석·코드설계 단계>
  * <코드 수정 단계>
  * <스샷/동작으로 확인하는 단계>
  * <카메라/프리셋 등 순회 확인 단계>
  * <틀어지면 재분석·재설계·재수정 하라는 재시도 조건>
  * 성공하면 최종 결과를 설명해줘.

Requirements:
  * <끝까지 지켜야 할 제약·수용 기준 1>
  * <제약·수용 기준 2>
  ...
```

## 각 섹션이 처리되는 방식
- **Goal** → goal로 그대로 채택(성공 정의).
- **Loop** → 리더가 순서대로 실행하는 이터레이션 절차. "틀어지면 다시…" 단계는 재시도 조건이 되어 성공까지 반복. "스샷 확인"은 sharp 렌더+Read, "순회"는 프리셋/카메라별 렌더·검수로 수행.
- **Requirements** → 설계·구현의 불변 제약이자 마감 시 항목별로 대조·보고하는 체크리스트. 미충족·미검증은 은닉 없이 명시.

## 예시 (실제 수행 사례)
```
Goal:
  SettingAgent\data\Place01\PtzCamRoi.json을 참조하여 바닥 ROI를 그려줘.

Loop:
  * PtzCamRoi.json 열어 분석 후 코드 설계를 한다.
  * 설계된 내용대로 코드를 수정한다.
  * 카메라 프리셋 별로 데이터대로 화면에 그려졌는지 스샷을 찍어 확인한다.
  * 카메라들의 모든 프리셋을 돌면서 확인한다.
  * 위치값이 많이 틀어져 있으면 다시 파일을 분석하고 코드설계·수정한다.
  * 성공하면 최종 결과를 설명해줘.

Requirements:
  * PtzCamRoi.json은 정밀수집 페이지 진입 시 바로 로딩해 데이터를 보유한다.
  * 각 카메라의 프리셋 단위로 출력한다.
  * 현재 해당 프리셋만 출력한다.
  * 프리셋마다 JSON 정보를 검수한다.
```

세부 실행 규약은 `SKILL.md`의 "goal/loop 실행 모드 (B) 상세" 참조.
